# 015：Web安全-part1

在本节课中，我们将要学习Web安全的基础知识，特别是两种最常见的Web攻击类型：注入漏洞和跨站脚本攻击。我们将从Web的基本工作原理开始，逐步深入到具体的攻击手法和防御策略。

## Web工作原理概述

上一节我们介绍了缓冲区溢出等内存攻击，本节中我们来看看Web应用的安全问题。首先，我们需要理解Web应用的基本交互模型。

当你在浏览器中输入一个URL时，浏览器会向指定的服务器发送一个HTTP请求。服务器接收到请求后，可能会执行一些服务器端代码（例如PHP），并可能查询数据库。最后，服务器将生成的HTML、CSS和JavaScript代码发送回浏览器，浏览器会解析这些内容并渲染成你看到的网页。

这个过程中涉及几个关键部分：
*   **客户端**：通常是用户的浏览器。
*   **服务器**：运行Web服务（如Apache）的机器。
*   **数据库**：存储应用数据的服务（如MySQL）。

## Web威胁模型

在深入具体攻击之前，我们需要明确Web环境下的威胁模型。与之前讨论的攻击不同，Web攻击的参与方和攻击目标更加多样。

以下是几种常见的攻击场景：
*   **恶意客户端攻击服务器**：例如，通过注入恶意输入来攻击服务器端应用。
*   **恶意服务器攻击客户端**：例如，通过钓鱼或点击劫持来欺骗用户。
*   **恶意用户攻击其他用户**：例如，在一个公共论坛中注入恶意脚本，攻击其他访问者。
*   **恶意中间人**：在通信链路上拦截或篡改数据。

## 注入漏洞

注入漏洞的核心问题是：**将不受信任的数据发送给解释器**。当用户输入被当作代码执行时，就会发生注入攻击。

### 命令注入

假设一个网站提供了一个“Ping工具”，允许用户输入IP地址来测试连通性。服务器端代码可能如下所示：

```php
$ip = $_POST[‘ip’];
system(“ping -c 3 $ip”);
```

![](img/7050550f8488bfe43058b5126a9e109e_1.png)

这段代码直接使用用户输入的 `$ip` 变量拼接成系统命令。如果用户输入的不是IP地址，而是 `127.0.0.1; ls`，那么实际执行的命令将是：

```
ping -c 3 127.0.0.1; ls
```

分号 `;` 在shell中用于分隔命令，因此服务器不仅会执行ping，还会执行 `ls` 命令，从而泄露服务器上的文件列表。攻击者甚至可以注入更危险的命令，例如开启一个反向shell。

### SQL注入

![](img/7050550f8488bfe43058b5126a9e109e_3.png)

SQL注入是另一种常见的注入攻击，它针对的是数据库查询。假设一个网站根据用户ID查询用户信息，PHP代码可能这样写：

```php
$id = $_GET[‘id’];
$query = “SELECT first, last FROM users WHERE user_id = ‘$id’”;
```

如果用户输入的 `id` 是 `1’ OR ‘1’=’1`，那么最终的SQL查询会变成：

```sql
SELECT first, last FROM users WHERE user_id = ‘1’ OR ‘1’=’1’
```

由于 `’1’=’1’` 永远为真，这个查询会返回 `users` 表中的所有记录，而不仅仅是ID为1的用户。

![](img/7050550f8488bfe43058b5126a9e109e_5.png)

#### 探测与利用SQL注入

攻击者通常不知道后端数据库的结构。以下是探测SQL注入的常用步骤：

![](img/7050550f8488bfe43058b5126a9e109e_7.png)

![](img/7050550f8488bfe43058b5126a9e109e_9.png)

1.  **确定列数**：使用 `ORDER BY` 子句。例如，注入 `1’ ORDER BY 1--`，然后递增数字（`ORDER BY 2`, `ORDER BY 3`…），直到引发错误，错误前的数字就是查询的列数。
2.  **获取数据库信息**：大多数数据库（如MySQL）有 `information_schema` 表，存储了所有数据库、表和列的信息。攻击者可以通过联合查询（`UNION`）来获取这些元数据。
3.  **盲注**：如果网站不显示详细的错误信息，攻击者可以使用“盲注”技术。通过构造条件查询，并根据页面响应时间或内容的细微差别来判断条件真假，从而逐位提取数据。

![](img/7050550f8488bfe43058b5126a9e109e_11.png)

![](img/7050550f8488bfe43058b5126a9e109e_13.png)

#### SQL注入防御

![](img/7050550f8488bfe43058b5126a9e109e_15.png)

防御SQL注入的最佳方法是使用**参数化查询**（或预编译语句）。这种方法将SQL代码与数据分离，确保用户输入永远被当作数据处理，而不是代码的一部分。

例如，在PHP中使用PDO：

![](img/7050550f8488bfe43058b5126a9e109e_17.png)

![](img/7050550f8488bfe43058b5126a9e109e_19.png)

![](img/7050550f8488bfe43058b5126a9e109e_21.png)

![](img/7050550f8488bfe43058b5126a9e109e_23.png)

```php
$stmt = $pdo->prepare(‘SELECT first, last FROM users WHERE user_id = :id’);
$stmt->execute([‘id’ => $id]);
```

![](img/7050550f8488bfe43058b5126a9e109e_25.png)

## 跨站脚本攻击

跨站脚本攻击允许攻击者将恶意脚本注入到其他用户浏览的网页中。当受害者的浏览器执行这些脚本时，攻击者可以窃取用户的会话Cookie、篡改网页内容或进行其他恶意操作。

### XSS工作原理

![](img/7050550f8488bfe43058b5126a9e109e_27.png)

浏览器在渲染网页时，会构建一个文档对象模型树。JavaScript可以动态地访问和修改这个DOM树。XSS攻击的本质就是**向DOM中注入恶意脚本**。

### 反射型XSS

![](img/7050550f8488bfe43058b5126a9e109e_29.png)

反射型XSS中，恶意脚本来自当前HTTP请求。例如，一个搜索页面将用户输入的搜索关键词直接显示在结果页面上。

如果搜索URL是：
`https://example.com/search?q=<script>alert(‘XSS’)</script>`

![](img/7050550f8488bfe43058b5126a9e109e_31.png)

而服务器端代码未对 `q` 参数进行过滤，直接将其嵌入到返回的HTML中，那么任何访问这个链接的用户都会执行这段脚本。

![](img/7050550f8488bfe43058b5126a9e109e_33.png)

攻击者可以构造一个包含恶意脚本的链接，并通过钓鱼邮件等方式诱骗用户点击。

### 存储型XSS

![](img/7050550f8488bfe43058b5126a9e109e_35.png)

存储型XSS中，恶意脚本被永久地存储在目标服务器上（如数据库、评论、用户资料）。当其他用户访问包含该恶意内容的页面时，脚本会自动执行。

例如，一个论坛允许用户在评论中使用HTML标签。如果未进行严格过滤，攻击者可以提交一条包含 `<script>` 标签的评论。此后，所有查看这条评论的用户都会中招。

### XSS攻击的危害

通过XSS，攻击者可以：
*   窃取用户的会话Cookie，从而劫持用户账户。
*   伪造请求，以用户身份执行操作（如转账、发帖）。
*   进行键盘记录或屏幕截图。
*   将用户重定向到恶意网站。

![](img/7050550f8488bfe43058b5126a9e109e_37.png)

### XSS防御

![](img/7050550f8488bfe43058b5126a9e109e_39.png)

![](img/7050550f8488bfe43058b5126a9e109e_41.png)

防御XSS非常困难，因为浏览器为了兼容性，会尝试“修复”不规范的HTML，这导致过滤规则极其复杂。

![](img/7050550f8488bfe43058b5126a9e109e_43.png)

核心防御原则是：**对输出进行编码或转义**。在将用户输入的内容放入HTML文档的不同位置（如HTML正文、属性、JavaScript、CSS）时，必须使用相应的编码函数。

![](img/7050550f8488bfe43058b5126a9e109e_45.png)

![](img/7050550f8488bfe43058b5126a9e109e_47.png)

![](img/7050550f8488bfe43058b5126a9e109e_49.png)

例如，在PHP中，可以使用 `htmlspecialchars()` 函数将特殊字符转换为HTML实体：
`echo htmlspecialchars($user_input, ENT_QUOTES, ‘UTF-8’);`

更安全的方法是使用成熟的、经过安全审计的模板引擎或框架，它们通常内置了安全的上下文输出编码。
