# 课程名称：BloackHat 2022 会议笔记 📝

在本节课中，我们将学习并整理来自BloackHat 2022安全会议的核心技术要点。这些笔记涵盖了多个关键的安全研究领域，旨在为初学者提供一个清晰、结构化的知识概览。

## 1. 内存安全漏洞 🧠

上一节我们介绍了课程的整体框架，本节中我们来看看内存安全漏洞这一基础但至关重要的领域。

内存安全漏洞源于程序对内存的不当访问。攻击者可以利用这些漏洞执行任意代码或窃取敏感数据。

以下是几种常见的内存安全漏洞类型：

*   **缓冲区溢出**：当程序向固定长度的缓冲区写入超出其容量的数据时，就会发生溢出，覆盖相邻的内存区域。
*   **释放后使用**：程序释放了某块内存后，后续代码又错误地引用了这块已被释放的内存。
*   **双重释放**：程序对同一块动态分配的内存进行了两次释放操作。
*   **越界读取/写入**：程序访问了数组或缓冲区边界之外的内存位置。

## 2. 现代缓解技术 🛡️

了解了传统漏洞后，我们来看看现代操作系统和编译器如何部署防御措施来缓解这些攻击。

现代系统采用了一系列缓解技术来增加漏洞利用的难度。

以下是几种关键的缓解技术：

*   **地址空间布局随机化**：在每次程序运行时，将关键数据区域的基地址随机化，使攻击者难以预测目标地址。
*   **数据执行保护**：将内存页标记为仅可读或仅可执行，防止在数据区域执行代码。
*   **控制流完整性**：通过校验间接跳转的目标地址，确保程序执行流不会被恶意篡改。

## 3. 旁道攻击 ⚡

当直接攻击变得困难时，攻击者会转向利用系统实现中的物理或微架构缺陷。本节将探讨旁道攻击。

旁道攻击不直接攻击算法或代码的逻辑错误，而是通过分析时间、功耗、电磁辐射等“旁道信息”来推断敏感数据。

一个经典的例子是缓存定时攻击，其核心思想可以简化为：
`if (数据在缓存中) { 访问速度快; } else { 访问速度慢; }`
通过精确测量内存访问时间，攻击者可以推断出密钥位等信息。

## 4. Web安全新挑战 🌐

过渡到应用层，Web安全始终是攻防前沿。本节聚焦于一些新兴的、复杂的攻击面。

随着Web技术栈的复杂化，新的攻击向量不断出现，对传统防御模型构成挑战。

以下是两个值得关注的高级Web攻击方向：

*   **针对客户端JIT编译器的攻击**：利用JavaScript引擎的即时编译优化过程，构造特殊代码触发漏洞，实现沙箱逃逸。
*   **WebAssembly的滥用**：攻击者可能利用WebAssembly的高性能特性来隐藏恶意代码逻辑，或构建更高效的密码货币挖矿程序。

## 5. 供应链攻击 ⛓️

最后，我们将视角从单点技术攻击提升到对整个软件生态的威胁——供应链攻击。

供应链攻击通过污染软件依赖、开发工具或分发渠道来植入后门，其影响范围极广。

以下是供应链攻击的常见切入点和防范思路：

*   **攻击点**：开源库、代码仓库、构建流程、更新服务器。
*   **防范建议**：实施严格的依赖审核、使用数字签名验证完整性、建立软件物料清单。
