## 概述

## 脚本与程序的区别
脚本本质上也是一种程序。脚本与编译型程序的一个基本区别在于，脚本不会被直接翻译成机器码。脚本依赖于一个解释器程序，该解释器会逐行读取并执行脚本代码。除此之外，脚本具备与程序相同的功能，例如可以包含函数、过程，并能进行外部调用。

**外部调用**指的是在一个脚本中引用并执行另一个程序（例如另一个脚本），并将其输出整合到当前脚本的执行流程中。

脚本的执行同样依赖于**标准变量**和**常量**。

---

## 变量
大多数脚本都依赖变量来生成输出。一个普遍的经验法则是，与大多数编程语言一样，大多数脚本解释器需要在使用前定义变量。

在像Bash这样的脚本语言中，变量定义可能不是强制的，但在Windows环境中，通常需要声明变量的类型。

**声明变量类型**意味着，如果你打算用变量存储文本，你需要告知脚本解释器该变量用于存储文本。反之，如果一个变量被声明用于存储数字，而你尝试为其分配文本，你将无法对该变量执行任何数学运算，否则会导致错误。

出于空间效率考虑，大多数变量类型在定义时就已经分配了固定的内存大小。这在大多数编程语言和解释器中都是如此。Bash是一个例外，它使用指针内存类型，因此不预先固定变量的大小。

对于变量，我们可以执行以下操作：赋值、引用、操作，并在程序结束时删除变量。

---

## 参数
有一种特殊的变量称为**参数**或**参数变量**。这正是脚本如此有用的原因。

当我们调用一个脚本时，可以在命令行中传递额外信息。例如，调用一个名为 `backup` 的脚本，并指定 `backup this_database`。在脚本内部，我们可以将跟在脚本名后面的 `this_database` 作为一个参数来编程处理。

参数本质上是脚本执行开始时接收到的输入，是脚本调用的一部分。我们将利用这些参数来确定操作的目标或要执行的具体命令。

---

## 条件语句（If语句）
`If` 语句是脚本中实现决策的基本结构。它为我们提供了一种框架，用于决定执行哪一种操作。

`If` 语句的标准定义如下：
```
if [condition is true]
then
    # 执行条件为真时的代码
else
    # 执行条件为假时的代码
fi
```
首先，我们设定一个条件，其结果为真或假。然后，定义当条件为真时要执行的代码块，以及当条件为假时要执行的代码块（`else` 部分）。最后，以 `fi` 结束整个 `if` 语句结构。

## 循环
在执行重复性任务时，我们会遇到需要多次重复同一操作的情况。有时我们知道需要重复的确切次数，有时则取决于一个变量或列表，在编写脚本时无法预知。

为此，我们使用循环。几乎所有编程语言都定义了三种主要的循环结构。

以下是三种主要循环的介绍：

**1. For 循环**
`For` 循环在循环开始时进行评估。它会执行一个预先确定次数或由参数决定的循环。
其基本结构是：
```
for variable in sequence
do
    # 要重复执行的代码
done
```

**2. While 循环**
`While` 循环在条件为真时持续执行。
其逻辑是：当某个条件成立时，重复执行一系列任务。例如，“当传感器被触发时，执行一系列加热传感器的任务”。只要条件保持为真，循环就会继续。

**3. Until 循环**
`Until` 循环在循环结束时进行评估，以判断某个条件是否变为真。从功能上讲，`until` 循环能做到的任何事情，`while` 循环也都能实现，因此 `until` 循环的使用相对较少。

## 总结
在本节课中，我们一起学习了脚本编程的核心概念。我们明确了脚本与编译程序的本质区别，理解了变量声明与类型的重要性，掌握了如何通过参数使脚本更具交互性和灵活性。我们还探讨了使用 `if` 语句进行条件判断的逻辑，并介绍了 `for`、`while` 和 `until` 这三种实现重复操作的循环结构。这些基础知识是进一步学习自动化脚本编写和安全分析工具使用的坚实起点。
