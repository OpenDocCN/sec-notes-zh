# 课程4：《网络安全与数据库漏洞》：62：3_06：有状态检测 🔍

![](img/d884b89b97ba8f95bcc5c816fd44b3e4_0.png)

在本节课程中，我们将学习有状态防火墙所提供的额外检测层级。我们将了解有状态检测如何工作，它与无状态检测的区别，以及数据包在防火墙中的处理流程。

![](img/d884b89b97ba8f95bcc5c816fd44b3e4_2.png)

---

上一节我们介绍了防火墙的基本概念，本节中我们来看看**有状态检测**。

有状态检测意味着，每个数据包的检查都基于对同一会话中已发送或接收的所有其他数据包的了解。一个会话包含双方在一次通信交换期间交换的所有数据包。

会话包含多个元素，例如：
*   源IP地址
*   目标IP地址
*   源端口
*   目标端口
*   在某些情况下，还有一个实例标识符（例如，如果路由器支持虚拟化）

这五个要素共同定义了一个会话。如果防火墙看到来自相同源IP地址、前往相同目标地址、来自相同源端口、前往相同目标端口的流量（无论是TCP还是UDP），它就会知道这些数据包属于同一会话，并可以自动允许或丢弃该数据包。

---

在以下示例中，可以看到一个已分配的会话ID。这意味着我们的防火墙维护着一个会话数据库，并跟踪会话中的所有数据包。

```plaintext
会话ID: 12345
策略名称: Allow_Web_Traffic
超时值: 1800秒
传入接口: ge-0/0/0.0
源IP: 192.168.1.100
目标IP: 203.0.113.5
VLAN/接口: VLAN10
数据包数: 150
字节数: 45000
网络地址转换（NAT）信息: 已启用
传出接口: ge-0/0/1.0
```

以下是与会话关联的一些信息：
*   **会话ID**和**策略名称**。
*   可以配置**超时值**，使会话不会永久保持打开状态。
*   可以看到**传入接口**、**源IP地址**、**目标IP地址**。
*   数据包来自哪个**VLAN或接口**。
*   会话中已使用的**数据包数量**和**字节数**。
*   如果存在**网络地址转换（NAT）**，也会看到相关信息。
*   还会看到**传出接口**。

> 注：此输出示例特定于Juniper防火墙。

---

那么，如果同时进行无状态检测和有状态检测，会发生什么？

首先会执行**无状态检测**，然后评估**有状态数据**。

以下是一个特定于Juniper防火墙的数据包处理流程图。数据包将遵循此流程：

**情况一：传入数据包与现有会话匹配**

如果传入数据包与现有会话匹配，防火墙将：
1.  评估**筛选器**。
2.  查看流量类型，并将其与会话进行匹配。
3.  应用所需的**NAT或其他服务**。图中显示的一些服务包括：
    *   AppT（应用跟踪）
    *   AppD（应用检测）
    *   AppQoS（应用服务质量）
    *   AppFW（应用防火墙）
    *   IDP（入侵检测与防御）
    *   （这些是更高级的安全服务）

**情况二：传入数据包与现有会话不匹配**

如果传入数据包不匹配任何现有会话，则将遵循不同的流程：
1.  首先应用**筛选器**。筛选器本质上是过滤器，用于防御洪泛攻击或拒绝服务攻击。
2.  如果需要，则应用**静态NAT**。
3.  如果已配置，则应用**目标NAT**。
4.  然后进行**路由和策略评估**。根据定义的传入和传出接口确定路由后，将定义**区域**，并查看是否有策略允许或丢弃该流量。
5.  在某些情况下，可能会看到**反向静态NAT**。策略评估后，源NAT信息将被移除。
6.  应用**服务**。
7.  最后，**建立会话**。

> 对于网络管理员而言，牢记此类流程非常有帮助。

---

通过对比可以看出，当传入数据包**不匹配**现有会话时，其通过防火墙的路径**更长**。如果数据包**匹配**现有会话，其路径则**更短**，因为防火墙已经知道该数据包的行为。

本节课中我们一起学习了有状态检测防火墙的工作原理。我们了解到，有状态防火墙通过维护会话数据库，能够基于完整的通信上下文来检查数据包，从而提供比无状态检测更深层次的保护。我们还分析了数据包在匹配或不匹配现有会话时的不同处理流程，这有助于理解防火墙如何高效地管理网络流量和安全策略。
