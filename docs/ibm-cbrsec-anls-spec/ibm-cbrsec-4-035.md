在本节课中，我们将学习识别典型组织中存在的多种数据源，并了解每个数据源中通常包含的数据类型。



![](img/c180024dc3be0b1e4acbd0223cb7fc9b_2.png)

上一节我们介绍了数据源的基本概念，本节中我们来看看一个典型组织中可能存在的各种数据源的具体例子。需要说明的是，以下列表并非详尽无遗，但它展示了与组织数据相关的各种事物，以及人们访问数据的不同途径。

一个典型的组织不会只有一个供数据库管理员（DBA）连接的数据库。实际上，组织会拥有许多连接到后端数据库的应用程序。例如，人力资源系统在员工入职和离职时使用，SAP或PeopleSoft等系统处理客户订单和全球物流运输，以确保准时交付。所有这些功能都依赖于数据库，而整个员工队伍每天都在登录这些应用程序来完成工作。

数据仓库通常用于处理大量数据计算。它们通常包含海量数据，例如Hadoop的Hive、亚马逊的Redshift，甚至是甲骨文的Exadata。Exadata配备了极其高速的处理器，专门用于高效、快速地处理数据。因此，你可以将Exadata理解为专门用于“数字运算”的工具。

大数据环境在组织中也很常见，它涉及巨量的数据。很多时候，人们并不完全清楚这些数据里有什么，或者打算用它来做什么。例如，一些已停用的遗留数据库中的数据，被归档存储起来，组织可能暂时不知道如何处理它们。有人可能会决定将其放入大数据环境中，希望未来能从中获取关于客户、产品、业务流程以及如何改进业务和互动方式的更多信息。这些信息被投入大数据环境，其理念通常是期望日后通过分析和挖掘来从中获取价值。

云环境则提供了与本地部署不同的数据托管方式。本地部署指的是你自己建立、控制和完全拥有的数据中心。

数据库工具是与数据库交互的不同方式，通常由DBA使用，但其用途可以多种多样。

内容管理器，例如SharePoint、Class One等，种类繁多。企业内容管理器可能是一个项目管理工具，比如Basecamp。

文件共享你可能比想象中更熟悉。即使是你的“下载”文件夹，也是一种文件共享。Linux、Unix、Windows系统中存储的所有不同文件，都位于文件共享中。当你连接到不同的FTP站点时，那些数据可以是非结构化数据。

以下是不同类型数据源的例子：

*   **分布式数据库**：例如 Oracle RAC、Microsoft SQL Server、MySQL、PostgreSQL 等。
*   **大数据数据库**：例如 Hadoop、MongoDB、Bigtable。
*   **数据仓库**：例如 Teradata、Exadata、Amazon Redshift、Apache Hive。
*   **文件共享**：例如网络附加存储（NAS）、网络文件共享（如EMC或NetApp）以及云存储共享（如Google Drive、Dropbox、Box.com和Amazon S3存储）。

现在，我们来思考一下这些数据源的类型。如果观察分布式数据库和数据仓库，这两者通常被认为是**结构化数据**。我们稍后会解释这意味着什么。

大数据数据库的例子通常是**半结构化数据**。这主要是因为很多时候，大数据是由许多不同的结构化数据源汇集而成，但这些数据源没有一个统一的方式来整体地看待所有被投入其中的不同类型的数据。我们将在讲解结构化和半结构化数据时进一步解释这一点。

文件共享的例子则纯粹是**非结构化数据**。想想你的“下载”文件夹：你下载文件总有原因，但仅此而已。它可能包含你工作相关的所有不同项目文件，也可能是你孩子第一次走路的可爱视频。这里有你下载的各种东西，但除了“被下载”这个事实外，它们本身没有任何真正的结构。

本节课中我们一起学习了典型组织内多种数据源的具体实例，包括应用程序数据库、数据仓库、大数据环境、文件共享等，并初步了解了结构化、半结构化和非结构化数据的概念区分。理解这些数据源的种类和特性，是分析其潜在安全漏洞的第一步。
