![](img/a98239e6f562c5e1a87ef1de3aa2bdee_1.png)

在本节课程中，我们将学习SOC（系统和组织控制）报告的审计流程。我们将了解审计师在评估组织控制措施时所关注的核心要素，以及如何通过持续监控来确保合规性。

---

上一节我们介绍了SOC报告的基本概念，本节中我们来看看审计师在执行SOC 2审计时的具体流程和关注点。

审计师在测试时，会关注控制的连续性。他们主要测试五个核心要素。SOC 2合规性要求非常严格，极具挑战性。审计师会逐一审视这些要素。

以下是审计师测试的五个主要方面：

1.  **准确性**：检查所有控制措施是否都得到处理，明确区分控制是通过还是失败，以及控制是否被完整执行。
2.  **完整性**：评估控制是否覆盖了整个服务范围。例如，针对系统的控制是否覆盖了所有系统和资产清单；针对访问管理的控制是否覆盖了所有人员。
3.  **及时性**：确保控制按时或提前执行，且覆盖范围没有间隙。例如，如果规定每周打补丁，延迟一天就会造成覆盖缺口，可能带来恶意攻击或意外错误的风险。审计师会特别关注及时性。
4.  **风险评估与记录**：如果因故无法按时执行控制，必须有基于客户业务需求的合理理由，并提前进行适当的风险评估和记录。例如，原定周五打补丁，但客户因促销活动要求推迟到周六。你需要在周四就完成风险评估并记录，而不是等到周六。
5.  **弹性与一致性**：审计师会寻找制衡机制，确保在主控制失效时有备用方案来保证任务按时完成。同时，他们希望确保没有因过多变数而引入的漏洞。例如，他们会测试主控制，并寻找支持或备份机制以确保其有效性。以访问管理为例，你是否定期核对审批记录与系统中的实时账户，以确保无人绕过系统私自创建账户。

---

这是一个用于审计的不同控制措施的总结列表。可以看到，它们可以分解为不同的章节。这些章节是我们出于自身目的使用的。你可以从AICPA（美国注册会计师协会）的原始要求集开始。

无论你选择哪种合规框架，无论是ISO还是SOC，在两次审计之间进行**持续监控**都至关重要。我们讨论了很多可能导致SOC 2审计失败的情况，但即使是ISO审计，你也需要确保控制措施按设计运行、发挥其功能，并且所有必要的沟通都已到位。你需要测试你的环境、流程和人员，以确保控制措施被持续执行。我们称之为**持续监控**，目的是发现任何偏离的风险、临时的失败或延迟，并有效地识别它们。因为，如果你不确保自己真正在执行标准，那么拥有标准就毫无意义。这是一个重要的方面。

本节课中，我们一起学习了SOC报告的审计流程。我们了解了审计师关注的五个核心要素：**准确性**、**完整性**、**及时性**、**风险评估与记录**以及**弹性与一致性**。同时，我们认识到在审计间隔期进行**持续监控**对于维持合规状态、及时发现和纠正偏差至关重要。
