## 📚 数制系统概述

我相信大家在学校里都接触过基数为2的二进制数。但考虑到对某些人来说那是很久以前的事了，我将快速回顾一下。

我们习惯于在基数为10的十进制系统中思考、交流和计算。人类采用十进制系统，很可能是因为我们有10根手指和10根脚趾。因此可以推测，早期的计数是在我们祖先的手指上完成的。

然而，计算机只知道两种不同的状态：开和关，或者高电压和低电压，或者门打开和关闭。我们在逻辑上用1或0来表示这些状态。计算机只能区分这两种状态。

## 🔢 十进制系统工作原理

以下是十进制系统的工作原理。我们有代表个位、十位、百位、千位等的数位。每个数位的值是前一个数位的10倍。每个数位可以容纳10个不同的值，范围从0到9。请记住，“decimal”这个词的意思是“十”。

## 🔲 二进制系统工作原理

另一方面，在二进制系统中，每个数位或占位符持有不同的值。有代表1、2、4、8、16等的占位符。每个数位的值是前一个数位的2倍。在二进制中，每个数位只能容纳两个不同的值：0或1。请记住，“binary”的意思是“二”。

从二进制转换为十进制很容易。只需将所有包含1的数位的值相加，忽略那些包含0的数位。

**示例：**
假设我们有一个八位的二进制数：`11011010`。
其十进制等效值为：`2 + 8 + 16 + 64 + 128 = 218`。

## 🔄 从十进制转换为二进制

从十进制转换为二进制稍微复杂一些，可能会让你想起小学时学的长除法。当然，有很多计算器和在线工具可以为你进行这些转换，但以下是手动方法。

**示例：**
将十进制数235转换为二进制。
1.  从最大的二进制占位符开始，看能否从你的十进制数中减去它。256太大，但可以减去下一个较小的占位符128。所以在128的位置放一个1。然后从235中减去128，剩下107。
2.  检查能否从107中减去64。可以，所以在64的位置加一个1。然后从107中减去64，剩下43。
3.  检查下一个占位符。能否从43中减去32？可以。所以在32的位置加一个1，并从43中减去32，剩下11。
4.  检查下一个占位符。能否从11中减去16？不能。所以在16的位置放一个0。
5.  检查下一个占位符。能否从11中减去8？可以。所以在8的位置放一个1，并从11中减去8，剩下3。
6.  检查下一个占位符。能否从3中减去4？不能。所以在4的位置放一个0。
7.  检查下一个占位符。能否从3中减去2？可以。所以在2的位置放一个1，并进行减法。现在剩下1。
8.  下一个占位符是最后一个，1。能否从1中减去1？可以。所以在1的位置放一个1，并进行减法。1-1=0。完成。

因此，235的二进制表示为：`11101011`。

## 🌐 其他常用数制系统

上一节我们介绍了二进制和十进制之间的转换。但不幸的是，在处理计算机时，你还会遇到其他数制系统。如果你已经熟悉二进制、八进制、十进制和十六进制系统，可以跳过本视频的最后五分钟。

*   **二进制（基数为2）**：每个数位可以有两个可能的值：0或1。
*   **八进制（基数为8）**：不如二进制常见，但偶尔会遇到。八进制可以取8个可能的值，范围从0到7。
*   **十进制（基数为10）**：可以取10个可能的值，范围从0到9。
*   **十六进制（简称Hex，基数为16）**：可以取16个可能的值，范围从0到F。当我们用完数字后，就开始使用字母。因此，10到15分别用A、B、C、D、E和F表示。

对于任何数制系统，不仅仅是这四种，计算给定位数可以表示多少个不同数字的方法很简单：只需将你正在使用的基数（base）提高到可用位数的幂次方。

**示例：**
假设我们有4个可用的数位。
*   在二进制中：`2^4 = 16` 个不同的数字（范围：0 到 1111，即十进制0到15）。
*   在八进制中：`8^4 = 4096` 个不同的数字（范围：0 到 7777，即十进制0到4095）。
*   在十进制中：`10^4 = 10000` 个不同的数字（范围：0 到 9999）。
*   在十六进制中：`16^4 = 65536` 个不同的数字（范围：0 到 FFFF，即十进制0到65535）。

很容易看出，使用更大的基数可以更紧凑地表示大数字。例如，十六进制中的3位数`FFF`在二进制中需要12位数来表示：`111111111111`。

## 🔗 数制间的转换优势

计算机硬件和机器代码级别的软件都基于二进制。因此，使用2的幂次方作为基数的数制系统，使得在这些系统之间转换变得更加容易。

*   3个二进制数位可以直接转换为1个八进制数位。
*   4个二进制数位可以直接由1个十六进制数位表示。

在十进制中，没有这种精确的数位到数位的转换可能。

要将任何基数系统转换为十进制，你可以绘制一个像我们之前使用的表格，但将右侧的列标题值替换为你正在使用的基数对应的值。

*   对于二进制：1, 2, 4, 8, ...
*   对于八进制：1, 8, 64, 512, ...
*   对于十进制：1, 10, 100, 1000, ...
*   对于十六进制：1, 16, 256, 4096, ...

请注意，所有这些列标题都是基数（base）的0次方、1次方、2次方、3次方……这对于你需要处理的任何基数都适用。
