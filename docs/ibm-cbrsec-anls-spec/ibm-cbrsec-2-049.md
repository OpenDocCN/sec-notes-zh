为了明确术语定义，我们先来看看这个基本场景。

在密码学文献中，你经常会看到爱丽丝、鲍勃和特鲁迪。他们通常用A、B、T来指代。早在20世纪60年代的一些论文中，这些角色就被赋予了爱丽丝、鲍勃和特鲁迪的名字，并沿用至今。

鲍勃和爱丽丝希望进行安全通信，原因可以是个人事务或商业事务。特鲁迪则是拦截者，她意图拦截、删除、添加或篡改消息，本质上是一个恶意行为者。

现在，我们来看幻灯片7上的图表。右侧是爱丽丝，她有一些数据。这些数据可能是一封电子邮件、一张便条或一个网页。她将这条消息从**明文**（`clear text`）加密为**密文**（`cipher text`），然后通过一个信道进行传输。

信道可以是任何我们能够想到的传输形式，例如电子邮件、直接传输、文件传输协议，或是如今的短信。回溯到拿破仑时期，这可能是一位年轻的海军候补军官在白厅与伦敦其他地区之间传递的一封信件。因此，信道就是传输机制，而信道内承载的则是数据，即有效载荷。

有效载荷包括控制信息：消息要发送给谁、有效期多长、接收者鲍勃的地址是什么。显然，在网络世界中，我们关注IP地址和MAC地址；在人工世界中，例如在英国情报开始崛起的拿破仑时代，这指的是姓名和实际的邮寄地址。所以，实际的邮寄地址就是控制信息的人工解读形式。

鲍勃收到消息后，对其进行解码，从而获得爱丽丝发送给他的明文。特鲁迪有能力在信道上拦截这些消息，但由于加密的安全保护，她无法读取、删除或篡改这些消息。

## 概述

在本节中，我们介绍了密码学中三个经典角色：爱丽丝、鲍勃和特鲁迪，并解释了他们在安全通信场景中的基本作用和关系。

## 核心角色定义

以下是这三个核心角色的详细说明：

*   **爱丽丝**：通常是消息的发送者。她拥有需要安全传输的原始数据（明文），并负责将其加密。
*   **鲍勃**：通常是消息的预期接收者。他负责接收加密后的消息（密文）并将其解密，恢复出原始的明文。
*   **特鲁迪**：代表攻击者或对手。她试图在信道上拦截通信，目的是窃听、篡改或破坏爱丽丝与鲍勃之间的消息。

## 通信过程解析

上一节我们定义了角色，本节中我们来看看典型的安全通信过程是如何进行的。

这个过程可以概括为以下几个步骤：
1.  **准备与加密**：爱丽丝将她的明文消息（`M`）使用加密算法和密钥（`K`）进行加密，生成密文（`C`）。公式可表示为：`C = Encrypt(K, M)`。
2.  **传输**：爱丽丝通过一个可能不安全的信道（如互联网）将密文`C`发送给鲍勃。
3.  **拦截尝试**：特鲁迪可能会在信道上截获密文`C`，但由于没有正确的密钥，她无法理解或有效篡改其内容。
4.  **接收与解密**：鲍勃收到密文`C`后，使用相应的解密算法和密钥（`K‘`，通常与加密密钥`K`配对）进行解密，恢复出原始明文`M`。公式可表示为：`M = Decrypt(K‘, C)`。
