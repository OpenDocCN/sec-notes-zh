在本节课中，我们将要学习数字签名的概念及其推荐用途。数字签名是一种确保数据来源真实性和传输完整性的关键技术。

## 什么是数字签名？

![](img/a9cc425ecd3f6011ff38db972b4dc37b_2.png)

上一节我们介绍了数据完整性的重要性，本节中我们来看看数字签名如何实现这一目标。数字签名是一种密码学技术，用于验证电子文档、消息或软件的真实性和完整性。它确保信息来自可信的来源，并且在传输过程中未被篡改。

数字签名的工作原理基于非对称加密。发送方使用其**私钥**对数据的哈希值进行加密，生成签名。接收方则使用发送方的**公钥**对签名进行解密，并将结果与接收到的数据重新计算出的哈希值进行比对。如果两者一致，则证明数据是完整且真实的。

![](img/a9cc425ecd3f6011ff38db972b4dc37b_4.png)

其核心过程可以用以下公式表示：
**签名生成：** `Signature = Encrypt(Hash(Data), Sender's Private Key)`
**签名验证：** `Decrypt(Signature, Sender's Public Key) == Hash(Received Data)`

## 数字签名的推荐用途

理解了数字签名的基本原理后，以下是数字签名在实践中的一些关键应用场景。

1.  **验证节点间数据交换的完整性**
    在网络通信中，数字签名可以确保服务器与客户端之间、或不同服务器之间交换的数据未被中途修改。

2.  **验证通过网络传输执行的代码**
    许多可执行代码在分发前都会进行数字签名，例如：
    *   Java 小程序（Applets）
    *   JavaScript 库
    *   Android 应用程序
    *   iOS 应用程序
    签名确保了用户下载和运行的代码来自官方开发者，且未被植入恶意软件。

3.  **保护服务包和修复补丁**
    软件供应商发布的服务包（Service Packs）和修复补丁（Fixed Packs）必须使用数字签名。这能防止攻击者篡改更新包，在其中植入恶意代码，从而确保客户安装的是安全、官方的更新。

4.  **保护临时保存在客户机上的数据**
    例如系统备份文件。对这些备份文件进行数字签名，可以在恢复数据时验证其完整性和来源，防止使用被篡改的备份导致系统受损。

## 数字签名的验证是关键

数字签名必须经过验证才能发挥作用。如果创建了数字签名但从不进行验证，那么这个签名就毫无意义。验证是确认数据真实性和完整性的最终步骤，是数字签名流程中不可或缺的一环。
