## 核心安全原则：最小权限原则

上一节我们介绍了本模块的学习目标，本节中我们来看看一个核心的安全原则。

![](img/08a020d639bd2bdf5459a3ce36c87ea1_2.png)

最小权限原则是什么？它为何对组织的安全至关重要？

![](img/08a020d639bd2bdf5459a3ce36c87ea1_3.png)

最小权限原则是指，用户和系统进程只应被授予执行其任务所必需的最低级别的访问权限。这有助于限制潜在攻击面，防止权限滥用，是纵深防御策略的关键组成部分。

## Linux服务器权限管理

了解了基本原则后，我们来看看它在具体系统中的实现。以下是Linux服务器上设置权限管理的关键方法：

*   **角色管理**：通过创建具有特定权限集的用户组（角色）来管理用户访问。
*   **权限委派**：使用如 `sudo` 命令，允许特定用户以提升的权限执行特定命令。
*   **根访问控制**：严格限制对 `root` 超级用户账户的直接访问，通常通过 `sudo` 机制进行控制。

![](img/08a020d639bd2bdf5459a3ce36c87ea1_5.png)

例如，在Linux中授予用户重启服务的权限可以使用以下命令：
```bash
# 在 /etc/sudoers 文件中添加规则
username ALL=(ALL) /bin/systemctl restart service_name
```
