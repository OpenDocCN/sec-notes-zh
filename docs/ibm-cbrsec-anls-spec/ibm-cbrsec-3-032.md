# 课程3：《网络安全合规框架与系统管理》：32：Windows审计概述 🔍

![](img/b9f326b0d934d1c1f835331afb65db7a_0.png)

在本节课中，我们将学习Windows系统中的审计与访问控制。审计是网络安全的一个重要方面，它涉及建立策略并监控对象的创建或修改，以追踪潜在的安全问题。通过审计，我们可以确保用户的责任性，并在发生安全事件时提供证据。

![](img/b9f326b0d934d1c1f835331afb65db7a_2.png)

上一节我们介绍了审计的基本概念，本节中我们来看看Windows系统下审计的具体应用和重要性。

## 审计与用户责任性

当我们谈论用户责任性时，需要了解大约50%的网络安全事件是由组织内部人员引起的。这并不意味着所有内部事件都是恶意行为造成的，很多时候可能是由于用户无意中点击了恶意链接或缺乏相关知识导致的。因此，监控和审计系统上的活动变得至关重要。

## Windows可审计的事件类型

在Windows系统中，有九种主要的事件类型可以被审计。这些事件记录在**安全日志**中，通常通过Windows事件查看器或日志分析工具（如SIEM系统）进行监控和分析。

以下是Windows系统中可审计的九种事件类型：

1.  **账户登录事件**：记录用户登录系统的行为，特别是远程登录。例如，短时间内出现大量失败的登录尝试可能表明存在暴力破解攻击。
2.  **账户管理事件**：审计账户的创建、删除、启用、禁用、重命名或密码更改。短时间内创建大量账户可能是可疑活动。
3.  **目录服务访问**：当有人访问Active Directory服务对象时触发。这些对象有自己的访问控制列表（ACL）。
4.  **登录事件**：记录用户登录或注销计算机的行为，无论是本地登录还是通过网络登录。可以监控登录频率、用户身份和来源IP地址。
5.  **对象访问**：审计对文件、文件夹、打印机或其他Active Directory内对象的访问。可以追踪谁在何时访问了何物。
6.  **策略更改**：跟踪对本地安全策略或用户权限分配的更改尝试。例如，管理员试图降低密码安全设置会被记录。
7.  **特权使用**：当用户执行某项用户权限时被记录。
8.  **进程跟踪**：跟踪端点上的进程活动，例如程序激活或进程退出。例如，用户通过任务管理器停止杀毒软件进程的行为会被记录。
9.  **系统事件**：记录计算机关机、重启，或应用程序/进程尝试执行其无权操作的行为。后者可能是恶意活动的迹象。

## 审计的重要性与实施

审计之所以重要，是因为许多安全事件始于一个文件被投放到系统或被修改。通过监控这些事件，安全团队可以及时发现异常链并采取行动。在Windows环境中，管理员通常使用事件查看器或集中式的日志管理解决方案来实施审计。

本节课中我们一起学习了Windows审计的核心概念。我们了解到审计是确保安全性和追责的关键工具，它通过监控九类特定事件来帮助识别潜在的内部和外部威胁。掌握如何配置和解读这些审计日志，对于任何系统管理员或网络安全分析师来说都是一项基本技能。
