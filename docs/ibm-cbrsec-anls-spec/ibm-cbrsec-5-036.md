在本节课中，我们将要学习渗透测试的基本概念、其重要性、可测试的不同系统类型以及执行测试时可采取的各种方法。

## 什么是渗透测试？🔍

美国国家标准与技术研究院将渗透测试定义为一种安全测试，其中评估人员模拟真实世界的攻击，以识别绕过应用程序、系统或网络安全特性的方法。它通常涉及使用黑客常用的工具和技术，对真实的系统和数据发起真实的攻击。

随着网络攻击成为常态，定期进行漏洞扫描和渗透测试以识别漏洞并确保现有网络安全控制措施有效，变得比以往任何时候都更加重要。由于这是对真实系统的真实攻击，此类测试通常不会每月或每季度进行。许多公司选择每年进行一次，以确保控制措施到位，同时将对业务的影响降至最低。

## 渗透测试的目标系统 💻

随着渗透测试的重要性日益增加，了解其可针对的操作系统也变得同样重要。

在桌面系统中，**Windows OS** 是目前最流行的操作系统，其次是 **Unix**、**Linux** 和 **Mac OS**。虽然也有系统运行 **Chrome OS** 和 **Ubuntu**，但其频率远不及前三种。

在移动设备领域，**Android** 是目前最流行的操作系统，其次是 **iOS** 和 **Blackberry OS**。**Windows Mobile** 较为少见，在非常古老的手机上，你仍可能发现 **Web OS** 和 **Symbian OS**。

## 渗透测试的不同方法 🛠️

上一节我们介绍了渗透测试的定义和目标系统，本节中我们来看看执行渗透测试时可以采取的不同方法。没有两次测试是完全相同的，因为每家公司使用的工具、系统和应用程序都不同，因此我们需要灵活调整方法。

以下是几种主要的测试方法：

*   **内部员工 vs. 外部黑客**：由于80%的攻击发生在公司内部，因此模拟员工或前员工的身份进行测试非常重要。内部人员可能已拥有某些访问权限，这比外部黑客仅使用常见工具能走得更远。
*   **Web或移动应用程序**：这类评估通常包括确保代码安全、符合安全策略，并进行大量的身份验证和密码破解尝试，以测试相关漏洞。
*   **社会工程学**：这是一种在许多方法中都会用到的策略。社会工程学旨在通过制造焦虑感，获取正常情况下无法访问的信息。历史上，我们熟知的社会工程学形式是电子邮件钓鱼攻击，但它也可以通过电话、当面交流进行，通常涉及威胁、最后通牒、虚假信息、制造紧迫感、升级事态等手段，诱使他人泄露信息或获取非常规访问权限。
*   **无线网络**：大多数公司都有供员工使用的内部无线网络，测试其是否符合安全策略至关重要。由于每位员工都连接到该网络，并且他们常会使用自己的设备（如手机或电脑），这些都成为我们需要测试的潜在漏洞点。
*   **物联网设备**：如今，不仅是人，许多设备也始终连接到互联网，如网络摄像头、恒温器甚至咖啡机。因此，网络本身、连接到网络的可能不安全的设备、以及接入网络的物理设备，都是我们可以采取的不同测试角度。
*   **工业控制系统**：这类系统的密码和操作系统通常非常过时，并且极易受到社会工程学攻击，因为它们经常使用默认密码和配置。工业控制系统并非无处不在，主要存在于石油、天然气和电力行业。可以想象，这些行业一旦被攻破将产生广泛影响，因此对其进行测试变得日益重要。

## 渗透测试的通用流程 📋

以上我们探讨了渗透测试的不同方法，现在让我们退一步，看看渗透测试本身通常包含哪些阶段。

正如在渗透测试的初始定义中提到的，测试可以针对应用程序、网络以及移动设备或不同操作系统等系统进行。虽然这些不同的测试方法可能有所差异，但我们使用的方法论大体相同。

一个典型的渗透测试流程通常包含以下阶段：

1.  **规划阶段**：这是初始阶段，你需要与公司商定合同或协议，明确攻击的范围。由于是对真实系统进行攻击，设定严格的边界并明确可能发生和不应发生的事情至关重要。
2.  **侦察/发现阶段**：在此阶段，你需要通过主动和被动方式，尽可能多地收集目标信息。
3.  **利用/攻击阶段**：当你认为信息收集足够时，便进入实际的渗透测试攻击阶段。
4.  **报告阶段**：攻击完成或达成规划阶段设定的目标后，报告对公司采取后续行动至关重要。报告需要明确指出哪些攻击成功了、哪些没有成功、存在哪些漏洞区域以及公司面临的风险。
