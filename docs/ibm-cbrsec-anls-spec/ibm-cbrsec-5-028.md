在本节课中，我们将学习脚本语言的核心概念。我们将探讨脚本是什么、变量如何工作、条件语句以及循环结构。这些是编写自动化任务和工具的基础知识，对于网络安全分析师至关重要。

## 脚本基础

脚本本质上是一个程序。它与传统编译程序的一个基本区别在于，脚本不会被翻译成机器码。脚本依赖于一个解释器程序，该解释器会逐行运行脚本。除此之外，脚本拥有与传统程序相同的功能，例如可以包含过程和外部调用。

外部调用是指引用另一个程序，在本例中是另一个脚本。这个被引用的脚本将从原始脚本中运行，并将其输出添加到原始脚本的输出中。

## 变量与常量

大多数脚本都依赖变量来执行其功能。一个经验法则是，大多数解释器（就像大多数编程语言一样）需要定义其变量。

在像Bash这样的脚本语言中，变量定义可能不是必须的。但在Windows环境中，通常需要声明变量的类型。声明意味着，如果你要存储文本，你需要告诉脚本解释器你正在使用该变量来存储文本。否则，如果你将一个变量用于数字，却为其分配了文本，你将无法对该变量使用任何数学函数，这会导致错误。

为了空间效率，大多数变量类型都有预定义的内存大小。这在大多数编程语言和解释器中都是如此。Bash使用了一种动态内存类型，因此它不使用固定大小，但这属于例外情况。

对于变量，我们可以执行以下操作：
*   赋值
*   引用
*   操作
*   在程序结束时删除变量

## 参数

有一种特殊的变量称为参数。这正是脚本如此有趣的原因。当我们调用一个脚本时，我们可以在命令行中传递参数。例如，我们调用一个名为 `backup` 的脚本，并告诉它“备份这个数据库”。在脚本内部，我们可以将脚本名称之后传入的变量编程为一个参数。参数是我们在脚本执行开始时获得的东西，作为脚本调用的一部分，我们将使用它来执行脚本。它可能是目标，也可能是我们要对目标执行的操作之一。

## 条件语句

`if` 语句是脚本中决策的基本结构。它为我们提供了一个框架，来决定是执行一个操作还是另一个操作。

`if` 语句的标准定义是：
```
if [条件]
then
    # 条件为真时执行的代码
else
    # 条件为假时执行的代码
fi
```
我们首先建立一个条件，其结果为真或假。然后定义当条件为真时要执行的操作，以及当条件为假时要执行的操作（`else` 部分）。最后以 `fi` 结束 `if` 语句。

## 循环结构

在执行重复性任务时，我们会遇到需要重复执行某个任务的情况。有时我们知道需要重复的次数，有时则不知道。

当我们编写脚本时，可能不知道任务会被重复多少次，但有一个变量或变量列表会决定这一点。为此，我们创建循环。

几乎所有编程语言都定义了三种主要循环：

1.  **`for` 循环**：在循环开始时进行评估。它会执行一个预先确定次数或由参数确定的循环。
    定义如下：
    ```
    for 变量 in 序列
    do
        # 要重复执行的代码
    done
    ```

2.  **`while` 循环**：只要条件为真，就会重复执行。例如，可以设定“当传感器被调用时，执行一系列任务来加热传感器”。只要条件为真，这个循环就会一直重复。

3.  **`until` 循环**：这个循环在循环结束时进行评估，以确定某个条件是否为真。实际上，`until` 循环能做的任何事情，`while` 循环也都能做到，因此 `until` 循环的使用已经较少。
