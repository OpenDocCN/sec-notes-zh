# 课程2：《网络安全角色、流程与操作系统安全》：51：真实性与可问责性 🔐

![](img/4ec6dc6cde6b5a39d30187b7f54be21f_0.png)

在本节课中，我们将学习在网络安全背景下“真实性”与“可问责性”的含义。这两个概念是确保数字交互可信和可追溯的基石。

上一节我们探讨了其他安全概念，本节中我们来看看本模块要讨论的最后两个定义：真实性与可问责性。

![](img/4ec6dc6cde6b5a39d30187b7f54be21f_2.png)

## 真实性的定义

真实性是指信息或实体是**真实且可验证**的特性。

具体来说，当爱丽丝向鲍勃发送一条消息时，鲍勃必须能够**证明**这条消息确实来自爱丽丝，符合正确的协议，是预期的内容，并且可以进行验证。

![](img/4ec6dc6cde6b5a39d30187b7f54be21f_4.png)

## 可问责性的定义

可问责性则侧重于**证明来源**。它意味着接收方（如鲍勃）能够证明收到的消息确实来自声称的发送方（如爱丽丝）。

## 在银行环境中的应用

我们可以通过一个银行环境的例子来理解这两个概念的实际应用。

在银行系统中，当对账户执行操作时（例如存款或转账），系统必须能够**绝对地将该操作映射到一个已知的个体**。这同时体现了真实性（操作是真实的、来自合法用户）和可问责性（操作可以追溯到具体的个人）。

本节课中，我们一起学习了网络安全中的“真实性”与“可问责性”。真实性确保信息是真实且可验证的，而可问责性则确保行动可以追溯到其发起者。理解这两个概念对于构建可信和安全的数字系统至关重要。
