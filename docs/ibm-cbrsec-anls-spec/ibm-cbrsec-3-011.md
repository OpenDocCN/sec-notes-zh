在上一节我们介绍了通用数据保护条例，本节中我们来看看一个行业特定的法规。HIPAA，即《健康保险便携和责任法案》，是一项美国联邦法案，专门针对医疗健康信息。

请注意，HIPAA的正确拼写是“H-I-P-A-A”。如果有人拼成“H-I-P-P-A”，那说明他们是这个领域的新手。

如今，医疗机构正利用云服务来节约成本并实现可扩展性。然而，将敏感数据置于云端引发了诸多担忧。这是否是件好事？这取决于风险。因此，理解安全性、了解您合作的云服务提供商，并确保他们理解保护数据完整性与安全的重要性，是至关重要的。当然，客户比以往任何时候都更关注云服务，以提升资源利用率、降低成本并加快响应时间。

HIPAA是美国联邦法律，用于管控个人健康信息，即PHI。它也与另一项名为HITECH的法案相关。

与HIPAA相关的隐私规则明确了个人对其医疗记录和健康信息的权利，并严格限制只有“需要知道”的人才能访问这些信息。该法案适用于健康保险公司、医疗保健提供者以及任何可能访问或需要共享医疗记录的个人或实体。

安全规则则建立了一套保护这些数据的标准。这些标准必须同时适用于“被覆盖实体”和“业务伙伴”。

HIPAA由美国卫生与公众服务部下属的民权办公室定义、管理和监督。该法案主要涉及两类主体：
*   **被覆盖实体**：指为客户管理医疗健康数据的公司，例如医院、保险公司或医生诊所。
*   **业务伙伴**：指支持被覆盖实体的任何供应商。例如，如果您为医院提供应用程序或云环境，那么您就是该被覆盖实体的业务伙伴。

受保护的健康信息是指关于个人健康状况的任何信息。确保其安全性和保密性是被覆盖实体的责任，或者由业务伙伴代表其履行。

在GDPR中我们提到了高额罚款，HIPAA同样对违规行为设有高额罚款。此外，还存在一个“耻辱墙”网站，会公布违规信息。

那么，为什么HIPAA合规至关重要？
1.  这是美国联邦法律，具有强制力。卫生与公众服务部会对业务伙伴或被覆盖实体进行突击审计。
2.  罚款可能高达数百万美元。
3.  可能面临刑事起诉，后果非常严重。

尽管HIPAA是美国法规，但需要注意的是，许多其他国家也有类似法律。例如，加拿大的《个人信息保护和电子文件法》。几乎每个地区都有类似的法律法规。美国许多州还会有比联邦HIPAA法案更严格或额外的要求。因此，在选择业务管辖区域和遵循的合规标准时，需要意识到这一点。一些国际公司也可能要求遵守HIPAA，无论他们是否与美国数据、美国客户有业务往来，或者仅仅视其为一项有价值且可信赖的标准。

HIPAA安全规则涵盖物理实体、技术控制和行政保障，所有措施都聚焦于保护健康信息。它关注**机密性、完整性和可用性**，并要求采取一切合理步骤和行动来预期对信息安全的威胁，防止不当使用和意外披露，并确保全体员工遵守规定。

以下是HIPAA安全规则的主要组成部分：

**行政保障措施**
这些是非技术性或操作性的控制措施。它们关注您的管理流程和人事流程，例如招聘实践、员工培训、背景调查等，旨在确保人员和所执行流程的完整性。

**技术保障措施**
顾名思义，这些是技术性控制。它们关注访问控制、审计控制、完整性控制、传输安全，以及对静态数据、传输中数据和使用中数据的加密。通过物理控制来确保技术控制有效，使软件按预期运行。

**物理保障措施**
这些措施围绕设施访问和设备存放位置展开。例如，数据存储在磁盘上，这些磁盘存放在哪里？是否处于适当的访问控制之下并确保安全？它们也关注访问点，如工作站和设备安全。例如，在医院里，所有放在公共区域办公桌上的电脑，其工作站设备安全就至关重要，需确保这些访问点得到妥善保护，并设置工作站超时锁定等功能，以防他人趁虚而入。
