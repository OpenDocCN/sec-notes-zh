## 概述
在本节课中，我们将要学习密码学的基础知识。我们将从定义核心概念开始，了解密码学如何在不安全的通信渠道上保护数据，并探讨不同的安全属性和威胁模型。本节课是密码学单元的起点，我们将为后续学习具体的加密方案打下坚实的理论基础。

## 从内存安全到密码学
上一节我们结束了内存安全单元的学习。一旦你完成了项目一，你在本课程中就不再需要编写C代码了。今天，我们开始一个全新的单元——密码学。

在内存安全单元，我们讨论了各种内存安全漏洞，它们都源于同一个根本原因：代码没有检查边界。由于不检查边界，你可以向数组末尾之外写入数据，从而引发各种类型的漏洞。我们详细讨论了这些漏洞及其修复方法，包括使用其他内存安全的语言，以及具体的缓解措施，如不可执行页、栈金丝雀、指针认证和地址空间布局随机化。

本节中，我们来看看密码学。这是本课程的四个主要单元中的第二个，我们将学习如何在不安全的信道上实现安全通信。

## 什么是密码学？
密码学是一门研究如何在不安全信道上进行安全通信的学科。随着发展，它的定义已不仅限于保护数据在传输过程中的安全，还包括保证数据在特定攻击者存在的情况下，仍能具备某些安全属性。

需要提醒的是，这是本课程中数学性最强的部分。我们将涉及一些简单的证明和特定的符号。虽然这听起来有些挑战，但我们会引导你逐步理解。此外，这是本课程中唯一一个需要“请勿在家尝试”警告的单元。原因在于，密码学算法非常脆弱，一个微小的错误就可能导致整个安全体系崩溃。在四周的学习中，我们只能让你从高层理解算法的工作原理和保证，但不足以让你自己安全地实现它们。在项目二和现实生活中，你应该使用由专家编写并经过严格审查的密码学库。

理解这些协议的高层工作原理，即使你不亲自编写，也能让你成为一个更好的“消费者”。当有人向你推销新的加密协议时，你将有能力判断它是否基于良好的基本原理。

## 核心角色与定义
为了在讨论密码学时使用共同的语言，我们首先需要定义一些核心角色和概念。

### 通信角色
在密码学中，我们通常使用一些角色来代表通信中的各方：
*   **Alice 和 Bob**： 他们是我们的“英雄”，是试图安全地相互发送消息的好人。
*   **Eve**： 窃听者。她可以读取在信道上发送的任何数据。
*   **Mallory**： 恶意攻击者。她不仅可以读取数据，还可以修改在信道上发送的数据。

根据具体问题的上下文，这些角色的能力可能会有所不同。

### 安全属性
我们希望通过密码学实现以下主要安全属性：
*   **机密性**： 攻击者（如Eve或Mallory）无法读取我们的消息内容。
*   **完整性**： 如果攻击者（如Mallory）修改了消息，接收方能够发现消息已被篡改，从而不会接受被修改的消息。
*   **真实性**： 接收方能够验证消息确实来自声称的发送方（例如，来自Alice）。

在初期，完整性和真实性看起来非常相似，它们都涉及防止攻击者在消息被接收前篡改消息。随着学习的深入，我们会在涉及多于两方的场景中区分它们。

### 密钥模型
密钥是密码学方案的基础，是需要保密的数值。主要有两种密钥模型：
1.  **对称密钥模型**： Alice和Bob共享同一个秘密值，称为对称密钥。只有他们两人知道这个密钥。
2.  **非对称密钥模型**： 每个人都拥有一对密钥：一个私钥（自己保密）和一个公钥（对外公开）。私钥和公钥是数学上相关联的一对。

在本周和下周讨论密钥交换之前，我们假设在对称密钥模型中，Alice和Bob已经通过某种方式神奇地拥有了相同的秘密密钥。

## 柯克霍夫原则
这是一个重要的安全原则，是“通过隐匿实现安全”这一理念在密码学中的具体体现。

**柯克霍夫原则**指出：一个密码系统的安全性必须建立在即使攻击者完全了解系统所有细节（除了密钥本身）的情况下。所有安全性的唯一来源应该是密钥本身。

这意味着，我们不应该依赖攻击者不知道我们使用何种算法来获得安全。因为如果算法被泄露，我们需要设计一个全新的算法，这非常困难。相反，如果安全性仅依赖于密钥，那么当密钥泄露时，我们只需要更换一个新的密钥即可，这要容易得多。在现实中，我们使用的算法通常是公开且标准化的，因此遵循柯克霍夫原则是合理且必要的。

## 机密性与完整性的直观理解
现在，让我们通过物理类比来直观地理解机密性和完整性。

### 机密性（加密）
想象Alice想通过不安全的邮政系统发送一封信给Bob。如果直接寄信，邮差（Eve）可以拆开阅读。

**物理类比**： Alice将信放入一个盒子，用只有她和Bob有的钥匙锁上，然后寄出盒子。Eve拿到盒子但无法打开。Bob收到后用同样的钥匙打开盒子，取出信件。

**计算机实现**： 在计算机中，我们使用**加密算法**来模拟“上锁”。加密算法接收**明文**（原始消息）和**密钥**，输出**密文**（加密后的消息）。然后，我们通过**解密算法**和相同的密钥，将密文恢复为明文。
*   **加密**： `密文 = 加密算法(明文， 密钥)`
*   **解密**： `明文 = 解密算法(密文， 密钥)`

### 完整性（消息认证）
现在，假设我们不在乎Eve是否读信，只在乎Mallory是否篡改了信件。

**物理类比**： Alice将信放入信封，然后用一种特殊的、只有她能制作的胶带密封信封。她寄出密封的信封。如果Mallory想篡改信，她必须破坏这个密封。由于她没有这种特殊胶带，她无法重新制作一个相同的密封。Bob收到后，检查密封是否完好。如果完好，则消息未被篡改。

**计算机实现**： 我们使用一个**标签生成算法**。该算法接收消息和密钥，生成一个唯一的**标签**（或称为消息认证码）。我们将`(消息， 标签)`这对数据一起发送。接收方Bob使用密钥和**验证算法**来检查标签是否与消息匹配。如果不匹配，则说明消息或标签被篡改。
*   **生成标签**： `标签 = 生成算法(消息， 密钥)`
*   **验证**： `结果 = 验证算法(消息， 标签， 密钥)` （结果应为“有效”或“无效”）

## 威胁模型：选择明文攻击
在设计安全系统时，我们必须明确攻击者的能力，即**威胁模型**。对于机密性，一个常见的增强威胁模型是**选择明文攻击**。

在这个模型中，我们赋予攻击者Eve额外的能力：
1.  **CPA 能力**： Eve可以要求Alice加密任何Eve选择的消息，Alice必须使用其真实密钥进行加密并返回密文。
2.  **CCA 能力**： Eve可以要求Bob解密任何Eve选择的密文，Bob必须使用其真实密钥进行解密并返回明文。

我们赋予Eve这些能力，是为了模拟现实世界中可能存在的漏洞。例如，一个网站可能允许用户输入一些内容，并返回其加密结果，这实际上就给了攻击者“选择明文”的能力。

在本课程中，我们将主要关注**选择明文攻击**模型，即Eve拥有CPA能力，但没有CCA能力。这是一个强大且现实的威胁模型。

## 密码学算法地图
以下是我们在接下来几周将要学习的密码学算法概览，它可以帮助我们根据需求选择算法：

| | **对称密钥** (Alice和Bob密钥相同) | **非对称密钥** (每人拥有公私钥对) |
| :--- | :--- | :--- |
| **机密性** (防止读取) | 流密码、分组密码（如AES） | 公钥加密（如RSA加密） |
| **完整性/真实性** (防止篡改/验证来源) | 消息认证码（如HMAC） | 数字签名（如RSA签名） |

选择算法时，你需要考虑两个轴：
1.  **密钥模型**： 你处于对称密钥环境还是非对称密钥环境？
2.  **安全需求**： 你主要需要机密性，还是完整性/真实性，或者两者都需要？

## 对称加密的形式化定义与安全性
本节课的剩余部分，我们将专注于地图的**左上角**：对称密钥模型下的机密性。

### 形式化定义
一个对称加密方案由三个算法定义：
1.  **密钥生成**： 产生密钥`K`。（目前我们假设密钥已神奇存在）
2.  **加密算法**： 接收明文`M`和密钥`K`，输出密文`C`。`C = Enc(K, M)`
3.  **解密算法**： 接收密文`C`和密钥`K`，输出明文`M`。`M = Dec(K, C)`

**正确性要求**： 对于任何密钥`K`和明文`M`，必须有 `Dec(K, Enc(K, M)) = M`。

### 定义“机密性”
仅仅说“攻击者无法读取消息”是不够精确的。我们需要一个严谨的、可证明的安全性定义。我们通过一个**挑战游戏**来定义**在选择明文攻击下的不可区分性**。

**游戏过程**：
1.  **学习阶段**： 攻击者Eve可以反复要求Alice加密任何Eve选择的消息`M_i`，并获得密文`C_i = Enc(K, M_i)`。
2.  **挑战阶段**： Eve选择两个**长度相等**的明文`M0`和`M1`，发送给Alice。
    *   Alice随机抛硬币，得到结果`b`（0或1，概率各50%）。
    *   Alice加密`M_b`，得到挑战密文`C* = Enc(K, M_b)`，并将其发送给Eve。
3.  **猜测阶段**： Eve可以继续要求加密其他消息（但不能直接要求解密`C*`）。最终，Eve输出一个猜测`b‘`，试图猜中`b`。

**安全性定义**： 如果对于**所有**多项式时间的攻击者策略，Eve猜中`b`的概率**非常接近**50%（即不比随机猜测好多少），那么我们就说这个加密方案是“CPA安全”的。

**定义中的关键点**：
*   **消息等长**： 要求`M0`和`M1`等长，是因为我们通常允许加密方案泄露消息的长度（否则效率极低）。如果不要求等长，Eve可以通过比较密文长度轻松赢得游戏。
*   **多项式时间**： 我们只关心在现实可行时间内（多项式时间）能完成的攻击。对于需要指数时间（如万亿年）才能完成的攻击，我们认为不构成实际威胁。
*   “非常接近”50%： 允许Eve有极其微小的优势（可忽略优势），只要这个优势小到无法被利用。

这个游戏化的定义非常强大，因为它不假设攻击者使用何种具体策略。如果我们能证明一个方案满足这个定义，那么它就安全地抵抗了所有符合该威胁模型的攻击者。

## 古典密码示例
最后，我们快速回顾一些古典密码，以理解上述概念。

### 凯撒密码
**加密**： 将每个字母在字母表中向后移动固定的位置（如移动3位，A->D, B->E）。
**破解**：
*   **暴力破解**： 尝试所有可能的偏移量（只有26种）。
*   **选择明文攻击**： 让Alice加密“AAA”，如果得到“DDD”，则立刻知道偏移量是3。

### 单表替换密码
**加密**： 使用一个随机生成的字母替换表（如A->N, B->Z）。
**破解**：
*   **暴力破解**： 可能的替换表数量巨大（26!），暴力破解不可行。
*   **选择明文攻击**： 让Alice加密完整的字母表“ABC...Z”，即可直接获得整个替换表（密钥）。

### 恩尼格玛密码机
这是二战时期德国使用的机械加密设备。它的教训深刻：
1.  **违反了柯克霍夫原则**： 德国人认为盟军不知道机器的内部结构，但盟军通过缴获获得了机器。
2.  **已知明文攻击**： 盟军知道德军每天早晨会发送格式固定的天气预报。
3.  **选择明文攻击**： 盟军通过布设水雷等方式，诱使德军在通信中提及特定地名，从而获得已知明密文对。

这些历史案例表明，即使复杂的系统，如果设计时没有考虑严谨的威胁模型（如CPA），也容易被攻破。
