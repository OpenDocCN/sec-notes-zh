## 概述
在本节课中，我们将学习几种低层网络协议及其相关的安全攻击。我们将从项目二的常见错误提醒开始，然后深入探讨ARP、DHCP和Wi-Fi（WPA2）协议的工作原理、存在的安全漏洞以及可能的防御措施。最后，我们会简要介绍网络层（IP）和BGP协议的基本概念与攻击方式。本节课的核心目标是理解这些基础协议为何容易受到攻击，以及攻击者如何利用这些漏洞升级其攻击能力。

## 项目二提醒：撤销访问权限的规则

上一节我们介绍了网络攻击的背景，本节我们先来看一个与课程项目相关的常见问题，以确保大家在实现时不会犯错。以下是关于`revoke_access`功能的两条关键要求，理解它们可以简化你的项目设计。

*   **唯一有权撤销文件访问权限的人是文件的所有者。** 我们永远不会在非文件所有者的用户上下文中调用`revoke`。如果你的代码在这种情况下崩溃或报错，都是可以接受的，因为我们不会测试这种情况。
*   **所有者只能撤销其直接共享对象的访问权限。** 如果你选择撤销某个文件的访问权限，你只能撤销你直接共享给的人。此外，如果你撤销了某个直接共享对象的权限，那么该对象所共享的所有下游用户也将失去访问权限。

为了更清晰地说明，请看以下示例图：
```
用户A (所有者)
    ├── 直接共享给 B
    │       └── B 共享给 D
    │               └── D 共享给 F
    └── 直接共享给 C
            └── C 共享给 E
                    └── E 共享给 G
```

根据规则：
*   只有用户A可以调用`revoke`。如果B或E调用`revoke`，行为是未定义的。
*   用户A只能对B和C调用`revoke`。我们永远不会测试A对D或F调用`revoke`的情况。
*   如果A撤销了C的权限，那么C和G都会失去访问权限。
*   如果A撤销了B的权限，那么B、D、E和F都会失去访问权限。

**重要补充**：当A撤销了C的权限后，B、D、E、F必须仍然能够继续使用该文件，**无需重新接受邀请**。从他们的视角看，访问权限未受影响。

这些规则通过限制`revoke`的调用者和目标，简化了项目的设计范围。请务必牢记。

## ARP协议与ARP欺骗攻击

现在，让我们回到低层网络协议。首先回顾一下地址解析协议。ARP存在的根本原因是，网络中的计算机同时拥有**IP地址**（第三层，用于全球寻址）和**MAC地址**（第二层，用于物理硬件标识）。ARP的作用就是在已知目标IP地址时，查询其对应的MAC地址。

协议流程很简单：
1.  主机A在本地区域网络内**广播**一个请求：“谁拥有IP地址 `X`？请告诉我你的MAC地址。”
2.  拥有该IP地址的主机B（如果在同一网络）会直接回复：“我是`X`，我的MAC地址是`Y`。”
3.  如果目标不在本地网络，路由器会回复自己的MAC地址，以便代为转发。

### ARP欺骗攻击
攻击就发生在这个简单的过程中。

1.  **漏洞**：ARP协议没有加密或身份验证机制。主机A无法验证回复消息的真实性。
2.  **攻击步骤**：
    *   当主机A广播ARP请求时，攻击者M（必须在同一本地网络）可以抢先回复一个**虚假的**ARP响应，声称：“IP地址 `X` 对应的MAC地址是 `Z`”（`Z`是攻击者自己的MAC地址）。
    *   由于A通常只接受第一个到达的回复（存在**竞态条件**），它会将这个映射缓存起来。
3.  **后果**：此后，当A想发送数据给IP地址 `X` 时，它会根据ARP缓存，将数据发往MAC地址 `Z`，也就是攻击者M。这样，M就成功介入了A与B的通信。

### 攻击者能力升级
*   **攻击前提**：攻击者必须是一个**在线路径攻击者**，即位于同一本地网络，能够监听到ARP广播请求。
*   **攻击效果**：成功实施ARP欺骗后，攻击者将自己升级为**中间人攻击者**。它不仅可以读取原本发给B的数据，还可以**篡改**、**丢弃**这些数据，然后再转发给B（或不转发）。

### 防御措施
完全杜绝ARP欺骗很困难，因为协议本身过于简单。但有一些缓解措施：
*   **使用交换机**：现代网络常用交换机作为中心设备。主机会先询问交换机，交换机维护着自己的ARP缓存。这减少了广播请求的次数，从而降低了攻击机会。
*   **虚拟局域网**：交换机可以将一个大的物理局域网划分为多个逻辑上隔离的VLAN，限制广播域的范围。
*   **监控工具**：使用如`arpwatch`等工具监控网络，当检测到异常（如一个IP对应多个MAC）时发出警报。

然而，这些都不是万无一失的防御。真正可靠的解决方案需要更高层的协议引入密码学机制，我们将在后续课程中看到。

## DHCP协议与攻击

动态主机配置协议用于主机首次加入网络时自动获取配置信息。当你连接到一个新网络（如Wi-Fi）时，你需要知道三件事：
1.  你自己的IP地址。
2.  本地路由器的IP地址。
3.  DNS服务器的IP地址。

DHCP协议通过四次握手来分配这些信息：
1.  **发现**：客户端广播“我需要配置信息”。
2.  **提供**：网络中的DHCP服务器（可能不止一个）回复“我可以为你提供这些配置：IP是`A`，路由器是`B`，DNS是`C`”。
3.  **请求**：客户端广播“我选择服务器X提供的配置”。
4.  **确认**：被选中的服务器发送最终确认，并保留该IP地址的分配。

### DHCP攻击
DHCP与ARP共享相同的核心漏洞：

1.  **漏洞**：客户端无法验证“提供”消息的来源是否合法。协议没有身份验证。
2.  **攻击步骤**：攻击者M可以伪装成DHCP服务器，在客户端广播“发现”消息后，抢先发送一个**恶意的“提供”消息**。
3.  **恶意配置示例**：
    *   将**默认网关**设置为攻击者自己的IP。这样，客户端所有发往外部网络的数据都会先经过攻击者。
    *   将**DNS服务器**设置为攻击者自己的IP。这样，客户端的域名查询会被攻击者控制（DNS攻击后续会讲）。
4.  **竞态条件**：同样，客户端通常接受第一个到达的“提供”消息。

### 攻击者能力升级
*   **攻击前提**：同样需要是**在线路径攻击者**，位于同一本地网络。
*   **攻击效果**：成功实施后，攻击者升级为**中间人攻击者**。例如，通过成为虚假网关，它可以拦截、篡改所有进出本地网络的数据。

### 防御措施
在协议层防御DHCP欺骗非常困难，因为主机初次加入网络时没有可信的基准。真正的防御同样需要依赖更高层的安全协议（如后续会学的TLS）来建立安全通道。目前，我们只能接受这种风险存在于网络底层。

## Wi-Fi安全与WPA2协议

Wi-Fi协议用于无线局域网连接。每个Wi-Fi网络有一个访问点、一个网络名称和一个密码（对于安全网络）。WPA2是保护Wi-Fi通信安全的密码学协议。

### WPA2四次握手（简化概念版）
目标是：让拥有密码的客户端安全地接入网络，并生成加密密钥，同时确保没有密码的人无法加入或窃听。

1.  **预共享密钥**：客户端和访问点都使用Wi-Fi密码，通过一个确定性算法（如哈希函数）独立生成一个**预共享密钥**。
2.  **交换随机数**：双方交换随机数。
3.  **生成成对传输密钥**：双方使用**预共享密钥**、交换的**随机数**以及双方的**MAC地址**，通过一个密钥派生函数，独立生成相同的**成对传输密钥**。此密钥用于加密客户端与访问点之间的单播通信。
4.  **验证与组密钥分发**：双方使用**成对传输密钥**生成并验证消息完整性码，确保握手过程未被篡改。最后，访问点将**组密钥**加密后发送给客户端。组密钥用于广播通信，网络内所有用户共享。

**优化版**：实际使用的WPA2握手对上述步骤进行了优化，将消息交换从6次减少到4次，但密码学本质不变。

### WPA2相关攻击

1.  **恶意访问点攻击**
    *   **前提**：攻击者知道Wi-Fi密码。
    *   **攻击**：攻击者伪装成合法的访问点。当客户端尝试连接时，与攻击者完成握手。
    *   **后果**：攻击者成为**中间人**，客户端的所有流量都经过它。

2.  **离线暴力破解攻击**
    *   **前提**：攻击者不知道密码，但可以**窃听**一次合法的握手过程（记录下交换的随机数和消息完整性码）。
    *   **攻击**：攻击者无需与访问点交互。他只需对密码进行猜测，然后用猜测的密码重复握手过程中的密钥派生计算。如果计算得到的消息完整性码与窃听到的匹配，则猜中了密码。
    *   **关键**：这是**离线**攻击，不受访问点速率限制，只受攻击者计算能力和密码强度的限制。

3.  **缺乏前向安全性**
    *   **问题**：如果攻击者现在记录了握手过程和加密的通信数据，未来一旦通过某种方式获得了Wi-Fi密码，他就可以用密码推导出**成对传输密钥**，从而解密过去记录的所有通信。
    *   WPA2不具备**前向安全性**。

### 防御：WPA3与WPA企业版
根本问题在于，所有安全性都源于一个共享的弱秘密——Wi-Fi密码。解决方案是改用WPA企业版（或更新的WPA3）。
*   **核心改进**：每个用户使用独立的用户名和密码进行认证。
*   **工作原理**：客户端通过一个安全的认证协议（如TLS，我们将在后面学习）连接到认证服务器。验证成功后，服务器分发一个一次性的、高强度的**预共享密钥**给客户端，用于后续的WPA握手。
*   **如何防御上述攻击**：
    *   **恶意AP**：TLS协议能认证服务器身份，防止冒充。
    *   **离线暴力破解**：预共享密钥是随机生成的高强度密钥，无法暴力破解。
    *   **前向安全性**：一次一密的预共享密钥保证了前向安全。

## 网络层与BGP简介

最后，我们快速上升到网络层。互联网协议是连接全球网络的第三层协议。

### 基本概念
*   **IP地址**：设备的逻辑地址，包含路由信息。IPv4为32位，IPv6为128位。
*   **路由**：数据包从源到目的地可能需要经过多个路由器（跳）。
*   **自治系统**：一个由单一机构管理的大型网络或网络集合，可以理解为一个“超级”本地网络。
*   **BGP协议**：边界网关协议，用于在自治系统之间交换路由信息，决定数据包在全球互联网中的传递路径。

### BGP劫持攻击
*   **攻击**：一个恶意的自治系统可以向其邻居**谎称**“我可以到达某个IP地址范围”。
*   **后果**：其他自治系统信以为真，就会将目的地为该范围的数据流量发送给这个恶意系统。攻击者从而能够拦截大量流量，成为**中间人**。

### IP地址欺骗及其防御
*   **攻击**：发送IP数据包时，伪造源IP地址。这通常是其他攻击（如DDoS）的基础。
*   **防御**：理论上，每个自治系统应该在边界路由器上检查进入的数据包，确保源IP地址是合理的（例如，来自其内部网络）。这称为**入口过滤**。但实际上，并非所有网络都部署了此措施。

## 总结

本节课我们一起学习了多种低层网络协议及其安全挑战：
1.  **ARP与DHCP**：由于缺乏身份验证和加密，且基于广播和竞态条件，两者都容易遭受欺骗攻击，使在线路径攻击者升级为中间人。
2.  **WPA2**：通过四次握手为Wi-Fi提供安全连接，但仍面临恶意AP、离线暴力破解和缺乏前向安全性的问题。WPA企业版通过引入每用户认证和一次性密钥来解决这些问题。
3.  **网络层**：IP协议负责全球路由，BGP协议管理自治系统间的路径。BGP劫持和IP欺骗是这一层的主要威胁，防御依赖于各个网络的自律部署。

![](img/722313aaeca767dd72bf82ea76168595_3.png)

一个贯穿始终的主题是：在网络的底层，由于协议设计的简单性和对效率的追求，**中间人攻击是一个必须接受的现实**。这些协议本身无法提供强大的安全保证。我们的希望在于更高层的协议，它们将运用密码学武器来对抗这些底层威胁。我们将在接下来的课程中探讨这些解决方案。
