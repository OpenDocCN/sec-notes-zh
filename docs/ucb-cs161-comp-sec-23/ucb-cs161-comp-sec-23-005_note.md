# 005：缓解内存安全漏洞

![](img/1672c3ee381fa3440197e1acc4f1215e_1.png)

在本节课中，我们将学习如何防御内存安全漏洞，以及攻击者如何绕过这些防御措施。完成本节课后，你将掌握完成项目一所需的所有知识。

## 概述：为何内存安全问题持续存在

上一节我们介绍了内存安全漏洞的基本原理。本节中，我们来看看为什么这些问题难以根除，以及防御的总体思路。

一种根本的解决方法是使用不会出现此类问题的编程语言。在C语言中，所有问题都源于它不进行边界检查。例如，如果你有一个大小为5的数组，却请求第6个元素，C语言不会阻止你。它没有边界的概念。

以下是几种防御思路：

*   **使用更安全的编程语言**：几乎所有现代语言（如Java、Python、C#、Go、Rust）都会进行边界检查。如果你尝试越界访问，程序会崩溃并报错，而不是像C语言那样允许越界并可能导致严重后果。这是唯一能100%阻止漏洞的方法。
*   **编程时极度谨慎**：理论上，你可以通过仔细编程来确保内存安全，例如总是检查空指针和边界，使用安全的库函数（如`strncpy`而非`strcpy`）。但这非常困难且繁琐，因为人类会犯错。
*   **使用自动化工具**：存在一些工具可以在程序运行时监控其行为，检测异常活动（如意外调用`execve`函数）。还有模糊测试（Fuzz Testing）等自动化方法，通过向程序输入大量随机数据来发现可能导致崩溃或越界写入的输入。例如，Valgrind就是一个检测内存泄漏和越界写入的工具。
*   **管理依赖库的安全**：即使你自己的C代码是安全的，你导入的第三方库也可能存在漏洞。软件需要不断更新以修补新发现的安全漏洞，这增加了确保整个软件栈安全的复杂性。

然而，现实中我们常常面对大量遗留的C代码，重写整个代码库或为庞大程序编写形式化证明通常不切实际。因此，本节课的重点不是完全阻止漏洞，而是探讨如何通过一些通用的**缓解措施**来增加攻击者的攻击难度。

## 缓解措施一：使攻击者难以执行注入的代码

上一节我们讨论了防御的总体哲学。本节中，我们来看看第一种具体缓解措施：如何阻止攻击者执行他们写入内存的恶意代码。

许多内存安全攻击遵循一个标准流程：
1.  发现漏洞。
2.  将Shellcode写入内存。
3.  覆盖返回指令指针（RIP），使其指向Shellcode的地址。
4.  当函数返回时，开始执行Shellcode。

我们的目标是让攻击者难以完成这些步骤。首先，我们针对**步骤5（执行Shellcode）** 进行防御。

### 不可执行页

核心思想是：我们可以为内存的不同区域设置权限位，规定哪些区域可写，哪些区域可执行。
*   **栈、堆、静态数据区**：这些区域通常需要存储和修改数据（即可写），但其中的数据不应被当作指令来执行。因此，我们可以将这些区域的**执行权限**关闭。
*   **代码区**：这里存放着程序的指令，需要**执行权限**。通常，编译后的代码在运行时不需要修改，因此可以将其**写权限**关闭。

操作系统通过虚拟内存机制提供了设置这些权限位（如“不可执行”位）的能力。这种防御措施有多个名称，如W^X（可写异或可执行）、NX位（不可执行位）等。

**公式/概念**：`W^X` - 内存页要么可写（W），要么可执行（X），但不能同时具备两者。

这种防御如何阻止经典缓冲区溢出攻击？在经典攻击中，攻击者将Shellcode写入栈（可写区域），然后使RIP指向该地址。如果栈被标记为不可执行，那么当程序尝试跳转到栈上的地址并执行时，操作系统会阻止并导致程序崩溃。

### 攻击者的绕过方法：返回到Libc

攻击者无法执行自己写入的代码，但他们可以尝试利用程序中已经存在的、具有执行权限的代码。一个常见的目标是C标准库（libc）中的函数，例如`system`函数。

`system`函数接收一个字符串参数，并在shell中执行该字符串。如果攻击者能控制传递给`system`的参数，他们就能执行任意命令。

攻击思路变为：
1.  利用缓冲区溢出漏洞覆盖RIP。
2.  将RIP的值覆盖为`system`函数在内存中的地址（而非Shellcode的地址）。
3.  同时，精心构造栈上的数据，使得当执行流跳转到`system`时，栈的布局恰好符合`system`函数对参数和返回地址的预期（通常需要在覆盖的数据中插入一个伪造的返回地址和真正的命令字符串地址）。

这样，攻击者就成功利用了已有的代码（`system`）来达到目的，而无需执行自己注入的代码。

### 更强大的绕过：面向返回的编程

攻击者还可以进行更精细的攻击，称为“面向返回的编程”。他们不再跳转到完整的函数，而是寻找程序中一系列以`ret`指令结尾的小代码片段（称为“gadget”）。

攻击思路是：
1.  覆盖RIP，使其指向第一个gadget的地址。
2.  该gadget执行一些有用操作（如设置寄存器）后，执行`ret`指令。
3.  `ret`指令会从栈中取出下一个值作为地址并跳转。
4.  攻击者提前在栈上布置好一连串gadget的地址。这样，通过连续执行`ret`指令，程序就会像“编程”一样，按顺序执行一系列gadget，最终组合成复杂的恶意功能。

这种方法允许攻击者在无法注入代码的情况下，复用程序中已有的指令片段来构建攻击逻辑。

## 缓解措施二：使攻击者难以覆盖关键指针

上一节我们介绍了如何阻止代码执行。本节中，我们来看看如何让攻击者更难成功覆盖返回地址等关键指针。

我们针对**步骤3（覆盖RIP）** 进行防御。许多漏洞（如基于`gets`的溢出）涉及从某个局部变量开始，连续向上覆写内存，直到覆盖RIP。栈金丝雀旨在检测这种连续的、未经授权的内存修改。

### 栈金丝雀

其灵感来源于矿坑中的金丝雀。矿工将金丝雀带入矿坑，如果有毒气积聚，金丝雀会先倒下，从而警示矿工危险。

在栈帧中，我们在**返回地址（RIP）和保存的帧指针（SFP）之下、局部变量之上**插入一个随机值，即“金丝雀”值。这个值在函数开始时被设置，在函数返回前被检查。

**工作流程**：
1.  **函数序言**：将随机生成的金丝雀值放入栈上的特定位置。
2.  **函数执行**：金丝雀值应保持不变。
3.  **函数尾声**：在恢复寄存器并返回之前，检查栈上的金丝雀值是否与原始值相同。
4.  **检测到篡改**：如果金丝雀值被改变（例如，被向上的缓冲区溢出所覆盖），则程序立即崩溃，而不是继续执行可能已被篡改的返回地址。

这使得那种需要连续、无间隔向上覆写内存才能到达RIP的攻击变得困难，因为攻击者必然会先改变金丝雀值从而触发崩溃。

**栈金丝雀的特性**：
*   每次程序运行时值都不同（防止预测）。
*   通常包含一个空字节（`\x00`），这可以意外地阻止一些基于字符串的漏洞利用，因为字符串函数遇到空字节会停止。

### 攻击者的绕过方法

栈金丝雀并非无懈可击：
*   **信息泄露**：如果攻击者能通过其他漏洞（如格式化字符串漏洞）**泄露**出金丝雀的具体值，那么他们在构造溢出时就可以在对应位置写入正确的金丝雀值，从而通过检查。
*   **非连续写入**：如果漏洞允许攻击者任意地址写（如某些格式化字符串漏洞），他们可以**绕过**金丝雀直接修改RIP，而不触动金丝雀。
*   **暴力破解**：理论上可以尝试猜测金丝雀值，但由于其随机性且程序可能崩溃，在实际中通常不可行。

### 更强大的变体：指针认证码

这是一种更新、更强大的防御，主要用于64位ARM架构。它利用了64位地址中高位通常为0的特点。

**核心思想**：
1.  当将一个地址（如返回地址）存入内存（栈）时，编译器/硬件会取出该地址中未使用的高位（零），并用一个加密生成的**指针认证码**替换它们。
2.  当后来使用该地址（如通过`ret`指令）时，硬件会检查附带的PAC是否有效。
3.  **如果有效**：将PAC位替换回0，然后跳转到该地址。
4.  **如果无效（被篡改）**：触发错误，程序崩溃。

与单一的金丝雀值不同，PAC可以为**不同的地址生成不同的认证码**，安全性更高。这需要密码学中消息认证码的支持，我们将在后续课程中详细讨论。

## 缓解措施三：使攻击者难以定位目标地址

上一节我们通过金丝雀增加了覆盖指针的难度。本节中，我们来看看最后一种缓解措施，它旨在破坏攻击链条的**步骤2（定位Shellcode地址）**。

攻击者需要知道他们注入的Shellcode（或要跳转的gadget）在内存中的确切地址，才能用这个地址覆盖RIP。地址空间布局随机化通过让这个地址变得不可预测来增加攻击难度。

### 地址空间布局随机化

在简单的内存布局中，栈、堆、库等的地址是固定或可预测的。ASLR会在**每次程序运行时**，随机化这些内存区域（栈、堆、共享库、代码段等）的**起始地址**。

**关键点**：
*   **随机化的是基址**：一个段（如栈）内部的布局和相对偏移保持不变。例如，栈帧中局部变量相对于栈顶的位置关系不变。
*   **增加猜测难度**：攻击者无法再假设`/bin/sh`字符串或`system`函数位于某个固定地址。他们需要先**泄露**出一个地址，然后根据相对偏移进行计算，才能确定目标地址。

### 攻击者的绕过方法

*   **信息泄露**：这是最主要的方法。利用另一个漏洞（如格式化字符串漏洞、缓冲区溢出读）来**泄露**某个已知的地址（如某个函数的地址、栈上的某个值）。一旦获得一个“锚点”地址，攻击者就可以根据已知的代码偏移或栈布局，计算出他们所需的目标地址。
*   **暴力破解**：在ASLR熵（随机化范围）不足的情况下，理论上可以尝试多次攻击来猜测地址。但这通常不切实际，因为每次尝试失败都可能导致程序崩溃，且现代系统的地址空间非常大。

### 组合防御与NOP雪橇

单独使用任何一种缓解措施都可能被绕过。但**组合使用**它们（如ASLR + NX + 栈金丝雀）可以产生协同效应，极大地增加攻击复杂度。攻击者可能需要同时利用多个不同类型的漏洞才能成功。

最后，介绍一个在漏洞利用中常用的技巧：**NOP雪橇**。当攻击者不确定Shellcode的确切起始地址时（例如由于ASLR），他们可以在Shellcode前面填充大量无操作指令。

**代码示例**：
```
[大量NOP指令] + [实际的Shellcode]
```

这样，只要RIP跳转到NOP雪橇范围内的任何一个地址，处理器都会执行一系列NOP（什么都不做），然后“滑行”到Shellcode并开始执行。这**扩大了成功的命中范围**，降低了攻击对地址精度的要求。

## 总结

![](img/1672c3ee381fa3440197e1acc4f1215e_3.png)

本节课我们一起学习了缓解内存安全漏洞的多种技术：

![](img/1672c3ee381fa3440197e1acc4f1215e_5.png)

1.  **不可执行页**：通过设置内存权限，阻止攻击者执行其注入到可写区域的代码。攻击者可能通过“返回到Libc”或“面向返回的编程”来绕过。
2.  **栈金丝雀**：在栈上插入随机值，检测连续的缓冲区溢出，在返回地址被覆盖前使程序崩溃。可通过信息泄露或非连续写入来绕过。
3.  **地址空间布局随机化**：随机化内存布局，使攻击者难以预测关键地址。主要依靠信息泄露来绕过。
4.  **组合使用**：同时启用多种缓解措施可以显著提高防御能力。
5.  **实用技巧**：NOP雪橇可用于提高漏洞利用的鲁棒性。

![](img/1672c3ee381fa3440197e1acc4f1215e_7.png)

这些缓解措施不能保证绝对安全，但能有效增加攻击者的成本和难度，是构建深度防御策略的重要组成部分。
