# 023：拒绝服务攻击与防火墙 🔥

![](img/c1c160d08f6730fb27cedaa398c00709_0.png)

在本节课中，我们将学习两个重要的计算机安全主题：拒绝服务攻击和防火墙。我们将探讨攻击者如何通过耗尽资源或使服务崩溃来阻止合法用户访问服务，以及如何通过防火墙来保护网络免受外部攻击。

---

## 拒绝服务攻击概述

拒绝服务攻击旨在破坏服务的可用性，使合法用户无法访问。攻击者可能出于各种动机，如经济利益、政治目的或恶意竞争。攻击主要分为两类：使运行服务的程序崩溃，或耗尽系统资源。

上一节我们介绍了拒绝服务攻击的基本概念，本节中我们来看看具体的攻击方式。

### 攻击向量分类

攻击可以从两个维度进行分类：攻击的层次和攻击的目标。

*   **攻击层次**：可分为应用层攻击和网络层攻击。应用层攻击针对运行服务的程序或计算机资源；网络层攻击则针对底层的网络协议。
*   **攻击目标**：可分为使程序崩溃和资源耗尽。前者利用程序漏洞使其停止运行；后者通过消耗计算、内存或带宽等有限资源来使服务不可用。

### 应用层攻击

应用层攻击直接针对运行服务的应用程序或其主机资源。这类攻击的一个关键思想是**不对称性**：攻击者希望以极小的代价，诱使目标服务器消耗大量资源。

以下是攻击者可能试图耗尽的资源类型示例：

*   **耗尽文件系统空间**：向磁盘写入大量垃圾数据。
*   **耗尽内存**：迫使程序分配大量内存，可能导致崩溃。
*   **耗尽线程/进程数**：创建大量并行进程，耗尽系统处理能力。
*   **耗尽文件描述符**：同时打开大量文件。

#### 算法复杂度攻击

这是一种特殊的应用层攻击。攻击者通过提供精心构造的输入，迫使目标算法进入最坏情况下的时间复杂度运行，从而消耗大量计算资源。

例如，**快速排序**在随机输入下平均时间复杂度为 `O(n log n)`，但在已排序（或逆序）的输入下，其最坏情况时间复杂度为 `O(n^2)`。如果一项排序服务使用了快速排序算法，攻击者只需提交一个已排序的数组，就能使其运行速度急剧下降。相比之下，**归并排序**在任何输入下的时间复杂度都是稳定的 `O(n log n)`，因此在面对此类攻击时更为安全。

### 防御应用层攻击

防御此类攻击的第一步是**检测攻击**。这可能通过要求用户认证，并监控异常请求模式来实现。但需注意，认证服务本身也可能成为攻击目标。

以下是其他防御策略：

*   **隔离与配额**：将不同用户的服务隔离开，并为每个用户设置资源使用上限，防止单个恶意用户影响整个服务。
*   **信任机制**：只允许经过验证的受信任用户执行高资源消耗的操作。
*   **工作量证明**：要求用户在发起请求前先完成一项计算任务（如验证码），增加攻击者的成本，降低攻击的不对称性。
*   **资源过载**：直接购买远超日常需求的巨大计算和带宽资源。虽然有效，但成本高昂，需根据威胁模型权衡。
*   **内容分发网络**：类似于保险机制，众多网站将资源汇集到一个大池中（如Cloudflare）。当某个成员遭受攻击时，CDN可以调动池中的大量资源来帮助抵御。

## 网络层攻击

上一节我们讨论了针对应用程序本身的攻击，本节中我们来看看如何攻击底层的网络协议。

网络层攻击旨在耗尽目标的网络资源，例如带宽或数据包处理能力。

### 分布式拒绝服务攻击

DDoS攻击是网络层攻击的主要形式。攻击者控制大量计算机（通常通过僵尸网络），同时向一个目标发起洪水般的请求。

DDoS攻击之所以危险且难以防御，原因在于：
1.  **攻击力强大**：攻击者汇聚了众多系统的资源。
2.  **来源分散**：流量来自四面八方，难以通过简单的源地址过滤来阻挡。

### 放大攻击

放大攻击利用了某些网络协议中请求小、响应大的**不对称性**。攻击者通过伪造源地址，将一个小请求发送给一个具有强大响应能力的服务器（如DNS服务器），诱使该服务器向受害者发送一个巨大的响应包。

**DNS放大攻击**是典型例子：
1.  攻击者向DNS服务器发送一个小的DNS查询数据包，但**伪造源IP地址为受害者的地址**。
2.  DNS服务器收到查询后，会向“请求者”（即受害者）发送一个包含大量记录的大响应包。
3.  攻击者仅付出发送一个小包的代价，就使受害者收到了巨大的流量冲击。

这种攻击不仅放大了攻击效果，还能隐藏攻击者的真实身份。

### 防御网络层攻击

防御网络层攻击的主要手段是**数据包过滤**。防火墙或专用设备检查每个入站数据包，根据规则决定是放行还是丢弃。

然而，数据包过滤面临挑战：
*   **难以区分**：在DDoS攻击中，恶意流量与合法流量混杂，难以精确识别。
*   **自身可能被攻击**：过滤逻辑如果过于复杂，其本身可能成为资源耗尽攻击的目标。
*   **欺骗攻击**：攻击者伪造源地址，使基于源地址的过滤失效。

与之前一样，**资源过载**（购买超大带宽）和使用**CDN**也是有效的终极防御手段。

## SYN洪水攻击与SYN Cookie防御

这是一个针对TCP协议的具体网络层攻击实例。

### SYN洪水攻击原理

TCP通过三次握手建立连接。在SYN洪水攻击中：
1.  攻击者向目标服务器发送大量TCP SYN包（握手第一步）。
2.  服务器每收到一个SYN包，都必须为这个“即将建立”的连接分配一些内存资源，并回复SYN-ACK包（握手第二步）。
3.  攻击者**不完成第三次握手**（不发送ACK）。服务器会等待一段时间，期间为这些“半开连接”保留资源。
4.  如果攻击者发送的SYN包速度足够快，服务器内存将被耗尽，无法为合法用户建立新连接。

攻击的**不对称性**在于：攻击者只需发送小SYN包，无需保存连接状态；服务器却必须为每个SYN包分配并保存资源。

### SYN Cookie防御机制

SYN Cookie是一种巧妙的防御技术，它消除了上述不对称性。其核心思想是：**服务器在收到SYN包时不立即分配内存，而是将必要的连接状态信息编码后“塞回”给客户端**。

具体流程（简化）：
1.  服务器收到SYN包后，不分配内存。
2.  服务器生成一个特殊的**初始序列号**（即SYN Cookie），这个数字由连接信息（如源/目的IP、端口、时间戳）通过哈希计算得出。
3.  服务器在SYN-ACK包中使用这个特殊序列号回复客户端。
4.  **合法的客户端**会在ACK包中返回这个序列号（加1）。
5.  服务器收到ACK后，验证该序列号是否有效（即重新计算哈希看是否匹配）。如果有效，则说明这是一个合法的连接请求，此时才分配内存正式建立连接。

对于SYN洪水攻击，攻击者不会回复ACK，或者无法回复正确的序列号，因此服务器从一开始就没有消耗任何内存资源。

## 防火墙 🔥

防火墙是位于内部可信网络和外部不可信网络之间的安全屏障。其核心安全原则是**完全中介**：所有进出内部网络的流量都必须经过防火墙的检查。

### 防火墙策略

防火墙根据一系列规则来决定是允许还是拒绝数据包通过。一个常见的策略模型是：
*   **出站流量（内部 -> 外部）**：默认允许。因为假设内部用户是可信的。
*   **入站流量（外部 -> 内部）**：默认拒绝，但允许例外。
    *   **允许例外1**：允许外部主机回复由内部发起的连接（例如，允许TCP ACK标志置位的包）。
    *   **允许例外2**：允许访问内部特定的公共服务（如IP地址为X.X.X.X的Web服务器）。

### 默认允许 vs. 默认拒绝

制定规则时，需要确定默认行为：
*   **默认允许（黑名单）**：列出所有不允许的事情，其他都允许。**用户体验好**（新应用默认可用），但**安全性较低**（未知威胁可能被放行）。
*   **默认拒绝（白名单）**：列出所有允许的事情，其他都拒绝。**安全性高**，但**用户体验差**（新应用需要手动配置规则）。

### 数据包过滤器类型

*   **无状态包过滤**：独立检查每个数据包，不关心数据包之间的关联。实现简单，但容易被绕过（例如，将恶意指令拆分到多个包中发送）。
*   **有状态包过滤**：跟踪网络连接的状态（如TCP连接）。能够理解数据包之间的关系，更智能，可以防御拆分攻击，但实现更复杂，消耗更多资源。

### 绕过防火墙的技术

攻击者会尝试各种方法绕过过滤规则：
1.  **数据包分割**：将恶意负载（如“root”命令）拆分到多个TCP数据包中发送。无状态过滤器单独检查每个包时无法发现威胁。
2.  **数据包乱序**：以乱序发送分割后的数据包，增加重组和识别的难度。
3.  **利用TTL**：精心设置数据包的生存时间，使得包含恶意内容的数据包在到达防火墙时因TTL耗尽被丢弃，而包含正常内容的数据包则能到达内部主机。防火墙看到的是混乱或无害的信息，而内部主机收到的却是完整的攻击载荷。

### 代理防火墙

为了应对复杂的绕过技术，出现了代理防火墙。它不作为单纯的过滤器，而是作为**中间人**：
1.  外部客户端不与内部服务器直接建立连接，而是与代理防火墙建立连接。
2.  代理防火墙代表客户端与内部服务器建立另一个独立的连接。
3.  代理防火墙可以深度检查应用层内容（如HTTP请求），再决定是否转发。

这种方式提供了更细粒度的控制，但会引入额外的延迟，并且需要代理支持特定的应用协议。

### 虚拟专用网络

VPN用于在不可信的网络（如互联网）上创建一个安全的“隧道”，让用户（如在外的员工）能够像在内部网络一样安全地访问公司资源。VPN流量是加密的，防火墙通常无法检查其内容。因此，VPN可以用于合法地绕过防火墙限制（访问内部资源），也可能被滥用来绕过安全策略。

### 防火墙的优缺点

**优点**：
*   **集中管理与扩展性**：通过单一控制点保护整个网络，易于更新策略。
*   **透明性**：可以明确告知用户网络访问规则。
*   **弥补终端漏洞**：为内部可能存在漏洞的主机提供一道外部防线。

**缺点**：
*   **内部威胁**：如果攻击者已经进入内部网络，防火墙将形同虚设。
*   **性能影响**：检查数据包会带来延迟。
*   **配置复杂性**：制定全面而准确的过滤规则非常困难。
*   **可能被绕过**：如前面所述，存在多种技术可以规避过滤。

## 课程总结 🎯

本节课我们一起学习了两个重要的安全主题。

**关于拒绝服务攻击**：
我们了解了攻击者如何通过**使程序崩溃**或**耗尽资源**来破坏服务的可用性。攻击可以在**应用层**（如算法复杂度攻击）或**网络层**（如DDoS、放大攻击）进行。防御手段包括检测、配额、工作量证明、资源过载以及使用CDN。我们深入分析了**SYN洪水攻击**及其防御方案**SYN Cookie**，该方案通过让客户端先保存状态，消除了攻击的不对称性。

**关于防火墙**：
我们探讨了防火墙作为网络边界守卫者的角色。它根据**安全策略**（如默认拒绝或默认允许）过滤流量。防火墙可以是**无状态**或**有状态**的，后者能提供更好的保护但更复杂。攻击者会使用**数据包分割、乱序和TTL操纵**等技术尝试绕过防火墙。**代理防火墙**和**VPN**是更高级或用于特定场景的网络访问控制方案。防火墙提供了集中化的安全管理和对终端主机的保护，但其有效性依赖于“内部可信、外部不可信”的威胁模型假设，并且无法防御内部威胁。
