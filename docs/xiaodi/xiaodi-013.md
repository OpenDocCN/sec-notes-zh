### 2. 常见源码泄漏途径

![](img/86e3942920d58359a6a2b379034a5f74_106.png)

由于管理员配置不当或开发习惯问题，可能导致源码泄漏。以下是几种常见情况：

#### 🔸 网站备份文件泄漏
管理员可能将网站源码压缩备份后，错误地存放在Web目录下。

**检测方法：**
使用目录扫描工具（如御剑、Dirsearch）扫描常见备份文件名，例如：
*   `www.zip`
*   `web.rar`
*   `site.tar.gz`

**示例命令：**
```bash
python3 dirsearch.py -u https://target.com -e zip,rar,tar,gz
```

![](img/86e3942920d58359a6a2b379034a5f74_108.png)

![](img/86e3942920d58359a6a2b379034a5f74_110.png)

![](img/86e3942920d58359a6a2b379034a5f74_112.png)

#### 🔸 GIT源码泄漏
开发中使用Git进行版本控制，如果上线时未删除 `.git` 目录，可能导致整个代码仓库被还原。

![](img/86e3942920d58359a6a2b379034a5f74_114.png)

**检测方法：**
访问 `https://target.com/.git/`，如果返回403（目录存在）而非404，则可能存在泄漏。

![](img/86e3942920d58359a6a2b379034a5f74_116.png)

![](img/86e3942920d58359a6a2b379034a5f74_118.png)

![](img/86e3942920d58359a6a2b379034a5f74_120.png)

![](img/86e3942920d58359a6a2b379034a5f74_122.png)

**利用工具：**
使用 `GitHack` 等工具可尝试还原源码。
```bash
python2 GitHack.py https://target.com/.git/
```

![](img/86e3942920d58359a6a2b379034a5f74_124.png)

#### 🔸 SVN源码泄漏
与GIT类似，Subversion（SVN）版本控制系统也可能遗留 `.svn` 目录。

#### 🔸 DS_Store文件泄漏
在macOS系统中，`DS_Store`文件用于存储文件夹属性。如果此文件被上传到服务器，可能泄露目录结构。

**检测方法：**
访问 `https://target.com/.DS_Store`。

### 1. 利用特征文件在代码托管平台搜索
分析目标网站加载的静态资源（如JS、CSS文件），寻找具有唯一性的文件名或路径，然后在GitHub、Gitee等平台搜索。

![](img/86e3942920d58359a6a2b379034a5f74_230.png)

![](img/86e3942920d58359a6a2b379034a5f74_232.png)

**操作流程：**
1.  使用浏览器开发者工具（Network面板）分析网站加载的静态文件。
2.  选取非通用、具有项目特征的文件名（如 `zb_system/function/c_system_version.php`）。
3.  在GitHub等平台搜索该文件名或路径。
4.  在搜索结果中寻找与目标网站匹配的源码仓库。

![](img/86e3942920d58359a6a2b379034a5f74_234.png)

![](img/86e3942920d58359a6a2b379034a5f74_236.png)

*   开发者邮箱（如 `*@qq.com`）
*   作者姓名
