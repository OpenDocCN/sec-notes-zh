# CTF夺旗赛教程：16：文件上传漏洞与一句话木马（上） 🚩

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_1.png)

在本节课中，我们将要学习CTF比赛中一个非常常见的漏洞类型——文件上传漏洞。我们将重点探讨如何利用此漏洞上传“一句话木马”，并演示其强大的控制能力。通过本课，你将理解文件上传功能的安全风险以及攻击的基本原理。

## 文件上传功能与安全风险

上一节我们介绍了课程概述，本节中我们来看看什么是文件上传功能及其潜在风险。

许多网站都具备文件上传功能。例如，用户上传QQ头像、网站头像，或者在电子邮件中发送附件，都属于文件上传的范畴。

如果网站在文件上传过程中，没有对上传文件的安全性进行充分校验，就会产生安全漏洞。正常情况下，上传文件时，服务器会生成一个临时文件，并对其进行合规性判断（例如，检查上传的是否为真实的图片文件）。判断成功则保存，失败则丢弃。

但若缺乏限制，攻击者就可能上传恶意文件，例如木马。

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_3.png)

## 一句话木马与大马

在了解漏洞利用方式前，我们需要先认识两种常见的Web木马：“一句话木马”和“大马”。

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_5.png)

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_6.png)

*   **一句话木马**：代码量通常在一行左右，不超过十行。其特点是隐蔽性强，但功能需要通过客户端连接工具来扩展。
*   **大马**：代码量通常很大，功能完善且强大，但正因为代码量大，其隐蔽性较弱，容易被安全防护系统发现。

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_8.png)

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_9.png)

本节课我们主要演示“一句话木马”。

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_11.png)

## 一句话木马示例与连接

以下是创建一个简单PHP一句话木马的示例。其核心是一段极短的代码，其中包含一个连接密码。

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_13.png)

```php
<?php @eval($_POST[‘GOK’]); ?>
```
在这段代码中，`GOK` 即为连接时使用的密码。

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_15.png)

将包含此代码的文件（例如 `eval.php`）上传到目标服务器的网站根目录下后，虽然通过浏览器直接访问该文件可能看不到任何内容，但它已经在服务器上运行。

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_17.png)

要利用这个木马，我们需要使用专用的连接工具，例如“中国菜刀”或“C刀”。以下是使用此类工具的基本步骤：

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_19.png)

1.  打开连接工具（如C刀）。
2.  右键添加一个新的连接。
3.  填写木马文件的完整URL地址。
4.  填写木马文件中设定的密码（本例中为 `GOK`）。
5.  选择正确的脚本类型（本例为PHP）和字符编码（通常为GB2312或GBK以支持中文）。

连接成功后，攻击者便可以通过工具对服务器进行多种操作。

## 一句话木马的强大功能

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_21.png)

成功连接一句话木马后，攻击者可以获得对服务器的强大控制权。以下是其主要功能：

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_23.png)

*   **文件管理**：可以浏览、下载、上传、删除服务器上的文件。
*   **虚拟终端**：可以在服务器上执行系统命令，例如查看网络配置(`ipconfig`)、关闭系统(`shutdown`)等。
*   **数据库管理**：如果服务器存在数据库，可以像利用SQL注入一样，查看和管理数据库内容。

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_25.png)

例如，攻击者可以通过文件管理功能，定位并下载服务器桌面上的一个名为 `flag.txt` 的文件，从而窃取关键信息。

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_27.png)

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_29.png)

理论讲完后，我们进入实战环节，看看如何在实际存在漏洞的网站中利用文件上传功能。

首先，我们尝试直接上传我们的一句话木马文件 `eval.php`。上传成功后，平台会返回文件的访问路径。

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_41.png)

然后，我们便可以在C刀工具中，使用此路径和密码添加连接，并成功获得服务器控制权，这证明了该上传点存在漏洞。

## 前端校验与绕过

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_47.png)

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_49.png)

然而，并非所有上传点都如此简单。很多网站会实施安全校验。一种常见的方式是**前端校验**。

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_51.png)

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_53.png)

当我们尝试上传一个非允许格式（如 `.php`）的文件时，浏览器可能会立刻弹出警告，提示“只允许上传特定格式”。这种校验是由浏览器端的JavaScript代码完成的，其特点是**拦截请求发生在页面发出之前**，因此使用抓包工具（如Burp Suite）拦截时，可能看不到这个上传请求被发送到服务器。

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_55.png)

**重要特征**：仅在前端进行的校验是极不安全的，因为它可以被轻松绕过。例如，攻击者可以禁用浏览器JavaScript，或使用抓包工具修改请求，将文件后缀名改为允许的类型（如 `.jpg`），然后在数据包中再将内容改回恶意代码。

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_57.png)

![](img/002ad5db6ecaf20e6b353e5f1609bdbc_59.png)
