# 网络安全面试突击：P18：应急与响应 - SSH被爆破解决办法 🔐

在本节课中，我们将学习当服务器SSH服务遭遇暴力破解攻击时，应如何系统地进行分析、响应和加固。这是一个在网络安全面试和实际运维中都非常常见的问题。

---

## 概述

SSH（Secure Shell）是一种用于远程登录和管理计算机的网络协议，它通过加密通道保证通信安全。然而，一旦SSH服务被攻击者发现并尝试暴力破解，就可能危及服务器安全。本节教程将分步讲解从发现到解决SSH爆破攻击的全过程。

---

## 第一步：进行日志分析 📊

上一节我们介绍了SSH的基本概念，本节中我们来看看如何通过日志发现攻击痕迹。日志是系统活动的记录，任何爆破尝试都会留下入侵记录，这些记录可能包含攻击者的IP地址。

以下是分析日志的关键步骤：

1.  **实时监控与分析**
    首先，我们需要实时监控SSH的认证日志，以便及时发现异常。在Linux系统中，通常可以使用 `tail` 命令实时查看日志文件。
    ```bash
    tail -f /var/log/auth.log
    # 或者对于使用systemd-journald的系统
    journalctl -f -u ssh
    ```

2.  **统计登录失败的IP**
    攻击者会使用密码字典进行大量尝试。我们可以通过分析日志，找出尝试次数最多的IP地址。
    ```bash
    grep "Failed password" /var/log/auth.log | awk '{print $(NF-3)}' | sort | uniq -c | sort -nr
    ```
    **命令解释**：
    *   `grep "Failed password"`：在日志中检索包含“Failed password”的行（即密码错误的记录）。
    *   `awk '{print $(NF-3)}'`：打印出这些行中倒数第三个字段，通常是IP地址。
    *   `sort`：对IP地址进行排序。
    *   `uniq -c`：统计每个不重复IP出现的次数。
    *   `sort -nr`：按尝试次数从高到低进行排序。

3.  **审查成功登录的IP**
    列出所有成功登录的IP地址，并审查其是否属于授权用户或可信IP范围。对于异常的成功登录，需要进一步分析其在系统内的操作行为，以便溯源。
    ```bash
    grep "Accepted password" /var/log/auth.log | awk '{print $11}' | sort | uniq
    ```

## 第二步：检查用户安全性 👤

在通过日志定位可疑IP后，我们需要检查系统用户账户是否安全，攻击者可能会创建后门账户或提升现有账户权限。

以下是检查用户安全性的方法：

1.  **查看用户登录历史**
    使用 `last` 命令查看所有用户的登录历史记录，寻找异常的时间、IP或用户。
    ```bash
    last
    ```

2.  **检查系统用户与权限**
    重点检查 `/etc/passwd` 和 `/etc/group` 文件，查看是否有新增的未知用户，特别是UID为0（root权限）的用户。
    ```bash
    # 查看/etc/passwd文件的后10行，新添加的用户通常会在末尾
    tail -n 10 /etc/passwd
    # 查看具有root权限的用户
    grep ‘:0:’ /etc/passwd
    ```

---

## 第三步：确定攻击影响与清理 🛡️

确认攻击情况后，我们需要立即采取措施阻断攻击、评估损失并清理后门。

以下是关键的响应步骤：

1.  **隔离异常IP**
    通过防火墙规则封禁可疑IP地址，阻止其继续访问。
    ```bash
    # 使用iptables封禁IP（例如：192.168.1.100）
    iptables -A INPUT -s 192.168.1.100 -j DROP
    # 使用firewalld封禁IP
    firewall-cmd --permanent --add-rich-rule=“rule family=‘ipv4’ source address=‘192.168.1.100’ reject”
    firewall-cmd --reload
    ```

2.  **检查内网安全性**
    确认受攻击的服务器是否被用作“跳板机”去攻击内网其他机器。可以检查内网流量或分析服务器上的其他连接。
    ```bash
    # 查看当前网络连接
    netstat -antp
    # 或使用ss命令
    ss -tunap
    ```

3.  **检查后门与恶意软件**
    使用杀毒软件（如ClamAV）或Rootkit检查工具（如rkhunter、chkrootkit）对服务器进行全面扫描，清除可能植入的后门程序。

---

## 第四步：加固SSH服务 ⚙️

处理完紧急事件后，必须对SSH服务进行加固，防止同类攻击再次发生。

以下是加固SSH的推荐措施：

1.  **修改默认端口**
    SSH默认使用22端口，将其修改为一个非标准的高位端口（如 2345）可以减少自动化扫描攻击。
    *   编辑SSH配置文件：`/etc/ssh/sshd_config`
    *   找到 `#Port 22`，修改为 `Port 2345`（去掉注释）。
    *   **重要**：修改后必须重启SSH服务。
        ```bash
        systemctl restart sshd
        ```
    *   同时需在防火墙中放行新端口。

2.  **限制访问来源**
    在防火墙中设置规则，仅允许特定的、可信的IP地址或IP段访问SSH端口。
    ```bash
    # 例如，仅允许192.168.1.0/24网段访问2345端口
    iptables -A INPUT -p tcp -s 192.168.1.0/24 --dport 2345 -j ACCEPT
    iptables -A INPUT -p tcp --dport 2345 -j DROP
    ```

3.  **使用密钥认证**
    禁用密码登录，强制使用SSH密钥对进行认证，这是最有效的防爆破方式。
    *   在 `/etc/ssh/sshd_config` 中设置：
        ```
        PasswordAuthentication no
        PubkeyAuthentication yes
        ```
    *   重启SSH服务。

---

## 总结

本节课中我们一起学习了应对SSH暴力破解攻击的完整流程：

1.  **日志分析**：通过监控和统计日志，定位攻击源IP。
2.  **用户检查**：审查登录历史和系统用户，发现可疑账户。
3.  **攻击处置**：隔离IP、检查内网扩散、查杀后门恶意软件。
4.  **服务加固**：修改默认端口、限制访问IP、改用密钥认证，从根本上提升SSH安全性。

遵循以上步骤，可以有效地响应SSH爆破攻击并加固服务器，保障系统安全。
