# 网络安全入门教程：P3：数据库字段、表操作 🗄️

![](img/fae3b31df21c90ce6356086f02a77c41_0.png)

在本节课中，我们将学习数据库的核心操作：字段和表的创建、删除以及数据的增删改查。这是理解数据库结构并进行后续SQL注入学习的基础。

上一节我们介绍了如何创建数据库，但创建出的数据库是空的。为了在数据库中存储信息，我们需要创建表和定义字段。本节中我们来看看如何具体操作。

## 创建数据表

创建数据表需要明确三个要素：**表名**、**表字段名**以及每个字段的**数据类型**。创建表主要有两种方法。

**方法一：使用图形化界面新建**
在数据库管理工具中，可以通过右键点击数据库，选择“新建表”来创建。

**方法二：使用SQL命令创建**
使用 `CREATE TABLE` 语句是更直接和通用的方法。其基本语法结构如下：

```sql
CREATE TABLE 表名 (
    字段名1 数据类型 [约束条件],
    字段名2 数据类型 [约束条件],
    ...
);
```

例如，创建一个名为 `user` 的用户表：

```sql
CREATE TABLE user (
    ID INT NOT NULL PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL
);
```

代码解释：
*   `INT`：整数类型。
*   `VARCHAR(50)`：可变长度字符串，最大50个字符。
*   `NOT NULL`：约束条件，表示该字段不能为空。
*   `PRIMARY KEY`：约束条件，表示该字段是主键。

## 删除数据表

当不再需要某个表时，可以将其删除。删除操作同样有两种方式，主要区别在于对不存在的表的处理。

**方法一：直接删除**
使用 `DROP TABLE 表名;` 命令。如果表不存在，系统会报错。

**方法二：安全删除**
使用 `DROP TABLE IF EXISTS 表名;` 命令。这条命令会先检查表是否存在，如果存在则删除，如果不存在则什么都不做，避免报错。

图形化操作同样简单：在工具中右键点击要删除的表，选择“删除”即可。

## 数据操作：增删改查

数据库的核心操作可以概括为“增删改查”（CRUD）。我们已经学习了“增”（创建表）和“删”（删除表），现在来看如何向表中“增”（插入）数据和“查”（查询）数据。

### 插入数据

使用 `INSERT INTO` 语句向表中添加新记录。

以下是插入数据的语法和示例：
```sql
INSERT INTO 表名 (字段1, 字段2, ...) VALUES (值1, 值2, ...);
```

例如，向 `user` 表插入一条数据：
```sql
INSERT INTO user (ID, username, email) VALUES (1, ‘test‘, ‘test@qq.com‘);
```
注意：字符串类型的值需要用**单引号**括起来。

### 查询数据

使用 `SELECT` 语句从表中检索数据。`*` 号表示选择所有字段。

以下是查询数据的语法和示例：
```sql
SELECT * FROM 表名;
```

例如，查询 `user` 表中的所有数据：
```sql
SELECT * FROM user;
```
执行后，将返回表中所有记录，包括我们刚刚插入的 `ID` 为1，`username` 为 ‘test‘ 的数据。

## 实战演练

现在，让我们将理论付诸实践，在数据库中完整地走一遍流程。

**第一步：创建数据库**
首先，我们创建一个名为 `test` 的数据库。
```sql
CREATE DATABASE test;
```

**第二步：创建数据表**
接着，在 `test` 数据库中创建一个 `user` 表。
```sql
USE test;
CREATE TABLE user (
    ID INT NOT NULL PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL
);
```

**第三步：插入数据**
然后，向 `user` 表中插入一条用户记录。
```sql
INSERT INTO user (ID, username, email) VALUES (1, ‘test‘, ‘test@qq.com‘);
```

**第四步：查询验证**
最后，查询表内容，确认数据已成功插入。
```sql
SELECT * FROM user;
```
此时，你应该能看到刚刚插入的数据行。

**第五步：扩展练习**
为了加深记忆，请尝试：
1.  再插入几条不同ID和用户名的数据。
2.  创建一个结构不同的新表（如 `product` 表）。
3.  使用 `DROP TABLE` 命令删除你创建的新表。

多动手练习是掌握这些命令的关键。图形化工具能辅助操作，但熟练书写SQL命令对后续学习网络安全（如SQL注入）至关重要。

## 总结

本节课中我们一起学习了数据库表的基本操作。我们掌握了如何使用 `CREATE TABLE` 和 `DROP TABLE` 来创建和删除表，并重点练习了使用 `INSERT INTO` 插入数据以及使用 `SELECT` 查询数据。这些“增删改查”操作是操作任何数据库的基石。

请大家务必在课后反复练习这些命令，做到熟记于心。扎实的数据库基础将是后续学习SQL注入等技术的重要前提。
