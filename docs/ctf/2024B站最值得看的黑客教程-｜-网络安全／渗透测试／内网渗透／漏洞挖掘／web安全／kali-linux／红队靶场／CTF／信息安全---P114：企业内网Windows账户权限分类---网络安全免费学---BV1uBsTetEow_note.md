# 网络安全：P114：企业内网Windows账户权限分类 🔐

在本节课中，我们将要学习Windows操作系统在企业内网环境中的账户权限分类。理解这些权限等级是进行渗透测试、权限提升（提权）和安全评估的基础。我们将详细解析四种核心权限类型，并通过实际操作演示它们之间的区别。

### 1. 本地普通用户权限

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_19.png)

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_21.png)

本地普通用户是指在Windows系统中没有管理员权限的账户。这类用户能执行的操作非常有限，通常只能进行基本的文件操作和运行非特权应用程序。

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_23.png)

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_25.png)

以下是本地普通用户的主要特征：
*   账户通常位于计算机管理工具中的 **Users** 组内。
*   无法执行系统级操作，如安装软件、修改系统设置、创建新用户等。
*   在渗透测试中，仅获得此类权限通常价值不大，操作空间极小。

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_27.png)

为了更直观地理解，我们登录一个名为“百里”的普通用户账户进行演示。打开命令提示符（CMD），尝试执行添加用户的命令：
```cmd
net user testuser Password123! /add
```
系统会返回“拒绝访问”的错误提示，这证实了该用户权限不足。

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_29.png)

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_30.png)

### 2. 本地普通管理员权限

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_32.png)

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_33.png)

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_34.png)

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_35.png)

在理解了普通用户的限制后，我们来看一种更常见但仍有约束的权限——本地普通管理员权限。大部分个人电脑的日常使用账户都属于此类。

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_37.png)

以下是关于本地普通管理员的关键信息：
*   账户位于 **Administrators** 组中，但并非该组内的最高权限账户。
*   虽然名为“管理员”，但在执行某些高危操作（如添加用户、修改系统文件）时，会受到**用户账户控制（UAC）** 的拦截，需要额外确认。
*   这是Windows为了提高系统安全性而引入的机制，旨在防止恶意软件在用户不知情的情况下获得完全控制权。

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_39.png)

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_40.png)

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_41.png)

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_42.png)

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_43.png)

我们切换到一个名为“oldmin”的普通管理员账户进行测试。再次执行添加用户的命令：
```cmd
net user testuser2 Password123! /add
```
尽管该账户在Administrators组中，命令依然可能被“拒绝访问”。这是因为操作触发了UAC，而当前上下文权限并未提升。

### 3. 本地最高管理员权限

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_49.png)

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_50.png)

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_52.png)

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_53.png)

既然普通管理员权限受限，那么渗透测试的终极目标之一就是获取本地最高管理员权限。这个权限通常对应着系统内置的 **Administrator** 账户。

以下是本地最高管理员权限的核心要点：
*   这是 **Administrators** 组中权限最高的账户，在Windows专业版/企业版中默认启用，在家庭版中可能默认禁用。
*   拥有此权限意味着可以无限制地执行任何操作，包括修改所有文件、安装驱动、创建用户、更改安全策略等，且通常不受UAC弹窗打扰。
*   获取此权限等同于完全控制该台计算机。

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_55.png)

我们可以通过以下命令检查Administrator账户的状态：
```cmd
net user Administrator
```
若账户启用，会显示“帐户启用 Yes”。我们启用并登录该账户后，再次执行添加用户命令，将会成功执行，没有任何“拒绝访问”的提示。

### 4. 本地服务用户权限

除了用于交互登录的用户账户，Windows还有一类用于运行系统服务和后台进程的账户，其中权限最高的是 **SYSTEM** 账户。

以下是本地服务用户权限，特别是SYSTEM账户的说明：
*   **SYSTEM** 是Windows操作系统本身使用的账户，用于运行核心系统进程和服务。它不能用于交互式登录（如桌面登录）。
*   在权限层级上，**SYSTEM** 账户与 **Administrator** 账户被认为是**同权**的。它们分别掌管不同领域（用户交互 vs. 系统服务），但权限高度相当，并且可以相互切换。
*   在渗透测试中，获取 **SYSTEM** 权限与获取 **Administrator** 权限具有同等价值，都意味着已完全掌控目标主机。

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_57.png)

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_58.png)

通过任务管理器查看进程，可以发现许多关键系统进程（如 `csrss.exe`, `services.exe`）的用户名都是 **SYSTEM**，这体现了其运行底层服务的角色。

## 核心概念总结与对比

本节课中我们一起学习了Windows内网环境下的四种核心账户权限。为了更清晰地理解它们之间的关系和目标，我们进行如下总结：

*   **权限提升目标**：在渗透测试中，我们的核心目标是摆脱**本地普通用户**或受UAC限制的**本地普通管理员**权限，最终获得**本地最高管理员（Administrator）** 或**本地服务最高权限（SYSTEM）** 账户的控制权。后两者权限等价，是系统完全控制的标志。
*   **UAC的作用**：用户账户控制（UAC）是Windows重要的安全机制，它在**本地普通管理员**尝试执行特权操作时进行拦截和提示，有效防止了恶意软件的自动提权。这也是普通管理员与最高管理员的关键行为区别。
*   **权限关系公式**：
    `完全控制系统权限 ≈ 获取 Administrator 权限 ≈ 获取 SYSTEM 权限 > 受UAC限制的普通管理员权限 > 普通用户权限`

![](img/4b2db27af9fb5d5fb58c217562ba4b0d_60.png)

理解这些权限分类及其差异，是后续学习令牌窃取、UAC绕过、服务提权等具体技术的前提。只有明确了目标，才能有效地选择和使用正确的提权方法。
