## 文件操作与隐写

上一节我们概述了杂项题目的特点，本节中我们来看看最基础的文件操作与隐写。

![](img/c4403960e668662504e2a7f3b35cedf0_6.png)

### 文件类型识别

处理文件的第一步是识别其类型。有时出题人会故意删除文件的后缀名，以增加难度。我们需要通过文件内容本身来判断其类型。

![](img/c4403960e668662504e2a7f3b35cedf0_8.png)

![](img/c4403960e668662504e2a7f3b35cedf0_10.png)

在Linux环境下，可以使用 `file` 命令来识别文件类型。该命令通过分析文件的头部信息（即文件头）来判断其格式。

![](img/c4403960e668662504e2a7f3b35cedf0_12.png)

![](img/c4403960e668662504e2a7f3b35cedf0_14.png)

![](img/c4403960e668662504e2a7f3b35cedf0_16.png)

**命令示例：**
```bash
file 文件名
```

![](img/c4403960e668662504e2a7f3b35cedf0_18.png)

![](img/c4403960e668662504e2a7f3b35cedf0_20.png)

执行此命令后，系统会返回该文件的类型信息，例如JPEG图像、PNG图像或ZIP压缩包等。

### 十六进制编辑器

工具识别文件类型的原理是解析文件头。不同的文件格式有特定的文件头标识。我们可以使用十六进制编辑器手动查看和编辑文件的原始数据。

![](img/c4403960e668662504e2a7f3b35cedf0_22.png)

![](img/c4403960e668662504e2a7f3b35cedf0_24.png)

![](img/c4403960e668662504e2a7f3b35cedf0_26.png)

以下是三个常用的十六进制编辑器工具：
*   **WinHex**：功能强大的十六进制编辑器。
*   **010 Editor**：另一款优秀的十六进制编辑器，推荐使用。
*   **Notepad++**：配合`Hex Editor`插件，也可以查看和编辑十六进制数据。

![](img/c4403960e668662504e2a7f3b35cedf0_28.png)

使用这些工具打开文件，可以查看其十六进制表示。文件头通常位于文件的开头部分。通过对比已知的文件头标识，我们可以手动判断或修复文件类型。

![](img/c4403960e668662504e2a7f3b35cedf0_30.png)

### 文件头修复

![](img/c4403960e668662504e2a7f3b35cedf0_32.png)

有时题目会故意破坏或错误设置文件头，导致文件无法正常打开或识别。这时需要进行文件头修复。

![](img/c4403960e668662504e2a7f3b35cedf0_34.png)

![](img/c4403960e668662504e2a7f3b35cedf0_36.png)

![](img/c4403960e668662504e2a7f3b35cedf0_38.png)

修复步骤如下：
1.  使用 `file` 命令查看，可能会显示为`data`文件。
2.  用十六进制编辑器（如010 Editor）打开该文件。
3.  根据文件应有的正确格式，查找或计算其标准的文件头。
4.  在文件开头相应位置，补充或修改为正确的文件头数据。
5.  保存文件后，即可正常识别和打开。

![](img/c4403960e668662504e2a7f3b35cedf0_40.png)

![](img/c4403960e668662504e2a7f3b35cedf0_41.png)

### 文件分离

![](img/c4403960e668662504e2a7f3b35cedf0_43.png)

一个文件中可能隐藏（合并）了其他文件。例如，一张图片的后半部分可能是一个压缩包。我们需要将隐藏的文件分离出来。

![](img/c4403960e668662504e2a7f3b35cedf0_45.png)

以下是几种文件分离的方法：

**方法一：使用 `binwalk` 工具**
`binwalk` 可以自动分析文件中嵌入的其他文件数据，并进行分离。

**命令示例：**
```bash
# 分析文件
binwalk 文件名
# 自动分离文件
binwalk -e 文件名
```

**方法二：使用 `foremost` 工具**
`foremost` 是另一个文件分离工具，有时在 `binwalk` 无效时可以尝试使用。

![](img/c4403960e668662504e2a7f3b35cedf0_47.png)

![](img/c4403960e668662504e2a7f3b35cedf0_49.png)

**命令示例：**
```bash
foremost -o 输出目录 文件名
```

**方法三：使用 `dd` 命令**
对于更复杂的、交错隐藏的文件，可以使用 `dd` 命令进行手动精确分离。这需要知道目标数据在文件中的起始偏移量和大小。

**命令示例：**
```bash
dd if=输入文件 of=输出文件 bs=1 skip=起始偏移 count=数据大小
```
其中：
*   `if`：输入文件
*   `of`：输出文件
*   `bs`：块大小（通常设为1字节）
*   `skip`：跳过输入文件开头多少字节后开始复制
*   `count`：要复制的块数

**方法四：手动选择保存**
在十六进制编辑器中，直接选中目标数据的十六进制区域，然后将其另存为一个新文件。

### 文件合并

与文件分离相反，有时题目会将一个完整文件分割成多个片段给出，需要我们将它们合并还原。

![](img/c4403960e668662504e2a7f3b35cedf0_51.png)

在Linux下，可以使用 `cat` 命令合并文件。

![](img/c4403960e668662504e2a7f3b35cedf0_53.png)

![](img/c4403960e668662504e2a7f3b35cedf0_55.png)

**命令示例：**
```bash
cat 文件1 文件2 文件3 ... > 合并后的文件名
```

![](img/c4403960e668662504e2a7f3b35cedf0_57.png)

![](img/c4403960e668662504e2a7f3b35cedf0_59.png)

![](img/c4403960e668662504e2a7f3b35cedf0_61.png)

在Windows下，可以使用 `copy` 命令。

![](img/c4403960e668662504e2a7f3b35cedf0_63.png)

**命令示例：**
```bash
copy /B 文件1+文件2+文件3 合并后的文件名
```

![](img/c4403960e668662504e2a7f3b35cedf0_65.png)

![](img/c4403960e668662504e2a7f3b35cedf0_67.png)

合并后，可能需要校验文件的MD5值以确保完整性，并使用十六进制编辑器检查或修复文件头。

### 字符串与十六进制搜索

最基础的隐写可能只是将flag以明文或十六进制形式藏在文件中。当没有其他思路时，可以尝试直接搜索关键字符串。

在编辑器中使用查找功能（Ctrl+F），搜索如`flag`、`key`、`ctf`等常见关键词。不仅要在文本中搜索，也要在十六进制视图中搜索对应的十六进制编码。

## 图片隐写

![](img/c4403960e668662504e2a7f3b35cedf0_71.png)

上一节我们介绍了通用的文件操作，本节中我们聚焦于图片文件特有的隐写方式。

### 图片隐写概述

图片隐写的方式多种多样，远比普通文件复杂。常见的考察点包括：
*   **细微颜色差别**：肉眼难以察觉的像素颜色变化。
*   **GIF多帧隐藏**：信息可能藏在动图的某一帧中。
*   **颜色通道**：利用RGB不同通道存储信息。
*   **EXIF信息**：图片的元数据，可能包含GPS定位、拍摄设备等提示信息或flag。
*   **图片修复**：图片长宽、CRC校验值被修改，需要修复。
*   **最低有效位隐写**：修改像素颜色值的最低比特位来隐藏信息，对图片观感影响极小。
*   **图片加密**：使用特定算法对图片进行加密。

### 图片处理工具

处理图片隐写题，除了通用工具，还需要一些专门的图片处理工具。
*   **Stegsolve**：Java编写的图片隐写分析工具，可用于分析通道、帧、以及执行图片间的运算（如异或、加减等）。
*   **画图工具**：系统自带的画图软件，可用于简单的格式转换、颜色取反、旋转等操作。
*   **Photoshop/GIMP**：专业的图像处理软件，用于复杂的图片操作（如拼图、高级修复）。

### EXIF信息查看

EXIF信息是嵌入在图片文件中的元数据。查看方法很简单：
*   **Windows**：右键图片 -> 属性 -> 详细信息。
*   **Linux命令行**：可以使用 `exiftool` 命令。
*   **在线工具或专用软件**：也可以查看和编辑EXIF信息。

其中可能直接包含flag，或提供解题的提示（如GPS坐标）。

![](img/c4403960e668662504e2a7f3b35cedf0_73.png)

![](img/c4403960e668662504e2a7f3b35cedf0_75.png)

### 图片运算

当给出两张看似相同或有关联的图片时，可以尝试对它们进行像素运算来发现隐藏信息。

使用 **Stegsolve** 的 `Analyse` -> `Image Combiner` 功能，可以尝试多种运算方式，如：
*   **XOR** （异或）
*   **ADD** （加法）
*   **SUB** （减法）

![](img/c4403960e668662504e2a7f3b35cedf0_77.png)

![](img/c4403960e668662504e2a7f3b35cedf0_78.png)

注意，减法的顺序不同结果也不同（A-B 与 B-A）。如果一种运算未得到结果，可以尝试交换图片顺序或尝试其他运算。

![](img/c4403960e668662504e2a7f3b35cedf0_80.png)

![](img/c4403960e668662504e2a7f3b35cedf0_82.png)

### 最低有效位隐写

LSB隐写是CTF中的常见考点。它将信息隐藏在像素颜色值（R、G、B）每个通道的最后一位（最低位）。

**解题工具与方法：**
1.  **Stegsolve**：在 `Analyse` -> `Data Extract` 功能中，勾选相应的颜色通道和位平面（通常是Red/Green/Blue的0位），点击`Preview`查看提取出的数据。
2.  **zsteg**：一个命令行工具，能自动尝试多种LSB隐写方案，并列出所有可能隐藏的信息。命令：`zsteg 图片文件名`。
3.  **wbStego**：一个图形化工具，支持BMP、PDF等格式的LSB隐写解密。对于JPG图片，需先用画图工具转换为BMP格式再处理。
4.  **Python脚本**：当题目对LSB隐写做了自定义改动时，可能需要编写或修改Python脚本来精确提取信息。核心是读取图片像素值，提取每个颜色通道的最低位，然后重组为字节数据。

![](img/c4403960e668662504e2a7f3b35cedf0_84.png)

![](img/c4403960e668662504e2a7f3b35cedf0_86.png)

### 图片修复

![](img/c4403960e668662504e2a7f3b35cedf0_88.png)

图片文件结构可能被破坏，导致无法正常显示。常见于PNG格式。

![](img/c4403960e668662504e2a7f3b35cedf0_90.png)

![](img/c4403960e668662504e2a7f3b35cedf0_92.png)

![](img/c4403960e668662504e2a7f3b35cedf0_94.png)

**修复工具与方法：**
1.  **TweakPNG**：一个PNG图片分析修复工具。打开图片后，它会提示CRC校验错误等信息。
2.  **修复CRC**：如果只是CRC校验值被篡改，TweakPNG可能会给出正确的CRC值。用十六进制编辑器找到错误的CRC值并替换为正确的即可。
3.  **修复长宽**：如果图片的高度或宽度被修改，会导致显示不全或CRC错误。需要计算正确的长宽值。
    *   可以使用提供的Python脚本，根据图片现有的IHDR块数据计算正确的长宽和CRC值。
    *   用十六进制编辑器找到IHDR块中的宽度和高度字段（通常各占4字节），修改为计算出的正确值。

### 图片加密

![](img/c4403960e668662504e2a7f3b35cedf0_104.png)

![](img/c4403960e668662504e2a7f3b35cedf0_105.png)

有些图片使用了特定算法加密。解题关键在于识别加密算法并使用对应工具解密。

![](img/c4403960e668662504e2a7f3b35cedf0_107.png)

![](img/c4403960e668662504e2a7f3b35cedf0_109.png)

**识别与解密流程：**
1.  **识别算法**：题目名称、描述或文件注释可能给出提示（如`outguess`、`F5`）。对于JPG，可以使用 `stegdetect` 工具进行自动检测。
2.  **使用解密工具**：
    *   **outguess**：命令行工具。解密示例：`outguess -r 加密图片.jpg 解密输出.txt`。
    *   **F5**：需要Java环境。将加密图片放入工具目录，运行`java Extract 加密图片.jpg`（可能需要密码）。
    *   **其他工具**：如`steghide`、`jphide`等，都有对应的解密命令或图形界面。

### 二维码处理

![](img/c4403960e668662504e2a7f3b35cedf0_111.png)

最终得到的flag可能编码在二维码中。但二维码可能被处理过：
*   **残缺/遮挡**：需要PS或画图工具补全缺失的定位点或图案。
*   **颜色反相**：黑白颜色反转，导致无法扫描。使用画图工具的“反色”功能即可纠正。
*   **彩色干扰**：二维码可能是彩色的，但关键的黑白对比信息藏在某个颜色通道中。使用Stegsolve查看各个颜色通道，找到可识别的黑白二维码图层。
*   **分块/多图**：信息可能分散在多个二维码中，需要分别扫描后组合。
