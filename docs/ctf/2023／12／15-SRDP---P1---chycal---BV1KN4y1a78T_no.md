![](img/ed97172ce8a5a73739d596786011df34_7.png)

![](img/ed97172ce8a5a73739d596786011df34_9.png)

首先，我们需要了解如何启动GDB来调试一个程序。大部分Linux系统已经预装了GDB程序。然而，一个普通的可执行文件如果没有在编译时添加特定参数，是无法被GDB直接调试的。

![](img/ed97172ce8a5a73739d596786011df34_14.png)

为了生成一个可被GDB调试的可执行文件，我们需要在编译时加上 `-g` 参数。这个参数会在可执行文件中包含调试信息。

以下是编译一个C语言源文件并生成可调试可执行文件的命令：
```bash
gcc -o c -g c.c
```
这条命令中，`-o c` 指定输出文件名为 `c`，`-g` 参数则添加了调试信息。

启动GDB调试有三种主要方式：
1.  直接调试可执行文件：`gdb ./c`
2.  附加到正在运行的进程：`gdb attach <进程ID>`
3.  调试核心转储文件：`gdb -c <core文件>`

对于初学者，最常用的是第一种方式。输入 `gdb ./c` 即可进入GDB的交互式调试界面。

至此，GDB的启动环节就结束了。接下来，我们将介绍GDB中一些常用的指令。

## GDB常用指令介绍

GDB主要提供四大功能：启动程序、在任意位置设置断点、检查程序运行时的状态（如变量值、调用栈），以及在程序运行时修改变量和执行条件。

以下是查看程序状态相关的一些指令。

### 查看源代码与汇编

`list` 指令用于显示指定范围内的源代码。
*   `list`：显示当前行附近的源代码（默认10行）。重复输入会继续向下显示。
*   `list -`：显示当前行之前的源代码。
*   `list <行号>`：显示指定行号附近的源代码。
*   `list <函数名>`：显示指定函数附近的源代码。
*   `list <文件名>:<函数名>`：显示指定文件中特定函数附近的源代码。
*   `list <起始行>,<结束行>`：显示指定行号区间的源代码。

默认显示行数可以通过 `set listsize <行数>` 来修改，并通过 `show listsize` 查看当前设置。

`disassemble` 指令（可缩写为 `disas`）用于查看指定位置的汇编代码。通常需要先让程序运行到某个断点处再使用。例如，`disas main` 可以查看 `main` 函数的汇编指令。

### 查看程序信息

![](img/ed97172ce8a5a73739d596786011df34_18.png)

![](img/ed97172ce8a5a73739d596786011df34_20.png)

`info` 是一个功能强大的指令，配合不同参数可以查看多种信息。
*   `info breakpoints`：查看所有已设置的断点信息。
*   `info locals`：查看当前函数内的局部变量。
*   `info registers`：查看当前所有寄存器的值。

`backtrace` 指令（缩写 `bt`）用于查看当前的函数调用栈，这对于理解程序执行流程和定位问题非常有帮助。

### 监视与打印变量

![](img/ed97172ce8a5a73739d596786011df34_28.png)

![](img/ed97172ce8a5a73739d596786011df34_30.png)

`watch` 指令用于监视一个变量或表达式，当其值发生变化时，GDB会暂停程序并通知用户。例如，`watch s` 会监视变量 `s` 的变化。

`print` 指令（缩写 `p`）用于在程序暂停时打印变量或表达式的当前值。例如，`p s` 会打印变量 `s` 的值。`ptype` 指令可以查看变量的类型，例如 `ptype s`。

上一节我们介绍了如何查看程序状态，本节中我们来看看如何控制程序的执行。

## 控制程序执行

在GDB中，程序需要被运行起来才能进行深入的调试。如果直接启动GDB而不运行程序，许多状态（如调用栈）是无法查看的。

### 运行与继续

`run` 指令（缩写 `r`）用于从头开始运行程序。如果不设置任何断点，程序会一直运行到结束。
`continue` 指令（缩写 `c`）用于在程序因断点等原因暂停后，让其继续运行，直到遇到下一个断点或程序结束。

### 断点管理

`break` 指令（缩写 `b`）用于设置断点，有多种设置方式：
*   `break <函数名>`：在函数入口处设置断点。
*   `break <行号>`：在指定行号设置断点。
*   `break *<地址>`：在指定的内存地址设置断点。
*   `break ... if <条件>`：设置条件断点，仅当条件满足时才在此处暂停。

![](img/ed97172ce8a5a73739d596786011df34_32.png)

![](img/ed97172ce8a5a73739d596786011df34_34.png)

`tbreak` 用于设置临时断点，该断点在被触发一次后会自动删除。

通过 `info breakpoints` 可以获取所有断点的编号。然后可以使用以下命令管理断点：
*   `disable <断点编号>`：禁用指定断点。
*   `enable <断点编号>`：启用指定断点。
*   `delete <断点编号>`：删除指定断点。

### 单步执行

![](img/ed97172ce8a5a73739d596786011df34_44.png)

![](img/ed97172ce8a5a73739d596786011df34_45.png)

`step` 指令（缩写 `s`）用于单步执行。如果当前行是函数调用，`step` 会进入该函数内部。
`next` 指令（缩写 `n`）同样用于单步执行，但遇到函数调用时，会将整个函数调用作为一步执行，不会进入函数内部。
`until` 指令（缩写 `u`）用于运行直到退出当前循环或到达指定行号，例如 `until 10` 会运行到第10行。

### 其他控制指令

![](img/ed97172ce8a5a73739d596786011df34_53.png)

`frame` 指令用于切换当前查看的栈帧。`up` 和 `down` 指令可以在调用栈中向上或向下移动。
`jump` 指令允许直接跳转到程序中的指定行或地址继续执行，这可以绕过某些代码段。

### 调用GDB调试

![](img/ed97172ce8a5a73739d596786011df34_74.png)

在Pwntools中，可以通过以下几种方式启动GDB调试：
1.  在脚本中启动并调试程序：
    ```python
    io = gdb.debug('./c')
    ```
    这会自动打开GDB并运行程序。
2.  附加到已运行的进程：
    ```python
    io = process('./c')
    gdb.attach(io, 'break main')
    ```
    这会先启动程序进程，然后GDB附加到该进程，并在main函数处断下。
3.  通过进程ID附加：
    ```python
    gdb.attach(pid, 'break main')
    ```

在Pwntools中使用GDB可以方便地为程序设置断点、检查寄存器和内存状态，从而辅助漏洞分析和利用脚本的编写。

## 栈溢出与ROP攻击原理

![](img/ed97172ce8a5a73739d596786011df34_104.png)

在掌握了调试工具之后，我们现在进入漏洞利用部分。栈溢出是一种常见的安全漏洞，而ROP（Return-Oriented Programming）是一种利用该漏洞的高级技术。

### 栈溢出基础

![](img/ed97172ce8a5a73739d596786011df34_114.png)

![](img/ed97172ce8a5a73739d596786011df34_116.png)

![](img/ed97172ce8a5a73739d596786011df34_117.png)

当一个函数被调用时，会在栈上分配空间，用于存储局部变量、函数参数、返回地址和旧的栈帧指针（EBP）。如果函数使用了不检查边界的安全函数（如 `gets`, `strcpy`），向缓冲区写入超长数据就可能覆盖栈上的返回地址。

攻击者通过精心构造输入数据，覆盖返回地址，使其指向恶意代码或特定的函数地址，从而控制程序执行流。最简单的利用方式是让程序跳转到已有的函数（如 `system("/bin/sh")`）来获取shell。

![](img/ed97172ce8a5a73739d596786011df34_119.png)

![](img/ed97172ce8a5a73739d596786011df34_121.png)

![](img/ed97172ce8a5a73739d596786011df34_123.png)

### ROP攻击链

在更复杂的情况下，程序中可能没有直接可用的危险函数，或者栈空间不可执行。这时就需要使用ROP技术。

![](img/ed97172ce8a5a73739d596786011df34_125.png)

![](img/ed97172ce8a5a73739d596786011df34_127.png)

![](img/ed97172ce8a5a73739d596786011df34_129.png)

ROP的核心思想是利用程序中已有的、以 `ret` 指令结尾的短指令序列（称为gadget），通过连续覆盖返回地址，将这些gadget串联起来，形成一条能够完成特定功能（如系统调用）的指令链。

例如，为了执行 `execve("/bin/sh", 0, 0)`，在32位系统中需要：
1.  将系统调用号 `0xb` 放入 `EAX` 寄存器。
2.  将字符串 `"/bin/sh"` 的地址放入 `EBX` 寄存器。
3.  将 `0` 放入 `ECX` 和 `EDX` 寄存器。
4.  执行 `int 0x80` 指令。

通过工具（如 `ROPgadget`）在二进制文件中搜索 `pop eax; ret`、`pop ebx; ret` 等gadget，并找到 `"/bin/sh"` 字符串的地址，就可以构造一个payload。这个payload会依次覆盖返回地址，指向这些gadget和所需数据，最终形成一个ROP链，实现系统调用。
