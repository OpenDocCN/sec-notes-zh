# 黑客技术入门：P161：真题练习—非洲之旅（进阶）

在本节课中，我们将一起分析一道网络安全进阶题目的解题思路。这道题是“非洲之旅”的高难度版本，代码结构更复杂，但我们将通过清晰的步骤来拆解它，帮助你建立解题的通用方法。

## 题目结构概览

上一节我们介绍了基础版本的解题方法，本节中我们来看看进阶版本。面对代码量大的题目时，不要逐行阅读，以免陷入混乱。首先，我们需要快速浏览代码，理解其整体框架。

通过浏览，我们可以将代码分为几个主要部分：
1.  文件开头的常规代码，可以暂时忽略。
2.  一个自定义的函数定义部分。
3.  标记为“level1”的第一关逻辑。
4.  标记为“第二关”的第二关逻辑。
5.  最后一部分的第三关逻辑。

整体来看，这道题与基础版类似，需要依次通过三个关卡才能获得flag。

## 分步解题思路分析

理解了整体结构后，我们进入下一步的详细分析。我们的策略是：先关注程序的主流程，再回头理解函数的具体作用。

### 第一关逻辑分析

首先，我们来看第一关。其核心逻辑是检查是否传递了名为 `num` 的参数。

```
if (isset($_GET[‘num’])) {
    // 进入关卡逻辑
} else {
    // 提示“带去非洲吧”
}
```

因此，解题的第一步是必须传递 `num` 参数。传递后，代码会使用 `intval()` 函数进行处理，但后续增加了更复杂的判断条件，例如要求 `num` 不等于某个 `project` 的运算结果。这一关的目标是找到一个特殊的数字，满足所有设定的条件。

### 第二关逻辑分析

接下来，我们分析第二关。这一关同样涉及MD5哈希值的验证，但难度有所提升。

其核心操作是对输入进行两次MD5计算，并寻找满足特定哈希碰撞的值。虽然听起来复杂，但可以明确的是，这样的特殊值是存在的。这需要你进行一些研究和尝试。

### 第三关逻辑分析

最后，我们来看第三关。这一关的核心是使用 `get` 参数传递命令来读取文件，但过滤的字符比第一题更多。

以下是它过滤掉的部分命令：

*   `cat`
*   `more`
*   `less`
*   `head`
*   `tail`
*   `nl`
*   `tac`

观察这个列表，我们可以发现它们有一个共同点：都是用于查看文件内容的Linux命令。题目不可能过滤掉所有查看文件的方法，因此，你的任务是思考并找到未被过滤的替代命令。

## 总结与练习

本节课中，我们一起学习了如何分析一个结构复杂的进阶题目。我们首先梳理了代码框架，然后分关卡讨论了其核心逻辑：第一关是寻找特殊数字，第二关是处理MD5碰撞，第三关是绕过命令过滤查看文件。
