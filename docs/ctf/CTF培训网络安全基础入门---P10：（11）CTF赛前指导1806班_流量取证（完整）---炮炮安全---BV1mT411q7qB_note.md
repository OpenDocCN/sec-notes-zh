在本节课中，我们将要学习网络安全中一个重要的领域——流量取证。流量是网络中数据传输的体现，英文称为“traffic”。在CTF比赛中，许多关键信息（如flag）常常隐藏在看似杂乱的网络流量数据包中。我们将学习如何分析这些流量包，提取出有用的信息。

## 什么是网络流量？ 📡

![](img/fdadb2f742278f8e001d8eca383f94f5_5.png)

网络流量是指在网络中传输的数据包。当我们使用网络时，无论是浏览网页还是传输文件，数据都会被分割成一个个小的数据包进行传输。每个数据包都包含源地址、目的地址、端口号、协议类型以及实际的有效数据。

一个数据包的有效载荷（payload）通常不超过1500字节。如果要传输一个较大的文件（如视频或压缩包），它会被拆分成多个数据包。因此，要还原原始文件或数据，我们需要将这些分散的数据包重新组合起来。

## 流量分析的基本结构 🧩

上一节我们介绍了网络流量的基本概念，本节中我们来看看如何分析一个流量包的结构。理解数据包的封装层次是分析的基础。

数据包的封装类似于寄信：
1.  **数据**：就像信纸上的内容，是我们要传输的核心信息。
2.  **信封（传输层）**：如同信封，包含了源端口和目的端口，确保数据能到达正确的应用程序。
3.  **地址（网络层）**：如同收件人地址，包含了源IP和目的IP，确保数据能在网络中路由。
4.  **物理传输（链路层）**：如同邮差送信，将数据转换成电信号或光信号在物理介质上传输。

在Wireshark等流量分析工具中，我们可以清晰地看到这些层次。分析流量时，我们主要关注“五元组”：
*   **源IP地址**
*   **目的IP地址**
*   **源端口号**
*   **目的端口号**
*   **协议类型**（如TCP、UDP、HTTP）

## 流量分析入门技巧 🛠️

了解了流量结构后，我们来看看分析流量的具体方法和技巧。掌握这些技巧能帮助我们快速定位关键信息。

### 1. 协议分级统计
这是分析流量的第一步。在Wireshark中，点击 **`统计 (Statistics)` -> `协议分级 (Protocol Hierarchy)`**。这个功能会展示当前流量文件中各种协议所占的比例。如果发现某种协议（如HTTP）或某种媒体类型（如`media type`）占比异常高，那么关键数据很可能就隐藏在这些数据包中。

![](img/fdadb2f742278f8e001d8eca383f94f5_7.png)

![](img/fdadb2f742278f8e001d8eca383f94f5_9.png)

![](img/fdadb2f742278f8e001d8eca383f94f5_10.png)

### 2. 应用过滤器
在协议分级中看到可疑的协议后，可以右键点击该协议，选择 **`作为过滤器应用 (Apply as Filter)` -> `选中 (Selected)`**。这样，Wireshark就会只显示属于该协议的数据包，大大缩小了分析范围。

### 3. 追踪数据流
对于TCP或HTTP等面向连接的协议，数据是连续传输的。Wireshark可以将属于同一次会话（流）的所有数据包重组，让我们以更直观的方式查看数据。
*   右键点击一个TCP或HTTP数据包。
*   选择 **`追踪流 (Follow Stream)` -> `TCP流`** 或 **`HTTP流`**。
*   弹出的窗口会显示这次会话中所有数据的明文（如果是未加密的），便于我们查找flag或敏感信息。

![](img/fdadb2f742278f8e001d8eca383f94f5_12.png)

![](img/fdadb2f742278f8e001d8eca383f94f5_13.png)

### 4. 查找特定内容
如果知道flag可能包含某些关键字（如`flag`、`key`、`password`），或者知道攻击者使用了特定命令（如`ls`、`dir`、`cat`），可以使用Wireshark的搜索功能。
在过滤栏使用 `tcp contains “关键字”` 或 `http contains “关键字”` 进行过滤。例如：
```bash
tcp contains “flag{”
```
或者针对具体字段：
```bash
http.request.uri contains “admin”
```

### 5. 导出对象
如果流量中包含文件传输（如通过HTTP上传/下载文件），可以直接将其导出。
*   点击 **`文件 (File)` -> `导出对象 (Export Objects)` -> `HTTP`**。
*   在列表中找到可疑的文件（如图片、压缩包、可执行文件），将其导出并进一步分析。

![](img/fdadb2f742278f8e001d8eca383f94f5_19.png)

### 1. 无线流量分析
无线网络（Wi-Fi）流量通常使用WPA/WPA2加密。分析这类流量包（协议为802.11）需要先破解其密码。
*   使用Kali Linux中的 `aircrack-ng` 工具进行破解。
*   基本命令格式为：
    ```bash
    aircrack-ng -w 密码字典.txt 无线流量包.cap
    ```
*   破解成功后，即可解密流量，然后按常规方法分析。

### 2. USB流量分析
USB流量捕获的是主机与USB设备（如键盘、鼠标）之间的通信数据。

![](img/fdadb2f742278f8e001d8eca383f94f5_23.png)

![](img/fdadb2f742278f8e001d8eca383f94f5_25.png)

![](img/fdadb2f742278f8e001d8eca383f94f5_26.png)

![](img/fdadb2f742278f8e001d8eca383f94f5_27.png)

**USB键盘流量**：
*   按键信息通常存储在数据包的特定偏移位置（例如第三个字节）。
*   每个按键值对应一个键位（如`0x04`对应`a`，`0x05`对应`b`）。
*   分析步骤：
    1.  在Wireshark中过滤USB流量：`usb.src == “设备地址”`。
    2.  将按键数据字段导出为文本。
    3.  使用Python脚本根据USB HID键值映射表将数据还原成敲击的字符。

![](img/fdadb2f742278f8e001d8eca383f94f5_29.png)

![](img/fdadb2f742278f8e001d8eca383f94f5_31.png)

![](img/fdadb2f742278f8e001d8eca383f94f5_33.png)

![](img/fdadb2f742278f8e001d8eca383f94f5_35.png)

![](img/fdadb2f742278f8e001d8eca383f94f5_37.png)

![](img/fdadb2f742278f8e001d8eca383f94f5_38.png)

**USB鼠标流量**：
*   鼠标移动和点击信息由坐标位移量表示。
*   通常包含四个关键字节：第一个字节表示按键状态（左键、右键），第二、三个字节表示X轴位移，第四个字-节表示Y轴位移。
*   分析步骤：
    1.  导出鼠标移动的坐标数据。
    2.  使用Python脚本处理数据，并利用`gnuplot`等工具将坐标点绘制成图像，从而还原鼠标的移动轨迹和点击操作。

### 3. HTTPS加密流量分析
HTTPS流量是经过TLS/SSL加密的HTTP流量，直接查看是乱码。要解密，需要获取服务器的私钥（RSA Key）。
*   在Wireshark中，点击 **`编辑 (Edit)` -> `首选项 (Preferences)` -> `Protocols` -> `TLS`**。
*   在`(Pre)-Master-Secret log filename`栏中，指定包含密钥的文件路径。
*   配置完成后，Wireshark会自动解密HTTPS流量，之后就可以像分析HTTP流量一样追踪和查看明文数据。

## 实战案例分析 🎯

![](img/fdadb2f742278f8e001d8eca383f94f5_58.png)

让我们通过一个简单的案例来串联以上技巧。假设我们拿到一个流量包，题目提示flag隐藏在Web后台的访问中。

![](img/fdadb2f742278f8e001d8eca383f94f5_60.png)

![](img/fdadb2f742278f8e001d8eca383f94f5_62.png)

![](img/fdadb2f742278f8e001d8eca383f94f5_63.png)

![](img/fdadb2f742278f8e001d8eca383f94f5_65.png)

![](img/fdadb2f742278f8e001d8eca383f94f5_67.png)

1.  **协议分级**：发现绝大部分是TCP和HTTP流量。
2.  **过滤HTTP请求**：使用过滤器 `http`，查看所有HTTP请求。
3.  **寻找可疑请求**：发现大量对 `/admin` 路径的访问，且伴有POST请求。
4.  **追踪流**：追踪其中一个向`/admin/login.php`发送POST请求的TCP流。
5.  **查看明文**：在流中直接看到明文的用户名和密码：`username=admin&password=flag{This_Is_Fake_Flag}`。
6.  **提交Flag**：成功获取flag。
