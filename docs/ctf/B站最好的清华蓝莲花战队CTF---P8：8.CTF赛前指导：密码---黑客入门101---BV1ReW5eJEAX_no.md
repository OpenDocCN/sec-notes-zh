## 密码学核心概念

密码学主要应用于数据的动态传输（如网络通信）和静态存储（如数据库密码）。其核心目标是：即使数据被截获，攻击者也看不懂其内容。

### 加密与编码的区别

*   **编码 (Encoding)**：一种映射关系，相当于一张公开的“替换表”。例如，Base64、ASCII、URL编码。目的是为了方便数据传输或存储，并非为了保密（因为“表”是公开的）。
*   **加密 (Encryption)**：涉及**算法**和**密钥**。目的是为了保密，将**明文 (Plaintext)** 转化为**密文 (Ciphertext)**。只有掌握算法和正确密钥的人才能解密。

### 加密算法的分类

根据加密和解密使用的密钥是否相同，主要分为两类：

**1. 对称加密 (Symmetric Encryption)**
*   **公式**：`加密密钥 = 解密密钥`
*   加解密使用同一把密钥。特点是**速度快**，但密钥必须严格保密，一旦泄露则安全性丧失。常见算法有DES、AES。
*   在实际应用中（如HTTPS），通常使用对称加密来加密实际传输的数据，以保证效率。

![](img/7aa1c6ee42347c17150da3c7697ea161_39.png)

![](img/7aa1c6ee42347c17150da3c7697ea161_41.png)

**2. 非对称加密 (Asymmetric Encryption)**
*   **公式**：`加密密钥 ≠ 解密密钥`
*   使用一对密钥：**公钥 (Public Key)** 和**私钥 (Private Key)**。公钥公开，私钥自己保存。
    *   用**公钥加密**的数据，只能用对应的**私钥解密**。（用于加密传输给特定对象）
    *   用**私钥加密**（通常称为签名）的数据，可以用对应的**公钥解密验证**。（用于验证发送者身份）
*   特点是安全性高，但**速度比对称加密慢很多**。常见算法有RSA。
*   在实际应用中，常利用非对称加密来安全地传输对称加密的密钥。

**3. 摘要算法 (Hash Algorithm)**
*   也称为散列函数或哈希函数。它能将任意长度的数据映射为固定长度的“指纹”（哈希值），如MD5（128位）、SHA-1（160位）。
*   **核心特性**：
    *   **雪崩效应**：输入数据的微小改变（即使1比特），会导致输出哈希值发生巨大变化。
    *   **不可逆性**：理论上无法从哈希值反推出原始数据。
*   主要用于验证数据完整性（如文件校验）和密码存储（数据库存哈希值而非明文密码）。注意：网上所谓的“MD5解密”实际上是利用彩虹表进行的碰撞查询，并非真正的逆向解密。

**URL编码**
*   将特殊字符和中文转换为 `%` 后跟两个十六进制数的形式。
*   **特征**：密文中包含大量 `%XX` 格式的片段。

**Unicode编码**
*   统一码，为世界上大多数文字系统提供标准编码。
*   **特征**：形式为 `\uXXXX`（如 `\u4e2d\u6587` 表示“中文”）。

**JS混淆与编码**
*   为了保护前端JavaScript代码，会进行混淆，使其难以阅读。
*   **常见形式**：
    *   使用 `eval()` 函数执行经过编码的字符串。
    *   使用 `[!+[]+...]` 等奇特字符组合表示代码。
    *   JJEncode/AAEncode：将代码编码成仅包含少量特定字符（如 `[]()!+`）或颜文字的形式。
*   **解决方法**：通常可直接在浏览器控制台（F12 -> Console）中执行混淆后的代码得到结果。

## 古典密码与常见加密

![](img/7aa1c6ee42347c17150da3c7697ea161_81.png)

![](img/7aa1c6ee42347c17150da3c7697ea161_83.png)

古典密码通常比较简单，是CTF中的常见考点。

![](img/7aa1c6ee42347c17150da3c7697ea161_85.png)

![](img/7aa1c6ee42347c17150da3c7697ea161_86.png)

**移位密码 (Shift Cipher)**
*   **算法**：将字母表中的字母移动固定位置。
*   **最著名**：凯撒密码 (Caesar Cipher)。
*   **示例（右移3位）**：`A -> D`, `B -> E`, ..., `X -> A`, `Y -> B`, `Z -> C`。

**替换密码 (Substitution Cipher)**
*   **算法**：按照一个固定的对照表，将明文中的每个字母替换为另一个字母。
*   **示例**：摩斯电码 (Morse Code)，用点(`.`)和划(`-`)的不同组合表示字母数字。
*   **特征**：摩斯电码的间隔通常用空格或斜杠(`/`)表示。

**换位密码 (Transposition Cipher)**
*   **算法**：不改变字母本身，只改变字母的排列顺序。
*   **常见类型**：
    *   **栅栏密码 (Rail-fence Cipher)**：将明文按“之”字形排列后，按行读取形成密文。密钥是栏数。
    *   **曲路密码**：将明文按特定路径（如回形）填入矩阵，再按另一顺序读取。
    *   **列移位密码**：将明文按行填入矩阵，然后按一个密钥决定的列顺序读取各列。

**其他经典密码**
*   **维吉尼亚密码 (Vigenère Cipher)**：使用一个关键词作为密钥，进行多表替换。需要查维吉尼亚方阵表。
*   **培根密码 (Bacon‘s Cipher)**：用两种字符（如A/B）的五位组合来代表一个字母。
*   **键盘密码**：利用键盘布局进行编码。例如：
    *   **QWE加密**：将字母按键盘顺序（Q-W-E-R-T...）对应回字母表顺序（A-B-C-D-E...）。
    *   **坐标加密**：用数字表示键盘上的行和列来定位字母。

## 实战与总结

本节课我们一起学习了密码学的基础概念、常见编码和古典加密算法。密码学在CTF中题型多变，可能独立出现，也可能与MISC（杂项）题目结合。

![](img/7aa1c6ee42347c17150da3c7697ea161_93.png)

![](img/7aa1c6ee42347c17150da3c7697ea161_94.png)

![](img/7aa1c6ee42347c17150da3c7697ea161_96.png)

**核心要点总结**：
1.  **区分编码与加密**：编码是公开映射，加密需要密钥。
2.  **掌握算法分类**：对称加密快，密钥需保密；非对称加密慢，分公钥私钥；摘要算法不可逆。
3.  **识别编码特征**：记住Base64的`=`、URL的`%`、Unicode的`\u`等关键特征。
4.  **熟悉古典密码**：凯撒（移位）、栅栏（换位）、摩斯（替换）是基础，需理解其原理。
5.  **善用工具**：安装好Java/Python环境，熟练使用提供的CTF工具包（如小葵工具、米斯特工具）或在线解码网站进行快速解题。
