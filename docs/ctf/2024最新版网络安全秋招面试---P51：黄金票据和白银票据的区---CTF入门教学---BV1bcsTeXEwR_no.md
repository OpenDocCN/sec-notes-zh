# 网络安全面试突击：P51：黄金票据与白银票据的区别 🔑

## 概述
在本节课中，我们将学习内网渗透中两种重要的权限维持技术：黄金票据与白银票据。理解它们的原理、区别和应用场景，对于回答相关面试题和进行实际渗透测试至关重要。

上一节我们介绍了Kerberos协议的基本认证流程，本节中我们来看看如何利用该协议的漏洞进行票据伪造。

## 黄金票据详解
黄金票据是一种伪造用户身份的票据，其核心在于利用域控制器上`KRBTGT`账户的哈希值来生成可被任意域控制器接受的票据授予票据。

### 核心原理
在Kerberos认证流程中，客户端访问服务前，需先向密钥分发中心申请TGT。黄金票据攻击直接绕过了身份验证的第一步，攻击者利用获取到的`KRBTGT`账户的NTLM哈希值，自行伪造TGT。

**关键公式/概念：**
伪造的TGT有效性基于一个事实：KDC使用`KRBTGT`账户的密钥来加密TGT中的部分数据。因此，只要拥有`KRBTGT`的哈希，就能生成被KDC认可的有效TGT。

### 利用条件
以下是成功实施黄金票据攻击所需的前提条件：
*   已获取域控制器的权限。
*   已导出`KRBTGT`账户的NTLM哈希值。
*   知晓域的SID（安全标识符）信息。

### 特点与影响
*   **权限极高**：可伪造域内任意用户身份，访问域内任何服务。
*   **持久性强**：只要`KRBTGT`账户的密码/哈希未更改，伪造的票据就长期有效。
*   **难以检测**：攻击发生在认证流程之外，生成的票据在Kerberos协议看来是合法的。

## 白银票据详解
白银票据是一种伪造服务访问权限的票据。与黄金票据不同，它直接伪造访问特定服务所需的服务票据，而无需与KDC进行交互。

### 核心原理
在Kerberos流程中，客户端获得TGT后，会用它向KDC申请访问特定服务的ST。白银票据攻击跳过了向KDC申请ST的步骤。攻击者利用获取到的目标服务账户的NTLM哈希值，直接为客户端伪造访问该服务的ST。

**关键公式/概念：**
服务端使用自身账户的密钥来验证ST的真实性。因此，只要拥有服务账户的哈希，就能生成被该服务认可的有效ST。

### 利用条件
以下是成功实施白银票据攻击所需的前提条件：
*   已获取目标服务所在服务器的权限，或通过其他方式获得了该服务账户的NTLM哈希值。
*   知晓目标服务的SPN（服务主体名称）。
*   知晓域的SID和要伪造的用户名。

### 特点与影响
*   **针对性**：只能用于访问特定的服务（如CIFS文件共享、MSSQL数据库等）。
*   **不与KDC交互**：整个攻击过程在网络上不可见，因为不涉及与域控制器的通信，隐蔽性更强。
*   **权限受限**：无法获取新的TGT或访问其他未授权的服务。

## 黄金票据与白银票据的核心区别
理解了两种票据的基本原理后，以下是它们的关键区别对比：

| 对比维度 | 黄金票据 | 白银票据 |
| :--- | :--- | :--- |
| **伪造对象** | 票据授予票据 | 服务票据 |
| **所需哈希** | `KRBTGT`账户的哈希 | **目标服务账户**的哈希 |
| **访问范围** | 可访问域内**所有**服务 | 只能访问**特定**服务 |
| **与KDC交互** | 需要与KDC交互（使用伪造的TGT申请ST） | **无需**与KDC交互，直接访问服务 |
| **检测难度** | 相对容易被监控发现（异常TGT请求） | 更难检测（无KDC通信日志） |
| **权限要求** | 需要已控制域控制器 | 需要已控制目标服务器或获取服务账户哈希 |

## 总结
本节课中我们一起学习了内网渗透中的两种重要票据攻击技术。

*   **黄金票据**通过伪造`KRBTGT`账户的TGT，实现了域内最高权限的伪装，是强大的权限维持手段。
*   **白银票据**通过伪造特定服务的ST，实现了对单一服务的直接访问，具有更高的隐蔽性。

两者的根本区别在于：黄金票据是“伪造身份”，从而获得全局访问权；而白银票据是“伪造门票”，只能进入特定的“房间”。掌握这些知识，不仅能从容应对相关面试题，更能深入理解内网Kerberos认证的安全边界与脆弱点。
