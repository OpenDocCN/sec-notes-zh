# 网络安全系统教程：P86：73. 可信任服务路径提权 🛡️

在本节课中，我们将要学习一种经典的Windows系统权限提升方法——可信任服务路径提权。我们将了解其原理、利用条件以及完整的操作步骤。

## 概述

上一节我们介绍了Windows系统服务漏洞提权的基本概念。本节中我们来看看一种具体的利用方式：可信任服务路径提权。这种提权方法利用了Windows服务在解析可执行文件路径时的一个特性，通过构造特定的恶意程序来获取系统（SYSTEM）权限。

## 漏洞原理

可信任服务路径提权的核心在于Windows服务路径解析机制的一个缺陷。

Windows服务通常以**SYSTEM**权限运行。当系统启动一个服务时，它会根据服务配置的二进制文件路径来查找并执行对应的程序。关键在于，如果该路径中包含**空格**，Windows会尝试寻找并执行与空格前名称相匹配的程序。

**解析过程公式化描述如下：**
假设服务路径为：`C:\Program Files\Some Folder\Service.exe`
系统会按顺序尝试查找并执行：
1.  `C:\Program.exe`
2.  `C:\Program Files\Some.exe`
3.  `C:\Program Files\Some Folder\Service.exe`

如果在前面的路径位置（如`C:\`或`C:\Program Files\`）存在一个与空格前名称匹配的可执行文件（例如`Program.exe`），系统会以**SYSTEM权限**执行它，而不是继续查找最终的正确路径。

## 利用条件

![](img/ea1d06385c41c8c3f1d68be7e50c7145_1.png)

要成功利用此漏洞，需要满足以下两个核心条件：

![](img/ea1d06385c41c8c3f1d68be7e50c7145_3.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_5.png)

1.  **存在有缺陷的服务**：目标系统上存在一个服务，其可执行文件路径中包含空格。
2.  **有可写目录权限**：在该服务路径的某个空格前的目录位置，当前用户拥有**写入（Write）权限**，能够上传我们构造的可执行文件。

![](img/ea1d06385c41c8c3f1d68be7e50c7145_7.png)

## 操作步骤

以下是利用可信任服务路径进行提权的完整流程。

![](img/ea1d06385c41c8c3f1d68be7e50c7145_9.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_11.png)

### 第一步：检测存在漏洞的服务

![](img/ea1d06385c41c8c3f1d68be7e50c7145_12.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_14.png)

首先，我们需要在已获得普通权限的Shell中，检查目标主机上是否存在可能存在漏洞的服务。

![](img/ea1d06385c41c8c3f1d68be7e50c7145_15.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_17.png)

使用以下命令查找服务路径中包含空格的服务：
```cmd
wmic service get name,displayname,pathname,startmode | findstr /i "Auto" | findstr /i /v "C:\Windows\\" | findstr /i /v """
```
这条命令会筛选出启动模式为“自动”（Auto）、路径不在`C:\Windows\`下且路径中包含空格（通过引号判断）的服务。

![](img/ea1d06385c41c8c3f1d68be7e50c7145_19.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_21.png)

### 第二步：检查目录写入权限

![](img/ea1d06385c41c8c3f1d68be7e50c7145_22.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_24.png)

找到可疑服务后，需要检查其路径中空格前的目录是否对当前用户可写。

![](img/ea1d06385c41c8c3f1d68be7e50c7145_26.png)

使用`icacls`命令检查目录权限：
```cmd
icacls "C:\Program Files"
```
在输出结果中，关注`F`（完全控制）权限。如果当前用户或`Everyone`组对该目录有`F`权限，则意味着可以在此目录写入文件。

![](img/ea1d06385c41c8c3f1d68be7e50c7145_28.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_30.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_32.png)

### 第三步：生成并上传恶意程序

![](img/ea1d06385c41c8c3f1d68be7e50c7145_34.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_36.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_38.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_40.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_42.png)

确认存在可写目录后，需要生成一个恶意可执行程序（如反弹Shell）。

![](img/ea1d06385c41c8c3f1d68be7e50c7145_44.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_46.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_48.png)

1.  使用MSFvenom等工具生成一个Windows可执行格式的反弹Shell载荷（Payload）。
2.  将该可执行文件命名为服务路径中空格前的名称。例如，如果目标目录是`C:\Program Files\Vulnerable`，则应将文件命名为`Program.exe`。
3.  利用当前用户的写入权限，将生成的`Program.exe`上传到可写目录（如`C:\`）。

![](img/ea1d06385c41c8c3f1d68be7e50c7145_50.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_52.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_54.png)

### 第四步：触发漏洞获取权限

![](img/ea1d06385c41c8c3f1d68be7e50c7145_56.png)

文件上传后，需要触发服务重启，使系统按照有缺陷的路径解析机制执行我们的恶意程序。

![](img/ea1d06385c41c8c3f1d68be7e50c7145_58.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_60.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_62.png)

1.  尝试重启目标服务。通常普通用户无权操作服务，可以使用以下命令查看服务状态，但可能无法重启：
    ```cmd
    sc qc [服务名]
    sc stop [服务名]
    sc start [服务名]
    ```
2.  如果无法手动重启，则需要等待系统重启或服务自动重启。当服务再次启动时，系统会解析路径，找到并执行我们上传的`Program.exe`，并以SYSTEM权限建立反弹Shell连接。

![](img/ea1d06385c41c8c3f1d68be7e50c7145_64.png)

### 第五步：稳定会话（进程迁移）

![](img/ea1d06385c41c8c3f1d68be7e50c7145_66.png)

成功接收到SYSTEM权限的Shell后，会话可能不稳定，因为我们的恶意程序并非真正的服务，会被Windows服务控制器（SCM）终止。

![](img/ea1d06385c41c8c3f1d68be7e50c7145_68.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_70.png)

因此，在接收到会话后，需要立即进行**进程迁移**，将Shell会话迁移到一个稳定的系统进程（如`lsass.exe`或`svchost.exe`）中。

![](img/ea1d06385c41c8c3f1d68be7e50c7145_72.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_74.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_76.png)

在Metasploit中，可以使用以下命令自动迁移：
```
set AutoRunScript migrate -f
```
或者在获得会话后手动迁移：
```
meterpreter > migrate -f
```

![](img/ea1d06385c41c8c3f1d68be7e50c7145_82.png)

![](img/ea1d06385c41c8c3f1d68be7e50c7145_84.png)

本节课中我们一起学习了可信任服务路径提权技术。我们首先理解了其核心原理：Windows服务在解析带空格的路径时，会错误地优先执行空格前的可执行文件。接着，我们明确了利用的两个必要条件：存在路径带空格的服务，并且空格前的某个目录可写。最后，我们一步步演练了从检测、上传到触发、稳定会话的完整提权流程。掌握这种方法，有助于在渗透测试中识别并利用此类配置缺陷，实现权限提升。
