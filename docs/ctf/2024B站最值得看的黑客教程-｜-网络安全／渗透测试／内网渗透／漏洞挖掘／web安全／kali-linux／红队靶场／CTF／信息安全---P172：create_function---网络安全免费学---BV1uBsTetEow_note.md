# 网络安全入门：P172：create_function函数详解

在本节课中，我们将要学习PHP中一个名为`create_function`的函数。这个函数不仅与反序列化知识相关，也常见于命令执行、代码审计等安全领域。理解它的工作原理对于后续学习各类安全漏洞至关重要。

## 函数概述与基本格式

上一节我们介绍了属性赋值等基础概念，本节中我们来看看`create_function`。`create_function`是PHP中用于创建匿名函数的一个内置函数。它的基本格式如下：

```php
create_function(string $args, string $code)
```

该函数接受两个字符串参数：
*   **第一个参数 `$args`**：定义所创建函数的参数列表。
*   **第二个参数 `$code`**：定义所创建函数的函数体，即要执行的代码。

它的作用是提供一种动态创建函数的简便方式。

## 函数使用演示

为了让大家更清晰地理解，我们将通过一个简单的例子来演示其用法。以下是创建一个执行加法运算的函数的步骤。

首先，我们按照格式编写代码。假设我们要创建一个将两个参数相加的函数。

```php
$f = create_function('$a, $b', 'return $a + $b;');
```

在这行代码中：
*   `'$a, $b'` 是第一个参数，它定义了新函数接受两个参数：`$a`和`$b`。
*   `'return $a + $b;'` 是第二个参数，它定义了函数体的行为：返回`$a`与`$b`的和。**请注意，函数体内的代码需要以分号`;`结尾**。
*   `$f =` 将`create_function`创建的函数赋值给了变量`$f`，`$f`就成为这个新函数的名称。

现在，我们可以像调用普通函数一样调用`$f`：

```php
echo $f(1, 2); // 输出：3
echo $f(2, 2); // 输出：4
```

## 与传统函数定义方式的对比

为了加深理解，我们将其与传统的函数定义方式进行对比。传统方式定义同样的加法函数如下：

```php
function f2($a, $b) {
    return $a + $b;
}
echo f2(1, 2); // 输出：3
echo f2(2, 2); // 输出：4
```

可以看到，`create_function('$a, $b', 'return $a + $b;')` 与 `function f2($a, $b) { return $a + $b; }` 所创建的函数在功能上是完全等效的。`create_function`的本质是将参数列表和函数体代码以字符串的形式动态组合，并返回一个可调用的函数。

## 安全相关的重要提示

在安全领域，`create_function`需要被重点关注，因为它可能引入安全风险。

*   **字符串参数与代码执行**：由于`create_function`的第二个参数是字符串形式的代码，如果这个字符串来源于不可信的用户输入且未经过严格过滤，攻击者就可能注入恶意代码，导致代码执行漏洞。
*   **与`eval()`的相似性**：其内部实现机制类似于使用`eval()`函数执行一段拼接后的代码字符串，因此具有相似的安全隐患。
