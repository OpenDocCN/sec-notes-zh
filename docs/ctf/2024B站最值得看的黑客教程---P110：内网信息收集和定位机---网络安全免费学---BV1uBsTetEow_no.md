# 网络安全：P110：内网信息收集和定位机器 🕵️

在本节课中，我们将要学习内网渗透的第一步：内网信息收集。我们将了解内网信息收集与外网信息收集的区别，并掌握一系列基础命令，用于在控制一台内网主机后，系统地收集网络架构、域环境、主机信息等关键情报。

---

## 概述

当我们成功进入一个企业的内网后，面对的是一个未知的网络环境。此时，我们需要进行内网信息收集。这与外网信息收集的目标不同：外网收集主要针对公司公开的资产，如网站、邮箱、C段等；而内网收集则聚焦于网络内部结构、域信息、主机存活情况、用户账号等，目的是摸清内网布局，为后续横向移动和权限提升打下基础。

## 第一步：收集当前用户与主机信息

![](img/ee3056e19a04bb3d73bc6680cc854404_11.png)

![](img/ee3056e19a04bb3d73bc6680cc854404_12.png)

首先，我们需要确认我们当前控制的是哪个用户，以及所在主机的名称。

![](img/ee3056e19a04bb3d73bc6680cc854404_14.png)

1.  **查看当前用户**：使用 `whoami` 命令。
    ```bash
    whoami
    ```
    执行后，我们得知当前用户是 `GOD\Administrator`（格式为`域名\用户名`）。

2.  **查看主机名**：使用 `hostname` 命令。
    ```bash
    hostname
    ```
    执行后，我们得知当前主机名为 `STU1`。结合用户名格式，可以初步判断这是一个域账户，主机已加入域。

## 第二步：确认域环境信息

![](img/ee3056e19a04bb3d73bc6680cc854404_22.png)

接下来，我们需要确认主机是否真的加入了域，并获取域的详细信息。

1.  **查看系统信息**：使用 `systeminfo` 命令。这个命令会输出详细的系统配置信息。
    ```bash
    systeminfo
    ```
    在输出的信息中，我们可以找到关键条目：
    *   **主机名**：再次确认主机名。
    *   **域**：找到“域”这一行，内容为 `GOD.ORG`。这确认了主机已加入名为 `GOD.ORG` 的域。
    *   **网卡信息**：在“网络适配器”部分，可以找到主机的IP地址，例如 `192.168.52.143`。

![](img/ee3056e19a04bb3d73bc6680cc854404_24.png)

![](img/ee3056e19a04bb3d73bc6680cc854404_25.png)

![](img/ee3056e19a04bb3d73bc6680cc854404_26.png)

至此，我们掌握了三个核心信息：当前用户、主机已加入域 `GOD.ORG`、以及主机的IP地址。

## 第三步：收集域内用户与计算机列表

![](img/ee3056e19a04bb3d73bc6680cc854404_32.png)

![](img/ee3056e19a04bb3d73bc6680cc854404_33.png)

现在，我们开始探索域这个“班级”里到底有哪些“成员”。

![](img/ee3056e19a04bb3d73bc6680cc854404_34.png)

![](img/ee3056e19a04bb3d73bc6680cc854404_36.png)

![](img/ee3056e19a04bb3d73bc6680cc854404_38.png)

1.  **查看域用户列表**：使用 `net user /domain` 命令。此命令向域控制器查询域内的所有用户账户。
    ```bash
    net user /domain
    ```
    执行后，会列出域中的所有用户，例如可能包含 `Administrator`、`Guest` 等。这帮助我们了解域内的账号体系。

![](img/ee3056e19a04bb3d73bc6680cc854404_40.png)

2.  **查看域计算机列表**：使用 `net group “domain computers” /domain` 命令。此命令查询所有加入域的计算机账户。
    ```bash
    net group "domain computers" /domain
    ```
    执行后，会列出域中的所有计算机名，例如 `DEV`、`ROOT`、`STU1`。其中 `STU1` 就是我们当前控制的主机。这意味着域内至少还有 `DEV` 和 `ROOT` 两台目标主机。

![](img/ee3056e19a04bb3d73bc6680cc854404_41.png)

![](img/ee3056e19a04bb3d73bc6680cc854404_42.png)

需要注意的是，此列表仅说明这些计算机账户在域中注册过，并不代表它们当前一定在线或存活。

![](img/ee3056e19a04bb3d73bc6680cc854404_44.png)

## 第四步：定位域控制器

域控制器（DC）是域环境的核心，相当于“班主任”。控制域控制器通常能获得域内的最高权限。

1.  **查询域控制器**：使用 `nslookup` 命令查询域的DNS记录，通常可以定位到域控制器。
    ```bash
    nslookup -type=SRV _ldap._tcp.dc._msdcs.GOD.ORG
    ```
    或者，在已加入域的主机上，一个更简单的方法是查看系统日志或使用命令：
    ```bash
    net time /domain
    ```
    执行 `net time` 命令时，回显的服务器名通常就是域控制器。通过此方法，我们得知域控制器的主机名为 `OWA`，其全名为 `OWA.GOD.ORG`。

## 第五步：探测网络结构与存活主机

![](img/ee3056e19a04bb3d73bc6680cc854404_52.png)

![](img/ee3056e19a04bb3d73bc6680cc854404_54.png)

我们需要了解当前主机所在的网段以及同一网段内存活的其他设备。

1.  **查看本机网络配置**：使用 `ipconfig /all` 命令。
    ```bash
    ipconfig /all
    ```
    此命令详细列出了所有网络适配器的信息。我们发现当前主机 (`192.168.52.143`) 可能还有另一个IP地址（例如 `192.168.111.128`），这表明它可能是一台双网卡主机，连接着两个不同的网络段。

2.  **查看ARP缓存**：使用 `arp -a` 命令。ARP缓存表显示了近期与本机通信过的IP地址和对应的MAC地址，有助于发现同一局域网内存活的主机。
    ```bash
    arp -a
    ```
    在 `192.168.52.0` 网段下，我们发现了两个IP地址：`192.168.52.138` 和 `192.168.52.141`。结合之前的信息，`138` 很可能就是域控制器 `OWA` 的IP。

3.  **验证主机与IP对应关系**：使用 `ping` 命令。通过ping域内计算机的全名，可以解析出其IP地址，从而确认存活状态和身份。
    ```bash
    ping DEV.GOD.ORG
    ping ROOT.GOD.ORG
    ```
    例如，ping `ROOT.GOD.ORG` 后返回的IP是 `192.168.52.141`，这就确认了该主机在线且IP为 `.141`。

## 信息整理与目标确定

通过以上步骤，我们收集并整理了以下关键信息：

![](img/ee3056e19a04bb3d73bc6680cc854404_59.png)

![](img/ee3056e19a04bb3d73bc6680cc854404_60.png)

*   **当前控制主机**：`STU1.GOD.ORG` (`192.168.52.143`)
*   **域控制器**：`OWA.GOD.ORG` (`192.168.52.138`)
*   **域内其他主机**：`ROOT.GOD.ORG` (`192.168.52.141`)，`DEV.GOD.ORG`（状态未知）
*   **域名称**：`GOD.ORG`
*   **域用户列表**：已获取
*   **网络拓扑**：当前主机位于 `52` 网段，并可能通过另一网卡连接其他网络。

我们的攻击思路变得清晰：首要目标是攻陷域控制器 (`OWA, 192.168.52.138`)，因为控制它就等于控制了整个域。次要目标是另一台已确认存活的主机 (`ROOT, 192.168.52.141`)。

## 总结

本节课中我们一起学习了内网信息收集的基本流程。我们从最基础的`whoami`、`hostname`命令开始，逐步使用`systeminfo`、`net user/group /domain`、`nslookup`、`ipconfig`、`arp`、`ping`等命令，像拼图一样逐步摸清了域环境的结构：域名、域控制器、域内用户、域内计算机以及网络存活主机。

![](img/ee3056e19a04bb3d73bc6680cc854404_62.png)

![](img/ee3056e19a04bb3d73bc6680cc854404_63.png)

这些信息是内网渗透的“地图”，为我们后续的横向移动、权限提升和最终控制整个网络奠定了坚实的基础。记住，清晰的信息收集是成功渗透的一半。
