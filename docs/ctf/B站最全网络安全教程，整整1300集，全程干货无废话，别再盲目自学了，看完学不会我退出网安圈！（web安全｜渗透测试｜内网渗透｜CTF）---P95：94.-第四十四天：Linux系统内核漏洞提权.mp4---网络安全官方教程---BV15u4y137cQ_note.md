## 信息收集：提权的基础

![](img/f9f2d98796809aa585687a436ef68cbe_9.png)

上一节我们介绍了课程的整体框架，本节中我们来看看提权的第一步：信息收集。在Linux系统中，进行提权前必须全面收集目标系统的信息，这有助于我们寻找可利用的漏洞或配置缺陷。

以下是Linux提权信息收集的关键点：

*   **系统与内核版本**：通过 `uname -a` 命令获取。内核版本是寻找内核漏洞提权的关键依据。
*   **环境变量**：通过 `env` 或 `printenv` 命令查看。配置不当的环境变量可能导致提权。
*   **网络与进程信息**：通过 `ifconfig`、`netstat -tulpn`、`ps aux` 等命令获取。了解开放端口、运行的服务（特别是以root身份运行的服务）至关重要。
*   **应用程序与配置**：检查已安装的软件版本（如 `sudo -V`）及其配置文件，寻找已知漏洞或错误配置。
*   **计划任务**：通过 `crontab -l` 查看。配置不当的计划任务可能被利用。
*   **可写目录与SUID/SGID文件**：寻找当前用户有写入权限的目录，以及设置了特殊权限位的文件。

为了提高效率，通常使用自动化脚本进行信息收集。一个常用的脚本是LinEnum，它能够批量执行上述检查并输出结构化的报告。

**获取与执行LinEnum脚本的示例命令：**
```bash
wget https://raw.githubusercontent.com/rebootuser/LinEnum/master/LinEnum.sh
chmod +x LinEnum.sh
./LinEnum.sh
```
执行后，脚本会输出系统基本信息、用户权限、敏感配置、潜在的可利用点等详细内容，为后续的提权尝试提供方向。

![](img/f9f2d98796809aa585687a436ef68cbe_11.png)

## 查找漏洞利用代码

在完成信息收集并分析出潜在的攻击路径后，我们需要找到对应的漏洞利用代码（Exploit）。以下是常用的漏洞信息与利用代码查询网站：

*   **Exploit Database**：一个全面的漏洞利用代码库，可通过漏洞编号、软件名称等搜索。
*   **SecurityFocus**：提供详细的漏洞公告、影响范围和解决方案。
*   **Rapid7**：Metasploit框架背后的公司，其数据库包含大量漏洞模块信息。
*   **国内平台**：如安全客，会及时更新最新的漏洞资讯和分析报告。

此外，还可以利用 `searchsploit` 工具在本地进行搜索。该工具基于Exploit Database的镜像，在Kali Linux中已预装。

![](img/f9f2d98796809aa585687a436ef68cbe_13.png)

**使用searchsploit搜索Linux本地提权漏洞的示例：**
```bash
# 更新本地数据库
searchsploit -u
# 搜索与Linux内核相关的漏洞
searchsploit linux kernel
# 结合版本信息进行更精确的搜索
searchsploit linux kernel 4.15
# 查看特定漏洞利用代码的详细信息及路径
searchsploit -p 12345
```
通过 `searchsploit` 找到对应的利用代码路径后，可以将其复制到当前目录进行编译或直接研究。

## Linux内核漏洞提权

系统内核漏洞是提权中最直接有效的方式之一。攻击者利用内核中的安全缺陷，执行精心构造的代码，从而直接获得root权限。

一个著名的资源是“Linux Kernel Exploits”项目，它汇总了历史上公开的Linux内核漏洞利用代码。在获得目标系统内核版本后，可以在此类资源库中查找对应的利用代码。

**内核漏洞提权的一般步骤：**
1.  **信息收集**：确定内核版本（`uname -r`）。
2.  **查找漏洞**：根据内核版本，在资源库或使用 `searchsploit` 查找公开的漏洞利用代码。
3.  **上传代码**：将找到的C语言 exploit 代码上传到目标服务器。
4.  **编译执行**：在目标服务器上使用gcc编译并运行。
    ```bash
    gcc exploit.c -o exploit
    ./exploit
    ```
5.  **获取权限**：如果漏洞存在且利用成功，当前shell将提升为root权限。

## 利用环境变量提权

上一节我们介绍了利用系统漏洞进行提权，本节中我们来看看另一种方法：利用环境变量配置不当进行提权。环境变量 `PATH` 定义了系统查找可执行文件的目录顺序。

如果我们将一个包含恶意代码的程序放在 `PATH` 变量中优先级较高的目录，并且该目录对当前用户可写，那么当系统或高权限用户执行某些命令时，就可能意外执行我们的恶意程序。

![](img/f9f2d98796809aa585687a436ef68cbe_36.png)

![](img/f9f2d98796809aa585687a436ef68cbe_37.png)

**环境变量提权示例场景：**
假设有一个SUID程序，它内部调用了 `system("ls")` 这样的命令，但没有使用绝对路径（如 `/bin/ls`）。那么，系统会在 `PATH` 环境变量指定的目录中寻找 `ls` 命令。

![](img/f9f2d98796809aa585687a436ef68cbe_39.png)

![](img/f9f2d98796809aa585687a436ef68cbe_41.png)

![](img/f9f2d98796809aa585687a436ef68cbe_42.png)

1.  **检查SUID程序**：找到具有SUID权限且可能调用外部命令的程序。
2.  **劫持PATH**：创建一个名为 `ls` 的恶意脚本，内容为 `/bin/bash` 或反弹shell命令。
    ```bash
    # 恶意 ls 脚本内容
    #!/bin/bash
    /bin/bash
    ```
3.  **设置权限与路径**：给脚本添加执行权限，并将当前目录（`.`）或我们可控的目录添加到 `PATH` 变量的最前面。
    ```bash
    chmod +x ls
    export PATH=.:$PATH
    ```
4.  **触发执行**：运行那个SUID程序。当它调用 `ls` 时，会优先执行我们伪造的 `ls` 脚本，从而启动一个具有该SUID程序所有者权限（通常是root）的shell。

## 密码哈希提权

![](img/f9f2d98796809aa585687a436ef68cbe_56.png)

![](img/f9f2d98796809aa585687a436ef68cbe_57.png)

![](img/f9f2d98796809aa585687a436ef68cbe_59.png)

除了利用漏洞和配置错误，另一种传统的提权方法是破解本地用户密码。Linux系统的用户密码哈希值存储在 `/etc/shadow` 文件中，但普通用户通常无法读取。然而，在某些情况下，我们可以通过其他方式获取哈希值。

![](img/f9f2d98796809aa585687a436ef68cbe_61.png)

![](img/f9f2d98796809aa585687a436ef68cbe_62.png)

**获取密码哈希的途径：**
*   **读取备份或配置**：如 `/etc/shadow` 的备份文件 `/etc/shadow-`，或错误的权限设置导致文件可读。
*   **内存转储**：从进程内存中提取哈希（需要一定权限）。
*   **数据库或Web应用配置**：从应用配置文件中发现数据库密码，该密码可能与系统用户密码相同。

![](img/f9f2d98796809aa585687a436ef68cbe_64.png)

![](img/f9f2d98796809aa585687a436ef68cbe_66.png)

获取哈希值后，可以使用 `john` 或 `hashcat` 等工具进行破解。

![](img/f9f2d98796809aa585687a436ef68cbe_68.png)

**使用john破解shadow文件的示例命令：**
```bash
# 首先需要将 /etc/passwd 和 /etc/shadow 文件合并为john可识别的格式
unshadow passwd.txt shadow.txt > hashes.txt
# 使用john进行破解
john hashes.txt
# 使用指定的字典文件进行破解
john --wordlist=/usr/share/wordlists/rockyou.txt hashes.txt
```
如果成功破解出root用户的密码，即可直接使用 `su` 或 `ssh` 切换到root身份。

## 提权实验与总结

![](img/f9f2d98796809aa585687a436ef68cbe_78.png)

![](img/f9f2d98796809aa585687a436ef68cbe_80.png)

本节课中我们一起学习了Linux系统下的多种提权技术。为了巩固知识，建议在可控的实验环境中进行练习。

**课后实验建议：**
1.  **实验一：内核漏洞提权**：搭建一个存在已知内核漏洞的Linux系统（如Metasploitable2），尝试利用 `searchsploit` 找到漏洞，编译并执行exploit，获取root shell。
2.  **实验二：环境变量提权**：在实验机中创建一个存在缺陷的SUID程序（编程调用 `system` 函数但不使用绝对路径），然后利用 `PATH` 环境变量劫持，完成提权。

![](img/f9f2d98796809aa585687a436ef68cbe_82.png)

![](img/f9f2d98796809aa585687a436ef68cbe_84.png)

**总结回顾：**
本节课的核心内容包括：
1.  **信息收集是基础**：使用自动化脚本（如LinEnum）全面收集系统信息。
2.  **查找利用代码**：掌握利用 `searchsploit` 和各大安全网站寻找漏洞利用代码的方法。
3.  **内核漏洞提权**：理解其原理与基本步骤：**确定版本 -> 查找漏洞 -> 编译执行**。
4.  **环境变量滥用**：利用 `PATH` 变量的优先级，通过伪造可执行文件进行提权。
5.  **密码哈希破解**：在获取哈希值的前提下，使用工具破解密码。
