## 服务路径漏洞利用原理

![](img/e4b1f357f30317df0d57d1b360b94938_17.png)

上一节我们介绍了Windows服务相关的漏洞。本节中我们来看看如何利用服务路径解析的缺陷进行提权。

![](img/e4b1f357f30317df0d57d1b360b94938_19.png)

![](img/e4b1f357f30317df0d57d1b360b94938_21.png)

![](img/e4b1f357f30317df0d57d1b360b94938_23.png)

如果服务对应的可执行程序路径没有被引号包裹，系统在启动该服务时，会处理该服务对应的二进制文件路径（BINARY_PATH_NAME）。系统会对该可执行程序路径中空格前面的字符串所对应的可执行程序进行加载。它会一层一层目录地查找，直到找到最终的服务程序。

![](img/e4b1f357f30317df0d57d1b360b94938_25.png)

![](img/e4b1f357f30317df0d57d1b360b94938_27.png)

![](img/e4b1f357f30317df0d57d1b360b94938_29.png)

我们可以对此漏洞进行利用。另一个方法是查找到该路径中的某一个目录。这个目录我们需要有写入权限。如果有写入权限，我们就可以尝试写入一个特定大小（例如15872字节）的程序到该目录下（例如`C:\Windows\Tasks`），并生成一个程序文件（例如`common.exe`）。

![](img/e4b1f357f30317df0d57d1b360b94938_31.png)

![](img/e4b1f357f30317df0d57d1b360b94938_33.png)

生成`common.exe`程序后，它会放到目标目录下。此时会出现一个错误，这个错误我们上节课讲过。当我们尝试启动该服务时，由于该漏洞需要等待服务重新启动，系统再次加载可执行程序路径时，才会查找到我们写入的木马程序并加载执行。由于我们是普通用户，没有权限启动该服务，所以需要等待服务自行启动，因此会处于等待状态。

## Cobalt Strike提权模块实战

上一节我们介绍了漏洞原理，本节中我们来看看如何在Cobalt Strike中实际操作。

![](img/e4b1f357f30317df0d57d1b360b94938_52.png)

![](img/e4b1f357f30317df0d57d1b360b94938_54.png)

以下是利用CS进行提权的基本步骤：

![](img/e4b1f357f30317df0d57d1b360b94938_56.png)

![](img/e4b1f357f30317df0d57d1b360b94938_58.png)

![](img/e4b1f357f30317df0d57d1b360b94938_60.png)

![](img/e4b1f357f30317df0d57d1b360b94938_62.png)

![](img/e4b1f357f30317df0d57d1b360b94938_64.png)

1.  **获取初始立足点**：首先通过其他方式（如钓鱼、漏洞利用）获得一个普通用户的Beacon会话。
2.  **使用提权模块**：在Beacon会话上右键，选择 `Access` -> `Elevate`。
3.  **选择监听器**：在弹出的界面中选择一个用于接收高权限会话的监听器（如 `http`）。
4.  **选择漏洞利用模块**：从列表中选择一个适合目标系统的提权模块（例如 `uac-dll`）。
5.  **执行与迁移**：模块执行后，如果成功，会弹回一个新的高权限Beacon。由于通过漏洞启动的进程可能不稳定，需要立即使用 `migrate` 命令将进程迁移到稳定进程（如 `explorer.exe`）中。

![](img/e4b1f357f30317df0d57d1b360b94938_65.png)

![](img/e4b1f357f30317df0d57d1b360b94938_67.png)

**关键配置说明**：
在 `Elevate` 模块的配置中，有几个重要选项：
*   `ExitOnSession`：设置为 `false`，确保提权成功后监听器Job不会退出，可以持续接收会话。
*   `AutoRunScript`：设置自动执行的脚本，例如 `migrate -f`，可以在获得会话后自动迁移进程。
*   这些配置对应了我们在原理部分讨论的等待服务启动和进程迁移的需求。

## 加载外部提权脚本套件

![](img/e4b1f357f30317df0d57d1b360b94938_77.png)

![](img/e4b1f357f30317df0d57d1b360b94938_79.png)

CS内置的提权模块有限，但可以通过加载外部脚本进行扩展。

![](img/e4b1f357f30317df0d57d1b360b94938_81.png)

![](img/e4b1f357f30317df0d57d1b360b94938_83.png)

一个强大的扩展是 `Elevate Kit` 脚本套件。加载后，`Elevate` 模块中会出现更多提权选项，例如针对较新系统（如Windows 10 1903/1909）的漏洞利用模块。

![](img/e4b1f357f30317df0d57d1b360b94938_85.png)

![](img/e4b1f357f30317df0d57d1b360b94938_87.png)

![](img/e4b1f357f30317df0d57d1b360b94938_89.png)

**加载方法**：
1.  下载 `elevate.cna` 脚本文件。
2.  在CS客户端中，点击 `View` -> `Script Console`。
3.  点击 `Load`，选择下载的 `.cna` 文件进行加载。
4.  加载成功后，在 `Elevate` 界面中即可看到新增的模块。

## 使用独立提权EXP

![](img/e4b1f357f30317df0d57d1b360b94938_97.png)

![](img/e4b1f357f30317df0d57d1b360b94938_99.png)

除了CS模块，我们也可以上传并使用独立的提权漏洞利用程序（EXP）。

![](img/e4b1f357f30317df0d57d1b360b94938_101.png)

![](img/e4b1f357f30317df0d57d1b360b94938_103.png)

![](img/e4b1f357f30317df0d57d1b360b94938_105.png)

![](img/e4b1f357f30317df0d57d1b360b94938_107.png)

以 `CVE-2019-0803`（Win32k漏洞）的EXP为例，这是一个影响范围很广的提权工具。

![](img/e4b1f357f30317df0d57d1b360b94938_108.png)

![](img/e4b1f357f30317df0d57d1b360b94938_109.png)

![](img/e4b1f357f30317df0d57d1b360b94938_111.png)

![](img/e4b1f357f30317df0d57d1b360b94938_113.png)

**使用方法**：
1.  将EXP（如 `exp.exe`）上传到目标机器。
2.  在普通用户的Beacon会话中，使用 `execute` 或 `shell` 命令运行该EXP。
3.  EXP的使用格式通常为：`exp.exe cmd “要执行的命令”`。例如，执行 `whoami` 会显示权限已变为 `NT AUTHORITY\SYSTEM`。
4.  **反弹SYSTEM Shell**：要获得一个SYSTEM权限的Beacon，可以先生成一个反向Shell负载（payload.exe），然后通过EXP以SYSTEM权限执行它。命令如下：
    ```bash
    shell exp.exe cmd “start C:\path\to\payload.exe”
    ```
    执行后，CS监听器就会收到一个SYSTEM权限的会话。

## PowerUp模块辅助提权

![](img/e4b1f357f30317df0d57d1b360b94938_130.png)

![](img/e4b1f357f30317df0d57d1b360b94938_132.png)

![](img/e4b1f357f30317df0d57d1b360b94938_136.png)

PowerUp是PowerShell环境下著名的提权脚本，可以集成到CS中使用。

![](img/e4b1f357f30317df0d57d1b360b94938_138.png)

![](img/e4b1f357f30317df0d57d1b360b94938_140.png)

![](img/e4b1f357f30317df0d57d1b360b94938_142.png)

![](img/e4b1f357f30317df0d57d1b360b94938_143.png)

![](img/e4b1f357f30317df0d57d1b360b94938_145.png)

**集成与使用**：
1.  **导入模块**：在Beacon中使用 `powershell-import` 命令加载本地的PowerUp脚本。
    ```bash
    powershell-import /path/to/PowerUp.ps1
    ```
2.  **执行检查**：使用 `Invoke-AllChecks` 方法对目标系统进行全面的提权漏洞检测。
    ```bash
    powershell Invoke-AllChecks
    ```
3.  **分析结果**：脚本会扫描不安全的服务权限、可写的服务路径、AlwaysInstallElevated策略等，并列出可能的利用点。我们可以根据其提示，手动或结合其他模块进行利用。
