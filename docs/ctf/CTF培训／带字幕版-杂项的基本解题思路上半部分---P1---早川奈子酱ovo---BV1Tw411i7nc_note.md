## 文件操作与隐写

![](img/bb78b82ce74d0619a97c65bdf8963c40_4.png)

### 文件类型识别

![](img/bb78b82ce74d0619a97c65bdf8963c40_16.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_18.png)

当我们拿到一个文件时，首先需要确定它是什么类型的文件。有时出题人会故意删除文件的后缀名来增加难度。

以下是识别文件类型的常用方法：

![](img/bb78b82ce74d0619a97c65bdf8963c40_20.png)

*   **`file` 命令**：这是Linux系统下的一个命令行工具，可以识别文件的真实类型。其基本用法是 `file 文件名`。例如，`file example` 会输出该文件的格式信息。
*   **十六进制编辑器**：通过查看文件的头部（Header）字节来判断类型。不同文件格式有特定的“魔数”（Magic Number）。例如，JPEG图片的头部通常是 `FF D8 FF`，ZIP压缩包的头部是 `50 4B 03 04`。常用的十六进制编辑器有 `010 Editor`、`WinHex` 和 `Notepad++`（配合 `Hex Editor` 插件）。

![](img/bb78b82ce74d0619a97c65bdf8963c40_22.png)

### 文件头修复

![](img/bb78b82ce74d0619a97c65bdf8963c40_24.png)

有时，出题人会破坏或修改文件的头部，导致文件无法正常打开或识别。这时就需要我们手动修复文件头。

![](img/bb78b82ce74d0619a97c65bdf8963c40_26.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_28.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_30.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_32.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_33.png)

修复步骤如下：
1.  使用 `file` 命令或十六进制编辑器确认文件本应属于的类型。
2.  用十六进制编辑器打开该文件。
3.  根据正确的文件格式，将文件头部的字节修改为标准的“魔数”。
4.  保存文件后再次尝试打开。

![](img/bb78b82ce74d0619a97c65bdf8963c40_37.png)

### 文件分离

一个文件可能由多个文件拼接而成。例如，一张图片的后半部分可能隐藏着一个压缩包。我们需要将隐藏的文件分离出来。

以下是几种文件分离的方法：

![](img/bb78b82ce74d0619a97c65bdf8963c40_39.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_41.png)

*   **`binwalk` 工具**：这是一个强大的自动化文件分析工具。
    *   **分析**：使用命令 `binwalk 文件名` 可以扫描文件中可能嵌入的其他文件。
    *   **分离**：使用命令 `binwalk -e 文件名` 可以自动提取扫描到的所有文件。
*   **`foremost` 工具**：另一个命令行文件恢复工具，用法与 `binwalk` 类似。常用命令为 `foremost -o 输出目录 文件名`。
*   **`dd` 命令**：这是一个更底层的工具，可以按指定大小和偏移量精确切割文件。当自动化工具有效时，手动使用 `dd` 命令更为灵活。其基本格式为：
    ```bash
    dd if=输入文件 of=输出文件 bs=块大小 count=块数 skip=跳过的块数
    ```
*   **手动分离**：使用十六进制编辑器（如 `010 Editor`）直接选中需要的数据区域，然后将其另存为一个新文件。

### 文件合并

与文件分离相反，有时题目会将一个完整的文件分割成多个碎片提供给我们，需要我们将其合并还原。

以下是文件合并的方法：

![](img/bb78b82ce74d0619a97c65bdf8963c40_43.png)

*   **Linux 环境**：使用 `cat` 命令。例如，将 `part1`、`part2`、`part3` 按顺序合并为 `whole`：`cat part1 part2 part3 > whole`。
*   **Windows 环境**：使用 `copy` 命令。例如：`copy /B part1+part2+part3 whole`。

![](img/bb78b82ce74d0619a97c65bdf8963c40_45.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_47.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_49.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_51.png)

合并后，可能需要使用 `MD5` 等校验工具来验证文件的完整性是否与题目给出的哈希值一致。

![](img/bb78b82ce74d0619a97c65bdf8963c40_53.png)

### 基础隐写与搜索

最简单的隐写方式就是将 `flag` 直接以文本或十六进制形式藏在文件的开头、末尾或中间。

解题方法如下：
1.  使用文本编辑器（推荐 `Notepad++`）或十六进制编辑器打开文件。
2.  利用软件的查找功能（通常是 `Ctrl + F`），搜索关键词如 `flag`、`key`、`ctf` 等。
3.  注意搜索时可能需要切换文本模式或十六进制模式。

## 图片隐写技术

![](img/bb78b82ce74d0619a97c65bdf8963c40_59.png)

上一节我们介绍了通用的文件操作，本节中我们聚焦于图片文件特有的隐写技术。图片因其结构复杂，为隐藏信息提供了更多可能性。

### 图片基本信息与EXIF

图片文件除了像素数据，还包含元信息（Metadata），其中最常见的是 **EXIF** 信息。EXIF 可能记录拍摄时间、相机型号、甚至 **GPS 地理位置坐标**。

查看方法：
*   **Windows**：右键点击图片 -> “属性” -> “详细信息”。
*   **Linux**：使用 `exiftool` 命令，如 `exiftool 图片名.jpg`。
*   **在线工具/专用软件**：也可以使用在线EXIF查看器或 `Google Earth` 来解析GPS坐标。

### 图片操作与通道分析

![](img/bb78b82ce74d0619a97c65bdf8963c40_61.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_63.png)

有时，`flag` 并非直接可见，而是通过对图片进行数学运算或分析颜色通道来获得。

*   **图片运算**：给出两张看似相同但有细微差别的图片，通过对它们进行**异或（XOR）、相加（ADD）、相减（SUB）** 等像素级运算，可能得到隐藏的二维码或文字。可以使用 `Stegsolve` 这类工具方便地进行各种运算。
*   **最低有效位隐写**：这是非常常见的考点。原理是修改像素RGB值每个字节的**最低有效位**，因为改动极小，人眼无法察觉，但可以编码隐藏信息。破解LSB隐写有以下工具：
    *   `Stegsolve`：在 `Analyse` -> `Data Extract` 界面中，勾选相应的位平面（Bit Planes）进行预览。
    *   `zsteg`：一个命令行工具，能自动尝试多种LSB组合，直接输出可能隐藏的数据。安装命令：`gem install zsteg`。
    *   `wbStego4.3open`：一个图形化工具，支持BMP等格式的LSB隐写解密。
    *   **Python脚本**：当现成工具无效时，可能需要根据题目细微改动，自行编写或修改Python脚本来提取信息。

![](img/bb78b82ce74d0619a97c65bdf8963c40_65.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_66.png)

### 图片修复

![](img/bb78b82ce74d0619a97c65bdf8963c40_68.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_70.png)

图片文件可能被故意损坏，导致无法正常显示。常见的修复点是 **PNG 图片的CRC校验** 或 **图片尺寸**。

![](img/bb78b82ce74d0619a97c65bdf8963c40_72.png)

修复步骤：
1.  使用 `TweakPNG` 等工具打开损坏的PNG图片，它会提示CRC错误。
2.  错误原因可能是CRC值本身被篡改，也可能是图片的宽度或高度被修改，导致计算出的CRC与存储的不符。
3.  **修复CRC**：若只是CRC错误，用十六进制编辑器找到存储CRC的字段，将其修改为 `TweakPNG` 提示的正确值即可。
4.  **修复尺寸**：如果是高度/宽度被修改，需要计算正确的值。通常可以借助已有的Python脚本，输入错误的CRC和图片数据，计算出正确的高度和宽度，再用十六进制编辑器修改对应的字段。

![](img/bb78b82ce74d0619a97c65bdf8963c40_74.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_76.png)

### 图片加密

![](img/bb78b82ce74d0619a97c65bdf8963c40_78.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_80.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_82.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_84.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_86.png)

有些图片会经过加密处理，需要先解密才能看到内容。解题的关键在于识别加密算法。

![](img/bb78b82ce74d0619a97c65bdf8963c40_90.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_92.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_93.png)

*   **识别工具**：对于JPG图片，可以使用 `stegdetect` 工具来探测是否使用了 `F5`、`outguess`、`jphide` 等常见隐写加密算法。命令如 `stegdetect -s 敏感度 图片名.jpg`。
*   **对应解密**：
    *   `F5` 加密：使用 `F5Steganography` 工具（Java编写）进行解密。
    *   `outguess` 加密：需要先编译 `outguess` 的源码，然后使用命令行解密。
    *   `jphide` 加密：使用 `JPHS` 工具套件进行解密。
    *   `BlindWatermark`：使用对应的Python脚本或工具进行解密。

![](img/bb78b82ce74d0619a97c65bdf8963c40_95.png)

![](img/bb78b82ce74d0619a97c65bdf8963c40_97.png)

### 二维码处理

![](img/bb78b82ce74d0619a97c65bdf8963c40_99.png)

千辛万苦提取出的二维码可能无法直接扫描，常见问题及处理如下：

![](img/bb78b82ce74d0619a97c65bdf8963c40_101.png)

*   **颜色反相**：二维码可能是反色的（浅底深码）。使用画图工具、`Photoshop` 或 `Stegsolve` 的“反色”功能即可。
*   **局部缺失/变形**：可能需要用图片编辑工具（如画图、PS）进行修补，将定位点等关键区域补全。
*   **分通道查看**：彩色二维码可能将信息藏在某一个颜色通道（如红色通道）中，使用 `Stegsolve` 可以单独查看每个通道的图像。

本节课中我们一起学习了杂项题型中文件操作与基础图片隐写的核心思路和工具。关键在于掌握文件类型识别、结构分析、信息提取的基本方法，并熟悉 `binwalk`、`Stegsolve`、十六进制编辑器等核心工具的使用。遇到题目时，从文件类型分析入手，逐步尝试分离、修复、通道分析等操作，是解决杂项题的有效路径。
