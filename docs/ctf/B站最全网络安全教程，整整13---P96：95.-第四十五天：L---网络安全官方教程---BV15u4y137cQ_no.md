### 1. 环境变量PATH

![](img/72d11d237ce8ae55994829cf3a389291_22.png)

![](img/72d11d237ce8ae55994829cf3a389291_23.png)

![](img/72d11d237ce8ae55994829cf3a389291_25.png)

`PATH`是Linux操作系统中的一个环境变量，它指定了系统查找可执行程序的所有目录。

![](img/72d11d237ce8ae55994829cf3a389291_27.png)

![](img/72d11d237ce8ae55994829cf3a389291_29.png)

![](img/72d11d237ce8ae55994829cf3a389291_31.png)

**作用**：当用户在终端执行任何命令时，系统会通过`PATH`变量来搜索可执行文件。
**查看方法**：使用命令 `echo $PATH`。
**输出示例**：其值通常是由冒号分隔的目录路径，例如 `/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin`。

![](img/72d11d237ce8ae55994829cf3a389291_33.png)

![](img/72d11d237ce8ae55994829cf3a389291_35.png)

![](img/72d11d237ce8ae55994829cf3a389291_37.png)

系统会按照`PATH`中目录的先后顺序进行查找，直到找到对应的可执行程序。

### 2. SUID权限

![](img/72d11d237ce8ae55994829cf3a389291_47.png)

![](img/72d11d237ce8ae55994829cf3a389291_48.png)

![](img/72d11d237ce8ae55994829cf3a389291_50.png)

![](img/72d11d237ce8ae55994829cf3a389291_52.png)

SUID（Set User ID）是一种赋予二进制可执行文件的特殊权限。

![](img/72d11d237ce8ae55994829cf3a389291_54.png)

![](img/72d11d237ce8ae55994829cf3a389291_56.png)

![](img/72d11d237ce8ae55994829cf3a389291_58.png)

![](img/72d11d237ce8ae55994829cf3a389291_59.png)

![](img/72d11d237ce8ae55994829cf3a389291_60.png)

![](img/72d11d237ce8ae55994829cf3a389291_61.png)

**特性**：
*   具有SUID权限的程序在运行时，其进程的有效用户ID（EUID）会被设置为该文件所有者的用户ID。
*   此权限**仅能作用于二进制程序**，对脚本或目录设置无效。
*   核心效果：**执行SUID程序的用户将继承该程序所有者的权限**。

![](img/72d11d237ce8ae55994829cf3a389291_62.png)

![](img/72d11d237ce8ae55994829cf3a389291_63.png)

![](img/72d11d237ce8ae55994829cf3a389291_65.png)

**查找方法**：使用以下命令可以查找系统中具有SUID权限的文件。
```bash
find / -perm -u=s -type f 2>/dev/null
```
**命令解释**：
*   `find /`：从根目录开始查找。
*   `-perm -u=s`：查找权限中包含SUID（用户位带s）的文件。
*   `-type f`：只查找文件类型。
*   `2>/dev/null`：将错误信息（如权限不足）重定向到空设备，使输出更清晰。

![](img/72d11d237ce8ae55994829cf3a389291_67.png)

## 环境变量提权实战演示

理解了核心概念后，我们来看一个具体的提权场景。假设我们通过信息收集，发现了一个具有SUID权限的自定义程序。

以下是利用环境变量提权的具体步骤：

1.  **发现SUID程序**
    执行查找命令后，我们发现一个名为 `shell` 的程序具有SUID权限，且其所有者是root。
    ```bash
    -rwsr-xr-x 1 root root 8392 May 15 10:00 /usr/local/bin/shell
    ```
    执行该程序，发现其功能是调用 `ps` 命令并显示结果。

2.  **分析提权思路**
    由于`shell`程序以root权限运行并调用了`ps`命令，而`ps`命令的查找依赖于`PATH`环境变量。如果我们能控制`PATH`，让系统优先执行我们伪造的`ps`程序，那么root权限的`shell`程序就会执行我们指定的代码。

![](img/72d11d237ce8ae55994829cf3a389291_69.png)

![](img/72d11d237ce8ae55994829cf3a389291_71.png)

3.  **实施提权操作**
    以下是操作的具体命令序列：
    ```bash
    # 1. 进入/tmp目录（通常普通用户有写权限）
    cd /tmp
    # 2. 创建一个名为ps的伪造程序，其内容为启动一个shell
    echo '/bin/bash' > ps
    # 3. 赋予伪造的ps文件可执行权限
    chmod 777 ps
    # 4. 修改当前会话的PATH环境变量，将/tmp目录置于最前
    export PATH=/tmp:$PATH
    # 5. 执行具有SUID权限的shell程序
    /usr/local/bin/shell
    ```
    **执行结果**：此时会获得一个root权限的bash shell。执行 `whoami` 命令会显示 `root`。

**原理总结**：
当root权限的`shell`程序执行时，它试图调用`ps`命令。系统根据当前被篡改的`PATH`（`/tmp:$PATH`）进行查找，首先在`/tmp`目录下找到了我们伪造的`ps`文件。于是，`shell`程序以root权限执行了`/tmp/ps`文件中的内容（`/bin/bash`），从而为我们创建了一个具有root权限的新shell。

## 其他提权方法补充

除了环境变量提权，Linux系统中还有其他常见的提权途径。

### 1. 密码哈希破解

最直接的提权方式是获取用户密码。Linux系统的用户密码哈希存储在以下两个文件中：
*   `/etc/passwd`：存储所有用户信息，所有用户可读。密码字段现在通常为`x`，表示密码哈希已移至`shadow`文件。
*   `/etc/shadow`：存储加密后的密码哈希，**只有root用户可读**。

![](img/72d11d237ce8ae55994829cf3a389291_75.png)

**利用方式**：如果因配置错误导致`/etc/passwd`中直接包含了密码哈希（第二字段不是`x`），或者通过某种方式读取到了`/etc/shadow`文件，就可以尝试使用工具（如`john`、在线CMD5平台）对哈希进行破解。

![](img/72d11d237ce8ae55994829cf3a389291_77.png)

![](img/72d11d237ce8ae55994829cf3a389291_79.png)

### 2. 内核漏洞提权

![](img/72d11d237ce8ae55994829cf3a389291_81.png)

这是通过利用操作系统内核本身的漏洞进行提权。

![](img/72d11d237ce8ae55994829cf3a389291_83.png)

**利用流程**：
1.  信息收集：使用 `uname -a` 获取系统内核版本。
2.  漏洞检索：根据内核版本，在漏洞数据库（如Exploit-DB）中搜索公开的提权漏洞利用代码（EXP）。
3.  编译与执行：将EXP上传到目标机器，编译后执行。
4.  **注意**：内核提权成功率取决于系统是否存在特定漏洞且未打补丁，具有一定运气成分。

**示例**：CVE-2019-13272漏洞，影响内核版本4.10-5.1.x。在符合条件的系统上，编译执行EXP即可获得root权限。

![](img/72d11d237ce8ae55994829cf3a389291_85.png)

![](img/72d11d237ce8ae55994829cf3a389291_87.png)

### 3. 服务漏洞提权

![](img/72d11d237ce8ae55994829cf3a389291_89.png)

通过利用系统中以root权限运行的服务（如Snap包管理器、Docker、MySQL等）的漏洞进行提权。

![](img/72d11d237ce8ae55994829cf3a389291_91.png)

![](img/72d11d237ce8ae55994829cf3a389291_93.png)

![](img/72d11d237ce8ae55994829cf3a389291_95.png)

**示例**：Ubuntu Snap包管理器本地提权漏洞（CVE-2019-7304）。在受影响版本的Ubuntu上，利用公开的EXP可以创建一个具有root权限的新用户（`dirty_socks`），从而完成提权。
