## 可信任服务路径漏洞提权 🔍
上一节我们介绍了提权的基本概念，本节中我们来看看如何利用“可信任服务路径”这一特性进行提权。

该漏洞的核心在于：Windows服务通常以`SYSTEM`权限运行。当系统解析服务二进制文件路径中的空格时，会按顺序尝试执行空格前的目录名与`.exe`组合的程序。如果攻击者能在有写入权限的目录下放置恶意程序，并在服务重启时被以`SYSTEM`权限加载，即可实现提权。

![](img/abad8b0599fccfe734439d92f669521d_49.png)

![](img/abad8b0599fccfe734439d92f669521d_51.png)

![](img/abad8b0599fccfe734439d92f669521d_53.png)

![](img/abad8b0599fccfe734439d92f669521d_55.png)

以下是利用此漏洞的基本步骤：
1.  **检查存在漏洞的服务**：使用`wmic service`命令列出服务，并筛选出二进制路径中包含空格且路径前段目录有写入权限的服务。
2.  **检查目录写入权限**：使用`icacls`命令检查目标目录的权限，确认`Everyone`或当前用户是否有完全控制(`F`)权限。
3.  **生成并上传Payload**：使用MSF生成反向Shell的exe文件，并将其以特定名称（如`Common.exe`）上传到有写入权限的目录。
4.  **重启服务**：等待或促使目标服务重启。服务重启时会以`SYSTEM`权限执行我们上传的恶意程序。
5.  **稳定会话**：由于服务控制管理器会终止未与其通信的进程，因此需要在MSF接收到Shell后，立即使用`migrate`命令将进程迁移到其他稳定进程（如`explorer.exe`）中。

## 不安全的服务权限提权 ⚙️

以下是操作流程：
1.  **使用AccessChk工具**：上传Sysinternals套件中的`accesschk.exe`工具，检查当前用户对哪些服务拥有`SERVICE_ALL_ACCESS`权限。命令为：`accesschk.exe -uwcqv *`
2.  **修改服务配置**：通过`sc config`命令修改目标服务的`binPath`参数。例如，将其改为添加用户的命令或反弹Shell的exe路径：`sc config "VulnerableService" binPath= "C:\Windows\System32\net.exe user hacker Password123! /add"`
3.  **重启服务**：尝试停止并启动服务以触发命令执行。普通用户可能无权限操作服务，需等待系统重启或服务自动重启。
4.  **验证提权**：如果命令是添加用户，则验证新用户是否创建成功；如果是执行Payload，则在MSF中接收`SYSTEM`权限的Shell。

## 不安全的注册表权限提权 📁
除了直接修改服务属性，我们还可以通过修改服务在注册表中的配置来实现相同目的。Windows服务的配置信息存储在注册表的 `HKLM\SYSTEM\CurrentControlSet\Services\` 键下。

![](img/abad8b0599fccfe734439d92f669521d_182.png)

![](img/abad8b0599fccfe734439d92f669521d_184.png)

![](img/abad8b0599fccfe734439d92f669521d_186.png)

![](img/abad8b0599fccfe734439d92f669521d_188.png)

![](img/abad8b0599fccfe734439d92f669521d_190.png)

![](img/abad8b0599fccfe734439d92f669521d_192.png)

如果当前用户对某个服务对应的注册表项拥有写入权限，就可以直接修改其`ImagePath`值，达到与修改服务`binPath`相同的效果。

以下是具体步骤：
1.  **检查注册表权限**：使用`subinacl`等工具检查`HKLM\SYSTEM\CurrentControlSet\Services\`下各子项的权限，寻找当前用户有写入权限的服务项。
2.  **修改注册表键值**：使用`reg add`命令修改目标服务的`ImagePath`值。例如，将其指向恶意程序：
    `reg add "HKLM\SYSTEM\CurrentControlSet\Services\VulnService" /v ImagePath /t REG_EXPAND_SZ /d "C:\temp\malware.exe" /f`
3.  **触发执行**：重启目标服务或等待系统重启，使修改生效。

## 其他提权相关知识点
- **AlwaysInstallElevated策略**：如果系统启用了此策略（对应注册表项值为1），则任何用户都能以`SYSTEM`权限安装MSI包。可以利用MSF生成恶意的MSI安装包，执行后即可提权。
- **启动项注册表**：`HKLM\Software\Microsoft\Windows\CurrentVersion\Run`等注册表位置用于设置开机自启动程序。如果有写入权限，可以在此处添加恶意程序路径实现权限维持。
- **内核漏洞提权**：通过系统信息收集（如`systeminfo`、`wmic qfe get`）和工具（如`Windows-Exploit-Suggester`、`Watson`、`Sherlock` PowerShell脚本）查找未修补的内核漏洞，并利用公开的EXP（如MS16-032）进行提权。
