# 护网行动红蓝攻防教程：P24：应急响应团队组建 👥

在本节课中，我们将要学习应急响应团队是如何组建的，了解团队内部的不同角色分工及其职责。这对于理解护网行动的实际运作至关重要。

## 团队构成

应急响应团队需要哪些人员？你可以根据自身情况，思考适合担任什么样的角色。团队主要分为内部团队和外部团队，这与我们参加护网行动的结构类似。

## 内部团队分工

内部团队是应急响应的核心力量，通常分为监控组、响应组、研判组和文档组。以下是各小组的具体职责介绍。

*   **监控组**：该组负责利用各类系统监控工具查看系统日志，对系统、应用、网络及数据安全进行持续检测。其工作类似于保安查看监控画面，发现异常情况。若问题能自行解决则处理，无法解决则需向上级报告。
*   **响应组**：该组接收来自监控组的报告，负责初步判断事件类型（如属于应用、系统或设备问题），并指派相应人员进行现场或远程响应处理。若响应组仍无法解决问题，则需继续上报。
*   **研判组**：该组由专家组成，负责处理响应组上报的复杂事件，进行深入的溯源分析，以确定攻击来源、手法和影响范围。
*   **文档组**：该组负责制定应急响应预案，并在事件处理后，整理输出完整的事件报告与经验总结文档。

基本上，初次参加护网行动的人员，很多会进入监控组或文档组。如果你的技术经验足够丰富，也可能进入研判组。

## 为何需要外部招募

上一节我们介绍了内部团队的构成，本节中我们来看看为何护网行动还需要从外部招募人员。这主要关系到工作强度与人力资源调配。

护网行动期间工作强度可能较大，但相应的薪资也较高。以银行为例，其内部员工通常有固定的朝九晚五工作时间。若要求他们在夜间或凌晨加班进行安全监控与响应，可能意愿不高。因此，相关单位会提供报酬，从外部招募人员来承担这部分工作。

所以，护网行动经常需要工作到凌晨，实行白班与夜班排班制度，这是很正常的现象。

## 外部团队构成

除了内部团队，应急响应体系还依赖于外部团队的支持。这些团队主要来自合作单位。

*   **安全厂商/安服团队**：许多公司会与专业的安全公司合作。这些安全公司会派遣由安全服务工程师组成的团队（即安服团队）外派并驻场，协助客户进行护网等安全行动。
*   **监管单位**：主要指网信办、公安部等政府监管机构，他们负责指导与监督整体的网络安全工作。

本节课中我们一起学习了应急响应团队的完整架构。我们明确了内部团队中监控、响应、研判、文档四个小组的职责与协作流程，也理解了为何需要从外部招募人员以补充人力资源，最后认识了外部合作团队与监管单位的角色。掌握这些知识，有助于你更好地定位自己在护网行动中的潜在角色。
