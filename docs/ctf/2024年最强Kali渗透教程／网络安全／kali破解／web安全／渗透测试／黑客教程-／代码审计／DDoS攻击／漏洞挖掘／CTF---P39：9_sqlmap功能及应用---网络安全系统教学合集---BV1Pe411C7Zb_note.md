## 漏洞检测

上一节我们介绍了基本参数，本节中我们来看看如何进行漏洞检测。检测漏洞主要使用 `-u` 参数。

![](img/35e707a2b79485026ca2452d42945504_26.png)

![](img/35e707a2b79485026ca2452d42945504_28.png)

`-u` 参数后面需要跟一个目标URL。我们也可以使用 `-m` 参数对一个包含多个URL的文本文件进行批量检测。

![](img/35e707a2b79485026ca2452d42945504_30.png)

![](img/35e707a2b79485026ca2452d42945504_32.png)

以下是使用 `-u` 参数进行检测的示例命令：
```bash
sqlmap -u "http://target.com/page.php?id=1"
```
执行命令后，SQLMap会开始检测。如果发现疑似注入点，它会进行交互式询问。例如，它可能提示后端数据库看起来像是MySQL，并询问是否跳过测试其他数据库。此时可以输入 `Y` 确认。

![](img/35e707a2b79485026ca2452d42945504_34.png)

随后，工具可能会询问是否包含所有针对MySQL的测试，可以根据需要选择 `Y` 或 `N`。

![](img/35e707a2b79485026ca2452d42945504_36.png)

如果检测到某个参数（例如 `id`）存在注入漏洞，SQLMap会提示该参数是脆弱的，并询问是否测试其他参数。选择 `N` 后，将看到最终的检测结果，例如确认存在基于GET请求的SQL注入漏洞。

![](img/35e707a2b79485026ca2452d42945504_38.png)

对于批量检测，可以先将多个URL保存到一个文本文件中（每行一个URL），然后使用 `-m` 参数：
```bash
sqlmap -m urls.txt
```

![](img/35e707a2b79485026ca2452d42945504_40.png)

## 数据枚举

![](img/35e707a2b79485026ca2452d42945504_42.png)

检测到注入漏洞后，下一步是枚举数据库中的信息。我们将按照数据库 -> 表 -> 列 -> 数据的顺序进行。

![](img/35e707a2b79485026ca2452d42945504_44.png)

### 枚举数据库

![](img/35e707a2b79485026ca2452d42945504_46.png)

![](img/35e707a2b79485026ca2452d42945504_48.png)

首先，我们可以枚举数据库管理系统中的所有数据库。使用 `--dbs` 参数。
```bash
sqlmap -u "http://target.com/page.php?id=1" --dbs
```
执行后，SQLMap会列出所有数据库名称。

我们也可以查询当前网站正在使用的数据库，使用 `--current-db` 参数。
```bash
sqlmap -u "http://target.com/page.php?id=1" --current-db
```
在实际漏洞提交（如SRC）时，通常枚举出当前数据库名就足以证明漏洞存在。

![](img/35e707a2b79485026ca2452d42945504_50.png)

### 枚举数据表

![](img/35e707a2b79485026ca2452d42945504_52.png)

确定了目标数据库后，接下来枚举该数据库中的所有表。使用 `-D` 指定数据库名，`--tables` 参数枚举表。
```bash
sqlmap -u "http://target.com/page.php?id=1" -D database_name --tables
```
命令执行后，会列出指定数据库中的所有表名。

![](img/35e707a2b79485026ca2452d42945504_54.png)

![](img/35e707a2b79485026ca2452d42945504_56.png)

### 枚举表的列

![](img/35e707a2b79485026ca2452d42945504_58.png)

![](img/35e707a2b79485026ca2452d42945504_60.png)

获取表名后，可以进一步枚举指定表中的列名。使用 `-T` 指定表名，`--columns` 参数枚举列。
```bash
sqlmap -u "http://target.com/page.php?id=1" -D database_name -T table_name --columns
```
例如，针对 `users` 表，可能会枚举出 `id`、`username`、`password` 等列。

![](img/35e707a2b79485026ca2452d42945504_62.png)

### 提取数据（脱库）

![](img/35e707a2b79485026ca2452d42945504_63.png)

最后一步是提取表中的具体数据，即“脱库”。使用 `-C` 指定要提取的列名，`--dump` 参数执行数据提取。
```bash
sqlmap -u "http://target.com/page.php?id=1" -D database_name -T table_name -C "username,password" --dump
```
**重要提示**：在未获得明确授权的渗透测试或漏洞挖掘中，**请谨慎使用 `--dump` 命令**。除非是在有合法授权的渗透项目中，否则提取用户数据可能涉及法律风险。在常规漏洞提交中，证明到列枚举步骤通常已足够。

![](img/35e707a2b79485026ca2452d42945504_65.png)

执行 `--dump` 后，数据会被提取并保存到本地（通常是CSV格式文件），便于查看和分析。

## 绕过过滤（Tamper脚本）

![](img/35e707a2b79485026ca2452d42945504_67.png)

![](img/35e707a2b79485026ca2452d42945504_69.png)

![](img/35e707a2b79485026ca2452d42945504_70.png)

在实际测试中，网站可能对SQL注入的关键词进行了过滤。SQLMap提供了Tamper脚本功能来绕过这些过滤。

![](img/35e707a2b79485026ca2452d42945504_72.png)

![](img/35e707a2b79485026ca2452d42945504_74.png)

Tamper脚本通过对注入载荷（Payload）进行编码、混淆或添加特殊字符来实现绕过。使用 `--tamper` 参数指定脚本。

以下是Tamper脚本的使用示例：
```bash
sqlmap -u "http://target.com/page.php?id=1" --tamper=space2comment
```
*   `space2comment` 脚本会将空格替换为注释符 `/**/` 来绕过空格过滤。
*   其他常见脚本如 `between` 用于绕过 `BETWEEN` 过滤，`randomcase` 用于将关键词随机大小写。

每个Tamper脚本都有其特定的绕过原理，例如双写关键词、插入百分号 `%` 等。使用时需要根据目标网站的过滤机制来选择合适的脚本。可以通过 `--eval` 参数查看Payload的具体变化过程。
