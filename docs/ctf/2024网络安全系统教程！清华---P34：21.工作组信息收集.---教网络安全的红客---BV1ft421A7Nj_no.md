![](img/936d583bfdb679597955ab677525dd06_1.png)

![](img/936d583bfdb679597955ab677525dd06_3.png)

在本节课中，我们将要学习内网渗透的第一步：工作组环境下的信息收集。我们将了解如何从一个已获得权限的跳板机出发，系统地收集目标内网环境中的各类信息，为后续的横向和纵向渗透打下基础。

## 内网渗透概述

上一节我们介绍了内网渗透的基本概念，本节中我们来看看如何开始具体的信息收集工作。

进行内网渗透的前提是获得一个能够访问内网的“入口”。这个入口通常是一个位于外网但能连通内网的服务器（跳板机）。例如，通过攻击一个对外提供Web服务的服务器，获得其Webshell权限，如果这台服务器恰好拥有连接内网的网卡（如IP地址为 `10.10.10.x`），那么我们就获得了进入内网的通道。

## 什么是工作组？

在深入了解信息收集方法前，我们需要理解“工作组”这个概念。工作组是Windows网络中最简单的一种资源管理模式，它将功能相近的计算机归入同一组以便管理。

![](img/936d583bfdb679597955ab677525dd06_5.png)

![](img/936d583bfdb679597955ab677525dd06_7.png)

![](img/936d583bfdb679597955ab677525dd06_9.png)

*   **默认设置**：所有Windows计算机默认都处于名为 **`WORKGROUP`** 的工作组中。你可以在“此电脑” -> “属性” -> “计算机名、域和工作组设置”中查看和修改。
*   **适用场景**：适用于计算机数量不多、管理要求不严格的小型网络环境。
*   **资源共享**：在同一个工作组内，用户可以通过对方主机的用户名和密码访问其共享的文件夹。在“网络”中启用网络发现后，可以看到同工作组内的其他计算机。

## 信息收集的核心思路

![](img/936d583bfdb679597955ab677525dd06_18.png)

![](img/936d583bfdb679597955ab677525dd06_20.png)

![](img/936d583bfdb679597955ab677525dd06_22.png)

信息收集的目的是全面了解目标环境。在内网中，我们尤其需要关注不同网段，这涉及到两个核心概念：

![](img/936d583bfdb679597955ab677525dd06_23.png)

![](img/936d583bfdb679597955ab677525dd06_25.png)

![](img/936d583bfdb679597955ab677525dd06_27.png)

*   **横向渗透**：指在**当前跳板机可直接访问的网段**内进行渗透。例如，跳板机在 `10.10.10.0/24` 网段，那么渗透此网段内的其他机器就属于横向渗透。
*   **纵向渗透**：指通过横向渗透拿下的机器作为**新的跳板**，去访问和渗透**原跳板机无法直接到达的其他网段**。例如，发现一台机器拥有 `10.10.11.0/24` 网段的网卡，通过它渗透该网段就属于纵向渗透。

![](img/936d583bfdb679597955ab677525dd06_29.png)

![](img/936d583bfdb679597955ab677525dd06_31.png)

发现不同网段是进行纵向渗透的基础。以下是发现内网网段的一些思路：

![](img/936d583bfdb679597955ab677525dd06_33.png)

![](img/936d583bfdb679597955ab677525dd06_35.png)

![](img/936d583bfdb679597955ab677525dd06_37.png)

*   对内网IP段进行扫描，发现存活主机和网段。
*   检查跳板机上的各种连接记录，如：
    *   文件共享、FTP连接记录
    *   浏览器历史记录（可能访问过内网服务）
    *   远程桌面（MSTSC）连接记录
*   检查路由器、交换机的配置信息（如果可获得权限）。

## 工作组信息收集实战命令

![](img/936d583bfdb679597955ab677525dd06_46.png)

以下我们将通过一系列命令，演示如何从一台已控制的Windows主机（跳板机）上收集信息。假设我们已通过CS（Cobalt Strike）等工具获得了该主机的Shell。

![](img/936d583bfdb679597955ab677525dd06_48.png)

### 1. 用户信息收集

了解系统上有哪些用户以及他们的权限。

![](img/936d583bfdb679597955ab677525dd06_50.png)

![](img/936d583bfdb679597955ab677525dd06_52.png)

*   **查看所有用户**：
    ```cmd
    net user
    ```
*   **查看本地管理员组用户**：
    ```cmd
    net localgroup administrators
    ```
*   **查看当前在线用户**：
    ```cmd
    query user
    ```
    或
    ```cmd
    qwinsta
    ```

### 2. 系统与网络信息收集

![](img/936d583bfdb679597955ab677525dd06_62.png)

![](img/936d583bfdb679597955ab677525dd06_64.png)

获取主机的基础配置和网络环境。

![](img/936d583bfdb679597955ab677525dd06_67.png)

![](img/936d583bfdb679597955ab677525dd06_69.png)

![](img/936d583bfdb679597955ab677525dd06_71.png)

![](img/936d583bfdb679597955ab677525dd06_73.png)

*   **查看详细的IP配置（主机名、DNS、网卡信息）**：
    ```cmd
    ipconfig /all
    ```
    此命令输出中的“DNS后缀”和“连接特定的DNS后缀”若为域名形式（如 `de1ay.com`），则暗示该主机可能加入了域。
*   **查看系统版本信息**：
    ```cmd
    systeminfo | findstr /B /C:"OS Name" /C:"OS Version"
    ```
    或使用WMIC命令：
    ```cmd
    wmic os get caption,version,osarchitecture
    ```
*   **查看安装的软件**：
    ```cmd
    wmic product get name,version
    ```

### 4. 密码、凭据与共享信息

![](img/936d583bfdb679597955ab677525dd06_120.png)

![](img/936d583bfdb679597955ab677525dd06_122.png)

寻找存储的密码、凭据和网络共享资源。

![](img/936d583bfdb679597955ab677525dd06_124.png)

![](img/936d583bfdb679597955ab677525dd06_125.png)

![](img/936d583bfdb679597955ab677525dd06_127.png)

*   **查看本机存储的凭据**：
    ```cmd
    cmdkey /list
    ```
*   **查看本机共享列表**：
    ```cmd
    net share
    ```
*   **映射网络驱动器**：在获得目标主机凭据后，可以将其共享目录映射到本地。
    ```cmd
    net use Z: \\10.10.10.10\C$ /user:administrator Password123
    ```
    此命令将 `10.10.10.10` 的C盘映射到本地的Z盘。

收集可能对后续渗透有帮助的零散信息。

![](img/936d583bfdb679597955ab677525dd06_146.png)

![](img/936d583bfdb679597955ab677525dd06_148.png)

![](img/936d583bfdb679597955ab677525dd06_150.png)

![](img/936d583bfdb679597955ab677525dd06_152.png)

![](img/936d583bfdb679597955ab677525dd06_155.png)

![](img/936d583bfdb679597955ab677525dd06_157.png)

*   **查看已安装的补丁**：
    ```cmd
    wmic qfe get caption,description,hotfixid,installedon
    ```
*   **查看杀毒软件**：
    ```cmd
    wmic /namespace:\\root\securitycenter2 path antivirusproduct get displayname,productstate
    ```
*   **查看代理设置**：
    ```cmd
    reg query "HKCU\Software\Microsoft\Windows\CurrentVersion\Internet Settings" /v ProxyServer
    ```
*   **获取已连接Wi-Fi的密码**：
    ```cmd
    netsh wlan show profile name="Wi-Fi名称" key=clear
    ```
    在输出中查找“关键内容”字段。
*   **读取回收站内容**：通过遍历用户SID，找到回收站目录（`$Recycle.Bin`），其中存放着被删除文件的元信息（`$I`开头的文件）和原始内容（`$R`开头的文件）。
*   **获取浏览器历史记录与Cookie**：Chrome浏览器的数据通常位于：
    ```
    %LocalAppData%\Google\Chrome\User Data\Default\History
    %LocalAppData%\Google\Chrome\User Data\Default\Cookies
    ```
    可以使用Mimikatz等工具读取这些SQLite数据库文件。

### 6. 自动化信息收集工具

![](img/936d583bfdb679597955ab677525dd06_161.png)

除了手动执行命令，也可以使用自动化脚本或框架模块。

*   **PowerShell 脚本**：如 `Get-Information.ps1` 可以批量收集多项信息。
*   **Metasploit 模块**：在获得Meterpreter会话后，可以使用 `run scraper` 或 `run winenum` 等后渗透模块进行自动化信息收集。
