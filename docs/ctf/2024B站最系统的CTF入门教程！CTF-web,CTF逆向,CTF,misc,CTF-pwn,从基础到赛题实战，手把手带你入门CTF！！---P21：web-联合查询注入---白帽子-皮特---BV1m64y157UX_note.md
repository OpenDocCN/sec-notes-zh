## 判断漏洞是否存在

![](img/6d867f6bfe50cc6723d35bca43d14093_12.png)

![](img/6d867f6bfe50cc6723d35bca43d14093_14.png)

上一节我们介绍了SQL注入的基本概念，本节中我们来看看如何判断一个Web应用是否存在SQL注入漏洞。

![](img/6d867f6bfe50cc6723d35bca43d14093_16.png)

判断漏洞是否存在的方法是：根据客户端返回的结果，来判断提交的测试语句是否被数据库引擎成功执行。如果语句被执行，说明存在漏洞。通常使用单引号或双引号进行判断。

![](img/6d867f6bfe50cc6723d35bca43d14093_18.png)

![](img/6d867f6bfe50cc6723d35bca43d14093_20.png)

**注意**：所有输入的命令都必须使用英文符号，切勿使用中文符号。使用中文符号可能导致难以排查的错误。

以下是判断漏洞是否存在的基本步骤：
1.  在输入点（如用户ID）后添加一个英文单引号 `'` 或双引号 `"`。
2.  提交请求，观察页面返回。
3.  如果页面返回了SQL语法错误信息，则很大可能存在SQL注入漏洞。

![](img/6d867f6bfe50cc6723d35bca43d14093_22.png)

![](img/6d867f6bfe50cc6723d35bca43d14093_24.png)

例如，在DVWA靶场的数据库注入板块，输入用户ID为 `1'` 时，页面返回了SQL语法错误，这提示我们此处可能存在注入点。

## 判断注入类型

![](img/6d867f6bfe50cc6723d35bca43d14093_30.png)

在确认存在漏洞后，我们需要判断注入点的类型，是数字型还是字符型，这决定了后续注入语句的构造方式。

![](img/6d867f6bfe50cc6723d35bca43d14093_32.png)

![](img/6d867f6bfe50cc6723d35bca43d14093_34.png)

判断注入类型的方法是：通过逻辑运算测试后端查询语句的处理方式。

以下是判断注入类型的测试方法：
1.  输入 `1 and 1=1`，观察页面正常返回。
2.  输入 `1 and 1=2`，观察页面返回。
    *   如果两次返回的结果**不同**，则可能是**数字型**注入。
    *   如果两次返回的结果**相同**，则可能是**字符型**注入。

![](img/6d867f6bfe50cc6723d35bca43d14093_36.png)

![](img/6d867f6bfe50cc6723d35bca43d14093_38.png)

**原理分析**：
*   对于数字型注入，后端查询语句可能类似：`SELECT ... WHERE id = 1`。`1 and 1=1` 结果为真，页面正常；`1 and 1=2` 结果为假，页面异常。
*   对于字符型注入，后端查询语句可能类似：`SELECT ... WHERE id = ‘1‘`。当我们输入 `1‘ and ‘1‘=‘1` 时，数据库在进行隐式类型转换后，`‘1‘ and ‘1‘=‘1` 和 `‘1‘ and ‘1‘=‘2` 都可能被转换为 `1`，导致两次查询结果相同。

![](img/6d867f6bfe50cc6723d35bca43d14093_40.png)

通过上述测试，我们可以确定当前靶场的注入点为字符型注入。

## 确定查询列数

![](img/6d867f6bfe50cc6723d35bca43d14093_52.png)

![](img/6d867f6bfe50cc6723d35bca43d14093_54.png)

确定了注入类型后，为了使用`UNION`联合查询，我们必须知道后端`SELECT`语句查询结果的列数。

![](img/6d867f6bfe50cc6723d35bca43d14093_55.png)

![](img/6d867f6bfe50cc6723d35bca43d14093_57.png)

![](img/6d867f6bfe50cc6723d35bca43d14093_59.png)

确定列数的方法是使用 `ORDER BY` 子句进行猜测。

以下是确定查询列数的步骤：
1.  构造注入语句，闭合前面的引号。例如对于字符型注入，输入 `1‘ order by 1 -- ` （`--` 用于注释掉后面的语句）。
2.  逐渐增加 `ORDER BY` 后面的数字（如 `order by 2`, `order by 3`...）。
3.  当页面正常显示时，说明该列存在。当页面报错（例如提示“Unknown column”）时，说明该列不存在。
4.  页面正常显示的最大数字，即为查询结果的列数。

在我们的靶场示例中，`order by 2` 页面正常，`order by 3` 页面报错，因此可以确定查询结果为 **2列**。

**注意**：在URL中，注释符 `--` 后需要加一个空格，或者使用 `#`（其URL编码为`%23`）。

## 联合查询注入

![](img/6d867f6bfe50cc6723d35bca43d14093_61.png)

![](img/6d867f6bfe50cc6723d35bca43d14093_63.png)

现在我们已经知道查询列数为2，并且是字符型注入。接下来，我们将利用 `UNION` 操作符进行联合查询注入，以获取数据库信息。

![](img/6d867f6bfe50cc6723d35bca43d14093_65.png)

![](img/6d867f6bfe50cc6723d35bca43d14093_67.png)

`UNION` 操作符用于合并两个或多个 `SELECT` 语句的结果集。

使用 `UNION` 有两个核心要求：
1.  `UNION` 内部的每个 `SELECT` 语句必须拥有**相同数量**的列。
2.  列也必须拥有相似的数据类型（通常我们使用数字或字符串占位）。
3.  结果集的列名总是等于第一个 `SELECT` 语句中的列名。

![](img/6d867f6bfe50cc6723d35bca43d14093_69.png)

![](img/6d867f6bfe50cc6723d35bca43d14093_70.png)

![](img/6d867f6bfe50cc6723d35bca43d14093_72.png)

以下是进行联合查询注入的步骤：
1.  **构造联合查询语句**：根据确定的列数（2列）和注入类型（字符型），构造语句。例如：`1‘ union select 1,2 -- `。
2.  **确定显示位**：提交上述语句后，观察页面中哪个位置显示了数字“1”和“2”。这两个位置就是我们可以用来回显查询数据的“显示位”。
3.  **获取数据**：将 `select 1,2` 替换为我们想要查询的数据。例如，想查询当前数据库的用户名和密码，可以构造：`1‘ union select user, password from users -- `。这样，用户名就会显示在第一个显示位，密码显示在第二个显示位。
4.  **处理单行回显**：有些网站只输出查询结果的第一行。为了让我们注入的查询结果能够显示出来，可以将原查询条件设置为一个不存在的值，例如 `-1‘ union select user, password from users -- `。这样，第一条（原）查询结果为空，页面就会显示我们注入的查询结果。

在我们的靶场演示中，通过 `1‘ union select 1,2 -- ` 确定了显示位，然后成功用 `1‘ union select user, password from users -- ` 查询到了用户名和密码信息。

## 总结

![](img/6d867f6bfe50cc6723d35bca43d14093_76.png)

本节课中我们一起学习了SQL联合查询注入的完整流程：
1.  **判断漏洞**：通过添加单引号`‘`，观察是否报错来判断是否存在SQL注入漏洞。
2.  **判断类型**：通过 `and 1=1` 和 `and 1=2` 的返回结果差异，判断是数字型还是字符型注入。
3.  **确定列数**：使用 `order by` 子句递增猜测，直到报错，以确定 `SELECT` 查询的列数。
4.  **联合查询**：利用 `UNION` 操作符，构造拥有相同列数的 `SELECT` 语句进行联合查询，先确定显示位，再获取数据库信息。
