在本节课中，我们将学习CTF比赛中流量取证（Traffic Analysis）的核心知识与实战技巧。流量取证旨在分析网络数据包（Traffic），从中发现隐藏的信息、提取文件或还原通信过程。我们将从基础概念讲起，逐步深入到USB、无线网络等特定流量的分析方法。

## 概述：什么是网络流量？

![](img/3179e83078da5a0852bf8144605437ab_7.png)

![](img/3179e83078da5a0852bf8144605437ab_8.png)

网络流量（Traffic）是在网络中传输的数据包集合。当数据通过网络发送时，会被封装成多个数据包进行传输。由于单个数据包的有效载荷（Payload）通常限制在1500字节以内，较大的数据（如文件）会被分割成多个包。因此，分析流量时，常常需要将相关的数据包组合还原，才能得到完整、可读的信息。

![](img/3179e83078da5a0852bf8144605437ab_10.png)

## 流量包的基本结构与查看方法

上一节我们介绍了流量的基本概念，本节中我们来看看如何查看和分析一个流量包。

一个网络数据包通常包含多个层次的信息，类似于寄信：数据本身是信的内容，而各种协议头（如IP地址、端口）则像是信封上的地址信息。在Wireshark等分析工具中，我们可以查看这些详细信息。

一个数据包的核心信息通常由“五元组”构成：
*   **源IP地址**：发送方的逻辑地址。
*   **目的IP地址**：接收方的逻辑地址。
*   **源端口**：发送方应用程序的端口。
*   **目的端口**：接收方应用程序的端口。
*   **协议**：使用的通信协议（如TCP、UDP、HTTP）。

在Wireshark中，我们可以通过统计功能（Statistics -> Protocol Hierarchy）快速了解流量中各种协议的分布比例，这有助于判断数据的藏匿之处。

## 明文协议流量的简单分析

![](img/3179e83078da5a0852bf8144605437ab_12.png)

![](img/3179e83078da5a0852bf8144605437ab_14.png)

有些协议（如SMTP、Telnet）在传输时是明文的，这意味着数据可以直接被读取。以下是分析此类流量的基本思路：

![](img/3179e83078da5a0852bf8144605437ab_16.png)

![](img/3179e83078da5a0852bf8144605437ab_18.png)

1.  **打开流量文件**，使用过滤器（如 `smtp` 或 `telnet`）筛选出相关协议的数据包。
2.  **追踪TCP流**：在关键数据包上右键，选择 `Follow` -> `TCP Stream`。这会将属于同一次通信的所有数据包按顺序组合显示。
3.  **查找关键信息**：在组合后的数据中，直接查找可能包含flag的字符串，如登录凭证（username, password）、文件内容或特定的命令输出。

![](img/3179e83078da5a0852bf8144605437ab_20.png)

![](img/3179e83078da5a0852bf8144605437ab_21.png)

![](img/3179e83078da5a0852bf8144605437ab_23.png)

![](img/3179e83078da5a0852bf8144605437ab_25.png)

![](img/3179e83078da5a0852bf8144605437ab_27.png)

例如，在一个SMTP流量包中，我们可能直接在追踪出的TCP流里看到用户输入的密码（可能是Base64编码），解码后即可获得flag。

### 1. 文件提取与重组
当流量中包含文件上传或下载时，文件数据可能分散在多个包中。我们需要将其导出并重组。
*   **方法**：在Wireshark中，使用 `File` -> `Export Objects` -> `HTTP...` 功能，可以自动识别并提取通过HTTP传输的文件。

![](img/3179e83078da5a0852bf8144605437ab_49.png)

### 2. 加密流量（HTTPS）的解密
HTTPS流量默认是加密的。但如果题目提供了服务器的私钥（RSA Key），我们可以解密它。
*   **方法**：在Wireshark的 `编辑` -> `首选项` -> `Protocols` -> `TLS` 中，添加提供的RSA密钥文件。之后，HTTPS流量就会被解密为HTTP流量，可以按明文方式分析。

### 3. 无线网络（Wi-Fi）流量分析
无线流量包（协议为802.11）通常经过加密（WPA/WPA2）。分析此类包往往需要破解密码。
*   **所需工具**：`aircrack-ng` 套件。
*   **基本命令**：`aircrack-ng -w <密码字典文件> <无线流量包文件>`。该命令会尝试使用字典中的密码来破解握手包，成功后即可获得密钥。

#### 键盘流量分析
USB键盘击键信息包含在数据包的特定字段中。每个按键都对应一个扫描码（Scan Code）。
*   **分析步骤**：
    1.  使用 `tshark` 命令提取USB键盘数据：`tshark -r usb.pcap -T fields -e usb.capdata > keyboard.txt`
    2.  使用Python脚本，根据扫描码映射表，将提取出的十六进制数据转换为实际的按键字符。

#### 鼠标流量分析
USB鼠标流量包含鼠标移动（X/Y坐标偏移）和按键（左键、右键）信息。
*   **分析步骤**：
    1.  同样使用 `tshark` 提取相关数据字段：`tshark -r mouse.pcap -T fields -e usb.capdata > mouse.txt`
    2.  使用Python脚本解析坐标偏移量。脚本通常需要指定关注哪种按键事件（如只记录左键按下时的移动轨迹）。
    3.  将解析出的坐标数据，使用绘图工具（如gnuplot）生成轨迹图，图像中可能隐藏着flag信息。

本节课中我们一起学习了CTF流量取证的核心方法。面对一个未知的流量包，我们可以遵循以下通用分析思路：

![](img/3179e83078da5a0852bf8144605437ab_128.png)

![](img/3179e83078da5a0852bf8144605437ab_130.png)

![](img/3179e83078da5a0852bf8144605437ab_132.png)

![](img/3179e83078da5a0852bf8144605437ab_134.png)

1.  **协议统计**：首先使用 `Statistics` -> `Protocol Hierarchy` 了解流量组成，判断重点。
2.  **过滤追踪**：根据协议统计结果，应用过滤器缩小范围，并对关键的TCP流或HTTP流进行追踪（Follow Stream）。
3.  **搜索关键字**：在追踪出的流或整个包中，搜索 `flag`、`key`、`password`、`exec`、`system` 等常见关键字。
4.  **文件导出**：检查是否有文件传输，尝试导出对象。
5.  **特定类型处理**：识别是否为USB、无线等特殊流量，并运用对应的工具和脚本进行分析。
6.  **数据重组**：对于分散的数据，思考是否需要组合、解码或绘图来获取最终信息。

![](img/3179e83078da5a0852bf8144605437ab_136.png)

记住，流量分析的核心在于耐心和细心，以及对网络协议层的理解。多加练习，你就能熟练地从数据包的海洋中捕获到隐藏的旗帜。
