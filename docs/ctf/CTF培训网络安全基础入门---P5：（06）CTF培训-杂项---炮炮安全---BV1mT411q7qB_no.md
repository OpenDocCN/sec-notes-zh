## 压缩文件处理 📦

上一节我们介绍了文件和图片隐写，本节中我们来看看如何处理加密或损坏的压缩文件。压缩文件（如ZIP、RAR）是CTF中常见的载体，题目可能考察伪加密、密码破解或文件修复。

### 伪加密识别与修复

![](img/9ae74f48fa1031e5a5166741b4302d8c_6.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_8.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_10.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_12.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_14.png)

有时压缩包提示需要密码，但这可能是“伪加密”。即文件头中某个标志位被人为修改，导致压缩软件误认为文件已加密。现代压缩软件（如360压缩、2345好压）可能自动修复此类问题，但了解原理仍有必要。

![](img/9ae74f48fa1031e5a5166741b4302d8c_16.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_18.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_20.png)

修复伪加密需要使用十六进制编辑器（如WinHex、010 Editor）修改特定字节。

![](img/9ae74f48fa1031e5a5166741b4302d8c_22.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_24.png)

*   **ZIP文件**：搜索十六进制值 `504B0102`，从该位置开始，第9到第10个字节若为 `0900`，则可能是伪加密。将其修改为 `0000` 即可。
*   **RAR文件**：从文件头开始，第24个字节的尾数若为 `1`，则可能是伪加密。将其修改为 `0` 即可。

以下是修改ZIP文件伪加密标志位的示意图：
```
原始字节：... 50 4B 01 02 ... [09 00] ...
修改为：... 50 4B 01 02 ... [00 00] ...
```

### 密码破解

如果压缩包是真正加密的，通常密码不会过于复杂，出题人会给出提示（如密码前几位）。我们可以使用工具进行暴力破解或字典攻击。

以下是常用的破解工具及方法：

*   **ARCHPR**：主要用于破解RAR文件密码。支持暴力破解、掩码攻击和字典攻击。
*   **ZIP密码破解工具**：针对ZIP文件的专用破解工具，使用方法类似。
*   **明文攻击**：如果你已知压缩包内某个文件的内容，可以利用此已知明文进行攻击，成功率更高。注意，用于攻击的明文文件必须使用与目标压缩包**相同的压缩算法**重新压缩。

### 压缩包文件修复

有时压缩包本身结构损坏，导致无法正常列出或解压所有文件。这需要你了解压缩文件格式，并手动修复文件头。

一个RAR文件的结构大致如下：
```
[文件块头] [数据] [文件块头] [数据] ... [结束块头]
```
每个文件块头包含类型字段。例如，RAR的文件头类型应为 `0x74`。如果该字段被错误修改（如改为 `0x7A`），解压软件就无法识别该文件。你需要使用十六进制编辑器找到损坏的文件头，并将其类型字段修正为正确的值（如将 `0x7A` 改为 `0x74`），从而恢复被隐藏的文件。

## 流量分析取证技术 🌐

![](img/9ae74f48fa1031e5a5166741b4302d8c_34.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_36.png)

压缩包处理告一段落，现在我们进入另一个重点——流量分析。流量包记录了网络通信的所有数据，可能隐藏着flag、传输的文件或攻击痕迹。

### 流量分析基础与工具

![](img/9ae74f48fa1031e5a5166741b4302d8c_42.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_44.png)

进行流量分析，**Wireshark** 是必不可少的工具。它能够抓取并深度解析网络数据包。

![](img/9ae74f48fa1031e5a5166741b4302d8c_46.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_48.png)

首先，打开一个流量包文件（格式通常为 `.pcap` 或 `.pcapng`）。界面会显示许多数据包，每个包包含源/目的IP、协议、长度等信息。面对大量数据包，我们需要使用**过滤语法**来聚焦关键信息。

![](img/9ae74f48fa1031e5a5166741b4302d8c_50.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_52.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_54.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_56.png)

以下是常用的Wireshark过滤语法：

![](img/9ae74f48fa1031e5a5166741b4302d8c_58.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_59.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_61.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_63.png)

*   **过滤IP地址**：`ip.src == 192.168.1.1` （源IP）或 `ip.dst == 192.168.1.2` （目的IP）。
*   **过滤端口**：`tcp.port == 80` （TCP 80端口）或 `udp.port == 53` （UDP 53端口）。
*   **过滤协议**：直接输入协议名，如 `http`、`tcp`、`dns`。
*   **过滤包内容**：`http contains “flag”` 或 `tcp contains “key”`。这是非常高效的技巧，可以快速定位包含特定关键词的包。
*   **过滤包长度**：`frame.len > 100` 可以筛选出长度较大的包，这类包可能正在传输文件。

### 协议分析与数据提取

面对复杂的流量包，可以先进行**协议分级统计**（点击“统计” -> “协议分级”），查看哪种协议流量占比最大，从而确定分析重点（例如，HTTP流量占比80%，则应优先分析HTTP）。

![](img/9ae74f48fa1031e5a5166741b4302d8c_67.png)

1.  **追踪流**：在感兴趣的包上右键，选择“追踪流” -> “TCP流”或“HTTP流”。这会将一次会话的所有相关数据包重组并显示，便于查看完整的请求和响应，flag或关键信息常藏于此。
2.  **导出对象**：对于HTTP等协议，可以点击“文件” -> “导出对象” -> “HTTP”，将传输的文件（如图片、文档）批量导出到本地。
3.  **手动导出分组字节流**：对于非标准协议或需要提取特定数据段的情况，可以选中数据区，右键选择“导出分组字节流”，将原始二进制数据保存为文件，再进行后续分析。

### 特殊流量分析

![](img/9ae74f48fa1031e5a5166741b4302d8c_96.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_98.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_100.png)

某些题目会涉及特定类型的流量，需要特殊处理方法。

![](img/9ae74f48fa1031e5a5166741b4302d8c_102.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_104.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_106.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_108.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_110.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_112.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_114.png)

![](img/9ae74f48fa1031e5a5166741b4302d8c_116.png)

*   **USB流量分析**：流量可能来自键盘或鼠标。
    *   **键盘流量**：数据包长度固定，按键信息藏在特定字节。你需要提取这些字节，并根据USB键盘键值映射表还原出输入的字符。可以使用 `tshark` 命令提取数据，再编写Python脚本进行解码。
    *   **鼠标流量**：数据包包含鼠标移动的坐标偏移量。提取出这些坐标后，可以使用绘图工具（如gnuplot）将移动轨迹画出来，轨迹可能就是flag或二维码。
*   **无线（Wi-Fi）流量分析**：目标是破解握手包中的Wi-Fi密码。使用 `aircrack-ng` 工具，指定捕获的握手包文件和密码字典，即可进行破解。
    ```bash
    aircrack-ng -w password_list.txt capture.pcap
    ```
*   **HTTPS流量分析**：HTTPS是加密的HTTP。如果题目给了解密密钥（如SSL私钥），你需要在Wireshark中导入（编辑 -> 首选项 -> Protocols -> TLS -> RSA keys列表），才能解密并看到明文的HTTP流量。
