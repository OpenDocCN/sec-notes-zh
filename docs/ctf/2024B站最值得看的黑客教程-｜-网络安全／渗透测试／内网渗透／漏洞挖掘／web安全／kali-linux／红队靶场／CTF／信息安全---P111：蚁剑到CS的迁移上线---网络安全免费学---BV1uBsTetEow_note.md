## 从蚁剑到Cobalt Strike的迁移

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_10.png)

上一节我们介绍了如何利用蚁剑进行内网信息收集。本节中我们来看看如何将控制权迁移到更专业的工具上。

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_12.png)

内网信息收集主要依靠系统命令和一些辅助工具，这与Web渗透中的端口扫描等方式有所不同。完成信息收集后，下一步就是进行内网渗透。

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_14.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_16.png)

在进行内网渗透之前，通常需要一个前提：使用MSF或Cobalt Strike这类专业工具。我们当前使用蚁剑控制了目标服务器，但蚁剑功能有限，主要执行命令，在进行提权、建立隧道等复杂操作时比较麻烦。

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_18.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_19.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_20.png)

因此，我们需要用一个更强大的工具来替换蚁剑。业内通常使用MSF或Cobalt Strike。虽然MSF在Kali Linux中非常有名，但它主要是命令行操作，需要输入各种命令和脚本。而Cobalt Strike是图形化界面，操作更直观便捷，通过点击、双击、右键等操作即可实现远程控制，因此应用更为广泛。

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_22.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_24.png)

接下来，我们将手把手安装并配置Cobalt Strike。

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_25.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_27.png)

### Cobalt Strike的安装与启动
以下是安装Cobalt Strike的步骤：

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_29.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_31.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_32.png)

1.  **准备工具**：需要两个文件：JDK（Java运行环境）和Cobalt Strike客户端。请确保已获取这两个工具。
2.  **安装Java环境**：如果电脑上没有Java环境，需要先安装JDK 8版本。运行安装程序，全程点击“下一步”即可完成安装。
3.  **放置Cobalt Strike**：将Cobalt Strike压缩包解压到电脑的某个目录（例如桌面）。为避免杀毒软件误报，建议暂时关闭杀软。
4.  **启动服务端**：
    *   以**管理员身份**打开命令提示符（CMD）。
    *   使用 `cd` 命令切换到Cobalt Strike所在目录。
    *   查看本机IP地址（例如 `192.168.1.100`）。
    *   输入启动命令：`teamserver.bat <你的IP地址> <密码>`。例如：`teamserver.bat 192.168.1.100 123456`
    *   回车后，服务端开始运行，并会显示监听的端口号（例如 `50050`）。
5.  **启动客户端**：
    *   运行解压目录中的 `CN` 文件（中文版客户端）。
    *   在弹出的连接界面中，填写以下信息：
        *   **主机**：填写你本机的IP地址（同上一步）。
        *   **端口**：填写服务端启动时显示的端口号（例如 `50050`）。
        *   **密码**：填写你设置的密码（例如 `123456`）。
    *   点击“连接”。

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_34.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_35.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_37.png)

至此，Cobalt Strike的客户端界面就成功打开了。

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_39.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_41.png)

### 生成木马并上线目标
现在，我们需要生成一个木马程序，并通过蚁剑在目标服务器上运行它，从而将控制权迁移到Cobalt Strike。

以下是具体操作步骤：

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_43.png)

1.  **创建监听器**：在Cobalt Strike客户端中，点击“监听器” -> “添加”。设置一个名称（如 `CS`），填写本机IP和任意端口（如 `7865`），然后保存。
2.  **生成木马**：点击“攻击” -> “生成后门” -> “Windows可执行文件”。在生成界面中，选择刚才创建的监听器，然后点击“生成”，将木马文件（例如 `shell.exe`）保存到桌面。
3.  **上传并执行木马**：
    *   回到蚁剑，连接到目标服务器的WebShell。
    *   将刚刚生成的 `shell.exe` 木马文件上传到目标服务器。
    *   在蚁剑的虚拟终端中，运行命令来执行这个木马：`shell.exe`。
4.  **等待上线**：如果一切正常，目标服务器运行木马后，在Cobalt Strike的“主机”视图里，很快就会看到一个新的上线主机（Beacon），其IP地址就是目标服务器的IP。

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_45.png)

**注意**：在实际操作中，可能会因为杀毒软件、网络配置或工具版本问题导致上线失败。需要根据具体情况排查，例如检查防火墙、使用更稳定的Cobalt Strike版本等。

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_47.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_49.png)

成功上线后，你可以在Cobalt Strike中选中该主机，在底部的命令交互框里输入命令（命令前需加 `shell`），例如 `shell ipconfig`，来验证控制是否成功。

## 内网横向移动控制域控

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_51.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_53.png)

我们已经成功将Web服务器的控制权迁移到了Cobalt Strike。接下来，我们的目标是利用这台Web服务器作为跳板，去控制内网中更重要的机器——域控制器。

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_55.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_57.png)

这种从一台已控机器出发，去攻击和控制内网其他机器的技术，称为**横向移动**。这是高级渗透测试和红队行动中的核心技能。

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_59.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_61.png)

横向移动有多种技术，如IPC$、WMI、PTH、PTT等。本节课我们重点学习利用 **IPC$** 进行横向移动。

### 什么是IPC$？
IPC$（进程间通信）共享是Windows系统的一个功能，可以理解为在两台机器之间建立了一个“命名管道”。一旦建立了IPC$连接，就可以像访问本地文件夹一样，在两台机器间上传和下载文件。

### 横向移动实战步骤
假设我们已经通过Cobalt Strike获取了Web服务器（IP: `192.168.111.128`）的控制权，并且通过信息收集知道了域控（IP: `192.168.52.138`）的管理员账号密码（例如：`Administrator/Redteam2020`）。

以下是利用IPC$横向移动的步骤：

1.  **建立IPC$连接**：在Cobalt Strike中，通过已控的Web服务器向域控建立连接。
    输入命令：
    ```cmd
    shell net use \\192.168.52.138\ipc$ "Redteam2020" /user:"Administrator"
    ```
    如果命令成功执行，则IPC$管道建立。

2.  **验证连接**：可以输入命令 `shell net use` 查看当前建立的网络连接，确认IPC$连接是否存在。

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_63.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_65.png)

3.  **上传木马**：现在可以通过这个管道，将木马文件从Web服务器复制到域控上。
    输入命令：
    ```cmd
    shell copy c:\path\to\your\shell.exe \\192.168.52.138\c$\windows\temp\shell.exe
    ```
    （请将路径替换为你木马在Web服务器上的实际路径）

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_66.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_68.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_70.png)

4.  **执行木马的挑战与解决方案**：
    *   **挑战**：直接运行上传到域控的木马可能无法让黑客的Cobalt Strike上线。因为域控（`52.138`网段）与黑客主机（`111.1`网段）可能不在同一网络，流量无法直接互通（即域控“不出网”）。
    *   **解决方案**：使用**隧道技术**。我们需要让木马的流量先“走”到已控的Web服务器（跳板机），再由Web服务器转发给黑客主机。这需要在Web服务器上建立流量转发隧道（例如使用socks代理、端口转发等）。这是内网渗透中更深层次的知识点。

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_72.png)

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_73.png)

**核心思路公式化**：
```
黑客控制(Web服务器) + IPC$(连接域控) + 隧道(转发流量) => 成功控制(域控)
```

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_75.png)

简单来说，我们通过已控机器建立通道，将攻击载荷传递到内网目标，并解决网络隔离问题，最终实现控制。

## 总结
本节课中我们一起学习了两个关键技能：
1.  **工具迁移**：将简单的WebShell控制（蚁剑）升级到功能全面的专业远控工具（Cobalt Strike），为后续复杂操作打下基础。
2.  **横向移动入门**：了解了内网渗透的核心——横向移动的概念，并学习了通过IPC$共享建立连接、传输文件的基本方法。同时，我们也认识到在实战中，解决网络隔离问题（如使用隧道）是成功实现横向控制的关键。

![](img/19a9cdb76b30bc6e0836ffa412db4c5b_77.png)

掌握这些基础，是迈向高级渗透测试领域的重要一步。
