## CTF简介

![](img/3e2e199465bdf0ab592499a795886e38_11.png)

![](img/3e2e199465bdf0ab592499a795886e38_13.png)

![](img/3e2e199465bdf0ab592499a795886e38_15.png)

![](img/3e2e199465bdf0ab592499a795886e38_17.png)

CTF是一种网络安全比赛形式，全称是“Capture The Flag”，中文常称为“夺旗赛”。比赛主要有以下几种模式：
*   **解题模式 (Jeopardy)**：参赛者解决不同方向的题目来获取分数。
*   **攻防模式 (AWD)**：参赛者在攻击对方服务器获取Flag的同时，也需要防守自己的服务器。
*   **混合模式**：结合了解题和攻防模式。

![](img/3e2e199465bdf0ab592499a795886e38_19.png)

![](img/3e2e199465bdf0ab592499a795886e38_20.png)

现阶段我们主要以**解题模式**为主。在比赛中，题目通常分为多个方向，例如：MISC（杂项）、CRYPTO（密码学）、WEB（网络安全）、PWN（二进制漏洞利用）等。

## 实战案例解析：二进制二维码

![](img/3e2e199465bdf0ab592499a795886e38_55.png)

![](img/3e2e199465bdf0ab592499a795886e38_57.png)

![](img/3e2e199465bdf0ab592499a795886e38_59.png)

现在，让我们通过一道具体的题目来实践。题目通常会给一个附件，例如一个名为`flag.txt`的文本文件。

![](img/3e2e199465bdf0ab592499a795886e38_61.png)

![](img/3e2e199465bdf0ab592499a795886e38_63.png)

![](img/3e2e199465bdf0ab592499a795886e38_65.png)

![](img/3e2e199465bdf0ab592499a795886e38_67.png)

打开文件后，你可能会看到大量由`0`和`1`组成的字符矩阵。例如：
```
0000000000000000000000000000
0000000000000000000000000000
0000001111111000011111110000
0000010000000100100000001000
... (更多行)
```

![](img/3e2e199465bdf0ab592499a795886e38_69.png)

![](img/3e2e199465bdf0ab592499a795886e38_71.png)

**解题思路**：
1.  观察数据特征：全是`0`和`1`，且矩阵的左上、右上、左下角通常为`0`，右下角数据不同。
2.  联想常见形式：这种特征非常像**二维码**。在二维码中，`0`可能代表白色模块，`1`代表黑色模块。
3.  转换与识别：我们需要将这些`01`文本转换成图片。

![](img/3e2e199465bdf0ab592499a795886e38_73.png)

![](img/3e2e199465bdf0ab592499a795886e38_75.png)

**解决方法**：
虽然可以编写Python脚本转换，但在比赛或练习中，使用现成工具更高效。我们可以使用 **QR Research** 工具。
*   将`01`文本文件导入工具。
*   工具会自动将其识别并渲染成二维码图片。
*   使用手机或扫码工具扫描生成的二维码，即可得到Flag。

![](img/3e2e199465bdf0ab592499a795886e38_77.png)

**核心概念（伪代码）**：
```python
# 概念性转换思路
for line in text_file:
    for char in line:
        if char == '0':
            image_pixel = WHITE
        elif char == '1':
            image_pixel = BLACK
        draw_pixel(image_pixel)
```

## 实战案例解析：NBT文件与Minecraft

接下来我们看另一类题目。题目附件是一个压缩包，解压后得到一系列文件，其中可能包含一张《我的世界》(Minecraft)游戏截图。

![](img/3e2e199465bdf0ab592499a795886e38_81.png)

**解题思路**：
1.  **注意文件名**：压缩包或文件的后缀名可能包含提示，例如`EZ_NBT`。
2.  **搜索与研究**：搜索“NBT”格式，会发现这是《我的世界》游戏使用的一种数据存储格式。
3.  **寻找解析工具**：到《我的世界》Wiki或相关社区，可以找到专用的NBT文件编辑器，例如`NBTExplorer`。

![](img/3e2e199465bdf0ab592499a795886e38_83.png)

**解题步骤**：
1.  下载并打开`NBTExplorer`。
2.  将题目给的NBT文件拖入工具中。
3.  在树状结构的数据中，寻找包含`flag`或类似关键词的`value`（值）。
4.  逐级展开父目录，定位到完整的Flag字符串。

### 案例1：文件头尾修复

![](img/3e2e199465bdf0ab592499a795886e38_119.png)

![](img/3e2e199465bdf0ab592499a795886e38_120.png)

拿到图片文件（如`simple.jpg`）后，首先使用 **010 Editor** 打开。

![](img/3e2e199465bdf0ab592499a795886e38_122.png)

**解题步骤**：
1.  **检查文件头尾**：常见的图片格式有固定的十六进制头尾标识。
    *   JPEG文件头：`FF D8 FF`
    *   JPEG文件尾：`FF D9`
    *   PNG文件头：`89 50 4E 47 0D 0A 1A 0A`
2.  **发现问题**：在文件末尾`FF D9`之后，可能附加了一段不规则数据（如`#107#107...`）。
3.  **提取与解码**：将异常数据段复制出来。这段数据可能是某种编码（如Unicode、Base64）。可以尝试使用在线解码工具或询问AI（如ChatGPT）进行识别和解码，最终得到Flag。

### 案例2：CRC校验与宽高修改

![](img/3e2e199465bdf0ab592499a795886e38_124.png)

有时图片因为CRC校验错误或宽高被修改而无法正常显示。

**解题步骤（使用010 Editor）**：
1.  打开图片文件，编辑器可能会直接提示CRC错误及其位置。
2.  根据提示，找到错误的CRC值，并将其替换为正确的值（工具可能自动提示）。
3.  **检查宽高**：PNG文件的宽高信息存储在固定的数据块（IHDR）中。如果宽高值被修改得不一致，图片会显示异常。
4.  在010 Editor中找到宽高字段，尝试将其修改为合理的相同值，保存后再次打开图片，Flag可能直接显示在图片中。

### 案例3：LSB隐写

更高级的隐写术会将信息藏在图片像素的最低有效位(LSB)，人眼难以察觉。

**解题步骤**：
1.  使用专门的隐写分析工具，如 **StegSolve**（一个Java工具）。
2.  运行工具需要Java环境（JDK），请确保已安装并配置好。
3.  在StegSolve中打开图片，使用工具提供的功能（如`Analyse -> Frame Browser`或`Extract`）对RGB颜色通道进行分离和组合分析。
4.  通过切换不同的位平面（Bit Planes），可能会在某个颜色通道的LSB层中发现隐藏的Flag信息。

## 实战案例解析：网络数据包分析

![](img/3e2e199465bdf0ab592499a795886e38_130.png)

最后，我们来看涉及网络流量的题目。这类题目会提供一个数据包文件（通常为`.pcap`或`.pcapng`后缀）。

![](img/3e2e199465bdf0ab592499a795886e38_132.png)

![](img/3e2e199465bdf0ab592499a795886e38_133.png)

**解题步骤（使用Wireshark）**：
1.  用 **Wireshark** 打开数据包文件。
2.  观察数据流。在简单的题目中，可能只有少量的、重复的协议请求（如TCP）。
3.  逐个检查数据包的应用层数据。**重点观察每次传输中变化的部分**。
4.  在详细数据中，查找可能以`f`、`l`、`a`、`g`等字符开头或包含`flag`格式的字符串。
5.  有时Flag可能被分段隐藏在多个数据包中，需要将它们按顺序拼接起来。
6.  对于更复杂的数据包，可以使用Wireshark的“追踪流”（Follow TCP Stream）功能，将所有相关通信内容整合在一起查看，便于发现规律和提取Flag。
