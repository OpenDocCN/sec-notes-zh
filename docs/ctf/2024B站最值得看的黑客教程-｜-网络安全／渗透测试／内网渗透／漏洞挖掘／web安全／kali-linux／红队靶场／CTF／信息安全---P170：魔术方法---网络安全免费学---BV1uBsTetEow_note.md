# PHP魔术方法详解：P170：魔术方法

在本节课中，我们将要学习PHP中一个非常重要的概念——魔术方法。理解魔术方法对于后续学习反序列化漏洞至关重要。

## 什么是魔术方法？

魔术方法，首先它是一个方法。方法就是类里面所定义的函数。例如 `function A()` 就是一个方法。

回顾我们昨天的题目，在 `index2` 这个题目里，`read` 类里面有 `get_file` 方法。`show` 类里面有 `__construct` 方法、`__toString` 方法和 `show` 方法。这些都是方法。

魔术方法是一类特殊的方法。特殊之处在于它可以自动调用。它不需要像我们昨天的例题一样，有一个手动的调用方式，它会自动触发。因此，它被称为特殊的方法，即魔术方法。

## 魔术方法的特征

以下是魔术方法的一个列表，可以将其理解为一个魔术方法的手册。

```php
__construct()    // 类的构造函数
__destruct()     // 类的析构函数
__call()         // 在对象中调用一个不可访问方法时调用
__callStatic()   // 用静态方式中调用一个不可访问方法时调用
__get()          // 获得一个类的成员变量时调用
__set()          // 设置一个类的成员变量时调用
__isset()        // 当对不可访问属性调用isset()或empty()时调用
__unset()       // 当对不可访问属性调用unset()时调用
__sleep()       // 执行serialize()时，先会调用这个函数
__wakeup()      // 执行unserialize()时，先会调用这个函数
__toString()    // 类被当成字符串时的回应方法
__invoke()      // 调用函数的方式调用一个对象时的回应方法
__set_state()   // 调用var_export()导出类时，此静态方法会被调用。
__clone()       // 当对象复制完成时调用
__debugInfo()   // 打印所需调试信息
```

观察这些魔术方法，可以发现它们都有一个共同特点：都以两个下划线开始。例如 `__construct`。

讨论区有同学提到构造方法。魔术方法是总称，其中 `__construct` 是构造方法，`__destruct` 是析构方法。这些是它的别名，了解与否都可以。它的总称都是魔术方法，特征就是以两个下划线为开始。

## 魔术方法的触发条件

每个魔术方法后面都有一段注释，这个注释就是告诉你这个魔术方法的触发条件。

魔术方法是自动调用的，不需要手动调用，但它也不是自动执行，它需要满足一定的条件才会执行。不同的魔术方法有不同的满足条件。

例如，构造方法 `__construct` 的执行条件是使用 `new` 关键字实例化对象时，就会自动调用。析构方法 `__destruct` 的触发条件是对象被销毁的时候，例如程序执行结束，要把对象销毁掉，就会触发 `__destruct` 方法。

每一种不同的魔术方法，都有它对应的触发条件。这里大家不需要把这个表背下来，背下来是一种比较笨的方法。可以将其作为一个手册放在这里。

后面我们在讲题目时，会用到这些魔术方法，然后再回过头来看。大家通过在做题的时候练习，在练习中熟悉这些魔术方法的触发条件。这样大家就真正理解了，而不是仅仅把它背下来，既难记又不是真正的理解。

## 魔术方法触发后做什么？

这里还需要给之前没接触过魔术方法的同学解释一下。魔术方法触发了是做什么呢？大家有没有想过，`__construct` 触发了是做什么？或者说 `__wakeup` 触发了，触发之后做什么呢？

我们可以回到昨天的题目。例如 `__construct` 方法触发之后干什么？它进行一个属性的赋值，然后输出一些内容。`__toString` 也是一个魔术方法，它先进行一个赋值，再输出内容并返回内容。

通过这两个例子是想告诉大家，每个魔术方法触发之后做什么是不一定的。这是由不同的程序，或者说不同的类它自己所定义的。

这里可以举个例子辅助大家理解。我们现在有很多智能家居。例如，你可以在智能家居里面设置，一旦关上门之后，就关闭所有的灯。那么“关上门”就是一个触发条件，“关闭所有的灯”就是触发条件满足后做什么。

这就好比我们这里，使用 `new` 的时候就会触发 `__construct`。那么智能家居一旦关上门，就会触发一个关灯的程序。`__construct` 具体做什么呢？每个类定义不一样。就好像我们每个人的智能家居的房间不一样。有的家庭设置关门之后就关灯，有的家庭设置关门之后扫地机器人开始工作，有的家庭设置关门之后关闭窗帘等等。

一旦触发之后，这个函数、这个方法是做什么，是由每一个对象他自己所定义的。当然，这里直接理解，可能大家只能理解个百分之六七十。我们后面在做题当中来彻底地理解它。现在大家先熟悉“魔术方法”这个概念，这在我们反序列化当中是非常重要的。

## 总结

本节课中我们一起学习了PHP魔术方法的核心概念。我们了解到魔术方法是一类以双下划线开头、能够自动触发的特殊类方法。每种魔术方法都有其特定的触发条件，例如 `__construct` 在对象创建时触发，而 `__destruct` 在对象销毁时触发。最重要的是，魔术方法被触发后执行的具体逻辑是由程序员在类中自定义的，这为编程带来了灵活性，但也需要在安全领域（如反序列化漏洞）中格外关注。在后续的实战练习中，我们将深入应用这些知识。
