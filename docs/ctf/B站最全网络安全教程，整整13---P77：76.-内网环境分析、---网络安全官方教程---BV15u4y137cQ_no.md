## 横向与纵向渗透概念

![](img/de3560a7bc1ae6a4fbf2e226387fec29_26.png)

上一节我们介绍了内网渗透的基本思路，本节中我们来看看横向与纵向渗透的具体定义和区别。

![](img/de3560a7bc1ae6a4fbf2e226387fec29_34.png)

![](img/de3560a7bc1ae6a4fbf2e226387fec29_36.png)

![](img/de3560a7bc1ae6a4fbf2e226387fec29_38.png)

![](img/de3560a7bc1ae6a4fbf2e226387fec29_40.png)

**横向渗透** 是指通过已控制的跳板机（入口点），去攻击和访问**该跳板机所在网络段内**的其他主机。关键在于“当前机器能够直接访问到的网络范围”。

![](img/de3560a7bc1ae6a4fbf2e226387fec29_42.png)

**纵向渗透** 则是在横向渗透的基础上，通过新拿下的机器，去访问**原先跳板机无法直接到达的其他网段**的主机。这通常意味着网络结构更深、权限更高。

![](img/de3560a7bc1ae6a4fbf2e226387fec29_44.png)

![](img/de3560a7bc1ae6a4fbf2e226387fec29_46.png)

![](img/de3560a7bc1ae6a4fbf2e226387fec29_48.png)

![](img/de3560a7bc1ae6a4fbf2e226387fec29_50.png)

**核心思路**：不要将横向渗透局限在单一C段。只要是从当前已控主机能够访问到的任何机器，都属于横向渗透的范畴。思路需要开阔，包括路由器、交换机等网络设备上的配置信息都可能成为突破口。

### 1. 用户信息收集
了解当前系统上的用户、用户权限以及在线状态。
以下是相关命令列表：
*   **`net user`**：查看本机所有用户账户。
*   **`net localgroup administrators`**：查看本地管理员组中的用户。
*   **`query user`** 或 **`quser`**：查看当前在线的用户。
*   **`whoami /priv`**：查看当前用户的具体权限。

### 2. 网络与系统信息
获取主机的网络配置、系统版本、运行的服务等关键信息。
以下是相关命令列表：
*   **`ipconfig /all`**：查看详细的网络配置信息（IP地址、DNS、主机名、域信息等）。其中`DNS后缀`和`DNS服务器`地址是判断是否在域内以及定位域控（Domain Controller）的重要线索。
*   **`systeminfo | findstr /B /C:“OS Name” /C:“OS Version”`**：查看操作系统名称和版本。
*   **`wmic os get caption,csarchitecture,version`**：通过WMIC命令查询系统信息。
*   **`tasklist`** 或 **`wmic process list brief`**：查看系统当前运行的进程列表。
*   **`netstat -ano`**：查看网络连接、监听端口及其对应的进程PID。常用组合命令 `netstat -ano | findstr “:445”` 来查找特定端口。
*   **`sc query`** 或 **`wmic service list brief`**：查看系统服务信息。

### 3. 密码与凭据信息
寻找系统中存储的各类密码和访问凭据。
以下是相关命令列表：
*   **`cmdkey /list`**：查看Windows保存的访问凭据。
*   **`netsh wlan show profile name=“WiFi名称” key=clear`**：查看已连接Wi-Fi的明文密码。
*   **回收站内容获取**：通过脚本遍历用户SID，检查 `C:\$Recycle.Bin` 目录下的文件，可能找到被删除的敏感文件。脚本示例如下：
    ```bash
    for /f “skip=1 tokens=1,2 delims= ” %%a in (‘"wmic useraccount get name,sid"’) do (
        if not “%%b”==“” (
            echo [+] Check user: %%a, sid: %%b
            dir /a “C:\$Recycle.Bin\%%b” 2>nul
        )
    )
    ```
*   **浏览器历史记录与Cookie**：使用工具（如`Mimikatz`）或脚本读取浏览器（如Chrome）的数据库文件（路径通常为 `%LocalAppData%\Google\Chrome\User Data\Default\History` 和 `Cookies`），获取访问历史和会话Cookie。

### 4. 其他重要信息
收集补丁、共享、计划任务等信息，用于评估漏洞利用可能性。
以下是相关命令列表：
*   **`wmic qfe get caption,description,hotfixid,installedon`**：查看已安装的系统补丁列表。
*   **`wmic product get name,version`** 或 **`powershell “Get-WmiObject -class Win32_Product | Select-Object -Property name,version”`**：查看安装的软件。
*   **`netsh winhttp show proxy`**：查看系统代理设置。
*   **`net share`**：查看本机共享列表。
*   **`schtasks /query /fo LIST /v`**：查看计划任务（新版系统）。老系统（如Win2003）使用 `at` 命令。
*   **`net use \目标IP\共享名C$ /user:用户名 密码`**：映射远程共享到本地驱动器（如Z:盘）。

### 域的基本概念
域是一个有安全边界的计算机集合，与工作组相比，它通过一台或多台**域控制器**进行集中管理。域控制器上运行着**活动目录**，存储着域内所有用户、计算机、组策略等对象的信息。

![](img/de3560a7bc1ae6a4fbf2e226387fec29_149.png)

**关键点**：
*   **域控制器**：域的核心，负责身份验证（使用Kerberos协议）和管理。
*   **活动目录**：域内的目录数据库，存储在域控上。
*   **DNS与域**：域环境严重依赖DNS来定位域控制器和其他服务。域控通常也充当DNS服务器。

![](img/de3560a7bc1ae6a4fbf2e226387fec29_151.png)

![](img/de3560a7bc1ae6a4fbf2e226387fec29_152.png)

### 域内信息收集命令
判断主机是否在域内，并收集域的相关信息。
以下是相关命令列表：
*   **`net config workstation`**：查看工作站信息，`域`字段显示当前所属域。
*   **`net view /domain`**：查看当前网络中的域列表。
*   **`net view /domain:域名`**：查看指定域内的所有计算机列表。
*   **`net group “domain controllers” /domain`**：查询域控制器列表。
*   **`net user /domain`**：查询域内的所有用户账户。
*   **`net group /domain`**：查询域内的所有组。
*   **`net group “domain admins” /domain`**：查询域管理员组成员。
*   **`nltest /dclist:域名`**：获取域控制器列表。
*   **`nslookup -type=SRV _ldap._tcp.dc._msdcs.域名`**：通过DNS查询定位域控制器。

![](img/de3560a7bc1ae6a4fbf2e226387fec29_154.png)

![](img/de3560a7bc1ae6a4fbf2e226387fec29_156.png)

### 定位域控制器的方法
综合运用上述命令，可以从多个维度定位域控。
以下是常用方法列表：
1.  **DNS解析**：直接解析域名或 `域控主机名.域名`，得到的IP通常是域控。命令：`nslookup 域名`。
2.  **查看DNS配置**：`ipconfig /all` 中 `DNS 服务器` 的IP地址很可能是域控。
3.  **查询域控制器组**：使用 `net group “domain controllers” /domain`。
4.  **端口扫描**：域控通常会开放 **389** 端口和 **53** 端口。
5.  **SPN扫描**：查询服务主体名称，域控会注册特定的LDAP服务。

### 1. NetBIOS协议探测
利用NetBIOS over TCP/IP协议进行主机发现，速度较快。
以下是相关工具列表：
*   **`nbtscan`** 工具：专用于NetBIOS扫描，效率高。命令：`nbtscan 网段`。
*   **Nmap脚本**：使用 `nmap -sU –script nbstat.nse -p137 目标IP`。

![](img/de3560a7bc1ae6a4fbf2e226387fec29_177.png)

### 2. ICMP协议探测
最经典的ping扫描，但可能被防火墙禁止。
以下是相关方法列表：
*   **`ping`** 命令：逐个ping IP地址。
*   **`for /L %i in (1,1,254) do @ping -n 1 -w 10 192.168.1.%i | findstr “TTL=”`**：批量化ping一个C段。
*   **Nmap**：`nmap -sn 网段`。

![](img/de3560a7bc1ae6a4fbf2e226387fec29_181.png)

![](img/de3560a7bc1ae6a4fbf2e226387fec29_187.png)

### 3. ARP协议探测
在局域网内最准确、最有效的探测方式，基于二层协议。
以下是相关工具列表：
*   **`arp -a`**：查看本机ARP缓存表，可发现近期有过通信的主机。
*   **Nmap**：`nmap -PR 网段`。
*   **`netdiscover`** 工具：主动ARP扫描工具。

![](img/de3560a7bc1ae6a4fbf2e226387fec29_189.png)

![](img/de3560a7bc1ae6a4fbf2e226387fec29_191.png)

### 4. 端口扫描
对特定IP进行端口扫描，以确认主机存活及开放服务。
以下是相关工具列表：
*   **Nmap**：功能强大，`nmap -sS -sV -O 目标IP` 可进行SYN扫描、版本探测和操作系统识别。
*   **Masscan**：号称最快的端口扫描器，异步传输。

![](img/de3560a7bc1ae6a4fbf2e226387fec29_193.png)

![](img/de3560a7bc1ae6a4fbf2e226387fec29_195.png)

![](img/de3560a7bc1ae6a4fbf2e226387fec29_196.png)

### 5. SMB/MSRPC协议探测
尝试连接Windows默认开放的135、445等端口，来发现Windows主机。
以下是相关方法列表：
*   **Nmap脚本**：`nmap –script smb-os-discovery.nse -p445 目标IP`。
*   **MSF**：使用 `auxiliary/scanner/smb/smb_version` 模块。

## 总结
本节课我们一起学习了内网渗透前期至关重要的两个阶段：信息收集与存活探测。
我们首先明确了横向与纵向渗透的概念，然后系统性地学习了在Windows工作组和域环境下，如何利用内置命令收集用户、网络、系统、凭据等各类信息。
接着，我们介绍了域环境的基础知识及定位域控制器的方法。
最后，我们讲解了使用NetBIOS、ICMP、ARP、端口扫描等多种协议和技术进行内网存活主机探测的实战方法。
掌握这些基础技能，是后续进行漏洞利用、横向移动和权限提升的前提。请大家务必在实验环境中多加练习，熟悉每条命令的输出和含义。
