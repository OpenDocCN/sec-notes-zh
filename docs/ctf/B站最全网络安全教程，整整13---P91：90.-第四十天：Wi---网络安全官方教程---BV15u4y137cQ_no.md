上一节我们介绍了渗透测试的多种攻击向量，本节中我们来看看权限提升的核心概念。权限提升是指攻击者通过安全漏洞，把获取到的受限制的低权限用户突破限制，提权至高权限的管理员用户，从而获得对整个系统的控制权。

![](img/ebdade265a6334f1a4dccb0a6104bc19_36.png)

![](img/ebdade265a6334f1a4dccb0a6104bc19_38.png)

![](img/ebdade265a6334f1a4dccb0a6104bc19_40.png)

![](img/ebdade265a6334f1a4dccb0a6104bc19_41.png)

![](img/ebdade265a6334f1a4dccb0a6104bc19_43.png)

权限提升的前提是已经获得了目标系统的某个低权限用户访问权限，例如通过Web漏洞获得的`www-data`或`apache`用户权限，或通过SQL注入获得的数据库用户权限。

![](img/ebdade265a6334f1a4dccb0a6104bc19_45.png)

提权的主要目标可以概括为：
*   **Windows系统**：从普通用户（如`User`）提权至管理员用户（`Administrator`），甚至系统用户（`SYSTEM`）。
*   **Linux系统**：从普通用户（`user`）提权至根用户（`root`）。

## 提权类别与条件

以下是提权操作的基本分类：

1.  **本地提权**：在已获得的低权限用户环境下，利用系统或应用程序漏洞直接提升到系统最高权限。
2.  **远程提权**：通过网络直接利用漏洞获取远程服务器的控制权限。
3.  **操作系统提权**：主要针对Windows和Linux系统内核或组件的漏洞。
4.  **应用程序提权**：利用如MySQL、SQL Server等应用程序的漏洞进行提权。

进行提权需要满足以下条件：
*   拥有一个基本的低权限用户Shell。
*   拥有某些软件或服务的账号密码（这是最直接的方法）。
*   本地或远程服务器上存在可利用的漏洞。
*   拥有对应的漏洞利用工具或代码。

## Windows提权基本思路

Windows系统提权通常遵循以下思路：
1.  **前期信息收集**：全面收集目标系统信息，这是决定攻击方向的关键。
2.  **利用提权模块**：使用如Metasploit（MSF）、Cobalt Strike（CS）等框架内置的提权模块。
3.  **内核漏洞利用**：利用操作系统内核的未修补漏洞。
4.  **服务/注册表漏洞利用**：利用配置不当的系统服务或注册表项。

## Windows提权信息收集

上一节我们介绍了提权的整体思路，本节中我们来看看如何进行详细的信息收集。首先需要获取一个初始的Meterpreter Shell或其他类型的低权限Shell，作为信息收集和后续攻击的起点。

信息收集可以通过系统内置命令或自动化脚本完成。

### 使用WMIC工具收集信息

WMIC是Windows管理命令行工具，可用于执行系统管理任务。以下是常用信息收集命令示例：

![](img/ebdade265a6334f1a4dccb0a6104bc19_47.png)

![](img/ebdade265a6334f1a4dccb0a6104bc19_49.png)

*   **查看系统用户账户**：
    ```cmd
    wmic useraccount list brief
    ```
*   **查看已安装补丁**：
    ```cmd
    wmic qfe get Caption,Description,HotFixID,InstalledOn
    ```
*   **过滤特定补丁（如KB4565554）**：
    ```cmd
    wmic qfe get Caption,Description,HotFixID,InstalledOn | findstr /C:"KB4565554"
    ```
*   **查看杀毒软件**：
    ```cmd
    wmic /namespace:\\root\securitycenter2 path antivirusproduct get displayname,pathToSignedProductExe
    ```

## 内核漏洞提权

![](img/ebdade265a6334f1a4dccb0a6104bc19_80.png)

![](img/ebdade265a6334f1a4dccb0a6104bc19_82.png)

![](img/ebdade265a6334f1a4dccb0a6104bc19_83.png)

在完成信息收集后，如果发现系统存在未修复的内核漏洞，就可以尝试利用它进行提权。内核漏洞提权通常涉及以下步骤：
1.  **识别系统版本和补丁**：使用`systeminfo`或WMIC命令确定操作系统版本和已安装的补丁。
2.  **查找对应漏洞**：根据系统版本和缺失的补丁号，在漏洞数据库（如Exploit-DB）或提权辅助工具（如Windows-Exploit-Suggester）中查找可用的本地提权漏洞。
3.  **编译/选择利用代码**：根据目标系统架构（x86/x64）选择合适的利用代码，可能需要在攻击机上交叉编译。
4.  **上传并执行利用程序**：将利用程序上传到目标机器并执行。如果成功，将获得SYSTEM权限的Shell。
5.  **验证权限**：执行`whoami`或`getuid`命令，确认当前用户是否为`NT AUTHORITY\SYSTEM`。

**核心公式**：`低权限Shell + 未修补的内核漏洞 + 正确的利用程序 => SYSTEM权限`

### 提权工具示例：Rotten Potato（烂土豆）

![](img/ebdade265a6334f1a4dccb0a6104bc19_97.png)

![](img/ebdade265a6334f1a4dccb0a6104bc19_99.png)

![](img/ebdade265a6334f1a4dccb0a6104bc19_101.png)

![](img/ebdade265a6334f1a4dccb0a6104bc19_103.png)

![](img/ebdade265a6334f1a4dccb0a6104bc19_104.png)

Rotten Potato是一个经典的Windows本地提权工具，它利用NTLM重放等技术，将服务账户的权限提升至SYSTEM。

![](img/ebdade265a6334f1a4dccb0a6104bc19_106.png)

![](img/ebdade265a6334f1a4dccb0a6104bc19_108.png)

![](img/ebdade265a6334f1a4dccb0a6104bc19_110.png)

![](img/ebdade265a6334f1a4dccb0a6104bc19_112.png)

![](img/ebdade265a6334f1a4dccb0a6104bc19_114.png)

![](img/ebdade265a6334f1a4dccb0a6104bc19_116.png)

**基本使用流程**：
1.  在已获得的Meterpreter会话中，上传`rottenpotato.exe`（或相关变种）到目标机器。
2.  在Meterpreter中加载`incognito`模块：`use incognito`
3.  执行烂土豆程序，生成一个SYSTEM权限的令牌。
4.  使用`list_tokens -u`列出可用令牌，应能看到`NT AUTHORITY\SYSTEM`。
5.  窃取该令牌：`impersonate_token "NT AUTHORITY\\SYSTEM"`
6.  此时，你的Meterpreter会话权限已提升为SYSTEM。

## Token窃取与利用

Windows使用Token（令牌）来标识进程或线程的安全上下文。提权过程中，窃取高权限Token是一种有效手段。

Token主要分为两类：
*   **授权令牌**：用于交互式会话登录（如远程桌面登录）。
*   **模拟令牌**：用于非交互式会话（如网络映射驱动器）。

![](img/ebdade265a6334f1a4dccb0a6104bc19_124.png)

**利用思路**：如果系统中有已登录的高权限用户（如Administrator），可以通过工具（如MSF的`incognito`模块）窃取其Token，然后模拟该用户身份执行操作，从而获得其权限。
