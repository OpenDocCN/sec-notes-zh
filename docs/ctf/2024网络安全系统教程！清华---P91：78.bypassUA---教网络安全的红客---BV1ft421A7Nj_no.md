# 网络安全系统教程：P91：78. 绕过UAC (Bypass UAC) 🛡️

![](img/9aaa7069e6da16287982ff9930a6bd42_1.png)

在本节课中，我们将学习如何绕过Windows的用户账户控制（UAC），这是一种在已获得普通用户权限后，进一步获取管理员权限的技术。

## 概述：什么是UAC？

![](img/9aaa7069e6da16287982ff9930a6bd42_3.png)

![](img/9aaa7069e6da16287982ff9930a6bd42_5.png)

上一节我们介绍了权限提升的基础概念，本节中我们来看看一个具体的提权场景——绕过UAC。用户账户控制（User Account Control, UAC）是微软Windows系统中的一个安全组件。它的主要作用是**缓解恶意软件的影响**，是一种安全策略。当用户或程序尝试执行需要管理员权限的操作时，系统会弹出UAC对话框请求确认。

![](img/9aaa7069e6da16287982ff9930a6bd42_6.png)

![](img/9aaa7069e6da16287982ff9930a6bd42_8.png)

![](img/9aaa7069e6da16287982ff9930a6bd42_10.png)

![](img/9aaa7069e6da16287982ff9930a6bd42_11.png)

例如，当你以普通用户身份尝试“以管理员身份运行”命令提示符（CMD）时，就会看到如下提示：
> “你要允许此应用对你的设备进行更改吗？”

只有点击“是”通过验证，才能获得管理员权限。因此，如果我们能**绕过（Bypass）** 这个验证机制，就能以普通用户身份直接获取管理员权限。

![](img/9aaa7069e6da16287982ff9930a6bd42_13.png)

## UAC绕过模块介绍

![](img/9aaa7069e6da16287982ff9930a6bd42_15.png)

在Metasploit Framework (MSF) 中，集成了多个用于绕过UAC的模块。使用这些模块有一个**前提条件**：我们已经获得了一个目标系统的初始访问权限，即一个普通的Meterpreter会话（Session）。

![](img/9aaa7069e6da16287982ff9930a6bd42_17.png)

![](img/9aaa7069e6da16287982ff9930a6bd42_19.png)

以下是使用MSF中`bypassuac`模块的基本流程：

![](img/9aaa7069e6da16287982ff9930a6bd42_21.png)

![](img/9aaa7069e6da16287982ff9930a6bd42_23.png)

![](img/9aaa7069e6da16287982ff9930a6bd42_24.png)

1.  首先，我们已获得一个普通权限的Meterpreter会话（例如 `session 1`）。
2.  在MSF中，我们使用 `use exploit/windows/local/bypassuac` 命令加载模块。
3.  然后，通过 `set SESSION 1` 指定要利用的会话。
4.  执行 `exploit` 命令，模块会尝试绕过UAC。
5.  如果成功，我们将获得一个新的、具有管理员权限的Meterpreter会话（例如 `session 2`）。

![](img/9aaa7069e6da16287982ff9930a6bd42_25.png)

![](img/9aaa7069e6da16287982ff9930a6bd42_27.png)

![](img/9aaa7069e6da16287982ff9930a6bd42_29.png)

核心操作代码如下：
```bash
use exploit/windows/local/bypassuac
set SESSION <已获得的普通会话ID>
exploit
```

![](img/9aaa7069e6da16287982ff9930a6bd42_30.png)

![](img/9aaa7069e6da16287982ff9930a6bd42_32.png)

![](img/9aaa7069e6da16287982ff9930a6bd42_34.png)

## 权限提升实战

成功绕过UAC并获得管理员会话后，我们的权限得到了提升，但尚未达到最高权限（SYSTEM）。此时，我们可以基于这个管理员会话，进一步获取SYSTEM权限。

![](img/9aaa7069e6da16287982ff9930a6bd42_36.png)

![](img/9aaa7069e6da16287982ff9930a6bd42_37.png)

![](img/9aaa7069e6da16287982ff9930a6bd42_39.png)

以下是具体的操作步骤：

![](img/9aaa7069e6da16287982ff9930a6bd42_41.png)

1.  进入新获得的管理员权限会话：`sessions -i <新会话ID>`。
2.  在该会话中，使用 `getsystem` 命令尝试提权至SYSTEM。
3.  提权成功后，我们便拥有了目标系统的最高控制权。

![](img/9aaa7069e6da16287982ff9930a6bd42_43.png)

![](img/9aaa7069e6da16287982ff9930a6bd42_45.png)

这个过程可以概括为：
**普通用户会话 → 绕过UAC → 管理员会话 → getsystem → SYSTEM权限**

## 总结

![](img/9aaa7069e6da16287982ff9930a6bd42_67.png)

![](img/9aaa7069e6da16287982ff9930a6bd42_69.png)

![](img/9aaa7069e6da16287982ff9930a6bd42_71.png)

本节课中我们一起学习了UAC绕过的原理与实践。我们了解到UAC是Windows的一项重要安全机制，而绕过UAC是在已取得初步立足点后实现权限提升的关键步骤。通过利用MSF中集成的 `bypassuac` 等模块，我们可以从普通用户权限提升至管理员权限，并最终通过 `getsystem` 命令获得SYSTEM最高权限。掌握这项技术对于理解Windows系统安全纵深防御和进行授权渗透测试至关重要。
