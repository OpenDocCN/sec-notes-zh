### 伪加密识别与修复
伪加密是指压缩包的文件头中，加密标志位被人为修改，导致解压软件误认为需要密码，而实际上并未加密。现代压缩软件（如360压缩、2345好压）可能自动修复此问题，但了解原理仍有必要。

![](img/1b8a11193883c76d9ca59ec56f632be0_16.png)

![](img/1b8a11193883c76d9ca59ec56f632be0_18.png)

![](img/1b8a11193883c76d9ca59ec56f632be0_20.png)

![](img/1b8a11193883c76d9ca59ec56f632be0_22.png)

修复伪加密的核心是使用十六进制编辑器（如WinHex、010Editor）修改特定的标志位。

![](img/1b8a11193883c76d9ca59ec56f632be0_24.png)

*   **对于ZIP文件**：
    1.  使用十六进制编辑器打开压缩包。
    2.  搜索十六进制值 `504B0102`（这是文件头标识）。
    3.  从找到的 `504B0102` 开始，向后数第9和第10个字节（一个字节占两位十六进制数）。
    4.  将这两个字节的值修改为 `00 00`。
    5.  保存文件后，压缩包即可正常解压。

*   **对于RAR文件**：
    1.  使用十六进制编辑器打开压缩包。
    2.  从文件开头向后找到第24个字节（通常在第2行第8个字节，因为一行有16个字节）。
    3.  将该字节的值修改为 `00`。
    4.  保存文件即可。

### 密码破解
当压缩包被真正加密时，我们需要尝试破解密码。通常题目不会设置过于复杂的密码，可能会给出部分提示。

以下是常用的破解工具和方法：

*   **ARCHPR**：主要用于破解RAR格式的压缩包。支持暴力破解、掩码攻击和字典攻击。
    *   **暴力破解**：尝试所有可能的字符组合。
    *   **掩码攻击**：当你知道密码的部分字符（如前缀）时，可以指定未知部分（如`ABC???`表示前三位是ABC，后三位未知）。
    *   **字典攻击**：使用常见的密码字典进行尝试。
*   **ZIP密码破解工具**：工具包中提供的专门针对ZIP格式的破解软件，使用方法类似，导入压缩包和字典即可开始破解。

**注意**：如果题目提示已知压缩包内某个文件的内容（明文），可以采用“明文攻击”。你需要将已知的明文文件用**相同的压缩算法**（如Store、Deflate）压缩成一个新文件，然后在破解工具中选择明文攻击并导入这个新压缩包，成功率会更高。

### 压缩包文件结构修复
有时压缩包本身结构损坏，导致无法列出或解压所有文件。这需要你了解压缩包的文件格式，并手动修复文件头。

![](img/1b8a11193883c76d9ca59ec56f632be0_26.png)

![](img/1b8a11193883c76d9ca59ec56f632be0_28.png)

![](img/1b8a11193883c76d9ca59ec56f632be0_30.png)

![](img/1b8a11193883c76d9ca59ec56f632be0_32.png)

*   **核心思路**：每个文件在压缩包中都有一个对应的“文件头”，其中包含“块类型”字段。例如，在RAR格式中，`0x74` 代表这是一个文件头。如果该字段被错误修改，解压软件就无法识别该文件。
*   **操作方法**：
    1.  用十六进制编辑器打开损坏的压缩包。
    2.  根据已知文件的结束位置，定位下一个文件的起始位置。
    3.  检查该位置的“块类型”字段是否正确（例如，应为 `0x74`）。
    4.  如果错误（如被改为 `0x7A`），将其修正为正确的值。
    5.  保存后，再次用压缩软件打开，隐藏的文件就会出现。

### 常用过滤语法
为了从海量数据包中快速定位关键信息，必须掌握过滤语法。以下是一些核心语法：

![](img/1b8a11193883c76d9ca59ec56f632be0_55.png)

![](img/1b8a11193883c76d9ca59ec56f632be0_57.png)

![](img/1b8a11193883c76d9ca59ec56f632be0_59.png)

![](img/1b8a11193883c76d9ca59ec56f632be0_60.png)

![](img/1b8a11193883c76d9ca59ec56f632be0_62.png)

![](img/1b8a11193883c76d9ca59ec56f632be0_64.png)

*   **过滤IP地址**：
    *   `ip.src == 192.168.1.1` （筛选源IP）
    *   `ip.dst == 192.168.1.100` （筛选目的IP）
*   **过滤端口**：
    *   `tcp.port == 80` （筛选TCP 80端口，常用于HTTP）
    *   `udp.port == 53` （筛选UDP 53端口，常用于DNS）
*   **过滤协议**：
    *   `http`、`tcp`、`dns`、`ftp` 等。
*   **过滤包内容**：
    *   `http contains “flag”` （在HTTP协议中搜索包含“flag”字符串的包）
    *   `tcp contains “key”` （在TCP负载中搜索“key”）
    *   `frame.len > 100` （筛选长度大于100字节的包，大包可能包含文件）

`contains` 语法非常实用，可以快速搜索可能包含提示信息的数据包。

### 分析技巧与步骤
1.  **协议分级统计**：
    打开数据包后，点击 **统计 (Statistics) -> 协议分级 (Protocol Hierarchy)**。这可以直观展示数据包中各种协议的占比，帮助你快速判断主要流量类型（如HTTP、TLS、TCP）。
2.  **追踪流**：
    右键点击一个数据包，选择 **追踪流 (Follow) -> TCP流/HTTP流/SSL流**。这会将一次完整会话的数据重组并显示出来，常用于查看完整的HTTP请求/响应、Telnet会话或提取可疑命令。
3.  **导出对象**：
    点击 **文件 (File) -> 导出对象 (Export Objects) -> HTTP…**。这可以一键导出所有通过HTTP协议传输的文件（如图片、文档、压缩包），是提取隐藏文件的快捷方法。
4.  **手动提取分组字节流**：
    对于非HTTP协议传输的文件，或文件被分片传输的情况，可以右键关键数据包，选择 **导出分组字节流 (Export Packet Bytes…)**。将原始数据保存为文件，后续可能需要合并或修复文件头。

### 特殊流量分析
*   **HTTPS流量**：
    HTTPS是加密的HTTP。在Wireshark中，你需要导入服务器的私钥（如果题目提供）才能解密。路径：**编辑 (Edit) -> 首选项 (Preferences) -> Protocols -> TLS/SSL**，添加RSA密钥。
*   **无线流量（WIFI握手包）**：
    包含WIFI握手过程的数据包，可用于破解WIFI密码。使用 `aircrack-ng` 工具，配合密码字典进行破解。
    ```bash
    aircrack-ng -w password_list.txt capture.pcap
    ```
*   **USB流量**：
    捕获自USB设备的数据，常考察键盘击键和鼠标移动轨迹的还原。
    *   **键盘流量**：数据通常为8个字节，其中第3个字节为键值。需要将键值映射到具体按键（可编写Python脚本实现）。
    *   **鼠标流量**：数据通常为4个字节，包含按键状态和X、Y轴的相对移动量。通过解析这些数据，可以绘制出鼠标移动轨迹，轨迹图形可能就是flag。
