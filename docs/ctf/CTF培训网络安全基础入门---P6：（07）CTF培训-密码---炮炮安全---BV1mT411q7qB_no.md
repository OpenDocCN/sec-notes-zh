## 密码学概述 📖

上一节我们介绍了密码学的基本概念，本节中我们来看看密码学的发展历程和主要分类。

密码学的发展经历了古典、近代和现代三个阶段。CTF题目中常涉及古典密码学和近代密码学，例如古代的数据加密方法或战争时期的电报加密。现代密码学题目通常难度较大，因为现代加密算法强调不可逆性和高强度。

密码学主要包含两个大概念：编码和加密。此外，还有一种称为摘要算法的技术。

*   **编码**：更像是一种映射关系。例如，可以创造一套自己的符号系统，每个符号代表一个特定含义。常见的编码包括ASCII、Unicode、URL编码和Base64编码。
*   **加密**：涉及更复杂的混淆过程。加密算法需要有密钥，并涉及明文（原始信息）和密文（加密后的信息）。加密算法主要分为两类：
    *   **对称密钥算法**：加密和解密使用同一个密钥。公式表示为：`C = E(K, P)` 和 `P = D(K, C)`，其中K是密钥，P是明文，C是密文，E是加密函数，D是解密函数。
    *   **非对称密钥算法**：使用一对密钥，一个公钥用于加密，一个私钥用于解密。公钥可以公开，而私钥必须保密。加密效率上，对称加密远高于非对称加密。

以下是常见的加密算法分类：
*   **对称密钥算法**：DES、3DES、IDEA、AES（目前最强的对称加密算法之一）。
*   **非对称密钥算法**：RSA（CTF中最常考）、ElGamal、椭圆曲线加密（ECC）。RSA算法涉及数论知识，如费马定理，理解起来较为复杂。

*   **摘要算法（哈希算法）**：主要用于验证数据的完整性，确保数据在传输过程中未被篡改。它具有“雪崩效应”，即输入数据的微小改变会导致输出结果的巨大差异。常见的摘要算法有MD5和SHA系列。它们是不可逆的。

## 常见编码 🔡

了解了密码学的整体框架后，我们来看看CTF中几种最常见的编码形式及其识别、解密方法。

![](img/3ed5593ae25d993c75dca09741c09f15_11.png)

### ASCII码

ASCII码是计算机最早期的编码标准，主要用于表示英文字母、数字和常见符号。标准ASCII码有128个字符（7位），扩展ASCII码有256个字符（8位）。

**特点与识别**：ASCII码本质上是字符到数字的映射。在解题时，如果遇到一串数字，可以尝试将其转换为对应的ASCII字符。

![](img/3ed5593ae25d993c75dca09741c09f15_13.png)

**工具与操作**：
*   **在线工具**：可以使用提供的在线编解码网站。
*   **离线工具**：CTF工具包中的“小葵多功能转换工具”或“密码机器”。
*   **编程实现**：在Python中，可以使用 `ord()` 函数将字符转为ASCII码，用 `chr()` 函数将ASCII码转回字符。
    ```python
    # 示例代码
    print(ord('A'))  # 输出: 65
    print(chr(65))   # 输出: 'A'
    ```

![](img/3ed5593ae25d993c75dca09741c09f15_15.png)

### Base64编码

Base64编码是一种基于64个可打印字符来表示二进制数据的方法。它常用于在HTTP等文本协议中传输二进制数据。

![](img/3ed5593ae25d993c75dca09741c09f15_17.png)

![](img/3ed5593ae25d993c75dca09741c09f15_18.png)

**原理与特点**：它将每3个8位字节（24位）的数据，重新编码为4个6位字节（24位）的数据，然后在每个6位数据前补两个0，形成4个8位字节。因此，编码后的数据长度通常会变长（约为原长的4/3），并且末尾常带有`=`或`==`作为填充标志。

**识别与解密**：如果看到一串由大小写字母、数字、`+`、`/`和`=`组成的字符串，很可能是Base64编码。如果解密失败，可以尝试在末尾补上`=`再解密。

![](img/3ed5593ae25d993c75dca09741c09f15_20.png)

**工具**：同样可以使用“小葵工具”、浏览器开发者工具（F12）中的控制台，或在线Base64编解码网站。

![](img/3ed5593ae25d993c75dca09741c09f15_22.png)

### URL编码

URL编码（百分号编码）主要用于将URL中不能直接传输的字符（如空格、中文、特殊符号）转换为安全格式。

**原理与特点**：它将特定字符转换为`%`后跟两位十六进制数（该字符的ASCII码）。例如，空格的ASCII码是32，十六进制是20，因此URL编码后为`%20`。

![](img/3ed5593ae25d993c75dca09741c09f15_24.png)

![](img/3ed5593ae25d993c75dca09741c09f15_26.png)

**识别与解密**：如果字符串中包含大量的`%xx`格式（xx为十六进制数字），则可以尝试URL解码。

![](img/3ed5593ae25d993c75dca09741c09f15_27.png)

![](img/3ed5593ae25d993c75dca09741c09f15_29.png)

**工具**：浏览器地址栏会自动进行URL编解码，也可以使用“小葵工具”或在线编解码器。

![](img/3ed5593ae25d993c75dca09741c09f15_31.png)

### Unicode编码

![](img/3ed5593ae25d993c75dca09741c09f15_33.png)

Unicode编码旨在为全世界所有字符提供统一的编码方案，以解决ASCII码容量不足的问题。

![](img/3ed5593ae25d993c75dca09741c09f15_34.png)

![](img/3ed5593ae25d993c75dca09741c09f15_36.png)

**特点与识别**：在表示时，常见的形式为`\u`后跟4位十六进制数（如`\u0041`代表‘A’）。如果看到大量这种格式的字符串，可能是Unicode编码。

![](img/3ed5593ae25d993c75dca09741c09f15_38.png)

![](img/3ed5593ae25d993c75dca09741c09f15_40.png)

**工具**：可以使用在线Unicode编解码工具或支持Unicode的文本编辑器进行转换。

![](img/3ed5593ae25d993c75dca09741c09f15_42.png)

![](img/3ed5593ae25d993c75dca09741c09f15_44.png)

### JavaScript混淆

严格来说，这不是一种编码，而是一种前端代码保护技术。在CTF的Web题中常见。

**原理与识别**：为了隐藏代码逻辑，开发者会将JavaScript代码进行混淆，例如将字符串转换为十六进制形式（`\x41`代表‘A’）或使用复杂的表达式。

**解密与执行**：混淆后的代码通常可以直接在浏览器的开发者工具（F12）的“控制台”（Console）中运行并得到结果。例如，一段形如 `\x61\x6c\x65\x72\x74\x28\x22\x58\x53\x53\x22\x29` 的代码，在控制台执行后会弹出一个显示“XSS”的警告框。
