# 网络安全实战：P47：实战模拟QQ、微信账号被盗取的全过程

在本节课中，我们将通过一个完整的实战演示，模拟QQ、微信账号被盗取的全过程。你将了解到攻击者如何部署钓鱼网站、生成伪装链接与二维码，以及如何收集和处理被盗的账号信息。本教程旨在揭示此类攻击的运作原理，以提升大家的安全防范意识。

## 准备公网服务器

上一节我们介绍了钓鱼网站的基本构成，本节中我们来看看如何将其部署到互联网上供他人访问。

![](img/80f0098a8539eba2690a497680588d3e_1.png)

首先，你需要一台拥有公网IP地址的服务器。家庭网络的IP通常是内网地址，无法被外部直接访问。公网IP则是一个可以被互联网上任何设备连接的地址。

![](img/80f0098a8539eba2690a497680588d3e_3.png)

以下是获取公网服务器的常见方式：

![](img/80f0098a8539eba2690a497680588d3e_5.png)

![](img/80f0098a8539eba2690a497680588d3e_6.png)

![](img/80f0098a8539eba2690a497680588d3e_8.png)

*   **云服务商**：例如阿里云、腾讯云等，它们提供按需购买的云服务器（ECS），通常会分配一个公网IP。
*   **传统运营商**：向移动、联通、电信申请公网IP，但通常费用较高且流程复杂。

![](img/80f0098a8539eba2690a497680588d3e_10.png)

在本演示中，我们使用了一台阿里云的Windows Server 2012服务器，其公网IP为：`118.178.134.226`。你可以通过远程桌面连接（RDP）工具管理这台服务器。

![](img/80f0098a8539eba2690a497680588d3e_11.png)

## 部署钓鱼网站源码

![](img/80f0098a8539eba2690a497680588d3e_13.png)

![](img/80f0098a8539eba2690a497680588d3e_14.png)

![](img/80f0098a8539eba2690a497680588d3e_16.png)

拥有了公网服务器后，下一步就是将钓鱼网站源码部署上去。

我们将准备好的“QQ空间”钓鱼网站源码复制到云服务器上。接着，在服务器上安装并运行一个集成的Web服务器环境（例如PHPStudy），将源码目录设置为该环境的网站根目录。

启动Web服务后，你的钓鱼网站就已经在公网可访问了。此时，在浏览器中直接访问服务器的公网IP（例如 `http://118.178.134.226`），就能看到伪装成QQ空间的登录页面。

![](img/80f0098a8539eba2690a497680588d3e_18.png)

## 生成伪装链接与二维码

![](img/80f0098a8539eba2690a497680588d3e_20.png)

![](img/80f0098a8539eba2690a497680588d3e_21.png)

![](img/80f0098a8539eba2690a497680588d3e_23.png)

直接发送IP地址链接很容易引起怀疑，因此我们需要对其进行伪装。

![](img/80f0098a8539eba2690a497680588d3e_25.png)

首先，使用短链接生成服务，将冗长的IP地址转换为一个更短、更不易察觉的链接。例如，将 `http://118.178.134.226` 转换为 `http://short.url/abc123`。

![](img/80f0098a8539eba2690a497680588d3e_27.png)

然后，利用二维码生成工具（如草料二维码），将这个短链接生成一张二维码图片。这样，攻击载体就从一串可疑的字符变成了一张看似无害的图片。

![](img/80f0098a8539eba2690a497680588d3e_29.png)

![](img/80f0098a8539eba2690a497680588d3e_30.png)

## 制作诱导性钓鱼文案

![](img/80f0098a8539eba2690a497680588d3e_32.png)

![](img/80f0098a8539eba2690a497680588d3e_34.png)

仅有二维码还不够，我们需要一个令人信服的“故事”来诱导受害者扫描。

以下是制作钓鱼文案的步骤：

![](img/80f0098a8539eba2690a497680588d3e_36.png)

![](img/80f0098a8539eba2690a497680588d3e_37.png)

1.  **设计文案**：编写一段具有诱惑力或紧迫感的文字，例如“怀旧经典：你是否还记得当年的QQ空间？扫码一键回顾青春岁月”。
2.  **图文混合**：使用Word、PPT或PS等工具，将生成的二维码图片嵌入到设计好的文案海报中。调整二维码的大小和位置，使其与背景图自然融合，尽可能消除拼接痕迹。
3.  **传播渠道**：将制作好的最终图片通过社交媒体、群聊、论坛等渠道进行传播。

## 模拟受害者上钩过程

现在，让我们模拟一个普通用户看到钓鱼信息后的完整流程。

1.  用户看到精心制作的“怀旧”海报，出于好奇扫描了其中的二维码。
2.  手机浏览器打开短链接，最终跳转到部署在云服务器上的钓鱼网站（伪装QQ空间登录页）。
3.  用户输入自己的QQ账号和密码，点击“登录”。
4.  页面弹出一个“账号或密码错误”的提示（无论密码对错都会弹出），点击确定后，页面会自动跳转到**真正的QQ空间官网**。
5.  用户在真官网再次登录，进入自己的空间，整个过程看似只是第一次登录时输错了密码，毫无察觉。

## 攻击者后台查看窃取信息

当受害者在钓鱼页面输入信息并点击“登录”的瞬间，其账号和密码已经被发送到攻击者服务器的后台数据库。

![](img/80f0098a8539eba2690a497680588d3e_42.png)

攻击者可以登录钓鱼网站的管理后台，查看所有收集到的凭证。记录通常包含以下信息：

*   **窃取的账号密码**：明文显示受害者输入的QQ号和密码。
*   **受害者IP地址**：记录下访问者当时的公网IP。

通过IP地址查询工具，攻击者可以大致定位受害者的地理位置（例如城市级精度）。这进一步增加了信息价值。

## 利用窃取信息进行后续攻击

获取QQ号和密码后，攻击者可能会尝试进行更深入的利用。

例如，通过“找回密码”等官方功能接口，利用已知的QQ号去尝试查询关联的手机号（部分接口验证不严，可能暴露部分手机号）。结合钓鱼网站源码中可能存在的“骗取短信验证码”的功能模块，攻击者有可能绕过二次验证，最终完全接管受害者的账号。

**核心攻击链公式可以概括为：**
`钓鱼网站 + 公网服务器 + 伪装引流 -> 窃取凭证 -> 信息拓展 -> 潜在账号接管`

## 总结与安全警示

本节课中我们一起学习了网络钓鱼攻击的完整实战流程。我们从部署公网服务器开始，逐步完成了钓鱼网站部署、链接伪装、文案制作、受害者交互模拟，并在后台查看了窃取到的敏感信息，最后探讨了可能的后续利用方式。

**请务必注意**：本教程仅用于网络安全知识学习与研究，旨在帮助大家理解攻击原理，从而更好地进行防御。**未经授权对他人实施此类行为是违法行为**。

通过了解攻击者的手法，我们希望你能：

1.  **警惕不明链接和二维码**：尤其是那些包装在情感煽动或利益诱惑文案中的。
2.  **仔细核对网站域名**：登录重要网站时，务必确认浏览器地址栏的域名是否正确。
3.  **启用多重验证**：为账号开启短信、邮箱或动态令牌等二次验证，即使密码泄露也能增加安全屏障。
4.  **保持软件更新**：确保操作系统和浏览器是最新版本，以修复可能被利用的安全漏洞。
