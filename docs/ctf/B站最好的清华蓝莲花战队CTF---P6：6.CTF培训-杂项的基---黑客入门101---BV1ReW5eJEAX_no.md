## 压缩文件处理 🗜️

上一节我们介绍了文件和图片隐写，本节中我们来看看如何处理加密或损坏的压缩文件。压缩文件（如ZIP、RAR）常作为题目的载体，考察点包括伪加密、密码破解和文件修复。

### 伪加密识别与修复

有时压缩包看似加密，要求输入密码，但这可能是“伪加密”。伪加密是指通过修改压缩文件头部的特定标志位，欺骗解压软件使其认为文件已加密。现代解压软件（如360压缩、2345好压）可能自动修复此类问题，但了解原理仍有必要。

![](img/65fe97e3cd76e26984c0c88868a5826a_6.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_8.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_10.png)

修复伪加密需要使用十六进制编辑器（如WinHex、010 Editor）修改特定字节。

![](img/65fe97e3cd76e26984c0c88868a5826a_12.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_14.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_16.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_18.png)

**ZIP文件修复步骤：**
1.  用十六进制编辑器打开ZIP文件。
2.  搜索十六进制值 `504B0102`（这是ZIP文件头的一个特征）。
3.  从找到的 `504B0102` 开始，向后数到第9和第10个字节（一个字节两位，即第9-10位字符）。
4.  将这两个字节的值改为 `00 00`。
5.  保存文件，伪加密即被解除。

![](img/65fe97e3cd76e26984c0c88868a5826a_20.png)

**RAR文件修复步骤：**
1.  用十六进制编辑器打开RAR文件。
2.  从文件开头向后找到第24个字节（第一行16个字节，第二行第8个字节）。
3.  将该字节的尾数（最低位）改为 `0`。
4.  保存文件。

### 密码破解

![](img/65fe97e3cd76e26984c0c88868a5826a_22.png)

如果压缩包是真正加密的，通常密码不会过于复杂，出题者会给出提示（如密码前几位），以减少暴力破解的搜索空间。

以下是常用的破解工具和方法：

**工具：ARCHPR (用于RAR) / ZIP密码破解工具**
这些工具支持多种攻击模式：
*   **暴力破解**：尝试所有可能的字符组合。
*   **字典攻击**：使用预定义的密码字典进行尝试。
*   **掩码攻击**：当你知道密码的部分字符（如前三位是`ABC`）时，可以指定已知部分，让工具只猜测未知部分，格式如 `ABC???`。
*   **明文攻击**：如果你拥有压缩包内某个未加密文件的原始内容，可以利用此文件进行更高效的破解。注意，用于攻击的明文文件必须使用与目标压缩包**相同的压缩算法**重新压缩一次。

### 文件修复与提取

有时压缩包本身结构损坏，导致无法正常列出或解压所有文件。这需要你了解压缩文件格式，手动修复文件头。

**关键概念：文件头类型**
在RAR格式中，不同的数据块有特定的“头部类型”标识。例如：
*   `0x74` 代表这是一个**文件头**，解压软件读到它才会解压对应的文件。
*   如果该标识被错误修改（如改成`0x7A`），该文件就不会被解压出来。

**修复步骤：**
1.  用十六进制编辑器打开损坏的压缩包。
2.  分析文件结构，找到应该是文件头但标识错误的位置。
3.  将错误的头部类型值改为正确的 `0x74`。
4.  保存后，即可正常解压出隐藏的文件。

## 流量分析取证 📡

![](img/65fe97e3cd76e26984c0c88868a5826a_26.png)

流量分析是CTF杂项题的重点和难点。题目会提供一个网络数据包捕获文件（如`.pcap`），你需要从中分析出隐藏的信息、提取文件或还原攻击过程。

### 核心工具：Wireshark

![](img/65fe97e3cd76e26984c0c88868a5826a_30.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_32.png)

Wireshark是进行流量分析必不可少的工具。你需要熟悉其基本界面和过滤语法。

![](img/65fe97e3cd76e26984c0c88868a5826a_34.png)

**常用过滤语法：**
以下是快速定位关键数据包的过滤命令：

![](img/65fe97e3cd76e26984c0c88868a5826a_36.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_38.png)

1.  **过滤IP地址**：
    *   `ip.src == 192.168.1.1` （源IP）
    *   `ip.dst == 10.0.0.1` （目的IP）
2.  **过滤端口**：
    *   `tcp.port == 80` （TCP 80端口，通常是HTTP）
    *   `udp.port == 53` （UDP 53端口，通常是DNS）
3.  **过滤协议**：
    *   直接输入协议名，如 `http`, `tcp`, `dns`, `ftp`。
4.  **过滤数据包内容（非常实用）**：
    *   `http contains “flag”` （在HTTP协议中搜索包含“flag”字符串的包）
    *   `tcp contains “key”` （在TCP负载中搜索“key”）
    *   使用 `contains` 可以快速搜索可能包含提示的关键字。

![](img/65fe97e3cd76e26984c0c88868a5826a_40.png)

### 分析流程与技巧

![](img/65fe97e3cd76e26984c0c88868a5826a_42.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_44.png)

面对一个数据包文件，可以按以下步骤进行分析：

![](img/65fe97e3cd76e26984c0c88868a5826a_46.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_48.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_50.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_52.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_53.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_55.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_57.png)

**1. 协议分级统计**
首先通过 **统计 (Statistics) -> 协议分级 (Protocol Hierarchy)** 功能，了解数据包中各种协议的分布比例。这能帮你快速判断主要流量类型（如HTTP占比很大），从而确定分析重点。

![](img/65fe97e3cd76e26984c0c88868a5826a_58.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_60.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_62.png)

**2. 应用显示过滤器**
根据协议分级结果或题目提示，使用上述过滤语法筛选出关键协议的数据流，例如 `http`。

**3. 追踪流 (Follow Stream)**
对于TCP或HTTP等有会话的协议，右键数据包选择 **追踪流 -> TCP流/HTTP流**。这会将整个会话的数据重组并显示在一个窗口中，便于查看完整的请求和响应，flag或关键信息常常直接出现在这里。

**4. 导出对象或分组字节流**
*   **导出HTTP对象**：如果主要是HTTP流量，可以通过 **文件 (File) -> 导出对象 (Export Objects) -> HTTP** 列出所有传输的文件（如图片、文档），并可直接保存。
*   **导出分组字节流**：对于非HTTP协议或分片传输的文件（如图片、压缩包），可以右键相关数据包，选择 **导出分组字节流**。你可能需要导出多个分段，然后使用工具（如`dd`命令）去除头部并合并，才能得到原始文件。

**USB流量分析**
USB流量可能包含键盘击键或鼠标移动轨迹数据。
*   **键盘流量**：数据通常为8个字节，击键信息在第3个字节。需要对照USB HID键值映射表，将十六进制数据转换为字符。可以编写或使用现有Python脚本自动化此过程。
*   **鼠标流量**：数据通常为4个字节，包含按键状态和X、Y轴偏移量。通过提取坐标数据，可以使用绘图工具（如gnuplot）还原出鼠标移动轨迹，轨迹可能构成flag或二维码。

![](img/65fe97e3cd76e26984c0c88868a5826a_77.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_79.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_81.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_83.png)

**无线（Wi-Fi）流量分析**
主要考察WPA/WPA2握手包的密码破解。
1.  使用 `aircrack-ng` 工具检查握手包：`aircrack-ng -J 捕获文件.pcap`
2.  使用字典进行破解：`aircrack-ng -w 密码字典.txt -J 捕获文件.pcap`
3.  若密码在字典中，工具会显示找到的密钥（KEY）。

![](img/65fe97e3cd76e26984c0c88868a5826a_85.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_87.png)

![](img/65fe97e3cd76e26984c0c88868a5826a_89.png)

**HTTPS流量分析**
HTTPS是加密的HTTP。如果题目给了解密所需的密钥（如RSA私钥），需要在Wireshark中导入才能解密查看明文。
*   路径：**编辑 (Edit) -> 首选项 (Preferences) -> 协议 (Protocols) -> TLS/SSL**。
*   在 “(Pre)-Master-Secret log filename” 中指定密钥文件。
*   导入后，原本加密的TLS流量会显示为解密的HTTP流量，即可按常规方法分析。
