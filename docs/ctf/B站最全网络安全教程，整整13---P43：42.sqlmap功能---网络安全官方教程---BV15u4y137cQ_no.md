## 概述

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_13.png)

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_15.png)

sqlmap是一款开源的自动化SQL注入检测与利用工具。它支持多种数据库，并能自动识别和利用多种类型的SQL注入漏洞。本节课将介绍其基本命令、漏洞检测流程以及数据枚举方法。

## 漏洞检测

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_34.png)

上一节我们介绍了如何查看帮助，本节中我们来看看如何使用sqlmap进行最基础的漏洞检测。核心是使用 `-u` 参数指定目标URL。

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_36.png)

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_38.png)

检测单个URL是否存在SQL注入漏洞，可以使用以下命令格式：
```bash
sqlmap -u "目标URL"
```
例如：
```bash
sqlmap -u "http://target.com/page.php?id=1"
```
执行后，sqlmap会进行一系列测试。如果发现疑似注入点，它会进行交互式询问，例如询问是否跳过检测其他数据库类型，或者是否包含针对特定数据库的所有测试。用户可以根据提示输入 `Y`（是）或 `N`（否）来引导扫描过程。最终，工具会报告参数（如`id`）是否存在注入漏洞。

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_40.png)

如果需要批量检测多个URL，可以使用 `-m` 参数，后面跟一个包含多个URL（每行一个）的文本文件。
```bash
sqlmap -m urls.txt
```

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_42.png)

## 信息枚举与数据获取

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_44.png)

在确认存在SQL注入漏洞后，我们可以利用sqlmap进一步获取数据库信息。这个过程通常是递进的：先获取数据库名，再获取表名，然后获取列名，最后导出数据。

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_46.png)

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_48.png)

以下是信息枚举的常用命令步骤：

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_50.png)

1.  **枚举所有数据库名**：使用 `--dbs` 参数。
    ```bash
    sqlmap -u "目标URL" --dbs
    ```
    此命令会列出数据库管理系统中的所有数据库名称。

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_52.png)

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_54.png)

2.  **枚举当前数据库**：使用 `--current-db` 参数。
    ```bash
    sqlmap -u "目标URL" --current-db
    ```
    此命令用于获取当前网站所使用的数据库名。在实际漏洞提交中，获取到数据库名通常就足以证明漏洞存在。

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_56.png)

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_58.png)

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_59.png)

3.  **枚举指定数据库中的表**：使用 `-D 数据库名 --tables` 参数。
    ```bash
    sqlmap -u "目标URL" -D security --tables
    ```
    此命令会列出指定数据库（例如 `security`）中的所有表名。

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_61.png)

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_62.png)

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_64.png)

4.  **枚举指定表的列**：使用 `-D 数据库名 -T 表名 --columns` 参数。
    ```bash
    sqlmap -u "目标URL" -D security -T users --columns
    ```
    此命令会列出指定表（例如 `users`）中的所有列名（字段名），如 `id`, `username`, `password`。

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_66.png)

5.  **导出表数据（拖库）**：使用 `-D 数据库名 -T 表名 -C “列名1,列名2” --dump` 参数。
    ```bash
    sqlmap -u "目标URL" -D security -T users -C "username,password" --dump
    ```
    **重要提示**：`--dump` 命令会导出并保存数据到本地（通常为CSV文件），这属于深度利用行为。**仅在获得明确授权的渗透测试中方可使用**，在未授权的漏洞挖掘（如SRC、众测）中应避免使用，以免触犯法律。

## 高级参数：检测等级（Level）与绕过脚本（Tamper）

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_68.png)

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_70.png)

在漏洞检测环节，sqlmap的 `--level` 参数决定了测试的深入程度。其默认值为1。

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_71.png)

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_73.png)

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_75.png)

**公式**：`--level` 数值越高，检测越全面，速度也越慢。
*   **Level 1**：默认值，仅测试GET和POST参数。
*   **Level 2及以上**：会额外测试HTTP Cookie、User-Agent、Referer等头部字段是否存在注入。

有时，网站会使用WAF（Web应用防火墙）或过滤机制来阻挡常见的SQL注入关键词（如 `sleep`, `union select`）。这时，我们可以使用 `--tamper` 参数调用内置的绕过脚本。

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_77.png)

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_79.png)

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_81.png)

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_82.png)

以下是 `--tamper` 参数的应用示例：
```bash
sqlmap -u “目标URL” --tamper=脚本名称
```
例如，使用 `space2comment` 脚本将空格替换为注释符：
```bash
sqlmap -u “目标URL” --tamper=space2comment
```
sqlmap内置了多种绕过脚本，其原理包括：
*   **关键词混淆**：如随机大小写（`randomcase`）。
*   **符号替换**：如用注释符`/**/`替换空格（`space2comment`）。
*   **编码绕过**：如对payload进行Base64编码（`base64encode`）。
使用 `--tamper` 参数需要根据目标网站具体的过滤规则来选择合适的脚本。可以结合 `-v` 参数查看payload的实时变化，以理解脚本的绕过原理。

![](img/5dd541679c1ce93d8d3c2e12e50eb2ca_86.png)
