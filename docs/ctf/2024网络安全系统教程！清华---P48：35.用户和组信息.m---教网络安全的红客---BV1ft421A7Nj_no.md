## 查看用户身份信息

![](img/a4900d9e69ac07161082e73018b85cb6_5.png)

![](img/a4900d9e69ac07161082e73018b85cb6_6.png)

上一节我们介绍了系统基础，本节中我们来看看如何查看用户身份信息。我们可以使用 `id` 命令来查看当前用户的身份标识。

![](img/a4900d9e69ac07161082e73018b85cb6_8.png)

![](img/a4900d9e69ac07161082e73018b85cb6_9.png)

![](img/a4900d9e69ac07161082e73018b85cb6_11.png)

执行 `id` 命令会显示以下信息：
*   **UID**：用户ID。例如，root用户的UID为0。
*   **GID**：主组ID。表示用户所属的主要组。
*   **组**：用户所属的所有组列表。

![](img/a4900d9e69ac07161082e73018b85cb6_13.png)

![](img/a4900d9e69ac07161082e73018b85cb6_15.png)

![](img/a4900d9e69ac07161082e73018b85cb6_16.png)

例如，对于root用户，`id` 命令的输出通常为：
```
uid=0(root) gid=0(root) groups=0(root)
```
这表示当前用户是root，其UID和GID都是0，并且属于root组。

![](img/a4900d9e69ac07161082e73018b85cb6_19.png)

## 查看登录用户信息

![](img/a4900d9e69ac07161082e73018b85cb6_25.png)

![](img/a4900d9e69ac07161082e73018b85cb6_27.png)

了解当前有哪些用户登录到系统是安全分析的重要环节。以下是相关的命令：

![](img/a4900d9e69ac07161082e73018b85cb6_29.png)

`w` 命令用于显示当前登录到Linux服务器的用户信息。

![](img/a4900d9e69ac07161082e73018b85cb6_31.png)

![](img/a4900d9e69ac07161082e73018b85cb6_32.png)

以下是 `w` 命令输出的关键信息：
*   **用户**：登录的用户名。
*   **TTY**：用户登录的终端。
*   **来源IP**：用户登录的来源IP地址。
*   **登录时间**：用户登录的起始时间。
*   **空闲时间**：用户空闲的时间。
*   **当前进程**：用户正在运行的命令。

![](img/a4900d9e69ac07161082e73018b85cb6_33.png)

![](img/a4900d9e69ac07161082e73018b85cb6_34.png)

![](img/a4900d9e69ac07161082e73018b85cb6_36.png)

![](img/a4900d9e69ac07161082e73018b85cb6_38.png)

`whoami` 命令非常简单，它只输出当前登录用户的用户名。这个命令在Windows和Linux系统下都通用。

![](img/a4900d9e69ac07161082e73018b85cb6_40.png)

![](img/a4900d9e69ac07161082e73018b85cb6_42.png)

`last` 命令用于显示系统近期的用户登录记录。

通过 `last` 命令，我们可以查看：
*   哪些用户曾登录过系统。
*   他们是从哪个IP地址登录的。
*   他们登录和退出的具体时间。
*   这对于追踪管理员活动或可疑登录行为非常有帮助。

`lastlog` 命令用于打印所有用户上次登录的日志。

![](img/a4900d9e69ac07161082e73018b85cb6_44.png)

![](img/a4900d9e69ac07161082e73018b85cb6_46.png)

该命令会读取 `/var/log/lastlog` 文件，并列出系统中每个用户的最后登录时间。对于从未登录过的用户（通常是系统服务账户），会显示 `**从未登录过**`。

![](img/a4900d9e69ac07161082e73018b85cb6_50.png)

## 用户配置文件解析

![](img/a4900d9e69ac07161082e73018b85cb6_55.png)

Linux系统的用户信息存储在特定的配置文件中，理解这些文件是深入掌握权限管理的基础。

![](img/a4900d9e69ac07161082e73018b85cb6_57.png)

`/etc/passwd` 文件存储了系统中所有用户的基本信息。

![](img/a4900d9e69ac07161082e73018b85cb6_59.png)

![](img/a4900d9e69ac07161082e73018b85cb6_60.png)

![](img/a4900d9e69ac07161082e73018b85cb6_62.png)

该文件中每一行代表一个用户，各字段由冒号 `:` 分隔。以root用户为例：
```
root:x:0:0:root:/root:/bin/bash
```
其各字段含义如下：
1.  **用户名**：`root`。
2.  **密码占位符**：`x`。实际密码哈希存储在 `/etc/shadow` 文件中。
3.  **UID**：用户ID，`0`。
4.  **GID**：主组ID，`0`。
5.  **描述信息**：`root`。
6.  **家目录**：`/root`。
7.  **登录Shell**：`/bin/bash`。

`/etc/shadow` 文件存储了用户的加密密码哈希。

由于 `/etc/passwd` 文件对所有用户可读，为了安全，密码哈希被移到了 `/etc/shadow` 文件。查看其权限：
```bash
ls -l /etc/shadow
```
输出通常为 `-rw-r-----`，表示只有root用户可读写，`shadow` 组的用户可读，其他用户无任何权限。这保护了密码哈希不被普通用户窃取。

`/etc/sudoers` 文件定义了哪些用户可以使用 `sudo` 命令获得root权限。

该文件非常重要，默认只有root用户可以编辑。使用 `visudo` 命令安全地编辑它。一个常见的配置是：
```
root    ALL=(ALL:ALL) ALL
%sudo   ALL=(ALL:ALL) ALL
```
这表示root用户和属于 `sudo` 组的用户可以在任何主机上，以任何用户的身份执行任何命令。

## Linux文件权限详解

在讨论用户和组时，不可避免地要回顾文件权限，这是Linux安全的核心。

![](img/a4900d9e69ac07161082e73018b85cb6_64.png)

使用 `ls -l` 命令查看文件详细信息时，第一列显示了权限，例如 `-rw-r--r--`。

![](img/a4900d9e69ac07161082e73018b85cb6_66.png)

![](img/a4900d9e69ac07161082e73018b85cb6_68.png)

权限字符串可以分解为四部分：
1.  **第一个字符**：表示文件类型。`-` 代表普通文件，`d` 代表目录。
2.  **第2-4字符**：文件**所有者（user）**的权限。
3.  **第5-7字符**：文件所属**用户组（group）**的权限。
4.  **第8-10字符**：**其他用户（other）**的权限。

每个权限位由三个字符表示：
*   `r`：读权限（read），数字表示为 **4**。
*   `w`：写权限（write），数字表示为 **2**。
*   `x`：执行权限（execute），数字表示为 **1**。
*   `-`：无对应权限。

![](img/a4900d9e69ac07161082e73018b85cb6_70.png)

![](img/a4900d9e69ac07161082e73018b85cb6_72.png)

权限可以用三位八进制数字表示，例如：
*   `rw-r--r--` = `644` （所有者：4+2=6， 组：4， 其他：4）
*   `rwxr-xr-x` = `755` （所有者：4+2+1=7， 组：4+1=5， 其他：4+1=5）

使用 `chmod` 命令修改权限：
```bash
chmod 755 filename    # 设置文件权限为 rwxr-xr-x
chmod u+x script.sh   # 给文件所有者增加执行权限
```

## 关于TTY终端

![](img/a4900d9e69ac07161082e73018b85cb6_74.png)

![](img/a4900d9e69ac07161082e73018b85cb6_76.png)

`TTY` 是Teletype的缩写，代表Linux系统中的虚拟控制台。

![](img/a4900d9e69ac07161082e73018b85cb6_78.png)

![](img/a4900d9e69ac07161082e73018b85cb6_80.png)

Linux系统默认提供多个虚拟控制台（如tty1, tty2...）。在图形界面下，通常可以通过快捷键切换：
*   `Ctrl + Alt + F1` 到 `F6`：切换到字符界面终端。
*   `Ctrl + Alt + F7` 或 `F8`：切换回图形界面。

在服务器环境或渗透测试中，了解用户通过哪个TTY登录有助于分析会话来源。

![](img/a4900d9e69ac07161082e73018b85cb6_82.png)
