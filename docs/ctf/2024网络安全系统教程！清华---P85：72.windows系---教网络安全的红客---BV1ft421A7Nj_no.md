# 网络安全系统教程：P85：Windows系统内核漏洞提权 🔧

![](img/60a033704edfd4087ca83e935b87d2ec_1.png)

![](img/60a033704edfd4087ca83e935b87d2ec_3.png)

在本节课中，我们将学习Windows系统内核漏洞提权的核心概念与常用方法。内核漏洞提权是渗透测试中获取系统级权限的关键技术之一，其核心在于识别目标系统未修补的安全漏洞并加以利用。

## 概述

![](img/60a033704edfd4087ca83e935b87d2ec_5.png)

![](img/60a033704edfd4087ca83e935b87d2ec_7.png)

上一节我们介绍了权限提升的基本概念，本节中我们来看看如何利用Windows内核漏洞进行提权。内核漏洞提权通常涉及检查系统版本、查找已知漏洞、验证补丁状态，并最终执行利用程序。

![](img/60a033704edfd4087ca83e935b87d2ec_9.png)

## 内核漏洞提权流程

![](img/60a033704edfd4087ca83e935b87d2ec_11.png)

![](img/60a033704edfd4087ca83e935b87d2ec_12.png)

内核漏洞提权的标准流程始于信息收集。我们需要首先检查Windows系统的版本，并确认其是否存在已知的可利用漏洞。

![](img/60a033704edfd4087ca83e935b87d2ec_14.png)

以下是进行信息收集的常用命令和方法：

![](img/60a033704edfd4087ca83e935b87d2ec_16.png)

![](img/60a033704edfd4087ca83e935b87d2ec_18.png)

![](img/60a033704edfd4087ca83e935b87d2ec_20.png)

*   **检查系统版本与补丁**：通过系统命令或脚本获取详细的系统信息和已安装的补丁列表。
*   **使用`wmic`或`Get-WmiObject`命令**：这两个命令功能类似，可用于查询系统信息，特别是补丁编号。
    *   示例命令：`wmic qfe get Caption,Description,HotFixID,InstalledOn`
    *   PowerShell等效命令：`Get-WmiObject -Class Win32_QuickFixEngineering`
*   **分析补丁信息**：通过对比已知漏洞的补丁编号（KB编号），判断系统是否缺失关键安全更新。

## 自动化提权探测工具

![](img/60a033704edfd4087ca83e935b87d2ec_26.png)

![](img/60a033704edfd4087ca83e935b87d2ec_28.png)

![](img/60a033704edfd4087ca83e935b87d2ec_30.png)

手动分析效率较低，因此我们常借助自动化工具来探测潜在的提权漏洞。以下是几个常用的工具：

![](img/60a033704edfd4087ca83e935b87d2ec_32.png)

![](img/60a033704edfd4087ca83e935b87d2ec_34.png)

![](img/60a033704edfd4087ca83e935b87d2ec_35.png)

*   **Metasploit 模块**：
    *   `post/windows/gather/enum_patches`：枚举系统已安装的补丁。
    *   `multi/recon/local_exploit_suggester`：探测目标系统可能存在并可用于提权的本地漏洞，并给出利用建议。
*   **Windows-Exploit-Suggester**：这是一个独立的Python脚本，它会根据系统信息（如`systeminfo`命令的输出）与漏洞数据库进行比对，列出可能存在的漏洞。
*   **PowerShell 脚本**：例如 `Sherlock.ps1` 脚本，可以枚举系统可能存在的提权漏洞。

![](img/60a033704edfd4087ca83e935b87d2ec_37.png)

![](img/60a033704edfd4087ca83e935b87d2ec_39.png)

![](img/60a033704edfd4087ca83e935b87d2ec_41.png)

![](img/60a033704edfd4087ca83e935b87d2ec_43.png)

**注意**：这些工具给出的结果是参考性的，提示的漏洞不一定100%可利用，需要进一步验证。

## 漏洞利用资源与实战

![](img/60a033704edfd4087ca83e935b87d2ec_47.png)

![](img/60a033704edfd4087ca83e935b87d2ec_48.png)

![](img/60a033704edfd4087ca83e935b87d2ec_50.png)

![](img/60a033704edfd4087ca83e935b87d2ec_52.png)

当我们识别出潜在的漏洞后，就需要找到对应的利用程序（Exploit）。

![](img/60a033704edfd4087ca83e935b87d2ec_54.png)

![](img/60a033704edfd4087ca83e935b87d2ec_55.png)

*   **漏洞利用代码库**：GitHub上存在许多收集Windows内核提权漏洞利用代码（Exp）的项目。例如，一个著名的仓库收集了从Windows XP到Windows 10的各种提权Exp。
    *   示例：`https://github.com/SecWiki/windows-kernel-exploits`
*   **提权辅助工具**：有些在线工具或本地工具可以辅助查询。例如，将`systeminfo`的输出粘贴到特定工具中，它可以快速匹配出未打补丁的漏洞及对应的Exp。
*   **Metasploit 实战**：在获得一个普通权限的Meterpreter会话后，可以运行`local_exploit_suggester`模块。如果它建议使用某个漏洞模块（如`exploit/windows/local/ms16_xxx`），配置并运行该模块，成功后通常会获得一个新的、权限更高的Meterpreter会话。在此会话中，便可使用`getsystem`命令尝试获取SYSTEM权限。

![](img/60a033704edfd4087ca83e935b87d2ec_57.png)

![](img/60a033704edfd4087ca83e935b87d2ec_59.png)

**关键点**：直接从普通用户权限的会话运行`getsystem`命令通常会失败。成功执行内核漏洞利用后，我们进入的会话本身权限可能已被提升，此时再执行`getsystem`命令则更容易成功。
