# 024：漏洞管理

![](img/8c90eef056f36f7aa8b05e5841193d2f_0.png)

![](img/8c90eef056f36f7aa8b05e5841193d2f_0.png)

在本节课中，我们将要学习漏洞管理的核心概念与流程。我们将了解什么是漏洞和利用，以及安全团队如何通过一个系统化的循环过程来发现并修复它们，从而在威胁造成损害之前保护资产安全。

对于每一个需要保护的资产，都存在许多漏洞。在漏洞引发问题之前找到并修复它们，是确保资产安全的关键。

我们之前已经介绍过漏洞的定义。漏洞是一个可以被威胁利用的弱点。描述中的“可以”一词非常重要。这是为什么呢？让我们一起来深入探讨。

想象一下，我交给你一份重要文件，并要求你确保它的安全。你会怎么做？有些人可能会首先想到把它锁在一个安全的地方。这背后的理解是，因为文件容易被移动，所以它容易遭到盗窃。当考虑到其他漏洞时，比如纸张易燃或不防水，你可能会增加其他保护措施。

与这个例子类似，安全团队根据资产的漏洞以及这些漏洞如何被利用来制定保护计划。利用是指利用漏洞的一种方式。除了发现漏洞，安全规划在很大程度上依赖于思考可能的利用方式。例如，存在想要造成伤害的窃贼。房屋有可以被窃贼利用的脆弱系统，窗户就是一个例子。玻璃容易被打破。窃贼可以通过用石头砸碎窗户来利用这个漏洞。提前思考这个漏洞和利用方式，能让我们提前做好计划。我们可以安装警报系统来吓跑窃贼并通知警方。

安全团队花费大量时间来寻找漏洞，并思考它们如何被利用。他们通过一个称为漏洞管理的过程来完成这项工作。

**漏洞管理**是发现和修补漏洞的过程。漏洞管理有助于保持资产安全。它是一种在威胁成为问题之前阻止威胁的方法。

漏洞管理是一个四步流程。第一步是识别漏洞。下一步是考虑这些漏洞的潜在利用方式。第三步是针对威胁准备防御措施。最后，第四步是评估这些防御措施。当最后一步结束时，流程重新开始。漏洞管理在一个循环中进行。这是安全团队工作的常规部分，因为总会有新的漏洞需要关注。

这正是为什么多样化的视角非常有用。拥有广泛的背景和经验只会加强安全团队及其发现利用方式的能力。然而，即使是庞大且多样化的安全团队也无法跟踪所有事情。新的漏洞不断被发现。这些被称为零日利用。

**零日利用**是指先前未知的利用方式。术语“零日”指的是这种利用正在实时发生，没有（零）天数来修复它。😊

这类利用非常危险。它们代表了尚未被计划应对的威胁。例如，我们可以预见到窃贼闯入家中的可能性。我们可以通过安装门锁和窗锁等防御措施来为这类威胁做计划。而零日利用则会是完全意想不到的事情，比如门锁因高温而脱落。零日利用通常是人们不会想到的事情。例如，这可能是一种新型间谍软件感染了一个流行网站。当零日利用发生时，它们可能使资产比原本更加容易受到威胁。

漏洞管理是发现漏洞并修复其利用方式的过程。这就是为什么这个过程在大多数组织中定期执行。或许这个过程最重要的一步是识别漏洞。我们下次见面时将更详细地探讨这一步。到时见。

**本节课总结**

![](img/8c90eef056f36f7aa8b05e5841193d2f_2.png)

本节课中，我们一起学习了漏洞管理的核心概念。我们明确了**漏洞**是可能被利用的弱点，而**利用**则是实际利用该弱点的行为。我们详细介绍了**漏洞管理**的四步循环流程：**识别漏洞**、**考虑潜在利用**、**准备防御措施**和**评估防御效果**。我们还了解了**零日利用**这种特殊且危险的威胁形式。理解并实践漏洞管理，是安全团队持续保护组织资产免受威胁的关键。
