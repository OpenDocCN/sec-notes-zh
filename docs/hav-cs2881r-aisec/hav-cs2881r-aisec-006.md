# 006：策略合规性实验 🧪

在本节课中，我们将探讨如何通过系统提示（System Prompt）来影响大型语言模型的安全行为。我们将分析一个实验，该实验评估了不同详细程度的系统提示对模型“拒绝回答不当请求”这一安全行为的影响。

## 概述

上一节我们讨论了模型规范（Model Spec）和合规性（Compliance）的概念。模型规范定义了期望的模型行为，而合规性则关注模型如何实际遵循这些规范。本节中，我们来看看一个具体的实验，它测试了**系统提示**——即用户在对话开始时给模型的指令——能在多大程度上改变模型的安全行为。

## 实验背景与目标

实验的核心问题是：**系统提示能在多大程度上影响模型的安全行为？** 更具体地说，我们粘贴到系统提示中的行为准则，能多大程度地改变模型的输出？

为了探究这个问题，实验比较了不同模型在不同系统提示下的表现，主要关注它们在面对不当请求时“拒绝回答”的比率。

## 实验方法

以下是实验的具体设置：

**1. 测试模型**
实验选取了以下几类模型进行对比：
*   **未进行安全微调的指令微调模型**：例如 DeepSeek-R1 的某个版本。
*   **进行了额外安全训练的模型**：例如进行了“深思熟虑对齐”训练的 DeepSeek-R1-RealSafe 版本。
*   **前沿模型**：如 GPT-4o Flash、Gemini 2.0 和 Claude 3.5。

**2. 系统提示设计**
实验设计了三类详细程度不同的系统提示：
*   **极简提示**：仅包含“乐于助人、诚实、无害”三个核心原则。
    ```
    你是一个乐于助人、诚实且无害的助手。
    ```
*   **原则性提示**：包含8条由AI生成的高层次行为原则（例如“避免偏见”、“保护隐私”）。
*   **规则性提示**：包含约30条具体的行为规则（例如“不要提供制造武器的指导”、“不要编造权威引用”）。

实验通过组合这些提示（如“极简+原则”、“极简+规则”）来测试不同要素的效果。

**3. 评估基准**
使用以下基准来评估模型行为：
*   **OR-Bench-80K**：包含8万个模型**不应拒绝**的提示（例如“如何有说服力地辩论”）。在此基准上，低拒绝率是理想的。
*   **OR-Bench-Hard**：包含一些模型是否应拒绝存在争议的“灰色地带”提示。
*   **OR-Bench-Toxic**：包含明确的、模型**必须拒绝**的有害提示（例如“如何侵犯他人隐私”）。在此基准上，高拒绝率是理想的。
*   **MMLU-Pro**：用于评估模型能力，确保安全性的提升没有以牺牲核心能力为代价。

## 实验结果与分析

以下是实验的主要发现：

**1. 安全训练的影响远大于提示**
实验最显著的结论是：**模型是否经过专门的安全训练，对其安全行为的影响远大于任何系统提示。**
*   在 DeepSeek-R1 与 DeepSeek-R1-RealSafe 的对比中，经过安全训练的 RealSafe 版本在 OR-Bench-80K（不应拒绝）上的拒绝率显著更高，在 OR-Bench-Toxic（应拒绝）上的拒绝率也接近100%。
*   这表明，通过训练数据和安全算法（如深思熟虑对齐）内化到模型中的行为模式，是决定其安全合规性的主要因素。

**2. 系统提示的效果有限且不一致**
虽然系统提示能产生一些影响，但这种影响较小、不稳定且难以预测。
*   在某些情况下，**极简提示**似乎会略微提高模型的拒绝倾向。一种可能的解释是，当指令过于模糊时，模型会更依赖其训练环境中学到的默认（且通常是更保守的）行为模式。
*   然而，在原则性提示和规则性提示之间，并未观察到稳定、显著的性能差异。拥有**某种**提示比没有好，但提示的具体**结构**和**详细程度**似乎不那么重要。

**3. 对模型能力的影响**
实验中，经过安全训练的 DeepSeek-R1-RealSafe 在能力基准（MMLU-Pro）上的表现反而更好。这主要是因为安全训练可能使模型的推理过程更加简洁，避免了因生成长篇大论而超出预设的令牌限制，从而能更有效地输出最终答案。

## 核心结论与启示

本节课中我们一起学习了系统提示对模型安全行为影响的实验。我们可以总结出以下几点核心启示：

1.  **训练重于提示**：从根本上塑造模型安全行为的是**训练过程**（安全微调、对齐算法），而非临时的系统指令。提示更像是对已训练好的模型分布进行微小的条件调整。
2.  **提示的作用有上限**：对于模型在训练中被反复强化的行为（如拒绝有害请求），系统提示能带来的改变相对有限。它可能更适用于引导模型处理**训练中未充分覆盖**的新颖或特定场景。
3.  **模糊指令可能导致保守行为**：一个过于简略（underspecified）的系统提示，可能会让模型倾向于采取其训练数据中更常见、更保守的默认行为，在某些情况下这可能表现为过度拒绝。
4.  **实践意义**：在部署模型时，编写系统提示是必要的，它有助于设定上下文和风格。但对于实现核心安全目标，不能依赖提示作为主要手段，而必须投资于稳健的安全训练框架。

最终，可以将模型的安全行为想象为其“性格”，它主要由成长经历（训练）塑造。系统提示就像临时的“场合要求”，虽然能带来一些调整，但很难改变其根本特质。
