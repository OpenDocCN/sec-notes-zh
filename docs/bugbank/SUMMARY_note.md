# 课程01：漏洞银行大咖面对面系列教程概览 📚

在本课程中，我们将系统性地学习“漏洞银行大咖面对面”系列分享的核心内容。该系列汇集了众多安全领域专家的实战经验，涵盖了从Web安全、内网渗透、漏洞挖掘到安全建设、逆向工程等多个方向。本教程旨在将这些宝贵的知识整理成结构化的学习路径，帮助初学者构建全面的网络安全知识体系。

## 课程内容结构

上一节我们介绍了本课程的整体目标，本节中我们来看看具体的课程模块划分。整个系列内容可以归纳为以下几个核心领域：

以下是主要的知识模块列表：

1.  **Web安全漏洞与利用**：涵盖SQL注入、文件上传、反序列化、SSRF、XXE等常见Web漏洞的原理、绕过技巧及实战案例。
2.  **内网渗透与权限维持**：深入讲解域渗透、横向移动、权限提升、隧道技术及各种权限维持方法。
3.  **漏洞挖掘与代码审计**：包括白盒/黑盒审计思路、CMS漏洞挖掘、Fuzz测试技术及自动化工具开发。
4.  **安全防御与体系建设**：从甲乙方视角探讨安全评估、日志分析、威胁情报、隐私保护及企业安全建设。
5.  **逆向工程与恶意代码分析**：涉及病毒样本分析、软件保护、Hook技术、二进制自动化等。
6.  **无线与硬件安全**：介绍WIFI安全、RFID攻击、IoT设备漏洞挖掘等。
7.  **安全开发与工具**：聚焦Python安全工具开发、Burp插件编写、扫描器优化等。
8.  **CTF与安全竞赛**：解析CTF中的PWN、隐写术等题型技巧。
9.  **安全职业与认证**：分享求职经验、CISP-PTE认证指南等职业发展相关内容。

## 核心概念与学习方法

本系列教程强调实战与原理结合。对于核心的技术概念，我们将使用**公式**或**代码**片段进行清晰描述。

例如，在理解SQL注入时，其本质是用户输入被拼接进SQL语句并执行：
```sql
-- 脆弱代码示例
query = "SELECT * FROM users WHERE id = '" + user_input + "'"
-- 如果 user_input 为 `1' OR '1'='1`，则查询逻辑被改变
```

学习时建议遵循以下步骤：
1.  **理解原理**：掌握每种漏洞或技术的基本运作机制。
2.  **搭建环境**：在安全的实验环境（如虚拟机、靶场）中复现。
3.  **动手实践**：跟随教程中的案例进行操作，观察输入与输出。
4.  **举一反三**：思考防御方案和在不同场景下的变种利用方式。
5.  **关注动态**：安全技术更新迅速，需持续关注新的绕过手法和防御策略。
