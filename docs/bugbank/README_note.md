在本节课中，我们将要学习安全测试的基本概念、核心流程以及如何开始你的第一次安全测试实践。课程内容旨在为初学者构建一个清晰、实用的知识框架。

## 概述

安全测试是评估计算机系统、网络或应用程序以发现潜在安全漏洞的过程。其目标是识别可能被恶意攻击者利用的弱点，从而在造成实际损害前进行修复。

上一节我们介绍了安全测试的基本定义，本节中我们来看看安全测试的核心目标。

## 安全测试的核心目标

安全测试主要围绕三个核心目标展开，它们共同构成了安全防护的基础。

以下是安全测试的三个核心目标：

1.  **机密性**：确保信息不被未授权用户访问。这通常通过访问控制机制实现，例如 `if (user.hasPermission('read_data')) { grantAccess(); }`。
2.  **完整性**：保护数据在存储或传输过程中不被未授权地篡改。常用哈希函数进行验证，例如 `SHA256(original_data) === received_hash`。
3.  **可用性**：确保授权用户在需要时可以访问系统和数据。这涉及到防御拒绝服务攻击等威胁。

理解了目标之后，我们需要了解实现这些目标的具体方法。接下来，我们将探讨安全测试的主要类型。

## 安全测试的主要类型

安全测试涵盖多种方法，每种方法从不同角度评估系统的安全性。

以下是几种常见的安全测试类型：

*   **静态应用程序安全测试**：在不运行代码的情况下分析源代码或二进制文件，以发现潜在漏洞。
*   **动态应用程序安全测试**：在应用程序运行时对其进行测试，模拟攻击者的行为来发现漏洞。
*   **渗透测试**：模拟真实世界的网络攻击，以评估系统的防御能力。这通常涉及授权的、目标明确的攻击尝试。
*   **漏洞评估**：使用自动化工具扫描系统，识别已知的漏洞和错误配置。

了解了测试类型，下一步就是将它们付诸实践。下面，我们将介绍一个标准的安全测试流程。

## 安全测试的基本流程

一个结构化的流程能确保测试的全面性和有效性。标准流程通常包括以下五个阶段。

以下是安全测试的五个基本阶段：

1.  **规划与侦察**：定义测试范围、目标并获得授权。收集关于目标系统的信息。
2.  **扫描**：使用工具分析目标，了解其开放的端口、运行的服务和可能的入口点。
3.  **漏洞利用**：尝试安全地利用已发现的漏洞，以验证其真实风险。
4.  **后渗透**（可选）：在成功利用漏洞后，评估其在系统内部进一步移动和造成影响的能力。
5.  **报告**：详细记录发现的所有漏洞、利用过程、风险等级以及修复建议。

流程是骨架，而具体的发现则需要工具和知识的支撑。最后，我们来看看安全测试中一些常见的关键发现。

## 常见安全漏洞示例

在实际测试中，某些类型的漏洞出现频率很高，了解它们对初学者至关重要。

以下是几个典型的安全漏洞示例：

> **SQL注入**：攻击者将恶意SQL代码插入输入字段，从而操纵后端数据库查询。例如：`SELECT * FROM users WHERE username = ‘admin’--’ AND password = ‘…’`，其中`--`注释掉了后续密码验证。
>
> **跨站脚本**：攻击者在网页中注入恶意客户端脚本，当其他用户浏览该页面时脚本会执行。
>
> **跨站请求伪造**：诱骗已认证的用户在不知情的情况下提交恶意请求，例如在用户登录银行网站的同时，诱使其点击一个转账链接。
