## 概述

在本节课中，我们将学习客户端安全领域的最后一部分内容，涵盖用户界面（UI）拒绝服务攻击、钓鱼攻击以及侧通道攻击。我们将通过一个名为“annoyingsite.com”的演示网站，了解攻击者如何滥用浏览器API来干扰用户，并探讨这些攻击背后的原理及防御措施。

## UI拒绝服务攻击

上一节我们介绍了客户端安全的基本概念，本节中我们来看看一种特殊的攻击形式——UI拒绝服务攻击。这种攻击通过覆盖浏览器的默认行为，使用户难以逃离攻击者设定的“陷阱”。

### 浏览器API的权限等级

为了更好地理解这些攻击，我们可以将浏览器提供的API按权限等级进行分类：

*   **等级0：无需用户交互即可使用的API。** 例如，操作DOM节点、移动窗口、触发文件下载、隐藏鼠标光标等。
*   **等级1：需要用户交互（如点击、按键）后才能触发的API。** 例如，打开新窗口（`window.open`）、全屏显示、复制到剪贴板等。
*   **等级2：基于用户与网站互动程度决定是否允许的API。** 例如，自动播放音频。如果用户经常在某个网站（如YouTube）与音频内容互动，浏览器可能会允许该网站自动播放。
*   **等级3：会触发明确权限提示的API。** 例如，请求获取地理位置、摄像头或麦克风权限。

### 攻击示例：annoyingsite.com

该网站演示了如何滥用这些API来制造混乱。以下是它使用的一些技术：

*   **滥用 `window.open` API：** 网站在检测到任何用户交互（如点击、按键）时，都会打开新的弹出窗口。
    ```javascript
    // 在用户点击时打开新窗口
    document.addEventListener('click', function() {
        window.open('', '', 'width=200,height=200');
    });
    ```
*   **阻止窗口关闭：** 通过监听 `beforeunload` 事件并返回一个真值，可以触发“确认离开”提示，干扰用户关闭标签页。
    ```javascript
    window.addEventListener('beforeunload', function(event) {
        event.returnValue = '你确定要离开吗？';
    });
    ```
*   **禁用后退按钮：** 通过监听 `popstate` 事件（用户点击后退按钮时触发）并立即导航前进，可以使后退按钮失效。
    ```javascript
    window.addEventListener('popstate', function() {
        history.forward();
    });
    ```
*   **劫持剪贴板：** 在用户交互后，自动将特定内容复制到用户剪贴板。
    ```javascript
    // 响应点击事件复制文本
    document.addEventListener('click', function() {
        navigator.clipboard.writeText('这是复制的文本');
    });
    ```

**过渡：** 了解了攻击者如何通过滥用API干扰用户后，我们接下来看看另一种更常见的、利用人性弱点的攻击——钓鱼攻击。

## 钓鱼攻击

钓鱼攻击的核心是伪装成可信实体，诱骗用户泄露敏感信息。攻击网站往往在视觉上极力模仿真实网站。

### 同形异义字攻击

这种攻击利用Unicode中来自不同语言但外观相似的字符来注册域名。

*   **原理：** 例如，西里尔字母的“а”（U+0430）与拉丁字母的“a”（U+0061）外观几乎一样，但计算机视其为不同字符。攻击者可以注册 `аррӏе.com`（使用特殊字符）来模仿 `apple.com`。
*   **浏览器防御：** 现代浏览器（如Chrome、Safari）会检测此类可能混淆的域名，并将其显示为“Punycode”编码形式（以 `xn--` 开头），从而暴露其真实面目。例如，`аррӏе.com` 可能显示为 `xn--80ak6aa92e.com`。
*   **当前状况：** 如果整个域名都由来自单一外语字符集的“形似”字母组成，某些浏览器（如旧版Firefox）可能仍会直接显示伪装后的域名，导致用户受骗。

攻击者利用很长的子域名来让用户忽略真正的域名。

*   **示例URL：** `https://paypal.com.security-update.activateuser.com/login`
    *   用户可能只注意到开头的“paypal.com”，而实际上他们位于 `activateuser.com` 这个攻击者控制的域名下。
*   **浏览器防御：** 部分浏览器（如Firefox）会尝试在地址栏中淡化显示子域名部分，突出显示注册的主域名（本例中为 `activateuser.com``），但效果有限，尤其在地址栏空间不足时。

### 画中画与界面伪造攻击

攻击者可以在自己的网页中内嵌真实网站的截图或iframe，并伪造浏览器UI元素（如地址栏、安全锁图标），使整个页面看起来像是在真实的网站上。

![](img/f681863924b7b9784688b178a0b3a611_24.png)

![](img/f681863924b7b9784688b178a0b3a611_26.png)

![](img/f681863924b7b9784688b178a0b3a611_28.png)

*   **防御措施：**
    1.  **使用密码管理器：** 密码管理器基于真实域名填充凭证，不会在钓鱼网站上自动填充，这是非常有效的防护手段。
    2.  **使用硬件安全密钥：** 像YubiKey这样的设备使用加密协议与特定网站通信，不会向钓鱼网站泄露秘密信息。

## 交互劫持攻击

![](img/f681863924b7b9784688b178a0b3a611_36.png)

![](img/f681863924b7b9784688b178a0b3a611_37.png)

![](img/f681863924b7b9784688b178a0b3a611_38.png)

这类攻击诱使用户在不知情的情况下执行危险操作。

### Cookie劫持

这是一个2011年在IE浏览器中被发现的已修复漏洞。其攻击步骤精巧地结合了多种技术：
1.  攻击者通过网络嗅探获取目标用户在本地网络中的用户名。
2.  构造一个URL，内嵌一个指向目标用户（如Google）Cookie页面的iframe。由于同源策略，攻击者页面无法直接读取iframe内容。
3.  利用点击劫持技术，将iframe设置为透明并覆盖在诱人的可拖动元素（如一个篮球游戏）之上。
4.  用户试图拖动篮球时，实际上却选中并拖出了iframe中自己的Cookie文本，并将其“放置”到了攻击者控制的页面上，从而泄露了Cookie。

### 文件劫持

攻击者通过触发“文件上传”对话框，并利用页面文字诱导用户，使其误以为是在选择“文件下载”的位置。用户选择文件夹并确认后，实际上是将整个文件夹的内容上传到了攻击者的服务器。

**过渡：** 上述攻击大多已被现代浏览器修复或缓解。面对层出不穷的钓鱼威胁，我们能否假设用户最终总会访问到恶意网站，并在此基础上提供保护呢？这就是“安全浏览”服务的思路。

## 安全浏览与隐私保护

![](img/f681863924b7b9784688b178a0b3a611_40.png)

![](img/f681863924b7b9784688b178a0b3a611_42.png)

谷歌安全浏览等服务维护着一个已知恶意网站的列表，并帮助浏览器阻止用户访问这些网站。

![](img/f681863924b7b9784688b178a0b3a611_44.png)

![](img/f681863924b7b9784688b178a0b3a611_46.png)

*   **隐私挑战：** 直接查询“某个URL是否安全？”会向服务提供商泄露用户完整的浏览历史。
*   **解决方案：** 使用隐私保护技术，如同态加密或私有信息检索，使得浏览器可以在不向谷歌发送具体URL的情况下，查询该URL是否存在于恶意名单中。这平衡了安全与隐私的需求。

## 总结

![](img/f681863924b7b9784688b178a0b3a611_56.png)

本节课我们一起学习了多种客户端攻击技术：
1.  **UI拒绝服务攻击：** 攻击者滥用浏览器API（如弹窗、阻止关闭、禁用后退）来骚扰或困住用户。
2.  **钓鱼攻击：** 通过同形异义字域名、子域名欺骗、伪造浏览器界面等手段，诱骗用户输入凭证。
3.  **交互劫持攻击：** 如Cookie劫持和文件劫持，利用UI混淆诱使用户执行非本意的危险操作。
4.  **防御措施：** 使用密码管理器、硬件安全密钥是防范钓鱼的有效手段。浏览器厂商也在通过改进URL显示、禁用危险API等方式不断提升安全性。
5.  **安全浏览服务：** 在保护用户隐私的前提下，利用共享的威胁情报帮助用户避开已知的恶意网站。

![](img/f681863924b7b9784688b178a0b3a611_58.png)

这些案例揭示了Web平台在赋予开发者强大能力（以构建丰富的应用）与保障用户安全、隐私之间所面临的持续挑战。
